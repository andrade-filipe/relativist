# TASK-0017: Add serde + bincode serialization support

**Spec:** SPEC-02

## Context

The Net must be serializable for transmission between nodes in the grid protocol. The confirmed technical decision is serde + bincode. The serde `#[derive(Serialize, Deserialize)]` attributes are already placed on the type definitions in earlier tasks (TASK-0002 through TASK-0008). This task adds the `serde` and `bincode` crate dependencies to `Cargo.toml` and implements convenience functions for serializing/deserializing a `Net` to/from bytes.

R24 requires serde + bincode. R25 requires self-contained format. R26 requires round-trip identity. R27 (SHOULD) recommends fixed-size per element (partially satisfied by bincode's compact format).

## Acceptance Criteria

- [ ] `Cargo.toml` includes `serde = { version = "1", features = ["derive"] }` and `bincode = "1"` dependencies
- [ ] `Net::to_bytes(&self) -> Result<Vec<u8>, RelError>` serializes the net using bincode
- [ ] `Net::from_bytes(bytes: &[u8]) -> Result<Net, RelError>` deserializes a net using bincode
- [ ] Round-trip test: `Net::from_bytes(&net.to_bytes()?) == net` (structural equality)
- [ ] Serialization errors are mapped to `RelError::Serialize` / `RelError::Deserialize`

## Key Types / Signatures

```rust
impl Net {
    /// Serializes the net to a bincode byte vector.
    ///
    /// The format is self-contained: the receiver can reconstruct
    /// the complete Net from the bytes (R25).
    pub fn to_bytes(&self) -> Result<Vec<u8>, crate::error::RelError> {
        bincode::serialize(self)
            .map_err(|e| crate::error::RelError::Serialize(e.to_string()))
    }

    /// Deserializes a net from a bincode byte slice.
    pub fn from_bytes(bytes: &[u8]) -> Result<Net, crate::error::RelError> {
        bincode::deserialize(bytes)
            .map_err(|e| crate::error::RelError::Deserialize(e.to_string()))
    }
}
```

- The serde derives on `Symbol`, `Agent`, `PortRef`, and `Net` must all be present for serialization to work. These were added in TASK-0002, TASK-0004, TASK-0005, and TASK-0008 respectively.
- Round-trip identity (R26) requires implementing `PartialEq` for `Net` (see TASK-0018).
