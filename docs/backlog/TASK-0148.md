## Context

SPEC-11 R6 mandates that `dispatch()` (coordinator module) and `handle_message()` (protocol module) are annotated with `#[tracing::instrument]`. These are the two remaining functions from the five required instrumented functions. `dispatch()` tracks partition distribution to workers; `handle_message()` tracks incoming protocol messages.

Both are combined in one task because they are small additions (a single attribute each) and belong to closely related modules in the protocol/coordinator domain.

## Acceptance Criteria

- [ ] `dispatch()` function has `#[tracing::instrument]` attribute
- [ ] `dispatch()` span includes fields: `worker_id`, `partition_index`, `size_bytes`
- [ ] `handle_message()` function has `#[tracing::instrument]` attribute
- [ ] `handle_message()` span includes fields: `message_type`, `peer`
- [ ] The `skip` parameter excludes large arguments (partition data, raw bytes) from span fields
- [ ] Both functions compile and existing behavior is unchanged
