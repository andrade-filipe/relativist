## Context

The metrics/health/ready HTTP server must run as a background tokio task within the coordinator's async runtime (R23). It binds to the same address as the main coordinator listener but on a separate port (default 9090, configurable via `--metrics-port` per R20). The server must NOT block the grid protocol event loop.

This task implements the `spawn_metrics_server()` function that starts the axum HTTP server in a detached tokio task. The function is called during coordinator startup (after `init_tracing` and metrics registration).

## Key Types / Signatures

```rust
use tokio::task::JoinHandle;
use std::net::SocketAddr;

/// Spawn the metrics/health/ready HTTP server as a background task (R20, R23).
///
/// The server runs on a dedicated port, separate from the grid protocol port.
/// Returns a JoinHandle that can be used to await server shutdown.
///
/// # Errors
///
/// Does not return an error. If binding fails, an ERROR event is logged
/// and the returned JoinHandle completes immediately.
#[cfg(feature = "metrics")]
pub async fn spawn_metrics_server(
    bind_addr: &str,
    port: u16,
    registry: std::sync::Arc<prometheus_client::registry::Registry>,
    coordinator_state: std::sync::Arc<std::sync::atomic::AtomicU8>,
) -> JoinHandle<()> {
    let router = metrics_router(registry, coordinator_state);
    let addr: SocketAddr = format!("{}:{}", bind_addr, port)
        .parse()
        .expect("valid bind address");

    tracing::info!(
        bind = %addr,
        "metrics HTTP server starting"
    );

    tokio::spawn(async move {
        let listener = match tokio::net::TcpListener::bind(addr).await {
            Ok(l) => l,
            Err(e) => {
                tracing::error!(
                    error = %e,
                    bind = %addr,
                    "failed to bind metrics HTTP server"
                );
                return;
            }
        };
        if let Err(e) = axum::serve(listener, router).await {
            tracing::error!(error = %e, "metrics HTTP server error");
        }
    })
}
```

## Notes

- `axum::serve` in axum 0.8 takes a `TcpListener` and a router. The exact API may vary; check the axum version documentation at implementation time.
- The background task is fully detached (`tokio::spawn`). The coordinator does not need to await it. The `JoinHandle` is returned for optional graceful shutdown support.
- The server binds to the same address as the main coordinator (respecting `--bind`, R24) but on the metrics port. In practice this is usually `0.0.0.0:9090`.
- If the port is already in use, the server fails gracefully with a log message. The coordinator continues to operate without metrics exposition. This is intentional: metrics are observability, not critical functionality.
