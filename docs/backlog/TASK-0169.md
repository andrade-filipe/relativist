## Context

The `net_summary` function computes statistics about a `Net`: total live agents, wire count, redex count, per-symbol counts (CON, DUP, ERA), free port count, and whether the net is in normal form (zero redexes). These statistics are used by the `inspect` subcommand (R29-R31) and as the `input`/`output` snapshots in `ReductionSummary`.

SPEC-12 Section 4.4 provides the computation algorithm. The wire count is the number of bidirectional connections divided by 2. The redex count requires filtering stale entries from the redex queue (per SPEC-02, R17).

## Acceptance Criteria

- [ ] `net_summary(net: &Net) -> NetSummary` computes all 8 fields correctly
- [ ] `agents` field counts live agents (`agents.iter().filter(Option::is_some).count()`)
- [ ] `con`, `dup`, `era` fields count agents by symbol type
- [ ] `wires` field counts the number of distinct agent-to-agent connections: pairs (A, B) where both A and B are `AgentPort` entries, `ports[A] == B`, and `A < B` by canonical ordering `(id * 3 + port)`. FreePort connections are NOT counted as wires (R29 v2)
- [ ] Wire count iterates only over ports `0..=arity(agent.symbol)` for each live agent, consistent with R61
- [ ] `redexes` field counts only valid (non-stale) entries in the redex queue
- [ ] `free_ports` field counts ports connected to `PortRef::FreePort(_)` (iterated over ports `0..=arity(agent.symbol)` per R61)
- [ ] `normal_form` is `true` when `redexes == 0`
- [ ] `format_summary_text(summary: &NetSummary) -> String` produces the human-readable format from R31
- [ ] `format_summary_json(summary: &NetSummary) -> String` produces the JSON format from R30

## Notes

- Counting wires is non-trivial: each bidirectional connection is represented by two entries in the port array (A->B and B->A). To count unique connections, iterate ports and count only where the "from" endpoint is canonically less than the "to" endpoint (same canonical ordering as the text DSL serializer in TASK-0166).
- Stale redex filtering: a redex `(a, b)` is stale if either `agents[a]` or `agents[b]` is `None`, or if the agents' principal ports no longer point to each other. The same check exists in `is_valid_redex` from TASK-0014.
- `format_summary_json` can use `serde_json::to_string_pretty` since `NetSummary` derives `serde::Serialize`. If `serde_json` is not available (TASK-0167 deferred), a manual JSON formatter is needed -- but this is unlikely since the dependency is small.
