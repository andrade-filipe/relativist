## Context

The `/metrics` endpoint serves Prometheus text exposition format (OpenMetrics) by encoding the `prometheus_client::registry::Registry` into a string response. This is the scrape target that Prometheus servers pull metrics from. The Content-Type must be `application/openmetrics-text` as required by R21.

This task implements the metrics handler function that serializes the registry and returns it as an HTTP response.

## Acceptance Criteria

- [ ] `metrics_handler()` async function encodes the registry into OpenMetrics text format
- [ ] Response Content-Type is `application/openmetrics-text; version=1.0.0; charset=utf-8`
- [ ] Response includes all registered metrics with `relativist_` prefix
- [ ] Response is a valid Prometheus text exposition format

## Key Types / Signatures

```rust
use axum::{extract::State, http::header, response::IntoResponse};

/// Prometheus metrics scrape endpoint (R21).
///
/// Encodes the entire registry into OpenMetrics text exposition format.
/// Content-Type: application/openmetrics-text
#[cfg(feature = "metrics")]
pub async fn metrics_handler(
    State(state): State<AppState>,
) -> impl IntoResponse {
    let mut buf = String::new();
    prometheus_client::encoding::text::encode(&mut buf, &state.registry)
        .expect("encoding metrics should not fail");

    (
        [(
            header::CONTENT_TYPE,
            "application/openmetrics-text; version=1.0.0; charset=utf-8",
        )],
        buf,
    )
}
```

## Notes

- `prometheus_client::encoding::text::encode()` writes OpenMetrics format, which is a superset of the classic Prometheus exposition format. Modern Prometheus servers (>= 2.5) support OpenMetrics natively.
- The `expect()` on encoding is acceptable because encoding to a String should never fail unless there is a bug in the prometheus-client crate.
- For large registries with many metrics, encoding is done synchronously in the handler. This is acceptable for v1 since the registry is small (< 20 metrics). If performance becomes an issue, consider caching the encoded output with a TTL.
