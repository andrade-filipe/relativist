## Context

SPEC-03 Section 4.4 defines pair normalization: before calling an interaction function, the agent pair `(a, b)` is reordered so that `a.symbol <= b.symbol` (using `Symbol` ordering: Con=0 < Dup=1 < Era=2). This guarantees that `interact_comm` always receives `(Con, Dup)` (never `(Dup, Con)`), `interact_eras` always receives `(X, Era)` where X is Con or Dup, and `interact_anni` always receives agents of the same symbol. Without normalization, each rule function would need an additional branch to determine which argument is which.

## Acceptance Criteria

- [ ] `normalize_pair` function implemented with signature `(AgentId, AgentId, &Net) -> (AgentId, AgentId)`
- [ ] For a pair where `sym_a > sym_b`, the IDs are swapped
- [ ] For a pair where `sym_a <= sym_b`, the IDs are returned unchanged
- [ ] Function is `#[inline]` for performance (O(1) -- two agent lookups + comparison)

## Key Types / Signatures

```rust
/// Normalizes the agent pair for dispatch (SPEC-03 Section 4.4).
/// Guarantees a.symbol <= b.symbol (by Symbol ordering: Con=0, Dup=1, Era=2).
/// Swaps arguments if necessary.
///
/// This allows:
/// - interact_anni to always receive (Con, Con) or (Dup, Dup), never inverted
/// - interact_comm to always receive (Con, Dup), never (Dup, Con)
/// - interact_eras to always receive (X, Era) where X is Con or Dup
/// - interact_void to receive (Era, Era)
#[inline]
pub fn normalize_pair(a: AgentId, b: AgentId, net: &Net) -> (AgentId, AgentId) {
    let sym_a = net.agents[a as usize].unwrap().symbol;
    let sym_b = net.agents[b as usize].unwrap().symbol;
    if (sym_a as u8) <= (sym_b as u8) {
        (a, b)
    } else {
        (b, a)
    }
}
```
