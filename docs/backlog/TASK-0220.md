## Context

SPEC-04 v3 added Section 3.6 (Root Port Propagation) with requirement R28. During split, the original net's `root` field must be propagated to exactly one partition. The root identifies the "output" of the net and must not be lost or duplicated across partitions. This requirement was identified during adversarial review (SC-010) as a gap in SPEC-04 v2.

## Acceptance Criteria

- [ ] Function `propagate_root` determines which partition inherits the root
- [ ] If `net.root` is `Some(AgentPort(id, p))`, only the partition containing agent `id` gets `subnet.root = net.root`; all other partitions get `subnet.root = None`
- [ ] If `net.root` is `None`, all partitions get `subnet.root = None`
- [ ] If `net.root` is `Some(FreePort(f))`, the root is preserved in the partition inheriting the interface wire (the partition containing the agent connected to `FreePort(f)`, if any); all other partitions get `None`
- [ ] The function is called during Step 5 of the split algorithm (TASK-0049)
- [ ] Exactly one partition has a non-None root (or zero, if net.root was None)

## Notes

- For the `FreePort(f)` case, the function needs to scan the original net's port array to find which agent is connected to `FreePort(f)`, then check if that agent belongs to this partition. This is an uncommon case (nets rarely have a FreePort as root).
- The `agent_ids` parameter may need to be a HashSet for O(1) lookup if the partition is large. For TCC scope (tens of thousands of agents), a linear search on the slice is acceptable.
- This function is pure and deterministic (R4, R5).
