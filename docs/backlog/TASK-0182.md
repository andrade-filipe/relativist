## Context

SPEC-09 Section 3.1 (R2) defines the `Benchmark` trait that all 9 benchmark workloads must implement. The trait is the core abstraction of the benchmark framework: each benchmark is a parametric net generator with a verifier. The trait defines 5 methods: `id`, `describe`, `make_net`, `default_sizes`, and `verify`.

The `verify` method is the hook for the Fundamental Property (G1) verification. Each benchmark implements its own correctness criterion: empty nets check agent count == 0; non-trivial results use graph isomorphism.

## Key Types / Signatures

```rust
use crate::net::Net;

/// Definition of a parametric benchmark (SPEC-09 R2).
///
/// Each benchmark generates input nets of varying sizes, executes
/// sequential and distributed reductions, and verifies correctness
/// of the distributed result against the sequential baseline.
pub trait Benchmark {
    /// Unique identifier of the benchmark.
    fn id(&self) -> BenchmarkId;

    /// Human-readable description for a given size.
    fn describe(&self, size: u32) -> String;

    /// Generate the input net for the specified size.
    ///
    /// The size parameter semantics are benchmark-specific:
    /// - EP benchmarks: number of pairs (N)
    /// - DualTree: depth (d)
    /// - TreeSum: number of items (N)
    /// - MixedNet: pairs per rule type (N)
    /// - ErasurePropagation: chain length (N)
    fn make_net(&self, size: u32) -> Net;

    /// Default sizes for this benchmark (logarithmic variation, R24).
    fn default_sizes(&self) -> Vec<u32>;

    /// Verify correctness: compare distributed result with sequential result.
    ///
    /// Returns true if the distributed result is correct.
    /// For benchmarks with empty normal forms: checks agent count == 0.
    /// For benchmarks with non-trivial results: uses graph isomorphism
    /// (SPEC-08, nets_isomorphic) or benchmark-specific metric.
    ///
    /// This is the Fundamental Property (G1) verification hook (SPEC-09 R4).
    fn verify(&self, sequential_result: &Net, distributed_result: &Net) -> bool;
}
```

## Notes

- The trait uses `&self` (not `self`) so that benchmarks can be stored in a `Vec<Box<dyn Benchmark>>` and reused across configurations.
- The `verify` method takes two `&Net` references: one from sequential `reduce_all`, one from distributed `run_grid`. The sequential result is the "ground truth."
- For benchmarks where the normal form is the empty net (0 agents), verification is trivial. For TreeSum, MixedNet, and ErasurePropagation, verification requires `nets_isomorphic` (TASK-0185).
