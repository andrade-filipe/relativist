## Context

Two more Profile A annihilation generators: CON-CON and DUP-DUP pairs. Both follow the same pattern as `ep_annihilation` but with non-ERA agents that have auxiliary ports connected to free ports.

`ep_annihilation_con(n)` produces N pairs of CON agents connected at principal ports, with auxiliary ports connected to free ports (R38). After reduction: 0 CON agents, free ports cross-reconnected.

`ep_annihilation_dup(n)` is the same pattern but with DUP agents. SPEC-12 R33 lists it as a separate `ExampleNet` variant (SPEC-09 R11).

## Acceptance Criteria

- [ ] `ep_annihilation_con(n: u32) -> Net` generates N CON-CON pairs as specified in R38
- [ ] Each CON pair has principals connected and 4 auxiliary ports connected to free ports
- [ ] Free port IDs are assigned sequentially: 0, 1, 2, 3 for the first pair, 4, 5, 6, 7 for the second, etc.
- [ ] The generated net has `2 * n` agents, `n` redexes, and `4 * n` free ports
- [ ] `ep_annihilation_dup(n: u32) -> Net` generates N DUP-DUP pairs (same structure, `Symbol::Dup`)
- [ ] DUP pair net has `2 * n` agents, `n` redexes, and `4 * n` free ports
- [ ] Both generators validate output in debug mode (R34)

/// Generate N CON-CON annihilation pairs (SPEC-12, R38).
///
/// Each pair: two CON agents, principals connected, auxiliaries to free ports.
/// After reduction: 0 CON agents, free ports cross-reconnected.
pub fn ep_annihilation_con(n: u32) -> Net {
    let mut net = Net::new();
    let mut free_id: u32 = 0;
    for _ in 0..n {
        let a = net.create_agent(Symbol::Con);
        let b = net.create_agent(Symbol::Con);
        net.connect(PortRef::AgentPort(a, 0), PortRef::AgentPort(b, 0));
        net.connect(PortRef::AgentPort(a, 1), PortRef::FreePort(free_id));
        free_id += 1;
        net.connect(PortRef::AgentPort(a, 2), PortRef::FreePort(free_id));
        free_id += 1;
        net.connect(PortRef::AgentPort(b, 1), PortRef::FreePort(free_id));
        free_id += 1;
        net.connect(PortRef::AgentPort(b, 2), PortRef::FreePort(free_id));
        free_id += 1;
    }
    net
}

## Notes

- The two generators are nearly identical except for the `Symbol` variant. A shared helper `fn ep_annihilation_symbol(n: u32, symbol: Symbol) -> Net` could be used internally to avoid duplication, but this is an implementation choice, not a requirement.
- After annihilation reduction, CON-CON pairs cross-reconnect auxiliary ports: `a.left <-> b.left` and `a.right <-> b.right` become direct free-to-free connections. DUP-DUP behaves the same way.
- The spec code in R38 uses explicit `free_id` incrementing. Follow this pattern for clarity.
