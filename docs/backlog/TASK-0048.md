# TASK-0048: split() trivial case (num_workers=1)

**Spec:** SPEC-04
**Requirements:** R1, R2, R4, R5
**Priority:** P0 (critical path)
**Status:** TODO
**Depends on:** TASK-0042 (PartitionPlan), TASK-0041 (Partition), TASK-0040 (IdRange), SPEC-02 types (Net)

**Estimated complexity:** S (< 50 LoC)

## Context

When `num_workers <= 1`, the split function must return the entire net as a single partition with no borders. This is the trivial case (R2) and should execute in O(1) modulo cloning the net. It is Step 1 of the split algorithm (Section 4.5). Implementing this first provides a working `split` function that can be tested end-to-end before the complex multi-partition logic is added.

## Acceptance Criteria

- [ ] `split()` function signature matches the spec: `fn split(net: &Net, num_workers: u32, strategy: &dyn PartitionStrategy) -> PartitionPlan`
- [ ] When `num_workers <= 1`, returns a PartitionPlan with exactly 1 partition
- [ ] The single partition contains a clone of the entire net
- [ ] The border map is empty
- [ ] The `id_range` covers the entire `u32` space: `[0, u32::MAX]`
- [ ] The `free_port_index` is empty (no borders)
- [ ] The `worker_id` is 0
- [ ] The strategy's `allocate()` is NOT called in the trivial case
