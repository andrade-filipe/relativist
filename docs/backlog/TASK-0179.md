## Context

SPEC-12 Section 7 defines 14 test requirements (T1-T14) that validate the I/O layer end-to-end.

> **Revision note (SPEC-12 v2):** Tests T10-T14 were added: T10 (size zero for all generators), T11 (root declaration edge cases), T12 (self-loop rejection), T13 (free-to-free rejection), T14 (empty net handling). This task implements integration tests that exercise the full pipeline: generation -> serialization -> deserialization -> reduction -> summary.

These tests are integration-level: they test the interaction between multiple modules (generators, formats, reduction engine, summary) rather than individual functions. They serve as the acceptance tests for Phase 9.

Individual unit tests are defined within each task. This task adds the cross-cutting integration tests that verify roundtrips and consistency.

## Acceptance Criteria

- [ ] **T1 - Binary roundtrip:** For each generator, `deserialize_binary(serialize_binary(generate(n)))` produces a structurally equivalent net, for N in {1, 10, 100}
- [ ] **T2 - Text DSL roundtrip:** For each generator, `parse_ic(format_ic(generate(n)))` produces a structurally equivalent net, for N in {1, 5, 10}
- [ ] **T3 - Text DSL error handling:** Parser rejects malformed inputs with descriptive errors:
  - Missing wire endpoint
  - Unknown agent name
  - ERA with auxiliary port
  - Duplicate port connection
- [ ] **T4 - Generator validity:** Each generator produces a valid net (invariants T1-T7) for N in {1, 10, 100, 1000}
- [ ] **T5 - Inspect correctness:** For `ep_annihilation(10)`, `net_summary` reports: 20 agents, 10 redexes, 0 CON, 0 DUP, 20 ERA, 0 free ports, not normal form
- [ ] **T6 - Reduce correctness:** For `ep_annihilation(10)`, after `reduce_all`, the output net has 0 agents and 0 redexes (Normal Form)
- [ ] **T7 - Max-interactions:** For `ep_annihilation(10)` with max_interactions=5, reduction stops early and the output is NOT in Normal Form
- [ ] **T8 - Format detection:** `.bin`, `.ic`, `.json` map to correct formats; `.xyz` returns `UnrecognizedFormat`
- [ ] **T9 - Generator consistency:** For each `ExampleNet` variant, `generate_example(variant, n)` produces the same net structure regardless of call site (CLI or benchmark)
- [ ] **T10 - Size zero:** For each generator, `generate(0)` MUST produce an empty net (0 agents, 0 redexes) and `reduce` on that net MUST yield 0 interactions (already in Normal Form) (R60)
- [ ] **T11 - Root declarations:** (a) two `root` declarations produce parse error; (b) no `root` means `root == None`; (c) `root free(0)` sets `net.root = Some(FreePort(0))` (R54-R57)
- [ ] **T12 - Self-loop:** `wire a.left a.left` produces parse error "port cannot be connected to itself" (R58)
- [ ] **T13 - Free-to-free:** `wire free(0) free(1)` produces parse error "free-to-free wires are not supported" (R59)
- [ ] **T14 - Empty net:** `inspect` on empty net reports (0 agents, 0 wires, 0 redexes, normal_form: true); `reduce` on empty net completes with 0 interactions (R60)

## Key Types / Signatures

```rust
// tests/io_integration.rs

use relativist::io::*;
use relativist::io::examples::*;
use relativist::io::binary::*;
use relativist::io::text::*;
use relativist::net::Net;

/// T1: Binary roundtrip for all generators.
#[test]
fn binary_roundtrip_ep_annihilation() {
    for n in [1, 10, 100] {
        let net = ep_annihilation(n);
        let bytes = serialize_binary(&net).unwrap();
        let restored = deserialize_binary(&bytes).unwrap();
        assert_eq!(net, restored);
    }
}

/// T2: Text DSL roundtrip for all generators.
#[test]
fn text_dsl_roundtrip_ep_annihilation() {
    for n in [1, 5, 10] {
        let net = ep_annihilation(n);
        let text = format_ic(&net);
        let restored = parse_ic(&text).unwrap();
        assert_eq!(net, restored);
    }
}

/// T3: Text DSL error handling.
#[test]
fn text_dsl_rejects_era_auxiliary() {
    let input = "agent e ERA\nwire e.left free(0)";
    assert!(parse_ic(input).is_err());
}

/// T5: Inspect correctness.
#[test]
fn inspect_ep_annihilation_10() {
    let net = ep_annihilation(10);
    let summary = net_summary(&net);
    assert_eq!(summary.agents, 20);
    assert_eq!(summary.redexes, 10);
    assert_eq!(summary.con, 0);
    assert_eq!(summary.dup, 0);
    assert_eq!(summary.era, 20);
    assert_eq!(summary.free_ports, 0);
    assert!(!summary.normal_form);
}

/// T6: Reduce correctness.
#[test]
fn reduce_ep_annihilation_10() {
    let mut net = ep_annihilation(10);
    let stats = reduce_all(&mut net);
    let summary = net_summary(&net);
    assert_eq!(summary.agents, 0);
    assert_eq!(summary.redexes, 0);
    assert!(summary.normal_form);
}

/// T7: Max-interactions limit.
#[test]
fn reduce_with_max_interactions() {
    let mut net = ep_annihilation(10);
    let stats = reduce_n(&mut net, 5);
    let summary = net_summary(&net);
    assert!(!summary.normal_form);
}

/// T8: Format detection.
#[test]
fn format_detection() {
    use std::path::Path;
    assert!(matches!(detect_format(Path::new("foo.bin")), Ok(NetFormat::Bin)));
    assert!(matches!(detect_format(Path::new("foo.ic")), Ok(NetFormat::Ic)));
    assert!(matches!(detect_format(Path::new("foo.json")), Ok(NetFormat::Json)));
    assert!(detect_format(Path::new("foo.xyz")).is_err());
}
```

## Notes

- This is marked as complexity L because it spans many generators and formats. However, each individual test is small (< 10 lines). The total may exceed 200 LoC when all 12 generators are tested across all formats.
- T6 and T7 depend on Phase 2 (reduction engine). If Phase 2 is not complete, these tests should be marked `#[ignore]` with a comment explaining the dependency.
- T9 (generator consistency) verifies that the `generate_example` dispatcher produces the same net as calling the individual generator function directly. This is a sanity check for the dispatch table.
- Text DSL roundtrip (T2) may not produce byte-identical text (e.g., different whitespace or comment ordering), but the parsed nets must be structurally equivalent via `PartialEq`.
- For large N (T4 with N=1000), the tests should complete in reasonable time (< 1 second). If a generator is slow for large N, optimize it or reduce the test size.
