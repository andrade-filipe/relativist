## Context

`run_worker` is the top-level function for a worker process. It connects to the coordinator (via `connect_with_retry`), then enters a loop: receive `AssignPartition` -> reduce locally -> rebuild `free_port_index` -> send `PartitionResult`. The loop exits when a `Shutdown` message is received. This implements the worker FSM from SPEC-13 R24: Init -> Idle -> Reducing -> Returning -> Idle -> ... -> Done. (R27 is historical, superseded by SPEC-13.)

> **SPEC-06 v3 changes:** WorkerRoundStats now has 6 fields (added `reduce_duration_secs` and `interactions_by_rule` per SPEC-11 Section 4.4). Worker must instrument timing around `reduce_all` and collect per-rule interaction counts.

## Notes

- The worker does NOT know the total number of rounds. It simply loops until receiving Shutdown.
- `rebuild_free_port_index` scans the subnet for FreePort references in the port array and rebuilds the `HashMap<u32, PortRef>`. This is necessary because reduction may change which ports are connected to FreePort sentinels (SPEC-05, Section 4.3).
- The worker logs each round with `tracing::info!` including round number, worker_id, agents_before, agents_after, local_redexes.
- Unlike the coordinator, the worker has no timeouts for receiving messages. It simply waits indefinitely for the next AssignPartition or Shutdown.
