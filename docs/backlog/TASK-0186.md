## Context

SPEC-09 Section 3.5 (R32) requires computing 5 aggregation statistics over repetitions: mean, standard deviation, median, minimum, and maximum. Additionally, the coefficient of variation (CV = std/mean) is needed for the high-variance warning (R34). These are pure mathematical functions with no external dependencies.

The functions are implemented in `src/bench/stats.rs` and operate on `&[f64]` slices, making them reusable for any metric (wall_clock_secs, mips, speedup, efficiency, overhead_ratio).

/// Arithmetic mean of a slice of f64 values.
/// Returns 0.0 for empty slices.
pub fn mean(values: &[f64]) -> f64 {
    if values.is_empty() {
        return 0.0;
    }
    values.iter().sum::<f64>() / values.len() as f64
}

/// Sample standard deviation (Bessel's correction, N-1).
/// Returns 0.0 for slices with fewer than 2 elements.
pub fn std_dev(values: &[f64]) -> f64 {
    if values.len() < 2 {
        return 0.0;
    }
    let m = mean(values);
    let variance = values.iter()
        .map(|v| (v - m).powi(2))
        .sum::<f64>() / (values.len() - 1) as f64;
    variance.sqrt()
}

/// Median of a slice of f64 values.
/// Returns 0.0 for empty slices.
pub fn median(values: &[f64]) -> f64 {
    if values.is_empty() {
        return 0.0;
    }
    let mut sorted = values.to_vec();
    sorted.sort_by(|a, b| a.partial_cmp(b).unwrap_or(std::cmp::Ordering::Equal));
    let mid = sorted.len() / 2;
    if sorted.len() % 2 == 0 {
        (sorted[mid - 1] + sorted[mid]) / 2.0
    } else {
        sorted[mid]
    }
}

/// Minimum value in a slice. Returns f64::INFINITY for empty slices.
pub fn min_f64(values: &[f64]) -> f64 { /* ... */ }

/// Maximum value in a slice. Returns f64::NEG_INFINITY for empty slices.
pub fn max_f64(values: &[f64]) -> f64 { /* ... */ }

/// Coefficient of variation: std_dev / mean.
/// Returns 0.0 if mean is zero (avoids division by zero).
pub fn coeff_of_variation(values: &[f64]) -> f64 {
    let m = mean(values);
    if m == 0.0 {
        return 0.0;
    }
    std_dev(values) / m
}

## Notes

- Using sample standard deviation (N-1 denominator, Bessel's correction) rather than population standard deviation (N denominator). This is the standard choice for experimental measurements where the sample is a subset of possible executions.
- `median` sorts a copy of the input, which is O(n log n). For the typical use case (5-30 repetitions), this is negligible.
- These functions do NOT use the `statistical` crate or similar -- keeping dependencies minimal for a simple set of calculations.
