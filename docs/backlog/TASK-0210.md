## Context

SPEC-14 R22-R25 define a `compute` subcommand for the Relativist CLI that combines encoding, reduction, and decoding into a single user-facing workflow. The subcommand accepts an arithmetic operation (add, mul, exp), two operands, and optional flags for distributed mode and output files.

This is essential for the TCC defense: the professor types one command and sees the complete pipeline from arithmetic expression to numeric result.

## Acceptance Criteria

- [ ] `ComputeArgs` struct defined with clap derives per R22
- [ ] `ArithmeticOp` enum defined with Add, Mul, Exp variants per R22
- [ ] `Compute(ComputeArgs)` variant added to main CLI enum
- [ ] Without `--workers`: uses local `reduce_all` (R23)
- [ ] With `--workers N`: uses distributed grid with N workers (R23) -- may delegate to Phase 5/6
- [ ] After reduction, decodes result with `decode_nat` and prints summary to stdout (R24)
- [ ] Local mode output matches the format in R24 (expression, encoding stats, reduction stats, result)
- [ ] Distributed mode output includes workers, rounds, speedup (R24)
- [ ] If `decode_nat` returns `None`, prints warning message per R25
- [ ] `--output` flag writes reduced net to file (R22)
- [ ] `--metrics` flag writes metrics JSON to file (R22)

## Files to Create/Modify

- `src/config/cli.rs` or equivalent -- **modify**: add `ComputeArgs`, `ArithmeticOp`, `Compute` variant
- `src/config/commands.rs` or equivalent -- **create/modify**: implement `run_compute_command`

## Key Types / Signatures

```rust
#[derive(Debug, clap::Args)]
pub struct ComputeArgs {
    #[arg(value_enum)]
    pub operation: ArithmeticOp,
    pub a: u64,
    pub b: u64,
    #[arg(long)]
    pub workers: Option<u32>,
    #[arg(long)]
    pub output: Option<PathBuf>,
    #[arg(long)]
    pub metrics: Option<PathBuf>,
}

#[derive(Debug, Clone, Copy, clap::ValueEnum)]
pub enum ArithmeticOp {
    Add,
    Mul,
    Exp,
}
```
