## Context

SPEC-09 Section 3.2.3 (R13) defines the **DualTree** benchmark: two perfect binary trees of CON agents with depth d, connected at the roots. Reduction cascades level by level with cross-connect. This is Profile C (sequential dependency) -- the worst case for distribution.

The DualTree is the maximally adversarial workload: each level can only be reduced after the previous level completes, creating inherent sequential dependency. The border ratio per level is ~50% (cross-connect butterfly pattern). The Haskell prototype measures 0.18x speedup with 4 workers (i.e., 5.5x slowdown). This benchmark demonstrates that strong confluence guarantees correctness even when distribution produces slowdown.

The net generator uses an array-based binary tree layout: node i has children at 2i+1 and 2i+2. Two trees of size 2^d - 1 are created and connected root-to-root.

Default sizes (depth): [4, 6, 8, 10, 12, 14]. Haskell comparison sizes: depth 8, 10, 12, 14 (R43).

```rust
use crate::bench::{Benchmark, BenchmarkId};
use crate::net::{Net, Symbol, PortRef};

/// DualTree: two perfect binary trees of CON connected at roots.
/// Profile C (sequential dependency). Cascading level-by-level reduction.
/// Rules exercised: CON-CON (annihilation cross, cascade).
/// Expected result: 0 agents.
///
/// SPEC-09 R13. Direct mapping from AC-005 `mkDualTreeNet`.
/// Maximally adversarial workload for distribution.
pub struct DualTree;

impl Benchmark for DualTree {
    fn id(&self) -> BenchmarkId {
        BenchmarkId::DualTree
    }

    fn describe(&self, depth: u32) -> String {
        let tree_size = (1u64 << depth) - 1;
        format!("DualTree: depth {}, {} agents, {} interactions",
                depth, tree_size * 2, tree_size)
    }

    fn make_net(&self, depth: u32) -> Net {
        if depth == 0 {
            return Net::new();
        }
        let tree_size = ((1u64 << depth) - 1) as usize;
        let total_agents = tree_size * 2;
        let mut net = Net::with_capacity(total_agents);

        // Create all agents for both trees
        let mut tree_a_ids = Vec::with_capacity(tree_size);
        let mut tree_b_ids = Vec::with_capacity(tree_size);

        for _ in 0..tree_size {
            tree_a_ids.push(net.create_agent(Symbol::Con));
        }
        for _ in 0..tree_size {
            tree_b_ids.push(net.create_agent(Symbol::Con));
        }

        // Helper: connect internal tree wires (array-based layout)
        let mut free_port_counter = 0u32;
        for tree_ids in [&tree_a_ids, &tree_b_ids] {
            for i in 0..tree_size {
                let left_child = 2 * i + 1;
                let right_child = 2 * i + 2;
                if left_child < tree_size {
                    // port 1 -> left child's port 0
                    net.connect(
                        PortRef::AgentPort(tree_ids[i], 1),
                        PortRef::AgentPort(tree_ids[left_child], 0),
                    );
                } else {
                    // Leaf: port 1 -> FreePort
                    net.connect(
                        PortRef::AgentPort(tree_ids[i], 1),
                        PortRef::FreePort(free_port_counter),
                    );
                    free_port_counter += 1;
                }
                if right_child < tree_size {
                    // port 2 -> right child's port 0
                    net.connect(
                        PortRef::AgentPort(tree_ids[i], 2),
                        PortRef::AgentPort(tree_ids[right_child], 0),
                    );
                } else {
                    // Leaf: port 2 -> FreePort
                    net.connect(
                        PortRef::AgentPort(tree_ids[i], 2),
                        PortRef::FreePort(free_port_counter),
                    );
                    free_port_counter += 1;
                }
            }
        }

        // Connect roots: tree_a[0].port0 <-> tree_b[0].port0
        net.connect(
            PortRef::AgentPort(tree_a_ids[0], 0),
            PortRef::AgentPort(tree_b_ids[0], 0),
        );

        net
    }

    fn default_sizes(&self) -> Vec<u32> {
        vec![4, 6, 8, 10, 12, 14]
    }

    fn verify(&self, _sequential_result: &Net, distributed_result: &Net) -> bool {
        count_live_agents(distributed_result) == 0
    }
}
```

## Notes

- The array-based binary tree layout (node i -> children 2i+1, 2i+2) is standard for complete binary trees. This matches the Haskell prototype's `mkDualTreeNet`.
- The inner node wiring uses port 1 for left child and port 2 for right child. This is consistent with the CON agent's auxiliary port semantics.
- For depth 14: tree_size = 16383, total agents = 32766. This is within memory limits but produces a large net.
- The reduction cascades level by level: the root-root CON-CON interaction exposes redexes at depth 1, which expose redexes at depth 2, etc. This creates inherent sequential dependency: exactly `d` rounds minimum in the grid loop.
