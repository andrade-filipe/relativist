## Context

SPEC-03 Section 4.1.3 defines the simplest of the 6 interaction rules: ERA-ERA (Void). When two ERA agents meet through their principal ports, both are simply removed. No new agents are created and no reconnections are needed because ERA has arity 0 (no auxiliary ports). This is the first rule to implement because it has zero dependencies on reconnection logic.

Agent balance: -2 (2 destroyed, 0 created). Link calls: 0.

## Key Types / Signatures

```rust
/// Void: two ERA agents annihilate without creating anything (SPEC-03 Section 4.1.3).
///
/// Precondition: both a_id and b_id are live Era agents connected port0<->port0.
/// Postcondition: both removed. No agents created, no reconnections.
///
/// Agent balance: -2. Link calls: 0.
/// Complexity: O(1).
///
/// Invariants preserved: T1 (ERA has no auxiliary ports, so removing them
/// leaves no dangling ports), I1/I2 (remove_agent cleans up port array slots).
pub fn interact_void(net: &mut Net, a_id: AgentId, b_id: AgentId) {
    net.remove_agent(a_id);
    net.remove_agent(b_id);
}
```

## Notes

- This is the simplest rule in the system. In HVM2 (AC-007), it is implemented as `interact_void` which returns `true` without node operations because ERA does not occupy a node slot there. In Relativist, agents occupy arena slots and MUST be explicitly removed.
- No auxiliary ports means no reconnection, no new redex detection, and no possibility of cascading.
