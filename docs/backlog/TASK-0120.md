# TASK-0120: Convert security module to directory structure

**Spec:** SPEC-10
**Requirements:** (structural, supports all SPEC-10 requirements)
**Priority:** P0 (critical path)
**Status:** TODO
**Depends on:** none
**Blocked by:** none
**Estimated complexity:** S (< 50 LoC)

## Context

The `security` module is currently a single file (`src/security.rs`) with only a doc comment. SPEC-10 Section 4.1 defines a three-file module structure: `mod.rs` (SecurityConfig, tier detection, re-exports), `token.rs` (AuthToken, generation, validation), and `tls.rs` (TlsServerConfig, TlsClientConfig, feature-gated). This task converts the flat file into a directory module, matching the pattern used by the protocol module (TASK-0080).

## Acceptance Criteria

- [ ] `src/security.rs` is replaced by `src/security/mod.rs`
- [ ] `src/security/token.rs` is created with a module doc comment placeholder
- [ ] `src/security/tls.rs` is created with a module doc comment placeholder and `#[cfg(feature = "tls")]` at the module level
- [ ] `mod.rs` declares `pub mod token;` and `#[cfg(feature = "tls")] pub mod tls;`
- [ ] `lib.rs` still has `pub mod security;` (no change needed)
- [ ] Project compiles with `cargo check` and `cargo check --features tls`

## Files to Create/Modify

- `src/security.rs` -- **delete** (replaced by directory)
- `src/security/mod.rs` -- **create**: module root with re-exports
- `src/security/token.rs` -- **create**: placeholder for AuthToken
- `src/security/tls.rs` -- **create**: placeholder for TLS types (feature-gated)
