## Context

SPEC-07 Section 4.9 defines the `generate` subcommand, which creates benchmark networks from pre-defined workloads and saves them as `.bin` files. The workload registry maps names ("tree-sum", "tree-sum-balanced", "era-chain", "con-dup-expansion", "dual-tree") to generator functions that produce deterministic `Net` instances for a given size. The actual generator functions are defined in Phase 10 (SPEC-09 benchmarks). This task implements the command entry point and the workload dispatch table (with stubs for generators not yet implemented).

## Acceptance Criteria

- [ ] `run_generate_command(args: GenerateArgs) -> Result<(), RelativistError>` is implemented (synchronous)
- [ ] Dispatches on `args.workload` name to the correct generator function
- [ ] Calls `generator(args.size)` to produce a `Net`
- [ ] Serializes the net via `save_net_to_file` (TASK-0104)

```rust
use crate::config::GenerateArgs;
use crate::io::{get_workload, save_net_to_file};
use crate::error::RelativistError;

/// Execute generate mode: create a benchmark network and save to file.
pub fn run_generate_command(args: GenerateArgs) -> Result<(), RelativistError> {
    let (generator, _verifier) = get_workload(&args.workload)
        .ok_or_else(|| RelativistError::Config(
            format!("unknown workload '{}'. Available: tree-sum, tree-sum-balanced, era-chain, con-dup-expansion, dual-tree",
                    args.workload)
        ))?;

    let net = generator(args.size);
    tracing::info!(
        workload = %args.workload,
        size = args.size,
        agents = net.agents.iter().filter(|a| a.is_some()).count(),
        "generated network"
    );

    save_net_to_file(&net, &args.output)?;
    Ok(())
}
```

## Notes

- SPEC-07 R33 requires generators to be deterministic: same size always produces same net. This is a property of the generator functions themselves (Phase 10), not of this wiring task.
- SPEC-07 R34 (SHOULD) defines verification functions. The `get_workload` return type includes a verifier, but it is not invoked during generation. It is used during benchmarks (Phase 10).
- Generator stubs may simply create an empty `Net::new()` until the real generators are implemented. This allows the CLI pipeline to be tested end-to-end.
