## Context

SPEC-07 Section 4.9 (Revised v3) defines the `generate` subcommand, which creates benchmark networks from pre-defined workloads and saves them as `.bin` files. Per SPEC-12 R33 (which supersedes SPEC-07 R8), the subcommand uses a positional `example` argument (value_enum) and `-n` for size instead of the old `--workload`/`--size` flags. The workload registry maps names ("tree-sum", "tree-sum-balanced", "ep-annihilation", "con-dup-expansion", "dual-tree", etc.) to generator functions that produce deterministic `Net` instances for a given size. Note: "era-chain" was renamed to "ep-annihilation" per SPEC-09/SPEC-12 alignment. The actual generator functions are defined in Phase 9 (SPEC-12) and Phase 10 (SPEC-09 benchmarks). This task implements the command entry point and the workload dispatch table (with stubs for generators not yet implemented).

//     pub n: u32,               // was --size/-s in v2
//     #[arg(short = 'o', long)]
//     pub output: PathBuf,

/// Execute generate mode: create a benchmark network and save to file.
pub fn run_generate_command(args: GenerateArgs) -> Result<(), RelativistError> {
    let (generator, _verifier) = args.example.get_workload();

    let net = generator(args.n);
    tracing::info!(
        example = %args.example,
        n = args.n,
        agents = net.agents.iter().filter(|a| a.is_some()).count(),
        "generated network"
    );

    save_net_to_file(&net, &args.output)?;
    Ok(())
}

## Notes

- SPEC-07 R33 requires generators to be deterministic: same size always produces same net. This is a property of the generator functions themselves (Phase 9/10), not of this wiring task.
- SPEC-07 R34 (SHOULD) defines verification functions. The `get_workload` return type includes a verifier, but it is not invoked during generation. It is used during benchmarks (Phase 10).
- Generator stubs may simply create an empty `Net::new()` until the real generators are implemented. This allows the CLI pipeline to be tested end-to-end.
- **SPEC-07 v3 change (SC-007, SC-009):** "EraChain"/"era-chain" renamed to "EpAnnihilation"/"ep-annihilation". `GenerateArgs` now uses positional `example` + `-n` instead of `--workload`/`--size`. See SPEC-12 R33 for the authoritative definition.
