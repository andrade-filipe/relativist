## Context

SPEC-09 Section 4.1 defines `BenchmarkSuiteConfig` (the configuration struct for a full benchmark run) and `AggregatedStats` (the statistical summary over repetitions). These two structs control the execution protocol (Section 4.3) and the aggregation output (R40).

`BenchmarkSuiteConfig` maps directly to the CLI parameters (R6): benchmark selection, sizes, workers, modes, warmup, repetitions, CSV paths, and round limits. `AggregatedStats` holds mean/std/median/min/max for the key metrics (R32-R33) plus the CV warning indicator (R34).

```rust
use std::path::PathBuf;

/// Configuration for a benchmark suite run (SPEC-09 R6, Section 4.1).
#[derive(Debug, Clone)]
pub struct BenchmarkSuiteConfig {
    /// Which benchmarks to execute (empty = all).
    pub benchmarks: Vec<BenchmarkId>,
    /// Worker counts to test (default: [1, 2, 4, 8]).
    pub workers: Vec<u32>,
    /// Modes to test (default: [Local]).
    pub modes: Vec<Mode>,
    /// Warmup runs discarded before measurement (default: 2).
    pub warmup: u32,
    /// Timed repetitions per configuration (default: 5, R31).
    pub repetitions: u32,
    /// Path for detail CSV output (one row per repetition).
    pub csv_detail: Option<PathBuf>,
    /// Path for aggregation CSV output (one row per configuration).
    pub csv_summary: Option<PathBuf>,
    /// Grid loop round limit (None = unlimited).
    pub max_rounds: Option<u32>,
}

impl Default for BenchmarkSuiteConfig {
    fn default() -> Self {
        Self {
            benchmarks: Vec::new(), // empty = all
            workers: vec![1, 2, 4, 8],
            modes: vec![Mode::Local],
            warmup: 2,
            repetitions: 5,
            csv_detail: None,
            csv_summary: None,
            max_rounds: None,
        }
    }
}

/// Aggregated statistics over repetitions for one configuration
/// (SPEC-09 R32, R33, R40).
#[derive(Debug, Clone, serde::Serialize)]
pub struct AggregatedStats {
    pub benchmark: BenchmarkId,
    pub input_size: u32,
    pub mode: Mode,
    pub workers: u32,
    pub num_repetitions: u32,
    pub correct_count: u32,

    // Wall-clock time (R33)
    pub time_mean: f64,
    pub time_std: f64,
    pub time_median: f64,
    pub time_min: f64,
    pub time_max: f64,
    pub time_cv: f64,

    // MIPS (R33)
    pub mips_mean: f64,
    pub mips_std: f64,

    // Speedup (R33)
    pub speedup_mean: f64,
    pub speedup_std: f64,

    // Efficiency (R33)
    pub efficiency_mean: f64,
    pub efficiency_std: f64,

    // Overhead ratio (R33)
    pub overhead_ratio_mean: f64,
    pub overhead_ratio_std: f64,

    // Memory
    pub peak_memory_mean: f64,

    // Communication
    pub bytes_sent_mean: f64,
    pub bytes_received_mean: f64,

    // Break-even indicator (R46)
    pub speedup_above_one: bool,

    // High variance warning (R34): true if CV > 0.10
    pub high_variance_warning: bool,
}
```

## Notes

- `BenchmarkSuiteConfig` does NOT derive `serde::Serialize` because it is a runtime configuration struct, not a data output.
- The `csv_detail` and `csv_summary` paths are `Option<PathBuf>`: when `None`, no CSV is written (terminal-only output).
- `AggregatedStats.high_variance_warning` is computed by the aggregation logic (TASK-0197) as `time_cv > 0.10`.
