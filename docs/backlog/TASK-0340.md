## Deliverable

`relativist-core::encoding::recipe` module exporting:

```rust
use serde::{de::DeserializeOwned, Serialize};
use crate::encoding::traits::{Encoder, EncodeError};
use crate::partition::Partition;

/// Optional extension of `Encoder` for codecs that can produce K independent
/// "recipes" — small descriptions a worker can use to generate its own
/// partition locally, instead of receiving a chunk of an already-built net.
///
/// SPEC-27 R24, R25.
///
/// **Not** part of the `Codec` supertrait: codecs that do not implement
/// `RecipeEncoder` fall back to centralized generation (coordinator builds
/// the full net and ships partitions). The registry stores `Box<dyn Codec>`,
/// so `RecipeEncoder` is checked separately (e.g., via `Any` downcast or
/// a parallel registry in a future SPEC-25 integration).
pub trait RecipeEncoder: Encoder {
    /// Recipe payload sent over the wire to each worker.
    /// Must be small (kilobytes), self-contained, and deterministic.
    type Recipe: Serialize + DeserializeOwned + Send + Sync;

    /// Whether this encoder supports recipe-based distributed generation.
    /// `false` means fall back to centralized generation.
    fn is_decomposable(&self) -> bool;

    /// Produce K recipes from the problem description and worker count.
    /// `recipes.len() == num_workers` on success.
    fn make_recipes(
        &self,
        input: &[u8],
        num_workers: u32,
    ) -> Result<Vec<Self::Recipe>, EncodeError>;

    /// Generate a local partition from a single recipe.
    /// Workers call this after receiving an `AssignRecipe` message
    /// (future SPEC-27 R27 integration).
    fn generate_partition(&self, recipe: &Self::Recipe) -> Result<Partition, EncodeError>;
}
```

Plus a demo implementation (`MinimalRecipeEncoder`) for tests, just enough
to exercise the trait. The demo does NOT need to be production-quality;
real recipe encoders will be added in M7 (SPEC-25 implementation).
