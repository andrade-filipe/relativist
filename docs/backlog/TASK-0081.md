## Context

The wire protocol needs a dedicated error type that covers all failure modes: connection loss, payload size violations, CRC32 checksum mismatches, serialization/deserialization failures, unexpected messages, timeouts, and authentication failures. This enum is used by every function in the protocol module. It must be defined first so that all other tasks can reference it.

> **SPEC-06 v3 changes:** The enum was restructured: `Io` renamed to `ConnectionLost`, `PayloadTooLarge.declared` renamed to `size`, `AuthFailed` added (from SPEC-10), and `WorkerError`/`WorkerCountMismatch` moved to `CoordinatorError` (SPEC-13 R16). The canonical enum now has 8 variants (was 8 in v2, but 2 removed and 1 added = 7 protocol-level variants + AuthFailed = 8 total).

## Acceptance Criteria

## Key Types / Signatures

```rust
use std::time::Duration;

/// Possible errors in the wire protocol.
///
/// This is the canonical definition (SPEC-06 Revised v3 Section 4.4).
/// SPEC-13 R16 provides a high-level sketch; this definition is
/// authoritative for field names and types.
#[derive(Debug)]
pub enum ProtocolError {
    /// Connection lost (I/O error in TCP communication).
    /// Named `ConnectionLost` (SPEC-13 convention) rather than `Io` for clarity.
    ConnectionLost(std::io::Error),

    /// Declared payload exceeds the maximum allowed size.
    /// Field types are `u32` consistent with the frame header's `length` field.
    PayloadTooLarge {
        size: u32,
        max: u32,
    },

    /// CRC32 checksum of the payload does not match the header declaration.
    ChecksumMismatch {
        expected: u32,
        computed: u32,
    },

    /// Bincode deserialization error.
    Deserialize(bincode::Error),

    /// Bincode serialization error.
    Serialize(bincode::Error),

    /// Unexpected message for the current FSM state.
    UnexpectedMessage {
        expected: &'static str,
        received: String,
    },

    /// Timeout exceeded in an operation.
    Timeout {
        phase: &'static str,
        elapsed: Duration,
    },

    /// Authentication failed (SPEC-10).
    /// Emitted when a worker's Register message contains an invalid
    /// or missing auth token.
    AuthFailed,
}

// NOTE: WorkerError and WorkerCountMismatch have been moved to
// CoordinatorError (SPEC-13 R16) as they are coordinator-level
// concerns, not wire protocol errors.
```

## Notes

- Consider using `thiserror` derive macro for Display implementations, consistent with the existing `RelError` in `src/error.rs`.
- The `Serialize` and `Deserialize` variants both wrap `bincode::Error`. They are separate for diagnostic clarity (knowing whether the error occurred during serialization or deserialization).
- `ProtocolError` does NOT derive `Clone` because `std::io::Error` and `bincode::Error` are not Clone.
