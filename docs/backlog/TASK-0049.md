## Context

This task implements the general case of `split()` for `num_workers > 1`. It orchestrates Steps 2 through 7 of the algorithm defined in SPEC-04, Section 4.5: compute the allocation function, classify wires, build sub-nets, compute ID ranges, and run debug assertions. This is the glue code that ties together all the helper functions built in previous tasks. It is the main entry point for the multi-worker partitioning logic.

## Key Types / Signatures

```rust
// Inside split(), after the trivial case:

// Step 2: Compute allocation function
let sigma = strategy.allocate(net, num_workers);

// Step 3: Group agents by worker
let worker_agents: Vec<Vec<AgentId>> = group_by_worker(&sigma, num_workers);

// Step 4: Classify wires and generate borders
let classification = classify_wires(net, &sigma, num_workers);

// Step 5: Build sub-nets
let id_ranges = compute_id_ranges(num_workers);
let mut partitions = Vec::with_capacity(num_workers as usize);
for i in 0..num_workers {
    let subnet = build_subnet(net, &worker_agents[i as usize], &sigma, &classification, i);
    let free_port_index = build_free_port_index(&classification.border_entries[i as usize]);
    // Step 6: Initialize next_id
    let max_id = worker_agents[i as usize].iter().copied().max();
    let mut subnet = subnet;
    subnet.next_id = std::cmp::max(
        id_ranges[i as usize].start,
        max_id.map(|m| m + 1).unwrap_or(0),
    );
    partitions.push(Partition {
        subnet,
        worker_id: i,
        free_port_index,
        id_range: id_ranges[i as usize],
    });
}

// Step 7: Debug assertions
#[cfg(debug_assertions)]
{
    assert_c1(net, &partitions);
    assert_c2_c3(&partitions, &classification.borders);
}

PartitionPlan {
    partitions,
    borders: classification.borders,
}
```

## Notes

- `group_by_worker` is a simple utility (iterate sigma, push each AgentId to `worker_agents[sigma[id]]`). It can be inlined in this function or extracted as a tiny helper -- developer's choice.
- The `subnet.next_id` initialization (Step 6) uses `max(id_range.start, max_agent_id + 1)` to ensure pre-existing IDs are not overwritten and new IDs fall within the reserved range.
- This is the integration point. Most of the algorithmic work is in the helper tasks. This task is primarily wiring.
