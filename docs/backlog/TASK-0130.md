## Context

The coordinator needs a TLS server configuration to accept encrypted connections from workers. `TlsServerConfig` wraps a `rustls::ServerConfig` pre-built from PEM-encoded certificate and private key files. It enforces TLS 1.3 exclusively (R22) and uses server-side TLS only (no client certificate requirement, R23). Self-signed certificates are supported because workers verify against a provided CA cert, not the system trust store (R27). The entire type is feature-gated under `tls`.

## Notes

- `rustls::ServerConfig` requires an `Arc<rustls::crypto::CryptoProvider>`. Use `rustls::crypto::ring::default_provider()` or `rustls::crypto::aws_lc_rs::default_provider()` depending on the rustls version and available features.
- The `Debug` implementation intentionally omits the internal config to avoid leaking any private key material in logs or debug output.
- PEM file parsing uses `rustls_pemfile::certs()` for certificates and `rustls_pemfile::private_key()` for the key.
- TLS 1.3 enforcement: `rustls` 0.23+ defaults to TLS 1.3. Verify by checking the `versions` field or using `.with_protocol_versions(&[&rustls::version::TLS13])`.
