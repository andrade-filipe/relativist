## Context

SPEC-19 ¬ß3.1 requires the worker to inspect its `free_port_index` after `reduce_all`
and report a single boolean `has_border_activity` indicating whether any local
border port is a principal port (`AgentPort(_, 0)`). This is the signal the
coordinator uses (in TASK-0351) to decide whether merge can be skipped for the
round.

This task adds the new field to `WorkerRoundStats` (the canonical per-round
worker statistics, SPEC-05 R37) and the pure helper that computes it. The field
crosses the wire as part of the existing `Message::PartitionResult { stats, ‚Ä¶ }`
variant ‚Äî no new `Message` variant is introduced in this bundle (R7: v1 protocol
compatibility).

## Key Types / Signatures

```rust
// merge/types.rs ‚Äî append at end of WorkerRoundStats
pub struct WorkerRoundStats {
    pub worker_id: WorkerId,
    pub agents_before: usize,
    pub agents_after: usize,
    pub local_redexes: usize,
    pub reduce_duration_secs: f64,
    pub interactions_by_rule: [u64; 6],
    /// SPEC-19 R1, R2: true iff at least one entry of
    /// `partition.free_port_index` points at a principal port
    /// (`AgentPort(_, 0)`) after local reduction. Computed by
    /// `merge::helpers::compute_border_activity`.
    pub has_border_activity: bool,
}

// merge/helpers.rs ‚Äî new helper
/// SPEC-19 R1: scans `partition.free_port_index` and returns `true`
/// if at least one local border endpoint is a principal port.
///
/// A border endpoint is "principal" when the `PortRef` stored in
/// `free_port_index` is `AgentPort(_, 0)` ‚Äî i.e. the local side of
/// some border wire could potentially participate in a border redex
/// (pending the remote side's state, which the coordinator tracks).
///
/// Pure / O(|free_port_index|). No allocation.
pub(crate) fn compute_border_activity(partition: &Partition) -> bool {
    partition
        .free_port_index
        .values()
        .any(|p| matches!(p, PortRef::AgentPort(_, 0)))
}
```
