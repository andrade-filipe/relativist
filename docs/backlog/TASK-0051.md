## Context

Each partition's redex queue must contain only Active Pairs that are internal to that partition (both agents belong to the same partition). Active Pairs that cross boundaries (border redexes) must NOT appear in any partition's redex queue (R24). Border redexes will be detected after merge via the `Net::connect` mechanism (SPEC-02, R13) and must not be lost (R25). This task populates each sub-net's redex queue by filtering the original net's redex queue.

## Acceptance Criteria

- [ ] Function populates the redex queue of a sub-net with internal Active Pairs only
- [ ] An Active Pair `(a, b)` is included if and only if both `a` and `b` are in the partition's agent set
- [ ] Border redexes (one agent in this partition, one in another) are excluded
- [ ] Stale redexes (agents already removed) are excluded
- [ ] The function does NOT add new redexes -- it filters the original queue

## Test Expectations

- Unit test: original has redex (a, b), both in partition -> subnet's queue contains (a, b)
- Unit test: original has redex (a, b), a in partition, b NOT -> subnet's queue does NOT contain it
- Unit test: original has stale redex (removed agent) -> subnet's queue excludes it
- Unit test: original has multiple redexes, mixed internal and border -> only internal ones in subnet

## Notes

- The agent set can be passed as a `HashSet<AgentId>` for O(1) membership check, or the sigma map can be used. Using a HashSet is simpler and avoids coupling to WorkerId.
- Complexity: O(Q) where Q is the size of the original redex queue.
- This can be called as a post-step after `build_subnet` or integrated into it. Keeping it separate is cleaner for testing.
