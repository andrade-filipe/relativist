## Context

Static ID space partitioning divides the `u32` space (~4.29 billion IDs) among workers so each can allocate new agent IDs during local reduction without coordination. This eliminates the `remapAllPartitions` phase from the Haskell prototype (AC-003). Each worker `i` receives an exclusive, contiguous range computed as `[i * chunk_size, (i+1) * chunk_size)`, with the last worker extending to `u32::MAX`. This task implements the computation of these ranges.

## Acceptance Criteria

- [ ] Function `compute_id_ranges(num_workers: u32) -> Vec<IdRange>` returns `num_workers` ranges
- [ ] Ranges are disjoint: no overlap between any two ranges
- [ ] Ranges cover the entire `u32` space: first starts at 0, last ends at `u32::MAX`
- [ ] `chunk_size = u32::MAX / num_workers` (integer division)
- [ ] Last worker's range extends to `u32::MAX` (inclusive, meaning `end = u32::MAX`)
- [ ] Works correctly for edge cases: `num_workers = 1` (entire space), `num_workers = u32::MAX`

## Notes

- Careful with integer overflow: `u32::MAX / num_workers` uses integer division. The last worker's range must extend to `u32::MAX` to avoid leaving IDs unassigned.
- For `num_workers = 1`, the range is `[0, u32::MAX]`.
- The `subnet.next_id` initialization (which depends on the maximum AgentId already present in the partition) is handled by the split orchestrator (TASK-0049), not by this function. This function only computes the raw ranges.
