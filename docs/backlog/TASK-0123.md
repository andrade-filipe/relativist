## Context

The `AuthToken` is a 256-bit (32-byte) cryptographically random value used for worker registration authentication. It is generated by the coordinator at startup using a CSPRNG (`rand::rngs::OsRng`), encoded as base64 for human display and file storage, and transmitted as raw bytes in the wire protocol. This task defines the `AuthToken` struct and its construction/serialization methods. Constant-time verification is in a separate task (TASK-0124).

## Acceptance Criteria

- [ ] `AuthToken` struct wraps `[u8; 32]` as a newtype
- [ ] `AuthToken` derives `Clone`, `PartialEq`, `Eq`
- [ ] `AuthToken` implements `Debug` manually to redact the token value (R36: token must not appear in debug output after initial display)
- [ ] `AuthToken::generate()` creates a new token using `rand::rngs::OsRng` and `rand::RngCore::fill_bytes` (R9)
- [ ] `AuthToken::from_base64(s: &str)` decodes a base64 string, returns `Result<Self, SecurityError>` (R10)
- [ ] `AuthToken::from_base64` returns `SecurityError::InvalidBase64` for non-base64 input
- [ ] `AuthToken::from_base64` returns `SecurityError::InvalidTokenLength` if decoded bytes are not exactly 32
- [ ] `AuthToken::to_base64(&self)` encodes the token as standard base64 (44 chars) (R10)
- [ ] `AuthToken::as_bytes(&self)` returns `&[u8; 32]` for wire protocol serialization (R19)
- [ ] Roundtrip: `AuthToken::from_base64(token.to_base64()) == Ok(token)` for any token (T2)
- [ ] `AuthToken` implements `serde::Serialize` and `serde::Deserialize` (for bincode wire format)

use rand::rngs::OsRng;
use rand::RngCore;
use serde::{Serialize, Deserialize};

use super::SecurityError;

/// Authentication token: 256-bit (32-byte) random value (SPEC-10, R9).
///
/// Generated by the coordinator at startup using a CSPRNG.
/// Workers present this token during registration to prove authorization.
/// Per-session lifetime: no rotation, no expiry (R18).
#[derive(Clone, PartialEq, Eq, Serialize, Deserialize)]
pub struct AuthToken([u8; 32]);

impl AuthToken {
    /// Generate a new random token using the OS CSPRNG (R9).
    ///
    /// Uses `rand::rngs::OsRng` which delegates to the operating system's
    /// cryptographically secure random number generator.
    pub fn generate() -> Self {
        let mut bytes = [0u8; 32];
        OsRng.fill_bytes(&mut bytes);
        Self(bytes)
    }

    /// Decode a token from a base64-encoded string (R10).
    ///
    /// Returns `InvalidBase64` if the string is not valid base64,
    /// or `InvalidTokenLength` if the decoded value is not 32 bytes.
    pub fn from_base64(s: &str) -> Result<Self, SecurityError> {
        /* ... */
    }

    /// Encode the token as a base64 string (R10).
    ///
    /// Produces a 44-character string using the standard base64 alphabet.
    pub fn to_base64(&self) -> String {
        /* ... */
    }

    /// Return the raw 32-byte value for wire protocol serialization (R19).
    pub fn as_bytes(&self) -> &[u8; 32] {
        &self.0
    }
}

/// Custom Debug implementation that redacts the token value (R36).
///
/// After initial startup display, the token must never appear in logs,
/// debug output, or error messages.
impl std::fmt::Debug for AuthToken {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        f.debug_tuple("AuthToken").field(&"[REDACTED]").finish()
    }
}

## Notes

- The `Debug` impl is critical for security: `AuthToken` will appear in structs like `SecurityConfig` that may be logged or printed. The redaction ensures the token value never leaks into logs after the one-time startup display (R11, R36).
- Base64 encoding uses the STANDARD alphabet (not URL-safe), producing 44 characters for 32 bytes (`ceil(32/3) * 4 = 44`).
- `serde::Serialize`/`Deserialize` on `AuthToken` serializes the raw `[u8; 32]` via bincode for wire protocol efficiency. The base64 encoding is only for human-facing display and file storage.
