## Context

`init_tracing()` is the single entry point that configures the entire tracing subscriber stack. It must be called exactly once at process startup, before any other tracing events. The function builds a `tracing_subscriber::Registry` with an `EnvFilter` (from TASK-0143) and a `fmt::Layer` whose output format (Text or JSON) is selected by `ObservabilityConfig::log_format` (from TASK-0142).

This task implements the always-on base layer. The optional OTel layer (feature `otel`) is wired in by TASK-0158, which modifies this function to conditionally add the `OpenTelemetryLayer`.

After initialization, the function logs an INFO event confirming the active configuration (R33).

## Key Types / Signatures

```rust
use tracing_subscriber::{fmt, layer::SubscriberExt, util::SubscriberInitExt, Registry};
use super::config::{LogFormat, ObservabilityConfig};

/// Initialize the tracing subscriber with all configured layers (R31).
///
/// MUST be called exactly once at startup, before any tracing events.
/// Panics if called more than once (R32).
///
/// After initialization, logs an INFO event confirming the active
/// configuration (R33).
pub fn init_tracing(config: ObservabilityConfig) {
    let env_filter = build_env_filter();

    match config.log_format {
        LogFormat::Text => {
            let fmt_layer = fmt::layer()
                .with_target(true)
                .with_thread_ids(true)
                .with_file(false)
                .with_line_number(false);

            Registry::default()
                .with(env_filter)
                .with(fmt_layer)
                .init(); // panics on second call
        }
        LogFormat::Json => {
            let fmt_layer = fmt::layer()
                .json()
                .with_target(true)
                .with_thread_ids(true)
                .with_file(false)
                .with_line_number(false);

            Registry::default()
                .with(env_filter)
                .with(fmt_layer)
                .init(); // panics on second call
        }
    }

    tracing::info!(
        log_format = ?config.log_format,
        role = ?config.role,
        metrics_enabled = cfg!(feature = "metrics"),
        otel_enabled = cfg!(feature = "otel"),
        filter = DEFAULT_LOG_FILTER,
        "observability initialized"
    );
}
```

## Notes

- The `match` on `LogFormat` with two separate `Registry` constructions is needed because `fmt::Layer<_, Full>` and `fmt::Layer<_, Json>` are different types in Rust's type system. A `Box<dyn Layer>` alternative exists but adds runtime overhead; the match-based approach is zero-cost.
- The `.init()` method is shorthand for `tracing::subscriber::set_global_default(subscriber).expect(...)`. It panics on a second call, satisfying R32 without additional bookkeeping.
- The R33 confirmation event uses `tracing::info!` with structured fields. This event itself is filtered by the EnvFilter (target `relativist::observability` defaults to INFO, so it will appear).
