## Context

SPEC-14 Section 8 defines 12 arithmetic benchmark scenarios (ARITH-ADD-S/M/L/XL, ARITH-MUL-S/M/L/XL, ARITH-EXP-S/M/L/XL) that test distributed reduction of Church arithmetic nets. These benchmarks must be integrated into the benchmark suite (SPEC-09).

Each benchmark point verifies the Fundamental Property (local result == distributed result) and collects metrics: correctness, speedup, MIPS, overhead ratio, rounds, and peak agents.

## Acceptance Criteria

- [ ] 12 benchmark scenarios defined per Section 8.2 table
- [ ] Each benchmark implements `Benchmark::make_net` using `build_add`, `build_mul`, or `build_exp`
- [ ] Verification property holds for every benchmark point (Section 8.3)
- [ ] Metrics collected per Section 8.4: correctness, speedup, MIPS, overhead ratio, rounds, peak agents
- [ ] Worker configurations match the table: S uses {1,2}, M uses {1,2,4}, L/XL use {1,2,4,8}
- [ ] Benchmarks are runnable via the benchmark suite runner (TASK-0198)

struct ArithAddLarge;

impl Benchmark for ArithAddLarge {
    fn id(&self) -> BenchmarkId { "ARITH-ADD-L" }
    fn make_net(&self) -> Net { build_add(5000, 5000) }
    fn verify(&self, net: &Net) -> bool {
        decode_nat(net) == Some(10000)
    }
    fn worker_counts(&self) -> Vec<u32> { vec![1, 2, 4, 8] }
}
