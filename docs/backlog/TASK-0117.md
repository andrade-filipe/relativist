# TASK-0117: Enforce Core/Infrastructure layer boundary

**Spec:** SPEC-13
**Requirements:** R6, R7, R8 (SPEC-13)
**Priority:** P1 (important but not blocking)
**Status:** TODO
**Depends on:** TASK-0107, TASK-0109, Phase 5
**Blocked by:** none
**Estimated complexity:** S (< 50 LoC)

## Context

SPEC-13 R6-R8 define a strict dependency boundary: Core Layer modules (`net`, `reduction`, `partition`, `merge`, `encoding`) must NOT depend on tokio, async runtimes, or I/O crates. Infrastructure Layer modules (`protocol`, `coordinator`, `worker`) MAY depend on tokio and MUST depend on Core Layer. The reverse dependency (Core -> Infrastructure) is forbidden.

This task audits all `use` statements across modules and adds compile-time guards to ensure the boundary is never violated. It also adds module-level documentation explaining the layer classification.

## Acceptance Criteria

- [ ] Core modules (`net`, `reduction`, `partition`, `merge`, `encoding`) contain NO `use tokio::` or `async fn` signatures
- [ ] No Core module imports from `protocol`, `coordinator`, or `worker`
- [ ] Infrastructure modules (`protocol`, `coordinator`, `worker`) import from Core modules (OK)
- [ ] `config` module may import from both layers (it is a bridge module)
- [ ] `lib.rs` module doc comments classify each module: `// Core Layer` or `// Infrastructure Layer`
- [ ] A CI-friendly check is documented (e.g., `grep -r "use tokio" src/net/ src/reduction/ src/partition/ src/merge/` should return nothing)

## Files to Create/Modify

- `src/lib.rs` -- **modify**: add layer classification comments to each `pub mod` declaration
- Core modules -- **audit**: verify no forbidden imports exist (read-only unless violations found)
- `docs/ARCHITECTURE.md` or inline comments -- **create/modify**: document the boundary rule

## Notes

- This is an audit/documentation task. If violations are found, they must be fixed (likely by moving imports or restructuring).
- The `encoding` module is listed in SPEC-13 R5 and corresponds to SPEC-14 (Revised v2). If TASK-0200 has been completed, the module already exists at `src/encoding/`. Otherwise, this task should add a placeholder `pub mod encoding;` with a stub file.
- The `merge` module calling `reduce_all` from `reduction` is Core -> Core, which is allowed.
- The boundary is enforced by convention and audit, not by Rust's module system (which does not prevent intra-crate circular deps). A future multi-crate workspace would enforce it at compile time.
