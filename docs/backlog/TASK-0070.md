## Context

This task wires Phase 1 (split) and Phase 2 (local reduction) into the `run_grid` function's loop body. Phase 1 calls `split` (SPEC-04) to decompose the net into partitions. Phase 2 iterates over each partition, calls `reduce_all` (SPEC-03) for local reduction, rebuilds the `free_port_index` (TASK-0063), and collects per-worker statistics. Timing and metrics are recorded per the spec.

In local simulation mode (no network), workers are processed sequentially in the same process. Distributed execution (SPEC-06) replaces Phase 2 with network send/receive.

## Key Types / Signatures

```rust
// Inside run_grid loop body:

// === PHASE 1: SPLIT ===
let t_partition = Instant::now();
let mut plan = split(&current_net, config.num_workers, strategy);
metrics.partition_time_per_round.push(t_partition.elapsed());

// === PHASE 2: LOCAL REDUCTION ===
let t_compute = Instant::now();
let mut local_interactions: u64 = 0;
let mut local_by_rule: [u64; 6] = [0; 6];
let mut worker_stats: Vec<WorkerRoundStats> = Vec::new();

for partition in &mut plan.partitions {
    let agents_before = count_live_agents(&partition.subnet);
    let redexes_before = partition.subnet.redex_queue.len();

    let t_reduce = Instant::now();
    let reduction_stats: ReductionStats = reduce_all(&mut partition.subnet);
    let reduce_duration = t_reduce.elapsed();
    local_interactions += reduction_stats.total_interactions;

    // Map ReductionStats to per-rule counts (see R37 note, OQ-4)
    let by_rule = reduction_stats_to_by_rule(&reduction_stats);
    for i in 0..6 { local_by_rule[i] += by_rule[i]; }

    // Rebuild free_port_index after local reduction (TASK-0063)
    // Uses border_id_start/end (R15a) to discriminate boundary vs Lafont FreePorts
    partition.free_port_index = rebuild_free_port_index(
        &partition.subnet,
        partition.border_id_start,
        partition.border_id_end,
    );

    let agents_after = count_live_agents(&partition.subnet);
    worker_stats.push(WorkerRoundStats {
        worker_id: partition.worker_id,
        agents_before,
        agents_after,
        local_redexes: redexes_before,
        reduce_duration_secs: reduce_duration.as_secs_f64(),
        interactions_by_rule: by_rule,
    });
}

metrics.compute_time_per_round.push(t_compute.elapsed());
metrics.local_interactions_per_round.push(local_interactions);
metrics.worker_stats_per_round.push(worker_stats);
```

## Notes

- In local simulation mode, Phase 2 processes workers sequentially. This is intentional: the purpose is correctness validation, not performance. Distributed parallelism is added by SPEC-06.
- `reduce_all` returns `ReductionStats` (not a simple integer). The `total_interactions` field provides the count, and per-rule breakdowns are mapped via `reduction_stats_to_by_rule`.
- `rebuild_free_port_index` must be called AFTER `reduce_all` completes, because local reduction may change FreePort connections.
- **SPEC-05 v3 changes (SC-001, SC-014):** WorkerRoundStats now includes `reduce_duration_secs` and `interactions_by_rule`. Phase 2 must time each worker's reduce_all and map ReductionStats to per-rule counts. `local_by_rule` accumulator added for per-rule totals passed to Phase 3.
