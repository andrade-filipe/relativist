## Context

SPEC-03 Sections 4.1.1 and 4.1.2 define the two annihilation rules. When two agents of the SAME symbol (both Con or both Dup) meet through their principal ports, both are destroyed and their auxiliary ports are reconnected:

- **CON-CON (cross):** `a.1 <-> b.2`, `a.2 <-> b.1` -- the auxiliary ports are cross-connected.
- **DUP-DUP (parallel):** `a.1 <-> b.1`, `a.2 <-> b.2` -- the auxiliary ports are parallel-connected.

The cross vs. parallel asymmetry is essential for universality (REF-002 p.90). Both rules are unified into a single function `interact_anni` with an internal branch on the symbol, following the HVM2 pattern (AC-007).

Agent balance: -2 (2 destroyed, 0 created). Link calls: 2. Up to 2 new redexes may form.

**Edge case (R25):** When auxiliary ports of `a` point back to `b` (or vice versa), the saved `PortRef` values reference removed agents after `remove_agent`. The `link` helper (TASK-0218) guards against this: if either endpoint is an `AgentPort` of a removed agent, the link is a no-op. This handles both full self-references (closed structure, both links are no-ops) and partial self-references (one link is a no-op, the other proceeds normally). See SPEC-03 Section 4.1.2 edge case diagram.

## Key Types / Signatures

```rust
/// Annihilation: two agents of the SAME symbol annihilate each other
/// (SPEC-03 Sections 4.1.1, 4.1.2).
///
/// CON-CON: reconnection in CROSS pattern (a.1<->b.2, a.2<->b.1).
///   The cross pattern distinguishes CON-CON from DUP-DUP and is
///   essential for universality (REF-002 p.90; SPEC-00 Sections 3.3, 3.4).
/// DUP-DUP: reconnection in PARALLEL pattern (a.1<->b.1, a.2<->b.2).
///
/// Precondition: both agent IDs MUST refer to live agents
///   (agents[id].is_some()). This precondition is guaranteed by
///   reduce_step's validity check (R12). Calling this function
///   with removed agents is undefined behavior.
/// Postcondition: a and b removed; auxiliary ports reconnected
///   (or no-op'd if self-referencing, per R25).
///
/// Agent balance: -2. Link calls: 2.
/// New redex detection: up to 2 (if reconnected targets are principal ports).
/// Complexity: O(1).
///
/// Invariants preserved: T1 (4 auxiliary ports reconnected in pairs, none
/// left disconnected), I1 (connect writes both sides), T3 (agents removed,
/// no other Active Pair affected).
pub fn interact_anni(net: &mut Net, a_id: AgentId, b_id: AgentId) {
    let sym = net.agents[a_id as usize].unwrap().symbol;

    let a1 = net.get_target(PortRef::AgentPort(a_id, 1));
    let a2 = net.get_target(PortRef::AgentPort(a_id, 2));
    let b1 = net.get_target(PortRef::AgentPort(b_id, 1));
    let b2 = net.get_target(PortRef::AgentPort(b_id, 2));

    net.remove_agent(a_id);
    net.remove_agent(b_id);

    match sym {
        Symbol::Con => {
            // CROSS: a.1 <-> b.2, a.2 <-> b.1
            link(net, a1, b2);
            link(net, a2, b1);
        }
        Symbol::Dup => {
            // PARALLEL: a.1 <-> b.1, a.2 <-> b.2
            link(net, a1, b1);
            link(net, a2, b2);
        }
        _ => unreachable!("interact_anni called with non-arity-2 symbol"),
    }
}
```

## Notes

- The algorithm follows the "read neighbors, remove old, reconnect" pattern (AC-001, Pattern T7). Neighbors MUST be read BEFORE agents are removed, because `remove_agent` disconnects ports.
- The function assumes both agents have the same symbol. This is guaranteed by `normalize_pair` + dispatch: `get_rule(Con, Con) == Anni` and `get_rule(Dup, Dup) == Anni`. The ERA case is unreachable because `get_rule(Era, Era) == Void`, not Anni.
- **SPEC-03 v3 change:** Uses `link()` helper instead of `net.connect()` directly. The `link` guard handles the self-referencing auxiliary port edge case (R25) where saved PortRef values point to agents that have already been removed. See SPEC-03 Section 4.1.2 for the detailed edge case diagram.
