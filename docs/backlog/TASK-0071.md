## Context

This task wires Phase 3 into the `run_grid` function: after local reduction of all partitions, the coordinator calls `merge` (TASK-0065/0066) to recombine the net, then calls `reduce_all` (SPEC-03) to resolve border redexes and any residual redexes (R15-R18). This is a deliberate improvement over the Haskell prototype, which only resolved border redexes selectively. Strong confluence guarantees that resolving everything at once is correct. After resolution, metrics are recorded and the round counter is incremented.

## Key Types / Signatures

```rust
// Inside run_grid loop body, after Phase 2:

// === PHASE 3: MERGE (structural) ===
let t_merge = Instant::now();
let (mut merged_net, border_redex_count) = merge(plan);
metrics.merge_time_per_round.push(t_merge.elapsed());
metrics.border_redexes_per_round.push(border_redex_count);

// === PHASE 4: RESOLVE BORDERS (reduce_all on merged net) ===
let t_border = Instant::now();
let border_stats: ReductionStats = reduce_all(&mut merged_net);
metrics.border_reduce_time_per_round.push(t_border.elapsed());

let border_by_rule = reduction_stats_to_by_rule(&border_stats);
metrics.border_interactions_per_round
    .push(border_stats.total_interactions);

// Accumulate per-rule totals (SC-004)
for i in 0..6 {
    metrics.total_interactions_by_rule[i] +=
        local_by_rule[i] + border_by_rule[i];
}
metrics.total_interactions +=
    local_interactions + border_stats.total_interactions;

current_net = merged_net;
metrics.rounds += 1;
```
