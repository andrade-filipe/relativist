## Context

SPEC-03 Section 4.3 defines a static 3x3 dispatch table that maps each pair of symbols `(Symbol, Symbol)` to a `Rule` constant in O(1). The `Rule` enum has 4 variants covering the 6 logical rules (annihilation, commutation, erasure, void). The dispatch table is symmetric: `TABLE[a][b] == TABLE[b][a]`. This is the foundation that `reduce_step` uses to determine which interaction function to call.

- [ ] `DISPATCH_TABLE` constant is a `[[Rule; 3]; 3]` array matching the spec's 3x3 matrix
- [ ] `get_rule(a: Symbol, b: Symbol) -> Rule` function performs O(1) table lookup

## Key Types / Signatures

```rust
/// Rule constants for dispatch (SPEC-03 Section 4.3).
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
#[repr(u8)]
pub enum Rule {
    /// Annihilation: same symbol, arity 2 (CON-CON or DUP-DUP).
    Anni = 0,
    /// Commutation: CON-DUP or DUP-CON (different symbols, both arity 2).
    Comm = 1,
    /// Erasure: ERA with CON or DUP (erasure propagation).
    Eras = 2,
    /// Void: ERA-ERA (erasers annihilate).
    Void = 3,
}

/// 3x3 dispatch table. Indexed by (Symbol as usize, Symbol as usize).
/// Symmetric: TABLE[a][b] == TABLE[b][a].
///
///          Con    Dup    Era
/// Con  [  ANNI   COMM   ERAS ]
/// Dup  [  COMM   ANNI   ERAS ]
/// Era  [  ERAS   ERAS   VOID ]
pub const DISPATCH_TABLE: [[Rule; 3]; 3] = [
    [Rule::Anni, Rule::Comm, Rule::Eras],   // Con row
    [Rule::Comm, Rule::Anni, Rule::Eras],   // Dup row
    [Rule::Eras, Rule::Eras, Rule::Void],   // Era row
];

/// Determines the interaction rule for a pair of symbols.
/// Complexity: O(1) (index lookup).
#[inline]
pub fn get_rule(a: Symbol, b: Symbol) -> Rule {
    DISPATCH_TABLE[a as usize][b as usize]
}
```

## Notes

- The table is intentionally static (`const`) rather than computed at runtime. For 3 symbols, there is no performance difference between table lookup and match, but the table cleanly separates dispatch data from rule logic.
- The spec allows replacing the table with a direct `match` if preferred (Section 5.1). The developer MAY use either approach as long as R8 (O(1), complete coverage) is satisfied.
