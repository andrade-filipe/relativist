## Context

SPEC-19 §3.6 R41 extends `GridConfig` (SPEC-05 §4.1) with a delta-protocol opt-in
flag. The sibling field `coordinator_free_rounds` is already present — shipped in
TASK-0350 when bundle 2.34 landed the coordinator-free round optimization, which
is a mergeable subset of the delta protocol. This task adds the remaining field:

```rust
/// Enable delta protocol (stateful workers).
///
/// - `false` (default): v1 full-partition protocol (SPEC-05 R24-R30a).
/// - `true`: delta BSP loop with stateful workers (SPEC-19).
pub delta_mode: bool,
```

The field is **inert** in this task — no call site reads it. The plumbing is
added later: TASK-0390 wires the CLI flag, and bundle 2.26-C's `run_grid_delta`
loop (separate task range) reads the field to branch between v1 and delta
behavior. This task's sole purpose is to land the type change so every
downstream bundle can reference the field without re-coordinating the struct.

## Key Types / Signatures

```rust
// merge/types.rs — updated GridConfig (append between strict_bsp and coordinator_free_rounds)
#[derive(Debug, Clone)]
pub struct GridConfig {
    pub num_workers: u32,
    pub max_rounds: Option<u32>,
    pub strict_bsp: bool,

    /// SPEC-19 §3.6 R41 (TASK-0389): opt-in for the delta-only BSP protocol
    /// (stateful workers).
    ///
    /// When `false` (default — R42 backwards compatibility), `run_grid` uses
    /// the v1 full-partition protocol (SPEC-05 R24-R30a): every round
    /// re-partitions the net and redistributes it to workers.
    ///
    /// When `true`, the grid loop dispatches the delta BSP loop
    /// (`run_grid_delta`, added in bundle 2.26-C), which keeps workers
    /// stateful across rounds and sends only border deltas between them.
    /// The delta loop is functionally equivalent to the v1 loop up to
    /// isomorphism (G1 amendment, R38), with the same Normal Form and the
    /// same total interaction count (T7).
    ///
    /// IC concept: the delta protocol exploits strong confluence (T4) —
    /// the *order* of independent reductions does not affect the Normal
    /// Form, so keeping local partitions stable and exchanging only
    /// border changes is safe.
    ///
    /// Inert until bundle 2.26-C lands `run_grid_delta`. In v2 up through
    /// TASK-0389 the field exists but is never read (see TASK-0391 for
    /// the R42 regression proof).
    pub delta_mode: bool,

    pub coordinator_free_rounds: bool,
}

impl Default for GridConfig {
    fn default() -> Self {
        Self {
            num_workers: 1,
            max_rounds: None,
            strict_bsp: false,
            // SPEC-19 §3.6 R42: opt-in only — default preserves v1 behaviour.
            delta_mode: false,
            // SPEC-19 §3.1 R4 (TASK-0350): opt-in only — defaults preserve v1 behaviour.
            coordinator_free_rounds: false,
        }
    }
}
```

## Notes

- **Scope:** this task adds the field only. Do NOT modify `run_grid` (bundle
  2.26-C territory), `BorderGraph` (shipped in 2.35), or any CLI code
  (TASK-0390 territory).
- **R42 literal interpretation:** adding a field to a struct is a source
  change; R42 forbids *behavioral* regression, which this task satisfies
  because no code path reads the new field yet.
- **R41 dual-shipment:** the spec lists both `delta_mode` AND
  `coordinator_free_rounds`. Half is already done (TASK-0350); this task
  completes the other half. After this task lands, `GridConfig` matches
  SPEC-19 §3.6 R41 verbatim.
- **No serde / bincode concern:** `GridConfig` is **not** serialized over the
  wire (it is a local control struct). No v2 wire format implication.
