## Context

The text DSL (`.ic` files) is a human-readable format for defining IC nets by hand. SPEC-12 Section 4.1 specifies a two-pass parser. This task implements Pass 1: scanning all lines, classifying them (comment, blank, agent declaration, wire declaration, root declaration), and collecting agent declarations into a map from name to (sequential AgentId, Symbol).

Pass 1 does NOT construct the `Net` -- it only builds the intermediate data structures (agent map and wire/root lists) that Pass 2 (TASK-0165) will use. Error reporting includes line numbers for all errors.

The grammar is defined in R7. Port names are mapped per R8: `principal`/`p0` -> 0, `left`/`p1` -> 1, `right`/`p2` -> 2. Agent IDs are assigned sequentially per R10.

## Notes

- This is a hand-written recursive-descent parser, not a parser combinator library. The grammar is simple enough that a library dependency is unnecessary.
- Port name aliases (`principal`/`p0`, etc.) are accepted interchangeably per R8. The parser normalizes to `PortId` values (0, 1, 2).
- The `ParsedPortRef` type uses strings for agent names (not yet resolved to `AgentId`). Resolution happens in Pass 2 (TASK-0165).
- Line numbers are 1-based for human-friendly error messages.
