## Context

SPEC-11 R14 defines four protocol-level metrics that the coordinator must maintain: messages sent/received by type, message size distribution, and heartbeat latency. These are separate from the grid-cycle metrics in `CoordinatorMetrics` (R12) because they track wire protocol behavior rather than reduction progress.

Protocol metrics use a `type` label with bounded cardinality (~10 message types, R16) and the same custom histogram buckets (R15). They are maintained by the coordinator and updated whenever messages are sent or received.

## Notes

- `prometheus-client`'s `Family` type creates metric instances on demand for each unique label combination. The `MessageTypeLabel` struct must derive `EncodeLabelSet` (or the equivalent trait in prometheus-client 0.23).
- The `r#type` field uses raw identifier syntax because `type` is a Rust keyword.
- Message type strings should match the protocol message names exactly (e.g., `"DispatchPartition"`, `"ReturnPartition"`, `"Heartbeat"`). These come from the `Message` enum defined in TASK-0082.
- Heartbeat latency has no labels because there is only one heartbeat mechanism per coordinator.
