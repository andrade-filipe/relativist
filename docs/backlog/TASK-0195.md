## Context

SPEC-09 Section 3.2.5 (R17) defines the **ErasurePropagation** benchmark: a chain of N CON agents with an ERA connected to one end. Reduction propagates the ERA through the chain, exercising the CON-ERA rule repeatedly. Each CON-ERA interaction destroys the CON and produces 2 new ERA agents, one of which propagates to the next CON in the chain.

This is Profile C (sequential cascade): each interaction in the chain depends on the previous one completing, creating inherent sequential dependency similar to DualTree but with a different rule (erasure instead of annihilation).

This pattern is common in real IC reductions: garbage collection of unused subterms propagates erasure through the net.

```rust
use crate::bench::{Benchmark, BenchmarkId};
use crate::bench::isomorphism::nets_isomorphic;
use crate::net::{Net, Symbol, PortRef};

/// ErasurePropagation: chain of N CON agents with ERA at one end.
/// Profile C (sequential cascade). ERA propagates through the chain.
/// Rules exercised: CON-ERA (erasure propagation, repeated N times).
/// Each CON-ERA produces 2 new ERA agents; one propagates forward.
///
/// SPEC-09 R17. New benchmark (not in Haskell prototype).
pub struct ErasurePropagation;

impl Benchmark for ErasurePropagation {
    fn id(&self) -> BenchmarkId {
        BenchmarkId::ErasurePropagation
    }

    fn describe(&self, size: u32) -> String {
        format!("ErasurePropagation: chain of {} CON + 1 ERA, {} interactions",
                size, size)
    }

    fn make_net(&self, n: u32) -> Net {
        let mut net = Net::with_capacity((n + 1) as usize);

        // Create ERA at the head
        let era = net.create_agent(Symbol::Era);

        // Create chain of N CON agents
        let mut cons = Vec::with_capacity(n as usize);
        for _ in 0..n {
            cons.push(net.create_agent(Symbol::Con));
        }

        // Connect ERA.port0 <-> CON_0.port0 (first redex)
        if !cons.is_empty() {
            net.connect(
                PortRef::AgentPort(era, 0),
                PortRef::AgentPort(cons[0], 0),
            );
        }

        // Chain: CON_i.port1 <-> CON_{i+1}.port0
        let mut fp = 0u32;
        for i in 0..(n as usize) {
            if i + 1 < n as usize {
                net.connect(
                    PortRef::AgentPort(cons[i], 1),
                    PortRef::AgentPort(cons[i + 1], 0),
                );
            } else {
                // Last CON: port1 -> FreePort
                net.connect(
                    PortRef::AgentPort(cons[i], 1),
                    PortRef::FreePort(fp),
                );
                fp += 1;
            }
            // Each CON: port2 -> FreePort (free auxiliary)
            net.connect(
                PortRef::AgentPort(cons[i], 2),
                PortRef::FreePort(fp),
            );
            fp += 1;
        }

        net
    }

    fn default_sizes(&self) -> Vec<u32> {
        vec![10, 50, 100, 500, 1_000, 5_000]
    }

    fn verify(&self, sequential_result: &Net, distributed_result: &Net) -> bool {
        nets_isomorphic(sequential_result, distributed_result)
    }
}
```

## Notes

- The erasure cascade is inherently sequential: each CON-ERA interaction produces 2 new ERAs, one of which becomes the new "head" that propagates to the next CON. This means the chain of N CON agents requires N sequential interactions minimum.
- In a grid with K workers, the chain will be split across workers. The propagation front crosses partition boundaries, generating border redexes. This makes it Profile C with high border ratio.
- The CON-ERA rule destroys the CON and produces 2 ERA agents: one is connected to where CON.port1 was (the next CON in the chain), and one is connected to where CON.port2 was (the FreePort). The FreePort-connected ERA becomes a dead end.
