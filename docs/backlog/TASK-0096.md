## Context

SPEC-06 introduces several new external crate dependencies that must be added to Cargo.toml before any protocol code can compile. These include: `tokio` (async runtime with net, io-util, time, macros features), `bincode` (binary serialization), `crc32fast` (CRC32C checksum), `serde` (serialization framework with derive feature), and `futures` (for `join_all` in concurrent sending). Some of these may already be partially declared (e.g., `serde` for SPEC-02), but the required features must be verified and added.

## Acceptance Criteria

- [ ] `tokio` dependency includes features: `rt-multi-thread`, `net`, `io-util`, `time`, `macros`
- [ ] `bincode` dependency is present (version 1.x for compatibility with `serde`)
- [ ] `crc32fast` dependency is present
- [ ] `serde` dependency includes `derive` feature
- [ ] `futures` dependency is present (for `futures::future::join_all`)
- [ ] `tracing` dependency is present (for structured logging, R38)

## Notes

- `bincode` version 1.x is chosen because version 2.x has a different API (uses `bincode::encode`/`bincode::decode` instead of `serde`-based `serialize`/`deserialize`). SPEC-06 pseudocode uses `bincode::serialize`/`bincode::deserialize`, which is the v1 API.
- `tokio::io::duplex` (used in TASK-0095 for testing) requires the `io-util` feature.
- `crc32fast` automatically uses hardware acceleration (SSE4.2 CRC32C instruction) when available on x86. No additional configuration needed.
- The `futures` crate is used only for `join_all`. If this is the only use, consider using `tokio::join!` macro instead and removing the `futures` dependency. The implementer may decide.
