# TASK-0152: Extend WorkerRoundStats with observability fields

**Spec:** SPEC-11, SPEC-05
**Requirements:** R13 (SPEC-11), R37 (SPEC-05)
**Priority:** P0 (critical path)
**Status:** TODO
**Depends on:** TASK-0061
**Blocked by:** none
**Estimated complexity:** S (< 50 LoC)

## Context

SPEC-11 Revised v2 (SC-002) eliminated the standalone `WorkerMetricsReport` struct. Instead, the canonical `WorkerRoundStats` from SPEC-05 R37 is extended with two additional fields for observability: `reduce_duration_secs: f64` and `interactions_by_rule: [u64; 6]`.

Workers report per-round measurements to the coordinator as part of the `PartitionResult` protocol message (SPEC-06), which already carries a `stats: WorkerRoundStats` field (SPEC-05 R37; SPEC-06 R12). By extending `WorkerRoundStats` directly (rather than introducing a separate struct), we avoid type duplication and message schema changes.

This task extends the `WorkerRoundStats` struct (defined in TASK-0061) with the two observability fields. The extension is additive and backward-compatible: all existing consumers of `WorkerRoundStats` continue to work since the original fields are preserved.

**Note:** This change is tracked in SPEC-11 OQ-1 as a cross-spec dependency. The extended definition in SPEC-11 Section 4.4 is normative for observability purposes until SPEC-05 R37 is revised.

## Key Types / Signatures

```rust
/// Statistics of a single worker in a specific round.
/// Canonical type defined in SPEC-05 R37, extended by SPEC-11 for observability.
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct WorkerRoundStats {
    // --- Fields from SPEC-05 R37 (canonical) ---
    /// Identifier of the worker.
    pub worker_id: WorkerId,
    /// Number of live agents in the partition before local reduction.
    pub agents_before: usize,
    /// Number of live agents in the partition after local reduction.
    pub agents_after: usize,
    /// Number of local redexes reduced by this worker.
    pub local_redexes: usize,

    // --- Fields added by SPEC-11 for Prometheus metrics (R13) ---
    /// Wall-clock reduction time in seconds.
    pub reduce_duration_secs: f64,
    /// Interactions broken down by rule type (indexed by rule ordinal:
    /// 0=CON-CON, 1=CON-DUP, 2=CON-ERA, 3=DUP-DUP, 4=DUP-ERA, 5=ERA-ERA).
    pub interactions_by_rule: [u64; 6],
}
```

## Notes

- The `interactions_by_rule` array uses a fixed-size `[u64; 6]` rather than a HashMap for zero-allocation, cache-friendly storage.
- This struct is intentionally NOT feature-gated. Even when the `metrics` feature is disabled, workers still populate and send this data (it is cheap to compute). The coordinator simply does not aggregate it into Prometheus counters when `metrics` is off.
- `PartialEq` is not derived because floating-point equality is unreliable. Tests should use approximate comparison for `reduce_duration_secs`.
- **Change history:** Originally this task defined a standalone `WorkerMetricsReport` struct. SPEC-11 Revised v2 (SC-002) eliminated that type in favor of extending `WorkerRoundStats` directly. See SPEC-11 OQ-1.
