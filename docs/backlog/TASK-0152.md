## Context

Workers do not expose HTTP endpoints for metrics scraping (R13). Instead, they report their per-round measurements to the coordinator as part of the `ReturnPartition` protocol message (SPEC-06). The `WorkerMetricsReport` struct carries these measurements: reduction duration, number of redexes reduced, and a breakdown of interactions by rule type.

This struct is always compiled (not feature-gated) because it is part of the wire protocol payload. The coordinator uses it to aggregate worker data into Prometheus metrics when the `metrics` feature is enabled. When `metrics` is disabled, the coordinator simply ignores the report fields.

## Key Types / Signatures

```rust
/// Metrics reported by a worker alongside its reduced partition (R13).
///
/// Included in the `ReturnPartition` protocol message payload.
/// The coordinator aggregates these into its Prometheus registry.
///
/// This struct is always compiled (not feature-gated) because it is
/// part of the wire protocol format (SPEC-06).
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct WorkerMetricsReport {
    /// Wall-clock reduction time in seconds.
    pub reduce_duration_secs: f64,

    /// Number of redexes reduced in this round.
    pub redexes_reduced: u64,

    /// Interactions broken down by rule type.
    ///
    /// Index mapping:
    ///   0 = CON-CON (annihilation)
    ///   1 = CON-DUP (commutation)
    ///   2 = CON-ERA (erasure)
    ///   3 = DUP-DUP (annihilation)
    ///   4 = DUP-ERA (erasure)
    ///   5 = ERA-ERA (void)
    pub interactions_by_rule: [u64; 6],
}

impl WorkerMetricsReport {
    /// Create an empty (zeroed) report.
    pub fn empty() -> Self {
        Self {
            reduce_duration_secs: 0.0,
            redexes_reduced: 0,
            interactions_by_rule: [0; 6],
        }
    }
}
```

## Notes

- The `interactions_by_rule` array uses a fixed-size `[u64; 6]` rather than a HashMap for zero-allocation, cache-friendly storage. The index-to-rule mapping is documented and corresponds to the `Rule` enum order.
- This struct is intentionally NOT feature-gated. Even when the `metrics` feature is disabled, workers still populate and send this data (it is cheap to compute). The coordinator simply does not aggregate it into Prometheus counters when `metrics` is off.
- `PartialEq` is not derived because floating-point equality is unreliable. Tests should use approximate comparison for `reduce_duration_secs`.
