## Context

Each message transmitted over TCP is encapsulated in a frame with an 8-byte header: 4 bytes payload length (little-endian u32) + 4 bytes CRC32 checksum (little-endian u32), followed by the payload. This task defines the `FrameHeader` struct and the constants needed by the framing functions (`send_frame`, `recv_frame`).

## Acceptance Criteria

- [ ] `FrameHeader` struct is defined in `src/protocol/frame.rs` with fields `length: u32` and `checksum: u32`
- [ ] `FrameHeader` derives `Debug`, `Clone`, `Copy`
- [ ] `FRAME_HEADER_SIZE` constant equals `8usize`
- [ ] `DEFAULT_MAX_PAYLOAD_SIZE` constant equals `268_435_456u32` (256 MiB)
- [ ] `FrameHeader` has methods to serialize to `[u8; 8]` (little-endian) and deserialize from `[u8; 8]`

/// Header of a frame in the wire protocol.
/// Precedes each payload transmitted over TCP.
#[derive(Debug, Clone, Copy)]
pub struct FrameHeader {
    /// Length of the payload in bytes (excluding the header itself).
    pub length: u32,
    /// CRC32C checksum of the payload.
    pub checksum: u32,
}

/// Header size in bytes: 4 (length) + 4 (checksum).
pub const FRAME_HEADER_SIZE: usize = 8;

pub const DEFAULT_MAX_PAYLOAD_SIZE: u32 = 268_435_456;

}

## Notes

- The header uses little-endian byte order because bincode defaults to little-endian, so the whole wire format is consistently LE.
- `FrameHeader` does NOT derive `Serialize`/`Deserialize` because it is not serialized with bincode -- it is manually encoded as raw bytes for efficiency and control.
- CRC32C (Castagnoli variant) is specified in R10. The checksum computation itself is in the `send_frame`/`recv_frame` tasks, not here.
