## Context

Relativist must bind to `127.0.0.1:9000` by default (R5, matching SPEC-13 R44) to prevent accidental exposure. This supersedes SPEC-07 R3/R12's `--host 0.0.0.0` default. The CLI flag is `--bind` (not `--host`). Binding to a non-localhost address requires `--bind` (R6). Binding to `0.0.0.0` (all interfaces) triggers a WARN-level log (R7). Binding to `0.0.0.0` without token authentication (Tier 1 on a public address) MUST emit a WARN log and proceed (no refusal, R8). The `--insecure` flag suppresses SHOULD-level warnings. This task implements the validation and warning logic for bind address configuration.

## Notes

- The bind address validation is called during coordinator startup, before actually binding the TCP listener. It is a pre-flight check that emits warnings but does NOT refuse (R8: MUST-warn-and-proceed).
- The `--insecure` flag suppresses SHOULD-level warnings. It is listed in the CLI extensions table (Section 4.5) for `CoordinatorArgs`.
- The CLI flag is `--bind` (matching SPEC-13 R44), NOT `--host` (SPEC-07 R3). SPEC-07 R3's `--host` is superseded.
- The check uses string matching on `"0.0.0.0"` rather than IP parsing, because the bind address comes from CLI as a string. A future improvement could parse it as `SocketAddr` and check `is_unspecified()`.
