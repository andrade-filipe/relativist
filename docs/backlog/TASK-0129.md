## Context

Relativist must bind to `127.0.0.1` by default (R5) to prevent accidental exposure. Binding to a non-localhost address requires `--bind` (R6). Binding to `0.0.0.0` (all interfaces) triggers a WARN-level log (R7). Binding to `0.0.0.0` without token authentication (Tier 1 on a public address) should be refused unless `--insecure` is provided (R8). This task implements the validation and warning logic for bind address configuration.

## Acceptance Criteria

- [ ] `validate_bind_address(addr: &str, tier: SecurityTier, insecure: bool) -> Result<(), SecurityError>` is implemented
- [ ] Default bind address when no `--bind` flag is provided: `"127.0.0.1"` (R5)
- [ ] `DEFAULT_BIND_ADDRESS` constant is `"127.0.0.1"`
- [ ] Binding to `0.0.0.0` emits WARN log: `"Binding to all interfaces (0.0.0.0). Ensure authentication is enabled for non-trusted networks."` (R7)
- [ ] Binding to `0.0.0.0` with Tier 1 (no auth) returns `Err(SecurityError::ConfigurationError(...))` (R8)
- [ ] Binding to `0.0.0.0` with Tier 1 and `insecure=true` emits WARN but succeeds (R8 override)
- [ ] Binding to `0.0.0.0` with Tier 2 or 3 succeeds (auth is enabled)
- [ ] Binding to a specific non-localhost IP (e.g., `192.168.1.10`) succeeds without warnings for Tier 2/3

## Notes

- The bind address validation is called during coordinator startup, before actually binding the TCP listener. It is a pre-flight check, not a runtime enforcement.
- The `--insecure` flag is a development convenience escape hatch. Its existence must be clearly documented as not for production use.
- The check uses string matching on `"0.0.0.0"` rather than IP parsing, because the bind address comes from CLI as a string. A future improvement could parse it as `SocketAddr` and check `is_unspecified()`.
