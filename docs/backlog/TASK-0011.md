## Context

`connect(a, b)` establishes a bidirectional connection between two ports. It writes both directions in the port array: `a -> b` and `b -> a`. If both ports are principal ports (`AgentPort(_, 0)`), the pair forms an active pair (redex) and is inserted into the redex queue for incremental reduction.

This is the core wiring operation. Bidirectionality (R18, invariant T1/I1) is maintained by always writing both sides. Incremental redex detection (R9) replaces the O(w) `findRedexes` scan of the Haskell prototype with O(1) detection at connection time.

## Acceptance Criteria

- [ ] `connect(a: PortRef, b: PortRef)` is a `pub` method on `Net`
- [ ] In debug mode, `connect` MUST reject same-port self-connections with `debug_assert_ne!(a, b)` (R18b)
- [ ] After `connect(a, b)`: `get_target(a) == b` and `get_target(b) == a`
- [ ] If both `a` and `b` are `AgentPort(_, 0)` (principal ports), the pair `(id_a, id_b)` is pushed to `redex_queue`
- [ ] If either port is not `AgentPort(_, 0)`, no redex is enqueued
- [ ] Intra-agent connections (different ports of the same agent) are VALID (R18b)
- [ ] O(1) complexity

## Key Types / Signatures

```rust
impl Net {
    /// Establishes a bidirectional connection between two ports.
    /// If both are principal ports (port_id == 0), inserts the pair
    /// into the redex queue.
    ///
    /// Self-loop policy (R18b): intra-agent connections (different ports
    /// of the same agent) are valid. Same-port self-connections are
    /// rejected by a debug assertion.
    ///
    /// Redex detection note: redex detection only fires when BOTH
    /// endpoints are `AgentPort(_, 0)`. Connections involving `FreePort`
    /// never produce redexes; border redexes are detected during merge
    /// when `FreePort` sentinels are resolved to `AgentPort` endpoints
    /// (SPEC-05 R5).
    ///
    /// Complexity: O(1).
    /// Postcondition: get_target(a) == b && get_target(b) == a.
    pub fn connect(&mut self, a: PortRef, b: PortRef) {
        #[cfg(debug_assertions)]
        {
            // R18b: same-port self-connections are invalid.
            assert_ne!(a, b, "Same-port self-connection is invalid: {:?}", a);
        }

        self.set_port(a, b);
        self.set_port(b, a);

        // Incremental redex detection: if both are principal ports,
        // an active pair is formed.
        if let (PortRef::AgentPort(id_a, 0), PortRef::AgentPort(id_b, 0)) = (a, b) {
            self.redex_queue.push_back((id_a, id_b));
        }
    }
}
```

## Notes

- `connect` does NOT check whether the ports are already connected to something else. The caller is responsible for disconnecting old connections first if needed (the reduction rules handle this).
- The redex detection uses pattern matching on `(a, b)`: both must be `AgentPort` with port_id `0`. This corresponds to invariant T2 (interaction exclusively via principal ports).
- `connect` does NOT validate port index against agent arity. Callers MUST provide valid port references. The `assert_refs_valid` debug assertion (TASK-0015) catches invalid port references after each operation in debug mode.
