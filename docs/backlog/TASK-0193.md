## Context

SPEC-09 Section 3.2.4 (R14, R15) defines two data-bound benchmarks: **TreeSum** and **TreeSumBalanced**. Both generate nets equivalent to the Haskell prototype's `mkTree` / `mkTreeBalanced` (AC-004): a chain of CON agents (adders) connected to ERA-ERA pairs (work units).

**TreeSum** places all ERA-ERA pairs at one end of the chain (unbalanced). **TreeSumBalanced** distributes ERA-ERA pairs evenly across the chain for better partition load balance.

These benchmarks exercise ERA-ERA (void) and CON-ERA (erasure propagation) rules. The result is verified by comparing the sequential and distributed normal forms via graph isomorphism.

Default sizes: [4, 8, 16, 32, 64, 128, 256]. Haskell comparison sizes: 50, 200, 1000, 2000 (R43). Since the Haskell sizes use different semantics, a subset overlap is ensured.

use crate::bench::{Benchmark, BenchmarkId};
use crate::bench::isomorphism::nets_isomorphic;
use crate::net::{Net, Symbol, PortRef};

/// TreeSum: chain of CON agents with ERA-ERA work units at one end.
/// Profile A/B (depends on size). Data-bound benchmark.
/// Rules exercised: ERA-ERA (void), CON-ERA (erasure propagation).
///
/// SPEC-09 R14. Direct mapping from AC-005 `mkTree` (AC-004).
pub struct TreeSum;

impl Benchmark for TreeSum {
    fn id(&self) -> BenchmarkId {
        BenchmarkId::TreeSum
    }

    fn describe(&self, size: u32) -> String {
        format!("TreeSum: {} items, ~{} agents", size, 2 * size + size)
    }

    fn make_net(&self, n: u32) -> Net {
        // Build a chain of n CON "adder" agents.
        // Each CON's port 2 connects to an ERA-ERA pair (work unit).
        // CON chain: CON_0.port1 -> CON_1.port0 -> ... -> CON_{n-1}
        // Each CON_i.port2 -> ERA_2i.port0 <-> ERA_{2i+1}.port0
        let mut net = Net::with_capacity((3 * n) as usize);

        // Create CON chain
        let mut cons = Vec::with_capacity(n as usize);
        for _ in 0..n {
            cons.push(net.create_agent(Symbol::Con));
        }

        // Chain: CON_i.port1 -> CON_{i+1}.port0
        for i in 0..(n as usize - 1) {
            net.connect(
                PortRef::AgentPort(cons[i], 1),
                PortRef::AgentPort(cons[i + 1], 0),
            );
        }

        // First CON's port0 -> FreePort (input)
        net.connect(PortRef::AgentPort(cons[0], 0), PortRef::FreePort(0));
        // Last CON's port1 -> FreePort (chain end)
        net.connect(
            PortRef::AgentPort(cons[n as usize - 1], 1),
            PortRef::FreePort(1),
        );

        // Each CON's port2 -> ERA-ERA pair
        for i in 0..n as usize {
            let era_a = net.create_agent(Symbol::Era);
            let era_b = net.create_agent(Symbol::Era);
            net.connect(
                PortRef::AgentPort(era_a, 0),
                PortRef::AgentPort(era_b, 0),
            ); // ERA-ERA redex
            // Connect one ERA to CON's port 2
            // (This creates a CON-ERA pair when the ERA-ERA resolves
            //  and propagates)
            // Actually: CON.port2 -> ERA_a's port0 would make a redex.
            // We need to wire CON.port2 into the ERA pair properly.
            // Use a FreePort as intermediary for the CON.port2 connection.
            net.connect(
                PortRef::AgentPort(cons[i], 2),
                PortRef::FreePort((i as u32) + 2),
            );
        }

        net
    }

    fn default_sizes(&self) -> Vec<u32> {
        vec![4, 8, 16, 32, 64, 128, 256]
    }

    fn verify(&self, sequential_result: &Net, distributed_result: &Net) -> bool {
        nets_isomorphic(sequential_result, distributed_result)
    }
}

## Notes

- The exact wiring of TreeSum follows the Haskell prototype's `mkTree` (AC-004). The pseudocode above is a sketch; the final implementation must match the Haskell semantics where ERA-ERA pairs represent "work units" that produce a numeric sum.
- TreeSumBalanced is a SHOULD requirement (R15). If it proves too complex for the size estimate, it can be simplified to a variant that just reorders agent creation.
- Both benchmarks use `nets_isomorphic` because their normal forms are non-trivial (remaining FreePort connections).
