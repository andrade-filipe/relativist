## Context

This task implements the construction of a single sub-net (`Net`) for a given worker, corresponding to Step 5 of the split algorithm. The sub-net contains only the agents assigned to the worker. Internal wires (both endpoints in the same partition) are copied directly from the original port array. Border wires are replaced: the agent's port is connected to `FreePort(borderId)` instead of the original remote agent. Interface wires (involving pre-existing Lafont FreePorts) are copied as-is. The result is a valid `Net` that can be reduced independently.

## Acceptance Criteria

- [ ] Function accepts the original net, the list of agent IDs for this worker, the allocation map, and the border entries for this worker
- [ ] Creates a new `Net` containing only the specified agents
- [ ] Internal wires: port array entries copied byte-for-byte from the original (R9)
- [ ] Border wires: `port_array[agent_port] = FreePort(bid)` for each border entry
- [ ] Interface wires: `FreePort(f)` connections copied unchanged from the original
- [ ] DISCONNECTED slots for agents not in this partition
- [ ] Port linearity (T1) is maintained: each FreePort(bid) appears exactly once in this sub-net (R14)

## Notes

- The sub-net's `agents` Vec must be large enough to hold agents at their original indices. Slots for agents NOT in this partition are `None`. This preserves the `AgentId == index` invariant from SPEC-02.
- The sub-net's `ports` Vec must be correspondingly sized. Ports of agents not in this partition are `DISCONNECTED`.
- The `redex_queue` is populated separately by TASK-0051, so this function should leave it empty or delegate to TASK-0051.
- The `root` field: if the original net's root points to an agent in this partition, the sub-net inherits it. Otherwise, `root = None`.
- Building a HashSet of border entries keyed by `(agent_id, port_id)` upfront enables O(1) lookup during port iteration.
