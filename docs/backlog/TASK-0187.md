## Key Types / Signatures

```rust
/// Obtain the peak memory usage (resident set size) of the current process.
///
/// On Linux: reads /proc/self/status, parses VmHWM (peak RSS in kB),
/// and converts to bytes.
/// On other OSes: returns 0 (metric unavailable).
///
/// SPEC-09 Section 4.4.
pub fn get_peak_memory_bytes() -> u64 {
    #[cfg(target_os = "linux")]
    {
        if let Ok(status) = std::fs::read_to_string("/proc/self/status") {
            for line in status.lines() {
                if line.starts_with("VmHWM:") {
                    // Format: "VmHWM:    12345 kB"
                    let parts: Vec<&str> = line.split_whitespace().collect();
                    if parts.len() >= 2 {
                        if let Ok(kb) = parts[1].parse::<u64>() {
                            return kb * 1024; // kB -> bytes
                        }
                    }
                }
            }
        }
        0
    }

    #[cfg(not(target_os = "linux"))]
    {
        0
    }
}
```

## Notes

- VmHWM reports the **peak** RSS, not the current RSS. This means calling it after a benchmark execution captures the maximum memory used during that execution.
- On Windows (the developer's machine), this returns 0. For the actual TCC benchmarks, the code runs in Docker on Linux where the metric is available. This is acceptable per SPEC-09 Section 4.4.
- A future improvement could use the `sysinfo` crate for cross-platform memory measurement, but that adds a dependency. For now, the Linux-only approach is sufficient.
