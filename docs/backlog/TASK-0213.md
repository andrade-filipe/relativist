# TASK-0213: Implement ERROR-level logging requirements (R9a)

**Spec:** SPEC-11
**Requirements:** R9a
**Priority:** P1 (important, not blocking)
**Status:** TODO
**Depends on:** TASK-0144, Phase 1, Phase 2, Phase 4, Phase 5, Phase 6
**Blocked by:** none
**Estimated complexity:** M (50-200 LoC)

## Context

SPEC-11 Revised v2 added R9a (SC-007), which specifies events that MUST be logged at ERROR level. These include invariant violations from `assert_all_invariants()` (SPEC-01 T1-T7), protocol errors (checksum mismatch, message too large, connection lost), coordinator/worker FSM transitions to `Error` state, `FatalError` events, and merge failures.

This task adds `tracing::error!` calls at all required locations. It is a cross-cutting concern that touches multiple modules, but each individual change is a single logging statement at an existing error-handling site.

## Acceptance Criteria

- [ ] Invariant violation events from `assert_all_invariants()` (SPEC-01 T1-T7) are logged at ERROR level in `net`, `reduction`, `partition`, `merge` modules
- [ ] Protocol errors are logged at ERROR level in the `protocol` module:
  - [ ] Checksum mismatch (SPEC-06, R29)
  - [ ] Message too large (SPEC-06, R9)
  - [ ] Connection lost (SPEC-06, R25)
- [ ] Coordinator FSM transitions to `Error` state are logged at ERROR level (SPEC-13, R19)
- [ ] Worker FSM transitions to `Error` state are logged at ERROR level (SPEC-13, R24)
- [ ] Any `FatalError` event (SPEC-13, R21) is logged at ERROR level
- [ ] Merge failures (unresolved borders, merge invariant violations) are logged at ERROR level in the `merge` module
- [ ] All ERROR log events include relevant structured fields (module, error details, context)
- [ ] No behavioral changes -- logging only

## Files to Create/Modify

- Multiple files across modules -- **modify**: add `tracing::error!` calls at existing error-handling sites
  - `src/net/*.rs` -- invariant violation assertions
  - `src/reduction/*.rs` -- invariant violations during reduction
  - `src/partition/*.rs` -- invariant violations during split
  - `src/merge/*.rs` -- merge failures, invariant violations
  - `src/protocol/*.rs` -- protocol errors
  - `src/coordinator/*.rs` -- FSM Error transitions, FatalError
  - `src/worker/*.rs` -- FSM Error transitions, FatalError

## Key Types / Signatures

```rust
// Example: invariant violation in net module
tracing::error!(
    invariant = "T1",
    details = %err,
    "invariant violation detected by assert_all_invariants"
);

// Example: protocol error
tracing::error!(
    error_type = "checksum_mismatch",
    expected = %expected,
    actual = %actual,
    peer = %peer_addr,
    "protocol error: checksum mismatch (SPEC-06, R29)"
);

// Example: FSM error transition
tracing::error!(
    from_state = %from,
    to_state = "Error",
    event = %trigger,
    "coordinator FSM entered Error state"
);

// Example: merge failure
tracing::error!(
    unresolved_borders = count,
    "merge failure: unresolved border connections"
);
```
