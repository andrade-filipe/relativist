## Context

Two summary data structures are needed throughout the I/O layer. `NetSummary` holds statistics about a net (agent count, wire count, redex count, symbol distribution, free ports, normal form status) -- used by the `inspect` subcommand (R29) and as input to the reduction summary. `ReductionSummary` holds the before/after `NetSummary` plus timing and interaction metrics -- used by the `reduce` subcommand (R44-R45) and optionally extended with grid metrics for distributed execution (R46).

This task defines only the data structures. The computation logic (populating these structs) is in TASK-0169 and TASK-0170.

## Key Types / Signatures

```rust
/// Summary statistics of a net (SPEC-12, R29).
///
/// Produced by the `inspect` subcommand and used as input/output
/// snapshots in ReductionSummary.
#[derive(Debug, Clone, serde::Serialize)]
pub struct NetSummary {
    pub agents: u32,
    pub wires: u32,
    pub redexes: u32,
    pub con: u32,
    pub dup: u32,
    pub era: u32,
    pub free_ports: u32,
    pub normal_form: bool,
}

/// Grid-specific metrics appended to a reduction summary
/// for distributed execution (SPEC-12, R46).
#[derive(Debug, Clone, serde::Serialize)]
pub struct GridReductionInfo {
    pub rounds: u32,
    pub workers: u32,
    pub speedup: f64,
    pub efficiency: f64,
    pub overhead_pct: f64,
}

/// Summary of a completed reduction (SPEC-12, R44-R45).
///
/// Printed to stdout after any reduction (local or distributed).
#[derive(Debug, Clone, serde::Serialize)]
pub struct ReductionSummary {
    pub input: NetSummary,
    pub output: NetSummary,
    pub total_interactions: u64,
    pub duration_secs: f64,
    pub mips: f64,
    pub max_interactions_reached: bool,
    pub grid: Option<GridReductionInfo>,
}
```

## Notes

- `NetSummary` is used in three places: (1) `inspect` output (R29-R31), (2) the `input`/`output` fields of `ReductionSummary`, (3) the `--metrics` JSON output (R21).
- The `mips` field is Millions of Interactions Per Second: `total_interactions as f64 / duration_secs / 1_000_000.0`.
- `max_interactions_reached` controls the warning message in the reduction summary (R47).
- `GridReductionInfo` maps to R46 output lines: Rounds, Workers, Speedup, Efficiency, Overhead.
