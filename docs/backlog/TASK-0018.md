# TASK-0018: Implement PartialEq for Net

**Spec:** SPEC-02
**Requirements:** R22, R26
**Priority:** P1 (important)
**Status:** TODO
**Depends on:** TASK-0008
**Blocked by:** none
**Estimated complexity:** S (< 50 LoC)

## Context

`Net` needs structural equality comparison for two purposes: (1) verifying the serialization round-trip identity (R26: `deserialize(serialize(net)) == net`), and (2) testing that operations produce expected results.

This task implements `PartialEq` for `Net`. The comparison is structural (field-by-field), NOT isomorphism (which would require agent ID renaming -- that is a separate concept from SPEC-00 Section 6.12, used in G1 testing). Structural equality means: same agents in same slots, same port array, same redex queue, same next_id, same root.

`PartialEq` is derived rather than manually implemented because all fields already implement `PartialEq` (`Vec<Option<Agent>>`, `Vec<PortRef>`, `VecDeque<(u32, u32)>`, `u32`, `Option<PortRef>`).

## Acceptance Criteria

- [ ] `Net` implements `PartialEq` (via derive)
- [ ] Two identical nets compare as equal
- [ ] Two nets differing in any field compare as not equal
- [ ] `PartialEq` also added to derive list for `Net` alongside existing derives

## Files to Create/Modify

- `src/net/net.rs` -- **modify**: add `PartialEq` to `Net`'s derive list

## Key Types / Signatures

```rust
// Change the derive on Net from:
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
// to:
#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub struct Net { /* ... */ }
```

## Test Expectations

- Unit test: `Net::new() == Net::new()` is `true`
- Unit test: two nets with same agents/ports are equal
- Unit test: nets differing in `next_id` are NOT equal
- Unit test: serialization round-trip: `Net::from_bytes(&net.to_bytes()?) == Ok(net)`

## Dependencies Context

- `Net` struct (defined in TASK-0008)
- All inner types implement `PartialEq`: `Agent` (TASK-0005), `PortRef` (TASK-0004), `VecDeque`, `Vec`, `Option`, `u32`

## Notes

- This is a simple derive addition. It is a separate task because the spec deliberately omits `PartialEq` from the initial `Net` definition (TASK-0008) to avoid premature equality semantics. Once serialization (TASK-0017) needs it, it is added.
- This is structural equality, not semantic/isomorphic equality. Two nets that are isomorphic (same structure, different IDs) will NOT compare as equal. Isomorphism comparison is a separate function in the testing infrastructure (SPEC-09).
