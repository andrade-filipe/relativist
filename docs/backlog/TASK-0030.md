# TASK-0030: Wire up reduction module re-exports

**Spec:** SPEC-03
**Requirements:** (ergonomic — enables clean public API for the reduction module)
**Priority:** P1 (important)
**Status:** TODO
**Depends on:** TASK-0020 (module structure), TASK-0021 through TASK-0029 (all types and functions)
**Blocked by:** none
**Estimated complexity:** S (< 50 LoC)

## Context

After all reduction types and functions are implemented in their sub-modules (`dispatch.rs`, `rules.rs`, `engine.rs`), the module root (`src/reduction/mod.rs`) needs to re-export the public API so that consumers (tests, benchmarks, other modules) can import items ergonomically via `use crate::reduction::*` rather than navigating the internal sub-module structure.

This task also ensures that the module compiles as a whole and that all pieces integrate correctly.

## Acceptance Criteria

- [ ] `src/reduction/mod.rs` re-exports all public types: `Rule`, `StepResult`, `ReductionStats`
- [ ] `src/reduction/mod.rs` re-exports all public functions: `get_rule`, `normalize_pair`, `interact_anni`, `interact_comm`, `interact_eras`, `interact_void`, `reduce_step`, `reduce_all`, `reduce_n`
- [ ] `cargo check` passes with the complete reduction module
- [ ] External code can use `use crate::reduction::{reduce_all, ReductionStats};` etc.

## Files to Create/Modify

- `src/reduction/mod.rs` -- **modify**: add `pub use` re-exports

## Key Types / Signatures

```rust
// src/reduction/mod.rs

//! Reduction engine for Interaction Combinators (SPEC-03).
//!
//! Implements the 6 interaction rules (CON-CON, DUP-DUP, ERA-ERA,
//! CON-DUP, CON-ERA, DUP-ERA), dispatch, and the reduction loop
//! (reduce_step, reduce_all, reduce_n).

pub mod dispatch;
pub mod rules;
pub mod engine;

// Re-exports for public API
pub use dispatch::{get_rule, normalize_pair, Rule, DISPATCH_TABLE};
pub use engine::{reduce_all, reduce_n, reduce_step, ReductionStats, StepResult};
pub use rules::{interact_anni, interact_comm, interact_eras, interact_void};
```
