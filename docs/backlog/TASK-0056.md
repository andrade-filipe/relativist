# TASK-0056: ID range exhaustion error handling

**Spec:** SPEC-04
**Requirements:** R20
**Priority:** P2 (nice to have)
**Status:** TODO
**Depends on:** TASK-0040 (IdRange), SPEC-02 types (Net)
**Blocked by:** TASK-0040
**Estimated complexity:** S (< 50 LoC)

## Context

If a worker exhausts its ID range (allocates more than `chunk_size` new agents during local reduction), the system must signal an error (R20). With 8 workers, each has ~537 million IDs, so exhaustion is not expected for practical nets within TCC scope. However, the check must exist for correctness. This task adds a validation method to `IdRange` and integrates the check into `Net::create_agent` when operating within a partition context.

## Acceptance Criteria

- [ ] `IdRange` has a method `contains(&self, id: AgentId) -> bool` that checks if an ID is within the range
- [ ] An error type or panic path exists for ID range exhaustion
- [ ] The check is documented as a safety net, not expected to trigger in practice
- [ ] The error message includes the worker ID and the exhausted range

## Files to Create/Modify

- `src/partition/types.rs` — modify: add `contains` and `is_exhausted` methods to `IdRange`

## Key Types / Signatures

```rust
impl IdRange {
    /// Returns true if the given AgentId falls within this range.
    pub fn contains(&self, id: AgentId) -> bool {
        id >= self.start && id < self.end
    }

    /// Returns the number of IDs available in this range.
    pub fn capacity(&self) -> u64 {
        (self.end as u64) - (self.start as u64)
    }
}
```

## Test Expectations

- Unit test: id within range -> contains returns true
- Unit test: id outside range -> contains returns false
- Unit test: id at start boundary -> contains returns true
- Unit test: id at end boundary -> contains returns false (exclusive end)
- Unit test: capacity of a range with 537M IDs

## Dependencies Context

- `IdRange` struct (TASK-0040): `{ start: AgentId, end: AgentId }`
- `AgentId` is `u32` (SPEC-02)

## Notes

- The actual integration with `Net::create_agent` to check range bounds during reduction is a cross-cutting concern. This task defines the `IdRange` methods. The reduction engine (SPEC-03 tasks) or the grid cycle (SPEC-05) should invoke the check when creating agents within a partition context.
- For the TCC scope, a `panic!` on exhaustion is acceptable. A more graceful `Result`-based approach can be added later.
- The `capacity()` method returns `u64` to handle ranges that span more than `u32::MAX / 2`.
