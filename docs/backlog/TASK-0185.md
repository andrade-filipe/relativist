## Context

The graph isomorphism function `nets_isomorphic` is THE critical verification tool for the TCC's core claim. The Fundamental Property (G1) states that `reduce_all(net) ~= run_grid(net, n)` where `~=` denotes structural equality modulo AgentId renaming. Without this function, correctness verification falls back to agent counting, which is insufficient (two nets with the same agent count but different topologies would pass).

SPEC-08 Section 3.11 (R23-R24) and Section 4.4 define the algorithm: a backtracking search for a bijection between AgentIds that preserves symbols and connectivity. The algorithm has O(n! * k) worst-case complexity but is practical for nets with < 100 agents. For larger nets, heuristic pruning (degree filtering, neighborhood signature) reduces the search space.

This function lives in `tests/common/isomorphism.rs` per SPEC-08 Section 4.3, but is also needed by the benchmark framework for production correctness verification (SPEC-09 R4). Therefore, it is implemented in `src/bench/` (library code) and re-exported for use by both benchmarks and tests.

**This is a key deliverable for the TCC -- the paper cannot verify its hypothesis without it.**

use crate::net::{Net, AgentId, PortRef, Symbol};
use std::collections::HashMap;

/// Verifies whether two nets are isomorphic: structurally equal modulo
/// AgentId renaming (SPEC-00 Section 6.12, SPEC-08 R23-R24).
///
/// Two nets are isomorphic if there exists a bijection f: AgentId_a -> AgentId_b
/// such that:
/// - For every live agent id in a, f(id) is live in b with the same symbol.
/// - For every port connection in a, the mapped connection exists in b.
/// - FreePort references are preserved without renaming.
///
/// Algorithm:
/// 1. Quick reject: count agents per symbol. If counts differ, return false.
/// 2. Group agents by symbol in both nets.
/// 3. Backtracking search: for each agent in a (ordered by symbol), try to
///    map it to an unmapped agent in b with the same symbol.
///    a. For each candidate, verify connectivity constraints:
///       every port's target, mapped by f, must match.
///    b. If consistent, proceed to next agent. Otherwise, backtrack.
/// 4. If complete bijection found, return true.
///
/// Complexity: O(n! * k) worst case. Practical for n < 100 with pruning.
/// For larger nets, falls back to heuristic pre-filtering by neighborhood
/// signature (degree + sorted neighbor symbols).
pub fn nets_isomorphic(a: &Net, b: &Net) -> bool {
    // 1. Quick reject by agent count per symbol
    let counts_a = count_agents_by_symbol(a);
    let counts_b = count_agents_by_symbol(b);
    if counts_a != counts_b {
        return false;
    }

    // 2. Group agents by symbol
    let groups_a = group_agents_by_symbol(a);
    let groups_b = group_agents_by_symbol(b);

    // 3. Backtracking search
    let agents_a: Vec<AgentId> = /* all live agent IDs in a */;
    let mut mapping: HashMap<AgentId, AgentId> = HashMap::new();
    let mut reverse: HashMap<AgentId, AgentId> = HashMap::new();

    backtrack(a, b, &agents_a, 0, &mut mapping, &mut reverse,
              &groups_b)
}

## Notes

- This function is placed in `src/bench/isomorphism.rs` rather than `tests/common/` because the benchmark framework (production code) needs it for correctness verification on every datapoint (R4, R36). Test code can import it from the library.
- For the TCC's empirical evaluation (~4200 datapoints), this function will be called at least 4200 times. For benchmarks with small normal forms (EP: 0 agents), the function is trivially fast. For TreeSum/MixedNet with non-trivial results, the normal forms should be small enough for backtracking.
- The heuristic for large nets (> 100 agents) is: compute a "neighborhood signature" for each agent (sorted list of neighbor symbols + degree), group agents by signature, then run backtracking only within matching groups. This drastically reduces the search space.
- **SPEC-08 v3 (SC-009):** The isomorphism checker SHOULD use canonical form computation (e.g., Weisfeiler-Leman refinement) or a polynomial-time heuristic that handles nets up to 500 agents within 100ms. The backtracking fallback MAY be used for nets where the heuristic is inconclusive. See SPEC-08 Section 4.4.
- **R37a (SPEC-09 v3):** Tiered isomorphism strategy: (a) lightweight check (same agent count per symbol, same wire count, same free port count, same redex count == 0) MAY be used when performance is a concern, (b) full structural isomorphism via `nets_isomorphic` SHOULD be used when lightweight passes and result is small (< 1000 agents), (c) for empty-result benchmarks (EP, DualTree), simply check `agent_count == 0`.
- **R37b (SPEC-09 v3):** Sequential baseline T6 validation: each sequential repetition MUST be verified against the first sequential result by graph isomorphism to confirm deterministic normal forms.
- **This is the most important function in Phase 10.** Without it, the TCC cannot verify G1 with structural precision, falling back to the weak agent-counting approach of the Haskell prototype.
