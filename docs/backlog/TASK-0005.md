## Context

An `Agent` is a node in the interaction net. Each agent has a `Symbol` (which determines its arity and applicable interaction rules) and a unique `AgentId`. Agents do not store their port connections directly -- connections are stored implicitly in the Net's flat port array. This is a deliberate design decision: separating agent identity from connectivity simplifies removal (mark slot as `None` instead of updating a complex structure).

## Notes

- `Agent` is intentionally lightweight (only 5 bytes: 1 byte symbol + 4 bytes id). This allows it to be `Copy` and stored efficiently in `Vec<Option<Agent>>`.
- The port connections for agent `id` live at port array indices `[id*3, id*3+1, id*3+2]`.
