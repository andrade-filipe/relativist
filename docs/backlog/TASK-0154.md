## Context

The HTTP endpoints for metrics, health, and readiness are served via `axum` on a dedicated port (R20). All HTTP functionality is feature-gated under the `metrics` Cargo feature (R19). This task creates the `metrics_router()` function that builds an `axum::Router` with the three required routes (`/metrics`, `/health`, `/ready`) and defines the shared application state.

The readiness check uses an `AtomicBool` flag (`is_ready`) per R22a, rather than an `AtomicU8` ordinal comparison on the coordinator state enum. This avoids fragile dependence on enum discriminant ordering and correctly handles the `Error` state.

The actual handler implementations are in TASK-0155 (health/ready) and TASK-0156 (metrics). This task provides the router skeleton with the state types and route declarations.

## Key Types / Signatures

```rust
#[cfg(feature = "metrics")]
use axum::{routing::get, Router};
#[cfg(feature = "metrics")]
use std::sync::{atomic::AtomicBool, Arc};

/// OpenMetrics Content-Type with version and charset parameters (R21).
#[cfg(feature = "metrics")]
pub const OPENMETRICS_CONTENT_TYPE: &str =
    "application/openmetrics-text; version=1.0.0; charset=utf-8";

/// Shared state for the metrics HTTP server.
#[cfg(feature = "metrics")]
#[derive(Clone)]
pub struct AppState {
    /// Prometheus metrics registry.
    pub registry: Arc<prometheus_client::registry::Registry>,
    /// Readiness flag, set by the coordinator FSM (R22a).
    /// true = ready (WaitingForWorkers..Done), false = not ready (Init, Error).
    pub is_ready: Arc<AtomicBool>,
}

/// Build the axum Router for observability HTTP endpoints (R19, R20, R21).
///
/// Routes:
/// - GET /metrics  -- Prometheus text exposition (TASK-0156)
/// - GET /health   -- Liveness check (TASK-0155)
/// - GET /ready    -- Readiness check (TASK-0155)
#[cfg(feature = "metrics")]
pub fn metrics_router(
    registry: Arc<prometheus_client::registry::Registry>,
    coordinator_is_ready: Arc<AtomicBool>,
) -> Router {
    let state = AppState {
        registry,
        is_ready: coordinator_is_ready,
    };

    Router::new()
        .route("/metrics", get(metrics_handler))
        .route("/health", get(health_handler))
        .route("/ready", get(ready_handler))
        .with_state(state)
}
```

## Notes

- The readiness flag uses `AtomicBool` per R22a. The coordinator FSM sets `is_ready = true` on `Init -> WaitingForWorkers` transition and `is_ready = false` on any transition to `Error` state. This is robust against enum reordering and correctly handles the `Error` state.
- `axum` 0.8 uses `Router::new().route(path, method_router)` syntax. Check exact API compatibility at implementation time.
- **Change history:** Originally used `Arc<AtomicU8>` with ordinal comparison. SPEC-11 Revised v2 (SC-012, R22a) replaced with `Arc<AtomicBool>` for robustness.
