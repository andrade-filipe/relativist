## Context

SPEC-05 Revised v3 (SC-010) promoted OQ-3 to R41 as a SHOULD requirement: after the final `reduce_all` in each round, and before declaring Normal Form, the grid loop SHOULD perform a full scan of the port array to detect any redexes not present in the queue. This serves as defense-in-depth against bugs in `connect()` (SPEC-02, R13) that might fail to insert a redex into the queue, which would cause `drain_stale_redexes` to declare Normal Form prematurely, violating G1.

The scan is enabled by default in debug mode (`#[cfg(debug_assertions)]`) and MAY be disabled in release mode via a configuration option (e.g., `GridConfig.verify_normal_form: bool`).

## Key Types / Signatures

```rust
/// Full-scan Normal Form verification (SPEC-05, R41).
///
/// Scans the entire port array to detect any redexes not present in the queue.
/// This is a defense-in-depth measure: if `connect()` has a bug that fails to
/// insert a redex, the drain approach would declare Normal Form prematurely.
///
/// Enabled by default in debug mode. Complexity: O(A * PORTS_PER_SLOT).
/// Panics if an undiscovered redex is found.
#[cfg(debug_assertions)]
pub fn verify_no_redexes_full_scan(net: &Net) {
    for (i, slot) in net.agents.iter().enumerate() {
        if let Some(agent) = slot {
            let target = net.get_target(AgentPort(agent.id, 0));
            if let AgentPort(other_id, 0) = target {
                if other_id != agent.id {
                    panic!(
                        "verify_no_redexes_full_scan: undiscovered redex \
                         ({}, {}) not in queue",
                        agent.id, other_id
                    );
                }
            }
        }
    }
}
```
