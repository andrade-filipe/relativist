# TASK-0402: Worker-side mint-then-wire — implement R24.1.6a/b/c in `worker.rs::handle_round_start`

**Spec:** SPEC-19 §3.3 R23a (5 substitution clauses + clause 6 HashSet pre-pass for NF-003), §3.3 R24.1.6a/b/c (three-step mint-then-wire protocol), §3.3 R24 ordering invariant (no `reduce_all` drain until step 2), §3.4 R33b (`Net::connect` sanctioned primitive), §3.4 R33c cases 1-7 (error paths + Case 4 dangling warn), §3.6 R48a (stray slot-marker guard), §3.6 R48b (empty `local_wiring` legal).
**Requirements:** R24.1.6a mint; R24.1.6b wire with clauses 3/4/6; R24.1.6c echo; R23a determinism; R24 ordering invariant; R33c case dispatch (1, 2, 3, 4-warn, 5, 6, 7); R48a + R48b boundary behaviors.
**Priority:** P0 (the actual bug-fix — workers currently mint agents but leave their local edges DISCONNECTED, breaking G1 parity on all three asymmetric rules).
**Status:** TODO
**Depends on:** TASK-0400 (consumes `PendingCommutation` Shape A + `LocalWiringHint` + `ProtocolError::MalformedLocalWiring`); TASK-0401 (consumes populated wire payloads end-to-end during integration QA).
**Blocked by:** TASK-0400, TASK-0401.

**Bundle:** D-005 Option A.

## Context

Today, `relativist-core/src/worker.rs::handle_round_start` (L406-446) executes R24.1.6a (mint) and R24.1.6c (echo) but skips R24.1.6b (wire). The minted siblings are inserted into the worker's partition with all of their aux ports set to `DISCONNECTED`. When `reduce_all` subsequently runs on that partition, the minted agents are inert: they have no principal-port redex with any other agent, and their aux ports cannot feed downstream rules. G1 parity therefore fails on every fixture that exercises CON-DUP, CON-ERA, or DUP-ERA across a partition boundary — exactly the three rules gated by `const SKIP_ASYMMETRIC: bool = true` in `grid_delta_integration_tests.rs`.

TASK-0402 ships the missing step. For each `PendingCommutation pc` in `RoundStart.pending_commutations`:

1. **R24.1.6a (mint):** allocate `pc.target_symbols.len()` fresh `AgentId`s, one per slot, minted with the slot's symbol (NF-001 Shape A).
2. **R24.1.6b (wire):** run the R23a clause-6 HashSet pre-pass on the whole `pc.local_wiring` vector; if duplicates detected → reject entire PC with `MalformedLocalWiringReason::DuplicateSourcePort`. Then walk entries in emitted order; for each hint, resolve source per slot vector, decode target per clauses 3/4/6, and call `Net::connect`.
3. **R24.1.6c (echo):** append `MintedAgent { request_id, minted_agent_id: minted_ids_per_pc[0] }` to the response buffer regardless of reduction activity (slot-0 is the canonical correlation anchor).

**R24 ordering invariant.** `Net::connect` may enqueue principal-principal pairs into `net.redex_queue` when the wired endpoints are both principal ports (R23a clause 5 — in-round reducibility of minted-to-minted pairs). Those redexes MUST NOT drain until `reduce_all` is called at R24.2. The mint-then-wire loop therefore MUST NOT itself invoke any reduction step.

## Acceptance Criteria

### 1. Insert the mint-then-wire block at the correct point

- [ ] Locate the insertion point: `relativist-core/src/worker.rs::handle_round_start`, immediately after the existing `local_reconnections` application (step R24.1.4) and BEFORE `reduce_all` (R24.2). Current code at L446 shows the existing mint loop — TASK-0402 REPLACES that loop with the three-step protocol.
- [ ] Preserve all existing step order: deltas (R24.1.1), resolved borders (R24.1.2), new borders (R24.1.3), local_reconnections (R24.1.4), **NEW pending_commutations (R24.1.5/6)**, then reduce_all (R24.2), then rebuild_free_port_index (R24.3), then compute deltas (R24.4), then send RoundResult (R24.5).
- [ ] The mint-then-wire loop runs EXACTLY ONCE per `PendingCommutation`. No recursion, no retry on error (errors bubble up as `ProtocolError::MalformedLocalWiring` which the caller maps to `RegisterNack`).

### 2. R24.1.6a — mint loop

- [ ] For each `PendingCommutation pc`:
  - Reject with `MalformedLocalWiring { request_id: pc.request_id, reason: MalformedLocalWiringReason::ZeroArity }` if `pc.target_symbols.is_empty()` (R33c case 7). Rejection MUST happen BEFORE any mint, so `minted_ids_per_pc[0]` is always well-defined for non-rejected requests.
  - Allocate one `AgentId` per slot via `net.create_agent(pc.target_symbols[k])` for `k in 0..pc.target_symbols.len()`. Store in `minted_ids_per_pc: Vec<AgentId>` in slot order.
  - Consume AgentIds from the worker's `IdRange` monotonically (existing behavior of `create_agent` — do NOT re-implement range exhaustion check; SPEC-04 handles it).

### 3. R24.1.6b — wire loop

- [ ] **HashSet pre-pass (R23a clause 6, R33c case 5, NF-003):**
  ```rust
  let mut seen = HashSet::<(u8, u8)>::with_capacity(pc.local_wiring.len());
  for hint in &pc.local_wiring {
      if !seen.insert((hint.src_slot, hint.src_port)) {
          return Err(ProtocolError::MalformedLocalWiring {
              request_id: pc.request_id,
              reason: MalformedLocalWiringReason::DuplicateSourcePort {
                  src_slot: hint.src_slot,
                  src_port: hint.src_port,
              },
          });
      }
  }
  ```
  The pre-pass MUST run BEFORE any `Net::connect` call on this PC (atomic-batch behavior: a malformed batch is never partially applied).
- [ ] **For each `hint in pc.local_wiring` (emitted order — R23a determinism guarantee):**
  - Validate `hint.src_slot < pc.target_symbols.len()`; else reject R33c case 1 (`SrcSlotOutOfRange`).
  - Validate `hint.src_port` is in range for `pc.target_symbols[hint.src_slot]`:
    - CON: port 0 = principal, ports 1..=2 = aux (3 total ports).
    - DUP: port 0 = principal, ports 1..=2 = aux (3 total ports).
    - ERA: port 0 = principal only (1 total port).
    - Use `total_ports(symbol)` or `arity(symbol)` helpers from `net/types.rs`.
    - Else reject R33c case 2 (`SrcPortOutOfRange`).
  - Compute `source = PortRef::AgentPort(minted_ids_per_pc[hint.src_slot], hint.src_port)`.
  - **Target decoding (per hint.target):**
    - `PortRef::AgentPort(x, p)` where `x >= SLOT_MARKER_BASE`: slot-marker placeholder path (R23a clause 3). Compute `slot_idx = x - SLOT_MARKER_BASE`. If `slot_idx >= pc.target_symbols.len() as u32`, reject R33c case 3 / R48a (`TargetSiblingOutOfRange`). Otherwise compute resolved target `PortRef::AgentPort(minted_ids_per_pc[slot_idx as usize], p)`.
    - `PortRef::AgentPort(x, p)` where `x < SLOT_MARKER_BASE`: concrete-AgentId pass-through (R23a clause 4). Pass the id unchanged to `Net::connect`. In `debug_assertions`, assert the agent exists in the partition; in release, the `Net::connect` call may silently succeed on a dangling id — emit `tracing::warn!("R33c case 4 dangling concrete agent id={} port={}", x, p);` instead of returning an error.
    - `PortRef::FreePort(bid)`: reject R33c case 6 (`ReservedForFuture { border_id: bid }`). Cross-partition edges MUST go through `PendingNewBorder`, never `local_wiring`.
  - Call `net.connect(source, resolved_target)`. `Net::connect` automatically enqueues principal-principal redexes on `redex_queue` (net/core.rs L198-200). DO NOT drain the queue here.

### 4. R24.1.6c — echo

- [ ] After the wire loop completes (no errors), push `MintedAgent { request_id: pc.request_id, minted_agent_id: minted_ids_per_pc[0] }` onto the response buffer. This is UNCONDITIONAL — even if `pc.local_wiring` is empty (R48b), even if `reduce_all` will immediately consume the minted agents.
- [ ] The response buffer is eventually attached to `Message::RoundResult.minted_agents`. Existing wiring for that field already ships (TASK-0394 + TASK-0398).

### 5. R24 ordering invariant preservation

- [ ] **NO call to `reduce_all`, `reduce_step`, or any redex-draining function INSIDE the mint-then-wire loop.**
- [ ] After all PCs are processed, the partition is in an intermediate state: minted agents exist, edges are wired, `redex_queue` may have principal-principal pairs pending. Then and only then does the existing `reduce_all` invocation at R24.2 drain the queue.
- [ ] Assertion: add a `debug_assert!(net.redex_queue.len() >= initial_queue_len, "R24 ordering: redex_queue MUST only grow during mint-then-wire");` pre- and post-loop, where `initial_queue_len` is captured at the top of R24.1.6a.

### 6. Error mapping to wire

- [ ] Any `ProtocolError::MalformedLocalWiring` from the mint-then-wire loop MUST propagate up the call stack. The caller (likely the TCP message handler for `RoundStart`) converts it to `Message::RegisterNack { reason: ... }` and closes the session — consistent with R33c / R48 NACK behavior.
- [ ] Do NOT catch or silently log MUST-reject errors. Case 4 (DanglingConcreteAgent) is the ONLY SHOULD-warn case and is handled via `tracing::warn!` without returning Err.

## Key Types / Signatures

This task adds a private helper (likely `fn apply_pending_commutation(net: &mut Net, pc: &PendingCommutation, response_buffer: &mut Vec<MintedAgent>) -> Result<(), ProtocolError>`) inside `worker.rs`. No new public API. No new struct definitions (all consumed from TASK-0400).

```rust
// worker.rs — private helper
fn apply_pending_commutation(
    net: &mut Net,
    pc: &PendingCommutation,
    response_buffer: &mut Vec<MintedAgent>,
) -> Result<(), ProtocolError> {
    // R33c case 7 (ZeroArity): reject before mint.
    if pc.target_symbols.is_empty() {
        return Err(ProtocolError::MalformedLocalWiring {
            request_id: pc.request_id,
            reason: MalformedLocalWiringReason::ZeroArity,
        });
    }

    // R24.1.6a — mint.
    let mut minted_ids_per_pc = Vec::with_capacity(pc.target_symbols.len());
    for k in 0..pc.target_symbols.len() {
        minted_ids_per_pc.push(net.create_agent(pc.target_symbols[k]));
    }

    // R23a clause 6 pre-pass (R33c case 5 DuplicateSourcePort).
    let mut seen = HashSet::<(u8, u8)>::with_capacity(pc.local_wiring.len());
    for hint in &pc.local_wiring {
        if !seen.insert((hint.src_slot, hint.src_port)) {
            return Err(ProtocolError::MalformedLocalWiring {
                request_id: pc.request_id,
                reason: MalformedLocalWiringReason::DuplicateSourcePort {
                    src_slot: hint.src_slot,
                    src_port: hint.src_port,
                },
            });
        }
    }

    // R24.1.6b — wire.
    for hint in &pc.local_wiring {
        // Validate src_slot (R33c case 1).
        if (hint.src_slot as usize) >= pc.target_symbols.len() {
            return Err(ProtocolError::MalformedLocalWiring { /* case 1 */ });
        }
        // Validate src_port (R33c case 2).
        let src_symbol = pc.target_symbols[hint.src_slot as usize];
        if hint.src_port >= total_ports(src_symbol) {
            return Err(ProtocolError::MalformedLocalWiring { /* case 2 */ });
        }
        let source = PortRef::AgentPort(minted_ids_per_pc[hint.src_slot as usize], hint.src_port);
        // Decode target per R23a clauses 3/4/6.
        let resolved_target = match hint.target {
            PortRef::AgentPort(x, p) if x >= SLOT_MARKER_BASE => {
                let slot_idx = (x - SLOT_MARKER_BASE) as usize;
                if slot_idx >= pc.target_symbols.len() {
                    return Err(ProtocolError::MalformedLocalWiring { /* case 3 / R48a */ });
                }
                PortRef::AgentPort(minted_ids_per_pc[slot_idx], p)
            }
            PortRef::AgentPort(x, p) => {
                if net.get_agent(x).is_none() {
                    tracing::warn!("R33c case 4 dangling concrete agent id={} port={}", x, p);
                }
                PortRef::AgentPort(x, p)
            }
            PortRef::FreePort(bid) => {
                return Err(ProtocolError::MalformedLocalWiring {
                    request_id: pc.request_id,
                    reason: MalformedLocalWiringReason::ReservedForFuture { border_id: bid },
                });
            }
            PortRef::DISCONNECTED => {
                /* undefined — either reject case 6 or silently ignore. DC: reject. */
            }
        };
        net.connect(source, resolved_target);
    }

    // R24.1.6c — echo (slot-0 canonical).
    response_buffer.push(MintedAgent {
        request_id: pc.request_id,
        minted_agent_id: minted_ids_per_pc[0],
    });

    Ok(())
}
```

## Notes

- **Why do the pre-pass at PC granularity, not round granularity?** R23a clause 1 pins slot namespaces to per-PC scope. A `(src_slot, src_port)` tuple in PC A has no relationship to the same tuple in PC B. Round-granularity deduplication would be a semantic bug.
- **What if `Net::connect` panics on some unrelated invariant?** Out of scope for D-005. `Net::connect` is covered by SPEC-02 + existing tests; this task consumes it as-is.
- **`DanglingConcreteAgent` log site.** The spec says SHOULD-warn symmetric to the resolver's lenient dangling-FreePort path at `border_resolver.rs:1061-1077`. Implementation uses `tracing::warn!` (NOT `eprintln!` — project prohibits `println!`/`eprintln!`). Verify a tracing subscriber is active under `cargo test`; use `tracing_test::traced_test` attribute on UT-0402-04 to capture the warning.
- **In-round reducibility test (UT-0402-09).** Under R23a clause 5, a CON-DUP batch that connects two principal ports will produce an in-round redex. `reduce_all` at R24.2 will consume it. The UT asserts the redex is QUEUED but not DRAINED inside the mint-then-wire loop. Easy inspection: snapshot `net.reductions_performed` (existing counter) before and after the loop; assert `== 0`.
- **NR3-003 follow-through.** If TASK-0400 landed the optional `TargetSymbolsTooLong` case, add a matching validation in §2: before the mint loop, check `pc.target_symbols.len() <= 16` (the resolver-side cap) and reject. If deferred at TASK-0400, nothing to add here.
- **Worker session semantics on error.** Returning `Err(ProtocolError::MalformedLocalWiring)` from this function means the entire round is aborted AND the worker session is NACKed. This matches R33c's prose "the worker MUST abort the round, emit `RegisterNack { reason: MalformedLocalWiring(...) }`, and the coordinator MUST treat this as a protocol violation." Implementation confirms: the calling message-handler path in the worker already maps `ProtocolError` → `Message::RegisterNack` and closes the TCP connection. Verify this during Stage 4 REVIEW.
