# TASK-0351: Coordinator skip-merge logic + Global Normal Form termination (R3, R4, R5, R6, R7)

**Bundle:** SPEC-19 §3.1 — Coordinator-Free Round (item 2.34)
**Spec:** SPEC-19 R3 (track per-worker activity, MAY skip), R4 (Global
Normal Form termination), R5 (no result drift — guaranteed by T4 strong
confluence), R6 (SHOULD under strict BSP), R7 (compatible with both v1
full-partition protocol AND v2 wire format from SPEC-18).
**Priority:** P0 (the actual feature delivered by this bundle)
**Status:** TODO
**Depends on:** TASK-0348, TASK-0349, TASK-0350
**Blocked by:** TASK-0348, TASK-0349, TASK-0350
**Estimated complexity:** M (~130 LoC inc. tests)

## Context

This is the **payoff** task for the bundle. After TASK-0348/0349/0350,
the coordinator has everything it needs:

- Each `WorkerRoundStats` carries an accurate `has_border_activity: bool`.
- `GridConfig.coordinator_free_rounds: bool` opts in.
- `GridMetrics.coordinator_free_rounds: u32` is ready to record skips.

The coordinator-side change (in `merge::grid::run_grid`) is small but
load-bearing:

1. After collecting all `WorkerRoundStats` for a round, compute two
   booleans:
   - `all_no_border_activity = stats.iter().all(|s| !s.has_border_activity)`
   - `all_no_local_redexes = stats.iter().all(|s| s.local_redexes == 0)`
2. **R4 — Global Normal Form:** if BOTH are true, the net has converged
   in the distributed sense. `run_grid` MUST terminate (return the
   merged result) regardless of the previous round count.
3. **R3 — Skip Merge:** if `all_no_border_activity` is true but
   `all_no_local_redexes` is false (workers still have local work),
   AND `config.coordinator_free_rounds == true`, AND we are in
   `strict_bsp` mode (R6 SHOULD), the coordinator SKIPS the
   merge-redistribute cycle for this round, increments
   `metrics.coordinator_free_rounds`, and proceeds to the next round
   leaving each partition unchanged on the worker side. Workers reduce
   locally in the next round (no new partition is sent — they retain
   what they had, which under v1 protocol means "the same partition
   already lives in their local buffer for this round's continuation",
   per R7 v1 compat note in spec line 67-68).
4. **R5 — Confluence safety:** the skip is **safe** because no border
   redex exists (every worker reported `has_border_activity == false`,
   which means no local border port is principal — by definition no
   border redex can be triggered until a worker creates one via local
   reduction in a later round). Strong confluence (T4) guarantees the
   final Normal Form is invariant to whether merge ran or not.
5. **R7 — protocol compatibility:** `run_grid` operates in
   "local-simulation" mode (no real wire) for the in-process tests;
   the coordinator FSM in `protocol/coordinator.rs` operates in
   "real-wire" mode. The skip logic is implemented at the `run_grid`
   (local) level for this bundle. The wire-level coordinator FSM uses
   the SAME `WorkerRoundStats` field (TASK-0349 populates both paths)
   so when item 2.26 / 2.34-wire ships, the FSM gains the equivalent
   skip branch for free. **No `Message` variant changes are required
   in this task** (R7 v1 compat).

## Key Types / Signatures

```rust
// merge/grid.rs — pseudocode for the new branch in run_grid
let stats_this_round: &[WorkerRoundStats] = /* collected above */;

let (all_no_border, all_no_redexes) =
    check_global_normal_form(stats_this_round);

// R4: Global Normal Form — distributed convergence.
if all_no_border && all_no_redexes {
    // No worker has anything left to do AND no border redex can be
    // produced. Terminate (T4 strong confluence guarantees the result
    // is the unique Normal Form regardless of remaining merge step).
    metrics.converged = true;
    break;
}

// R3 + R6: skip merge under strict BSP when no border redex is possible.
if all_no_border
    && config.coordinator_free_rounds
    && config.strict_bsp
{
    // Confluence-safe (R5): no border redex can be triggered until a
    // worker creates one in a later round; partitions stay valid as-is.
    metrics.coordinator_free_rounds += 1;
    // Do NOT call merge() / reduce_all() / split() this round.
    // Workers retain their partitions (R7 v1 compat: under v1 protocol
    // this means the coordinator simply does not send AssignPartition
    // for this round; the local-simulation path here mirrors that by
    // skipping the merge-redistribute block entirely).
    round += 1;
    continue;
}

// Otherwise: existing v1 merge / reduce_all / split / dispatch path.
// (unchanged)
```

```rust
// merge/grid.rs — small testable helper
/// SPEC-19 §3.1 R3, R4: derive the two booleans the coordinator
/// uses to decide between (a) Global Normal Form termination,
/// (b) merge skip, (c) full v1 merge.
///
/// Returns `(all_no_border_activity, all_no_local_redexes)`.
pub(crate) fn check_global_normal_form(
    stats: &[WorkerRoundStats],
) -> (bool, bool) {
    let all_no_border = stats.iter().all(|s| !s.has_border_activity);
    let all_no_redexes = stats.iter().all(|s| s.local_redexes == 0);
    (all_no_border, all_no_redexes)
}
```

## Notes

- **No `Message` variant changes** in this task. The `has_border_activity`
  field rides inside the existing `Message::PartitionResult { stats, … }`
  payload (TASK-0349 already wired this).
- **No real-wire FSM change in `protocol/coordinator.rs`** for this
  bundle. The `run_grid` skip handles the local-simulation path (which
  is what every existing benchmark and integration test exercises).
  The wire-level FSM gains the same skip branch when item 2.26
  (Delta-Only Protocol) ships; the type plumbing is already in place
  thanks to TASK-0348/0349. Document this scope boundary in the
  `run_grid` code comment.
- **Confluence proof** (R5): the skip is SOUND because the predicate
  `all_no_border_activity` means "no worker has a principal port
  connected to a border" → no border redex can fire this round → merge
  would not produce any new redex queue entries → skipping is the
  identity transformation on the merged net. Strong confluence (T4)
  guarantees that any subsequent reduction order produces the same
  Normal Form. Cite SPEC-01 T4 and ARG-001 P2/P3 in the code comment.
- **R6 SHOULD** allows the optimization in either BSP mode; we restrict
  to `strict_bsp = true` for this bundle because lenient mode already
  collapses the cycle into 1 round on most inputs and adding skip logic
  there has zero observable benefit. This is a **documented design
  choice**, not a deviation from the spec.
- **DEFERRED-WORK note (NOT for this bundle):** R45's per-round
  delta byte / time vectors require coordinator-side aggregation and
  the BorderGraph; they ship with item 2.35 (BorderGraph) and 2.26
  (Delta-Only Protocol). If the reviewer flags this as a gap, file a
  DEFERRED-WORK row on bundle close.
