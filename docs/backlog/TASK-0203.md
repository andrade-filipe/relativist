## Context

`decode_nat(net: &Net) -> Option<u64>` performs structural traversal of a Church numeral IC net in Normal Form to extract the encoded natural number. It follows the algorithm specified in SPEC-14 Section 4.4:

1. From `net.root`, find the outer lambda CON agent
2. From `lam_f.p2`, find the inner lambda CON agent
3. Check for n = 0 case (self-loop on inner lambda, ERA on f)
4. Walk the application chain from `lam_x.p2`, counting CON application agents
5. Return the count as `Some(n)`, or `None` for unrecognizable structures

The function uses `net.get_target(port) -> PortRef` (SPEC-02) and checks against `DISCONNECTED` sentinel. A local `get_agent` helper accesses `net.agents`.

## Notes

- The decode algorithm in Section 4.4 provides complete Rust pseudocode. The key insight is that in Normal Form, `lam_x.p2` connects to the outermost application's `p2`, and following each application's `p1` leads to the next inner application's `p2`.
