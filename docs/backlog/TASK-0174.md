## Context

`dual_tree(depth)` produces two perfect binary trees of CON agents with the given depth, connected at the roots via principal ports (R39). This is the Profile C benchmark (SPEC-09, R13) and the most structurally complex generator. It tests the reduction engine's ability to handle tree-structured nets with cascading reductions.

A perfect binary tree of depth `d` has `2^d - 1` internal CON nodes. Two such trees connected at the roots create a net with `2 * (2^d - 1)` agents and a single initial redex (the root pair). Reduction expands into an exponential cascade of annihilation rules, eventually eliminating all agents and cross-connecting the leaves.

The Haskell prototype reference is `mkDualTreeNet` from AC-005.

## Acceptance Criteria

- [ ] `dual_tree(depth: u32) -> Net` generates two perfect binary trees of CON agents
- [ ] Each tree has depth `depth` (depth 1 = single root node, depth 2 = root + 2 children, etc.)
- [ ] The roots of the two trees are connected at principal ports (creating 1 redex)
- [ ] Leaf auxiliary ports are connected to free ports
- [ ] Tree structure: each internal CON node has `left` port connected to the left child's `principal`, and `right` port connected to the right child's `principal`
- [ ] Total agents: `2 * (2^depth - 1)` (two complete trees)
- [ ] Total initial redexes: 1 (the root pair)
- [ ] Output validates in debug mode (R34)
- [ ] `generate_example` dispatch is updated for the `DualTree` variant

## Key Types / Signatures

```rust
/// Generate two perfect binary trees of depth `depth`, connected at roots
/// (SPEC-12, R39; SPEC-09 R13).
///
/// Each tree has `2^depth - 1` CON agents. Roots connected at principal ports.
/// Leaf auxiliary ports connected to free ports.
/// After full reduction: 0 agents, leaves cross-reconnected.
pub fn dual_tree(depth: u32) -> Net {
    let mut net = Net::new();
    let mut free_id: u32 = 0;

    // Build tree A, get root agent ID
    let root_a = build_tree(&mut net, depth, &mut free_id);
    // Build tree B, get root agent ID
    let root_b = build_tree(&mut net, depth, &mut free_id);

    // Connect roots at principal ports
    net.connect(
        PortRef::AgentPort(root_a, 0),
        PortRef::AgentPort(root_b, 0),
    );

    net
}

/// Recursively build a perfect binary tree of CON agents.
/// Returns the AgentId of the root.
fn build_tree(
    net: &mut Net,
    depth: u32,
    free_id: &mut u32,
) -> AgentId {
    let node = net.create_agent(Symbol::Con);
    if depth <= 1 {
        // Leaf: connect auxiliary ports to free ports
        net.connect(PortRef::AgentPort(node, 1), PortRef::FreePort(*free_id));
        *free_id += 1;
        net.connect(PortRef::AgentPort(node, 2), PortRef::FreePort(*free_id));
        *free_id += 1;
    } else {
        // Internal: connect auxiliary ports to children's principal ports
        let left_child = build_tree(net, depth - 1, free_id);
        let right_child = build_tree(net, depth - 1, free_id);
        net.connect(
            PortRef::AgentPort(node, 1),
            PortRef::AgentPort(left_child, 0),
        );
        net.connect(
            PortRef::AgentPort(node, 2),
            PortRef::AgentPort(right_child, 0),
        );
    }
    node
}
```

## Notes

- The recursive `build_tree` helper may cause stack overflow for very large depths (e.g., depth > 30). For the TCC benchmarks, depths up to 15-20 are expected. A stack overflow guard or iterative implementation can be added as a P2 enhancement if needed.
- Depth 0 is an edge case. It could be defined as "no tree" (empty net) or "single node with free ports". The spec says "depth N" without defining depth 0. Implementation should handle it gracefully (e.g., `dual_tree(0)` produces a net with 0 agents or panics with a clear message).
- The Haskell prototype `mkDualTreeNet` in AC-005 builds trees bottom-up. The recursive top-down approach here is equivalent and more readable.
- Total leaf count per tree: `2^depth`. Total free ports: `4 * 2^(depth-1)` = `2^(depth+1)` for both trees combined.
