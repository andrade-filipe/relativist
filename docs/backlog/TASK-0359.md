## Context

SPEC-18 §7.2 enumerates four feature-gated tests (T11..T14) that prove the
rkyv path's correctness end-to-end, beyond the per-task unit tests already
specified by TASK-0353..0358. These four tests are the **acceptance gates
for closing DEFERRED-WORK D-002**, and they must live as a single coherent
test module so a reviewer can audit the spec-to-test mapping in one read.

All four tests are gated under `#[cfg(all(test, feature = "zero-copy"))]`
and run only with `cargo test --workspace --features zero-copy`. They live
alongside the existing protocol tests (e.g.,
`relativist-core/src/protocol/frame.rs` test module, or a new file
`relativist-core/src/protocol/zero_copy_tests.rs` under `#[cfg]` — developer
choice). Recommendation: a new file keeps the existing 1700+ line `frame.rs`
test module from growing further.

## Acceptance Criteria

- [ ] A new test module (file or inline `mod`) exists under
      `#[cfg(all(test, feature = "zero-copy"))]` with the four T11-T14 tests:
  - **T11 (`rkyv_round_trip_partition`):** for a realistic `Partition`,
    assert `rkyv::deserialize::<Partition, rkyv::rancor::Error>(
    rkyv::access::<ArchivedPartition, rkyv::rancor::Error>(
    rkyv::to_bytes::<rkyv::rancor::Error>(&p)?.as_slice())?)? == p` (R27).
  - **T12 (`rkyv_with_lz4_round_trip`):** same as T11, but pipe the archived
    bytes through `compress_payload` → `decompress_payload_aligned` (the
    helper introduced by TASK-0357) before `rkyv::access`. Assert the final
    deserialized value equals the original.
  - **T13 (`rkyv_validation_rejects_corruption`):** call
    `rkyv::access::<ArchivedAssignPayload, rkyv::rancor::Error>(&corrupted)`
    on a buffer obtained by flipping a single byte mid-archive. Assert the
    `Err` variant and assert that the wrapping `recv_frame` call returns
    `ProtocolError::ArchiveValidationFailed(reason)` whose `reason`
    contains the rkyv error's Display (tuple variant per spec-critic
    DC-2, 2026-04-16).

      sent one.

      `send_frame_v2` since R22 makes that impossible legitimately).
      Assert `recv_frame` returns
      `ProtocolError::ArchiveValidationFailed(reason)` per R22

//! SPEC-18 §7.2 — T11-T14 rkyv test suite.
//! Gated under `#[cfg(feature = "zero-copy")]`; runs only with
//! `cargo test --features zero-copy`.

#![cfg(all(test, feature = "zero-copy"))]

use crate::partition::Partition;
use crate::protocol::error::ProtocolError;
use crate::protocol::frame::{
    recv_frame, send_frame_v2, FLAG_ARCHIVED, FrameHeader, FRAME_HEADER_SIZE,
};
use crate::protocol::types::Message;

/// T11 (SPEC-18 §7.2 R27) — rkyv round-trip on a realistic Partition.
#[test]
fn rkyv_round_trip_partition() -> Result<(), Box<dyn std::error::Error>> {
    let p = sample_partition_for_zero_copy();
    let bytes = rkyv::to_bytes::<rkyv::rancor::Error>(&p)?;
    let archived = rkyv::access::<rkyv::Archived<Partition>, rkyv::rancor::Error>(
        bytes.as_slice(),
    )?;
    let round: Partition =
        rkyv::deserialize::<Partition, rkyv::rancor::Error>(archived)?;
    assert_eq!(p, round);
    Ok(())
}

/// T12 (SPEC-18 §7.2) — rkyv + LZ4 round-trip.
#[test]
fn rkyv_with_lz4_round_trip() -> Result<(), Box<dyn std::error::Error>> {
    let p = sample_partition_for_zero_copy();
    let archived = rkyv::to_bytes::<rkyv::rancor::Error>(&p)?;
    let compressed = crate::protocol::frame::compress_payload(archived.as_slice());
    let decompressed = crate::protocol::frame::decompress_payload_aligned(&compressed)?;
    let view = rkyv::access::<rkyv::Archived<Partition>, rkyv::rancor::Error>(
        decompressed.as_slice(),
    )?;
    let round: Partition =
        rkyv::deserialize::<Partition, rkyv::rancor::Error>(view)?;
    assert_eq!(p, round);
    Ok(())
}

/// T13 (SPEC-18 §7.2 R26) — rkyv validation rejects corruption.
#[test]
fn rkyv_validation_rejects_corruption() {
    let p = sample_partition_for_zero_copy();
    let mut bytes = rkyv::to_bytes::<rkyv::rancor::Error>(&p)
        .expect("test fixture serializes");
    let mid = bytes.len() / 2;
    bytes[mid] = bytes[mid].wrapping_add(0xAA); // flip
    let result = rkyv::access::<rkyv::Archived<Partition>, rkyv::rancor::Error>(
        bytes.as_slice(),
    );
    assert!(result.is_err(), "rkyv must reject the corrupted buffer");
}

/// T14a (SPEC-18 §7.2) — round-trip via the wire (FLAG_ARCHIVED legitimate).
#[tokio::test]
async fn archive_flag_legitimate_round_trip()
    -> Result<(), Box<dyn std::error::Error>>
{
    let (mut tx, mut rx) = tokio::io::duplex(64 * 1024);
    let original = Message::AssignPartition {
        round: 7,
        partition: sample_partition_for_zero_copy(),
    };
    send_frame_v2(&mut tx, &original, usize::MAX, true).await?;
    let received = recv_frame(&mut rx, 1 << 30).await?;
    assert_eq!(original, received);
    Ok(())
}

/// T14b (SPEC-18 §7.2 R22) — FLAG_ARCHIVED on Shutdown is rejected.
#[tokio::test]
async fn archive_flag_on_control_message_rejected() {
    let (mut tx, mut rx) = tokio::io::duplex(4096);
    // Manually craft: header with FLAG_ARCHIVED + arbitrary archived bytes
    // (we forge the bytes since send_frame_v2 would refuse to set
    // FLAG_ARCHIVED for Shutdown per R22).
    let archived_bytes = rkyv::to_bytes::<rkyv::rancor::Error>(&42u32)
        .expect("test fixture serializes");
    let payload = archived_bytes.as_slice();
    let header = FrameHeader {
        length: payload.len() as u32,
        checksum: crc32fast::hash(payload),
        flags: FLAG_ARCHIVED,

    use tokio::io::AsyncWriteExt;
    tx.write_all(&header.to_bytes()).await.expect("write header");
    tx.write_all(payload).await.expect("write payload");

    match result {
        Err(ProtocolError::ArchiveValidationFailed(_)) => {} // ok (tuple per DC-2)
        other => panic!("expected ArchiveValidationFailed, got {:?}", other),
    }
}

## Notes

- **Why a separate file (`zero_copy_tests.rs`) instead of inline:** the
  T11-T14 suite is a coherent block of evidence for D-002 closure. Putting
  it in its own file makes the spec-to-test traceability obvious and
  avoids growing `frame.rs` (already ~1700 lines including tests). The
  per-task unit tests from TASK-0354..0358 stay close to their respective
  module to keep refactor blast radius small.
- **`rkyv::Archived<Partition>` vs custom `ArchivedPartition`:** rkyv 0.8
  generates the `Archived<T>` associated type for any `T: Archive`.
  Spelling it as `rkyv::Archived<Partition>` is more portable than the
  generated type name `ArchivedPartition` (which depends on the macro's
  naming convention). Both work; the developer should pick one and be
  consistent.
- **T13's "single byte flip" caveat:** rkyv's validation primarily checks
  pointer / offset bounds and discriminant validity. Flipping a byte in
  the middle of a `u32` field may not be detected (it's still a valid
  `u32`, just a different value — round-trip correctness, not byte-level
  integrity, is what rkyv guarantees). T13 should target the rkyv root
  pointer (last 4-8 bytes of the archive) where corruption produces an
  out-of-bounds access. TEST-SPEC-0359 will pin the exact offset.
- **T14b's "forged archived Shutdown" caveat:** `send_frame_v2` won't
  legitimately produce this frame (R22 enforced on send), so the test
  uses the byte-level frame constructor. The test's purpose is the
  receiver's R22 enforcement, which is the half that can be exploited
  by a malicious or buggy peer. The forged payload is `42u32` (a
  conveniently small archived primitive) — the discrimination logic in
  `recv_frame` (try `ArchivedAssignPayload`, then
  `ArchivedPartitionResultPayload`) will fail both attempts and fall
  through to the rejection branch.
- **Closing D-002:** after this test suite is GREEN under
  `--features zero-copy` AND TASK-0358's CLI flags are wired AND the
  Stage 6 REFACTOR sign-off is recorded in `pipeline-state.md`,
  TASK-UPDATER edits `docs/DEFERRED-WORK.md` to mark D-002 closed with a
  pointer to TASK-0359's test count delta as evidence. That edit is
  explicitly out of scope here.
