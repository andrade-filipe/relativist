## Context

The top-level I/O API (R51) consists of four dispatch functions: `load_net`, `load_net_with_format`, `save_net`, `save_net_with_format`. These functions auto-detect the format from the file extension (R48) or accept an explicit format override (R50), then delegate to the appropriate format-specific implementation (binary, text, or JSON).

This task wires together the format detection logic (from TASK-0161) with the format-specific modules (TASK-0163 for binary, TASK-0165/0166 for text, TASK-0167 for JSON) into a unified API in `io/mod.rs`.

## Acceptance Criteria

- [ ] `load_net(path: &Path) -> Result<Net, FileIoError>` auto-detects format by extension and loads
- [ ] `load_net_with_format(path: &Path, format: NetFormat) -> Result<Net, FileIoError>` loads with explicit format
- [ ] `save_net(net: &Net, path: &Path) -> Result<(), FileIoError>` auto-detects format by extension and saves
- [ ] `save_net_with_format(net: &Net, path: &Path, format: NetFormat) -> Result<(), FileIoError>` saves with explicit format
- [ ] `.bin` extension dispatches to `binary::load_binary` / `binary::save_binary`
- [ ] `.ic` extension dispatches to `text::parse_ic` (load) / `text::save_ic` (save, which calls `format_ic` and writes)
- [ ] `.json` extension dispatches to `json::load_json` / `json::save_json`
- [ ] Unrecognized extension returns `FileIoError::UnrecognizedFormat` (R49)
- [ ] For `.ic` loading, the file is read as a string then passed to `parse_ic`
- [ ] For `.ic` saving, `format_ic` result is written to the file
- [ ] All four functions are re-exported from `io/mod.rs`

## Key Types / Signatures

```rust
use std::path::Path;
use crate::net::Net;

pub use types::{FileIoError, NetFormat, InspectOutputFormat};
pub use summary::{NetSummary, ReductionSummary};

/// Load a Net from a file, auto-detecting format by extension (SPEC-12, R51).
pub fn load_net(path: &Path) -> Result<Net, FileIoError> {
    let format = types::detect_format(path)?;
    load_net_with_format(path, format)
}

/// Load a Net from a file with explicit format (SPEC-12, R51).
pub fn load_net_with_format(path: &Path, format: NetFormat) -> Result<Net, FileIoError> {
    match format {
        NetFormat::Bin => binary::load_binary(path),
        NetFormat::Ic => {
            let contents = std::fs::read_to_string(path)?;
            text::parse_ic(&contents)
        }
        NetFormat::Json => json::load_json(path),
    }
}

/// Save a Net to a file, auto-detecting format by extension (SPEC-12, R51).
pub fn save_net(net: &Net, path: &Path) -> Result<(), FileIoError> {
    let format = types::detect_format(path)?;
    save_net_with_format(net, path, format)
}

/// Save a Net with explicit format (SPEC-12, R51).
pub fn save_net_with_format(
    net: &Net,
    path: &Path,
    format: NetFormat,
) -> Result<(), FileIoError> {
    match format {
        NetFormat::Bin => binary::save_binary(net, path),
        NetFormat::Ic => {
            let text = text::format_ic(net);
            std::fs::write(path, text)?;
            Ok(())
        }
        NetFormat::Json => json::save_json(net, path),
    }
}
```

## Notes

- This is the "glue" task that ties all format implementations together. It is the last step in the I/O pipeline before CLI integration.
- The text DSL load path reads the entire file into a string before parsing. This is acceptable because `.ic` files are intended for small, hand-crafted nets (not large benchmarks).
- If JSON is deferred (TASK-0167 ships as stub), the `Json` branch will return `FileIoError::UnsupportedFormat`, which is the correct behavior per R17.
