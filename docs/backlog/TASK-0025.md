## Context

SPEC-03 Sections 4.1.5 and 4.1.6 define the erasure rules. When an ERA agent meets an arity-2 agent (CON or DUP) through their principal ports, both are destroyed and 2 new ERA agents are created, one for each auxiliary port of the arity-2 agent. This "propagates" erasure through the sub-net.

The topology is identical for CON-ERA and DUP-ERA because it depends only on the arity of the non-ERA agent (always 2). Both are unified into `interact_eras`, following the Haskell prototype pattern (AC-001, `ruleErase`).

Agent balance: 0 (2 destroyed, 2 created). Link calls: 2. Up to 2 new redexes may form (erasure cascade).

The function assumes normalization: the first argument is the arity-2 agent (Con or Dup), the second is ERA. This is guaranteed by `normalize_pair` (Con=0 < Era=2, Dup=1 < Era=2).

**FreePort behavior (R26):** In partitioned sub-nets, old neighbors `a1`/`a2` may be `FreePort(bid)` boundary sentinels. The `link` helper handles this correctly: `connect` writes `FreePort(bid)` to the new ERA agent's port array slot, but the FreePort side is a no-op in `set_port`. Invariant I1 is temporarily violated; restored after `rebuild_free_port_index` (SPEC-05).

## Key Types / Signatures

```rust
/// Erasure: ERA encounters an arity-2 agent (CON or DUP)
/// (SPEC-03 Sections 4.1.5, 4.1.6).
///
/// Propagates erasure: creates 2 new ERA agents on the auxiliary ports
/// of the arity-2 agent. Initiates an erasure cascade that terminates
/// when ERA meets other ERA (void rule) or reaches free ports.
///
/// Precondition: both agent IDs MUST refer to live agents
///   (agents[id].is_some()). node_id MUST be Con or Dup, era_id
///   MUST be Era (normalized by normalize_pair). This precondition
///   is guaranteed by reduce_step's validity check (R12).
/// Postcondition: both removed; 2 new ERA connected to old neighbors.
///
/// Agent balance: 0 (2 destroyed, 2 created). Link calls: 2.
/// New redex detection: up to 2 (if old neighbors are principal ports).
/// Complexity: O(1) amortized (create_agent may trigger Vec realloc).
///
/// Invariants preserved: T1 (2 auxiliary ports of the arity-2 agent are
/// replaced by principal ports of 2 new ERA agents), I3 (2 new IDs via
/// create_agent).
pub fn interact_eras(net: &mut Net, node_id: AgentId, era_id: AgentId) {
    let a1 = net.get_target(PortRef::AgentPort(node_id, 1));
    let a2 = net.get_target(PortRef::AgentPort(node_id, 2));

    net.remove_agent(node_id);
    net.remove_agent(era_id);

    // Create 2 new ERA agents
    let e1 = net.create_agent(Symbol::Era);
    let e2 = net.create_agent(Symbol::Era);

    // Connect new ERA principal ports to old neighbors
    // Note: a1/a2 may be FreePort(bid) in partitioned sub-nets (R26).
    link(net, PortRef::AgentPort(e1, 0), a1);
    link(net, PortRef::AgentPort(e2, 0), a2);
}
```

## Notes

- The algorithm reads neighbors BEFORE removing agents (Pattern T7 from AC-001). This is critical because `remove_agent` disconnects ports.
- Only the auxiliary ports of `node_id` (the arity-2 agent) are read. ERA has arity 0, so `era_id` has no auxiliary ports to read.
- The function does NOT inspect `node_id`'s symbol. It works identically for Con and Dup because the erasure topology depends only on arity (always 2 for both).
- **SPEC-03 v3 change:** Uses `link()` helper instead of `net.connect()` directly for external wires. While the removed-agent guard (R25) is never triggered for erasure (auxiliary ports of the active pair always point outside the pair), the `link` helper also provides uniform handling of FreePort targets (R26) in partitioned sub-nets. Complexity updated from O(1) to O(1) amortized per R20 (Vec reallocation from create_agent).
