## Context

In debug mode (`#[cfg(debug_assertions)]`), the net must verify its representation invariants after each mutation. These assertions catch bugs early during development and testing by validating:

- **I1 (Bidirectional port array, R18):** If `ports[a] == b`, then `ports[b] == a`. This implements the linearity invariant T1.
- **I2 (Reference validity, R19):** Every `AgentPort(id, p)` reference in the port array points to an existing agent with `p <= arity(agent.symbol)`.
- **I3 (Monotonic IDs):** `next_id` is strictly greater than any `AgentId` in use.

Additionally, a `count_stale_redexes` diagnostic function and a convenience `assert_all_invariants` that calls all checks.

In release mode (R21), these assertions are compiled out for performance.

## Acceptance Criteria

- [ ] `assert_adjacency_consistent()` verifies I1 for all live agents, panics with a descriptive message on violation
- [ ] `assert_refs_valid()` verifies I2 for all live agents, panics with a descriptive message on violation
- [ ] `assert_next_id_valid()` verifies I3 for all agent slots, panics with a descriptive message on violation
- [ ] `count_stale_redexes() -> usize` counts stale entries in the redex queue
- [ ] `assert_all_invariants()` calls all three assertion functions
- [ ] All functions are gated by `#[cfg(debug_assertions)]`
- [ ] In release mode, none of these functions exist (compile-time elimination)

## Key Types / Signatures

```rust
#[cfg(debug_assertions)]
impl Net {
    /// Verifies I1: bidirectionality of the port array.
    pub fn assert_adjacency_consistent(&self) {
        for slot in self.agents.iter() {
            if let Some(agent) = slot {
                let num_ports = total_ports(agent.symbol);
                for p in 0..num_ports {
                    let port = PortRef::AgentPort(agent.id, p);
                    let target = self.get_target(port);
                    if target == DISCONNECTED { continue; }
                    let reverse = self.get_target(target);
                    assert_eq!(reverse, port,
                        "I1 violated: {:?} -> {:?}, but {:?} -> {:?}",
                        port, target, target, reverse);
                }
            }
        }
    }
```

## Notes

- The `#[cfg(debug_assertions)]` gate means these functions only exist in debug/test builds. In release builds (`cargo build --release`), the entire `impl` block is removed at compile time, adding zero overhead (R21).
- The `debug.rs` file needs to import types from `super::types` and use `super::net::Net`.
- These assertions are called by the reduction engine (SPEC-03) after each rule application. Getting them right here prevents bugs from propagating downstream.
