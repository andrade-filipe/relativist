## Context

This task lands the two **large-payload** delta-protocol variants:

- `InitialPartition { round: u32, partition: Partition }` at
  discriminant **7** — coordinator → worker. Sent exactly once at round 0
  to give each worker its initial partition; thereafter the worker
  holds state locally and rounds carry only deltas.
- `FinalStateResult { round: u32, partition: Partition }` at
  discriminant **11** — worker → coordinator. Sent during Final State
  Collection (R27-R29) so the coordinator can assemble the final merged
  net.

Both payloads carry a full `Partition` (the same type used by v1's
`AssignPartition` / `PartitionResult`, already `Serialize + Deserialize`
via SPEC-04 / SPEC-06). Per R35, these payloads MUST benefit from
SPEC-04 `CompactSubnet` encoding (which is already baked into
`Partition`'s serde impl), bincode v2 varint encoding (already applied
per SPEC-18 R1-R3), and the optional LZ4 compression path in
`send_frame_v2` / `send_frame_with_threshold` (SPEC-18 R9).

**No wire-level integration in this task.** The wire integration
(`send_frame_v2` round-trip, CRC32, compression threshold) lands in
TASK-0370. This task only ships the enum variants + in-memory bincode
round-trip tests, which is sufficient to exercise `#[derive(Serialize,
Deserialize)]` correctness per R34.

## Notes

- **Why no CRC32 test here:** CRC32 lives at the frame layer
  (`send_frame_v2`), not the bincode layer. The CRC integration test
  ships in TASK-0370.
- **Why no compression test here:** LZ4 compression is applied by
  `send_frame_with_threshold`, also frame-layer. Test ships in
  TASK-0370.
- **Why no `PartialEq` on `Message`:** the existing enum intentionally
  doesn't derive it because `Net` doesn't — adding it would cascade
  across the full `net/` module, which is out of scope for a wire-
  protocol task. Tests match via `match { variant_pattern => { ... } }`
  (same style as existing tests).
- **R35 compliance:** `Partition` already serialises via SPEC-04's
  `CompactSubnet` path (confirmed by the existing
  `test_message_with_agents` test round-tripping a partition with a
  real `Net`). No additional work in this task. The LZ4 layer is
  automatically engaged by `send_frame_with_threshold` when the
  encoded payload exceeds the configured threshold; R35 compliance is
  verified at the wire layer in TASK-0370.
- **Discriminant 7 vs 11 ordering:** R31 places `InitialPartition` at
  7 and `FinalStateResult` at 11 (SPEC-19 §3.4 tables). The enum source
  order must be: InitialPartition (7), RoundStart (8), RoundResult (9),
  FinalStateRequest (10), FinalStateResult (11). This task inserts 7
  and 11; TASK-0368 inserts 8 and 10; TASK-0369 inserts 9. To keep the
  enum compilable at each task boundary, the three tasks should land
  in one PR OR the developer should add placeholder markers (not
  source variants) in the intermediate states. Simplest: squash all
  three tasks' variant-addition diffs into a single commit within the
  Stage 3 DEV step.
