## Context

SPEC-03 Section 4.1.4 defines the commutation rule: CON-DUP. This is the ONLY rule that INCREASES the number of agents in the net (agent balance +2). When a CON and a DUP meet through their principal ports, both are destroyed and 4 new agents are created: 2 DUP + 2 CON. The new agents are connected in a crossed configuration with 8 wires (4 external, 4 internal).

This rule is the engine of computation expansion. Without it, nets would only shrink. The granularity of partitioning depends directly on how many CON-DUP expansions have occurred.

Agent balance: +2 (2 destroyed, 4 created). Link calls: 8 (4 external via `link`, 4 internal via `net.connect`). Up to 4 new redexes may form (from external wires connecting principal ports to context).

The function assumes normalization: the first argument is CON, the second is DUP. This is guaranteed by `normalize_pair` (Con=0 < Dup=1).

**FreePort behavior (R26):** In partitioned sub-nets, old neighbors (`a1`, `a2`, `b1`, `b2`) may be `FreePort(bid)` boundary sentinels. The external wires use `link()` to handle this correctly. Internal wires are always AgentPort-to-AgentPort (freshly created agents), so they use `net.connect()` directly for clarity and efficiency.

**Vec reallocation safety:** `PortRef` values are index-based (AgentId + PortId), not pointer-based. Vec reallocation during `create_agent` does NOT invalidate previously read PortRef values because they store integer indices, not raw pointers.

```rust
/// Commutation: CON and DUP commute, creating 4 new agents
/// (SPEC-03 Section 4.1.4).
///
/// This is the ONLY rule that INCREASES the number of agents (balance +2).
/// Creates 2 new DUP + 2 new CON in a crossed configuration.
///
/// Precondition: both agent IDs MUST refer to live agents
///   (agents[id].is_some()). con_id MUST be Con, dup_id MUST be Dup
///   (normalized by normalize_pair). This precondition is guaranteed
///   by reduce_step's validity check (R12).
/// Postcondition: CON and DUP removed; 2 new DUP + 2 new CON created
///   and reconnected. PortRef values are index-based, not pointer-based:
///   Vec reallocation during create_agent does NOT invalidate previously
///   read PortRef values.
///
/// Agent balance: +2. Link calls: 8 (4 external + 4 internal).
/// New redex detection: up to 4 (from external wires).
/// Complexity: O(1) amortized (create_agent may trigger Vec realloc).
///
/// Invariants preserved: T1 (all 12 ports of 4 new agents connected),
/// I1 (each connect writes both sides), I3 (4 new IDs via create_agent).
pub fn interact_comm(net: &mut Net, con_id: AgentId, dup_id: AgentId) {
    let a1 = net.get_target(PortRef::AgentPort(con_id, 1));
    let a2 = net.get_target(PortRef::AgentPort(con_id, 2));
    let b1 = net.get_target(PortRef::AgentPort(dup_id, 1));
    let b2 = net.get_target(PortRef::AgentPort(dup_id, 2));

    net.remove_agent(con_id);
    net.remove_agent(dup_id);

    // Create 4 new agents: 2 DUP + 2 CON
    let p = net.create_agent(Symbol::Dup);   // DUP: inherits side of con.1
    let q = net.create_agent(Symbol::Dup);   // DUP: inherits side of con.2
    let r = net.create_agent(Symbol::Con);   // CON: inherits side of dup.1
    let s = net.create_agent(Symbol::Con);   // CON: inherits side of dup.2

    // External wires: principal ports of new agents <-> old neighbors
    // Note: old neighbors (a1, a2, b1, b2) may be FreePort(bid) in
    // partitioned sub-nets. The link helper handles this correctly:
    // connect writes FreePort to the AgentPort side's port array,
    // and the FreePort side is a no-op in set_port (R26).
    link(net, PortRef::AgentPort(p, 0), a1);
    link(net, PortRef::AgentPort(q, 0), a2);
    link(net, PortRef::AgentPort(r, 0), b1);
    link(net, PortRef::AgentPort(s, 0), b2);

    // Internal wires: auxiliary ports in crossed configuration
    // These are always AgentPort-to-AgentPort (never FreePort).
    net.connect(PortRef::AgentPort(p, 1), PortRef::AgentPort(r, 1));
    net.connect(PortRef::AgentPort(p, 2), PortRef::AgentPort(s, 1));
    net.connect(PortRef::AgentPort(q, 1), PortRef::AgentPort(r, 2));
    net.connect(PortRef::AgentPort(q, 2), PortRef::AgentPort(s, 2));
}
```

## Notes

- The crossed internal wiring is the key topological feature. `p` (DUP) connects its aux ports to BOTH new CONs (`r` and `s`), not to the same one. Similarly for `q`. This ensures the correct duplication semantics.
- The 4 internal wires connect auxiliary ports (port 1, port 2), NOT principal ports (port 0). Therefore, none of the internal wires can produce new redexes. Only the 4 external wires (which connect principal ports) can create new Active Pairs.
- This is the most complex rule in terms of operations (8 connects, 4 creates, 2 removes). It is still O(1) amortized because all operations are O(1) amortized.
- For Grid Computing: this rule is responsible for work expansion. Pre-reduction CON-DUP expansions increase the net's parallelism, enabling more effective partitioning.
- **SPEC-03 v3 change:** External wires use `link()` helper instead of `net.connect()` directly, to uniformly handle FreePort targets (R26). Internal wires remain `net.connect()` since they are always AgentPort-to-AgentPort. Complexity updated from O(1) to O(1) amortized per R20 (Vec reallocation). Vec reallocation safety note added to docstring. Explicit preconditions added.
