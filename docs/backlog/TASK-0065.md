## Context

The `merge` function is the core of SPEC-05. It recombines locally-reduced partitions back into a single Net. This task implements Steps 1-2 of the merge algorithm (SPEC-05, Section 4.2): computing capacity, uniting agents from all partitions into a single arena, and copying internal connections. Boundary connections are NOT restored here (that is TASK-0066). The function skeleton is created here with the full signature; Step 3 (boundary restoration) is added in the next task.

The merge accepts `PartitionPlan` by value (consuming both partitions and borders in a single move, avoiding Rust partial-move issues) and returns `(Net, u32)` where the u32 is the border redex count (populated in TASK-0066). Internally, it destructures: `let PartitionPlan { partitions, borders, .. } = plan;`

## Notes

- Static ID space partitioning (SPEC-04 R16-R19) guarantees no AgentId collisions. The merge does NOT need to check for collisions or perform remapping. This is a major simplification over the Haskell prototype.
- The port array must be expanded to accommodate agents from all partitions. The engineer should ensure the result port array has size `max_next_id * 3` (or the maximum AgentId * 3 + 3).
- Partition redex queues are discarded during merge. After `reduce_all`, they contain only stale entries (R9). New border redexes are detected by `connect` during boundary restoration (Step 3).
- **SPEC-05 v3 changes (SC-003, SC-005, SC-007, SC-012):** Merge now accepts `PartitionPlan` by value (not separate args). Step 2 distinguishes Boundary vs Lafont FreePorts using border map as discriminator. Debug assertion on partition queue staleness added.
