## Context

`recv_frame` is the core function for reading a message from the TCP socket. It reads the 8-byte header, extracts the length and checksum, validates the payload size against `max_payload_size` (defense against OOM), reads the payload, verifies the CRC32C checksum, and deserializes the message with bincode. It returns the deserialized `Message` and the total bytes read (for network metrics). The function is generic over `AsyncReadExt` to support both TCP streams and in-memory buffers.

## Acceptance Criteria

- [ ] `recv_frame` is implemented in `src/protocol/frame.rs`
- [ ] Function is async and generic over `R: AsyncReadExt + Unpin`
- [ ] Reads exactly 8 header bytes using `read_exact`
- [ ] Extracts length and checksum from header (little-endian u32)
- [ ] Rejects payload if `length > max_payload_size` with `ProtocolError::PayloadTooLarge` BEFORE allocating memory (R9)
- [ ] Reads exactly `length` bytes of payload using `read_exact`
- [ ] Verifies CRC32C checksum; rejects with `ProtocolError::ChecksumMismatch` if mismatch (R29)
- [ ] Deserializes payload with `bincode::deserialize` to `Message`
- [ ] Returns `Ok((message, total_bytes))` where `total_bytes = FRAME_HEADER_SIZE + length`

```rust
use tokio::io::AsyncReadExt;

/// Reads a frame from the TCP socket and deserializes the message.
///
/// Returns the deserialized message and the total number of bytes read.
///
/// Steps:
/// 1. Read exactly 8 header bytes.
/// 2. Extract length and checksum from the header.
/// 3. Reject if length > max_payload_size (defense against OOM).
/// 4. Read exactly `length` bytes of payload.
/// 5. Verify CRC32C of payload against header checksum.
/// 6. Deserialize payload with bincode -> Message.
pub async fn recv_frame<R: AsyncReadExt + Unpin>(
    reader: &mut R,
    max_payload_size: u32,
) -> Result<(Message, usize), ProtocolError>
```

## Notes

- The size check (`length > max_payload_size`) MUST happen BEFORE `vec![0u8; length as usize]` to prevent an attacker or corrupt frame from causing the receiver to allocate up to 4 GiB of memory.
- `read_exact` handles TCP fragmentation internally (loops until all bytes are read or an error occurs). This replaces the manual `recvExact` loop in the Haskell prototype (AC-003).
- The order of operations (check size -> read payload -> verify checksum -> deserialize) is critical for security and correctness.
