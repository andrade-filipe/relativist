# TASK-0379: Worker persistent state `WorkerDeltaState` struct (R22, R25)

**Bundle:** SPEC-19 §3.3 — Stateful Worker Lifecycle (item 2.26-C)
**Spec:** SPEC-19 R22 (persistent partition across rounds), R25 (`previous_border_state` map)
**Priority:** P0 (foundational — every other task in 2.26-C reads or writes this struct)
**Status:** TODO
**Depends on:** none at SPLITTING time. At DEV time requires 2.26-A (wire types) merged.
**Blocked by:** 2.26-A TASK-0366..0371 (for compile-time access to `Partition`/`PortRef`
  imports already exist; only `crate::protocol::BorderDelta` re-export requires 2.26-A).
**Estimated complexity:** S (~100 LoC inc. tests)

## Context

R22 of SPEC-19 mandates that the worker retain its partition in local state across
BSP rounds: "the worker's partition state MUST survive across rounds. The worker MUST
NOT require a full partition retransmission for any subsequent round."

R25 mandates that the worker track the last-reported endpoint for each border ID in
a `previous_border_state: HashMap<u32, PortRef>`, so that in each round it can diff
the freshly rebuilt `free_port_index` against the previous report and emit only
changed entries as `BorderDelta`s.

This task introduces the struct and its constructors. **It does NOT wire the struct
into any runtime path** — that happens in TASK-0380 (Round 0 handler) and TASK-0381
(delta-round handler). The goal here is a pure-data type with clear ownership and
testable construction.

## Key Types / Signatures

```rust
// relativist-core/src/worker.rs — append after existing v1 FSM types

use std::collections::HashMap;
use crate::net::PortRef;

/// SPEC-19 R22, R25: persistent state a stateful worker retains across
/// BSP rounds in delta mode (`GridConfig.delta_mode = true`).
///
/// v1 workers are stateless: each `AssignPartition` delivers a fresh
/// `Partition` and the worker returns the reduced result via
/// `PartitionResult`. Under the delta protocol, the worker receives the
/// partition ONCE at Round 0 (`Message::InitialPartition`) and thereafter
/// receives only `BorderDelta`s in `Message::RoundStart`. The worker
/// reduces its stored partition in-place and reports only changed border
/// endpoints in `Message::RoundResult`.
///
/// The `previous_border_state` map holds the last-reported endpoint for
/// every border ID, used by TASK-0382's delta computation to emit only
/// changed entries (R25). It is seeded at Round 0 from
/// `partition.free_port_index` so that Round 1's first delta-dispatch
/// reports only borders that local reduction actually moved.
///
/// This struct is never serialized; only the `partition` field crosses
/// the wire (via `Message::InitialPartition` at Round 0 and
/// `Message::FinalStateResult` at convergence).
#[derive(Debug, Clone)]
pub struct WorkerDeltaState {
    pub partition: Partition,
    pub previous_border_state: HashMap<u32, PortRef>,
    pub round: u32,
}

impl WorkerDeltaState {
    /// Initialize from the Round-0 `InitialPartition` payload. Stores
    /// the partition and seeds `previous_border_state` from its
    /// `free_port_index` (DC-C4 option (b)).
    pub fn from_initial_partition(partition: Partition) -> Self {
        let previous_border_state = partition.free_port_index.clone();
        Self {
            partition,
            previous_border_state,
            round: 0,
        }
    }
}
```

## Dependencies Context

- `Partition` — defined in `crate::partition::Partition`. Already imported
  at the top of `worker.rs` (v1 FSM uses it in `WorkerEvent::ReceivePartition`).
- `PortRef` — defined in `crate::net::types::PortRef`. Imported on demand
  in this task.
- `Partition.free_port_index: HashMap<u32, PortRef>` — the seed source.
  Populated by `split()` at partitioning time and rebuilt by
  `rebuild_free_port_index()` after each local reduction.

## Notes

- **No wire-format impact.** `WorkerDeltaState` is in-process only. The
  `partition` field is serialized indirectly when the worker answers
  `FinalStateRequest` (via `Message::FinalStateResult`, shipped under
  2.26-A TASK-0370).
- **Pure-core compliance:** `worker.rs` is not pure-core (it sits at
  the crate root alongside `coordinator.rs` and is permitted to use
  tokio). This task does not introduce any tokio dependency — the
  struct is pure data. Runtime glue is added in TASK-0380 / TASK-0381.
- **No FSM changes.** The existing v1 `WorkerContext` / `transition`
  / `WorkerState` enum are not touched. TASK-0380 may add a new enum
  variant `WorkerState::DeltaActive { state: WorkerDeltaState }` or
  may store the delta state as a sibling field on `WorkerContext`;
  that design choice is deferred to TASK-0380 so this task can ship
  a minimal type.
- **DC-C4 rationale:** seeding from `free_port_index` at storage time
  avoids a Round 1 redundant re-dispatch of border endpoints the
  coordinator already knows (the coordinator's `BorderGraph` was
  initialized from the same `PartitionPlan`). An empty-seed alternative
  (DC-C4 option (a)) is simpler but wastes one round's bandwidth.
  Spec-critic ruling on DC-C4 may amend this task.

## Spec-Critic Amendments (2026-04-17)

| DC | Verdict | Change |
|---|---|---|
| DC-C4 | B (ratified) | Seed `previous_border_state` from `partition.free_port_index` at storage time (NOT empty map) |

**Details:**
- Ratified by spec-critic per DC-C4 of `docs/spec-reviews/SPEC-19-section-3.3-2.26C-design-choices-2026-04-17.md`.
- Seeding from `free_port_index` is the semantic identity with the coordinator's `BorderGraph::from_partition_plan` initialization — at Round 0 coordinator and worker hold identical knowledge of border endpoints.
- Rationale hardens from "may amend" to "ratified": the "last-reported" wording in R25 is from the coordinator's perspective, and the coordinator has already SEEN the initial state via `InitialPartition.partition`.
- Current `TASK-0379.md` draft at L119 (`let previous_border_state = partition.free_port_index.clone();`) already implements the ratified option. No code change required.
- Seeding semantics already covered by `workerdeltastate_from_initial_partition_seeds_previous_border_state` and `workerdeltastate_from_initial_partition_empty_freeports` tests.
