## Context

After the merge produces the result net, SPEC-05 R11 requires that in debug mode (`#[cfg(debug_assertions)]`), the result net is verified with `assert_all_invariants()` (SPEC-02, Section 4.6). This checks at minimum: T1 (linearity/bidirectionality), I1 (bidirectional port array), I2 (reference validity). R10 requires the merge to satisfy invariant D1 (split/merge identity). The debug assertions are the runtime check for this.

## Acceptance Criteria

- [ ] After merge completes (end of the `merge` function), a `#[cfg(debug_assertions)]` block calls `assert_all_invariants()` on the result net (R11)
- [ ] The assertion checks T1 (linearity), I1 (bidirectionality), I2 (reference validity)
- [ ] Assertions are compiled out in release mode (R11 uses `#[cfg(debug_assertions)]`)
- [ ] The module compiles with `cargo check` in both debug and release modes

## Notes

- This is a configurable feature per the project guideline: features with runtime cost (logging, assertions) must be configurable and disableable. Using `#[cfg(debug_assertions)]` satisfies this requirement.
- The assertion runs AFTER boundary restoration but BEFORE returning. At this point, all FreePort sentinels should be resolved, and the net should be a valid IC net.
- If any FreePort entries remain unresolved (indicating a bug), the assertion should catch the reference validity violation (I2).
