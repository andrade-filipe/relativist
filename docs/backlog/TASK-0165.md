## Context

This task implements Pass 2 of the text DSL parser (SPEC-12 Section 4.1) and the top-level `parse_ic` function (R51). Pass 2 takes the intermediate data structures from Pass 1 (TASK-0164) -- the agent declaration map and wire/root lists -- and constructs a `Net` by calling `create_agent` and `connect` operations.

After construction, the parser validates the net against invariants T1 (port linearity), I2 (reference validity), I6 (ERA auxiliary slot cleanliness), and I7 (root port consistency) from SPEC-01 Revised v3 (R11). If validation fails, the parser returns a descriptive error with the line number of the offending declaration.

R9 is enforced here: ERA agents with auxiliary port references (`left`, `right`, `p1`, `p2`) are rejected during wire resolution.

## Acceptance Criteria

- [ ] `parse_ic(input: &str) -> Result<Net, FileIoError>` is the top-level entry point that runs both passes and returns a valid `Net`
- [ ] Pass 2 creates agents via `Net::create_agent` in declaration order (R10)
- [ ] Pass 2 resolves `ParsedPortRef::AgentPort` to `PortRef::AgentPort(id, port_id)` using the declaration map
- [ ] Pass 2 resolves `ParsedPortRef::FreePort(n)` to `PortRef::FreePort(n)`
- [ ] ERA agents with auxiliary port references (`left`, `right`, `p1`, `p2`) are rejected with an error including the line number (R9)
- [ ] Unknown agent names in wire/root declarations produce an error with line number
- [ ] After construction, the net is validated for port linearity (T1), reference validity (I2), ERA auxiliary slot cleanliness (I6), and root port consistency (I7)
- [ ] Validation errors include the line number of the offending wire declaration when possible
- [ ] `root` declarations set the net's `root` field to the resolved `PortRef`
- [ ] At most one `root` declaration per file; duplicate produces parse error: `"duplicate root declaration at line {line}"` (R54)
- [ ] No `root` declaration means `net.root = None` (R55)
- [ ] Root port reference must be valid; both AgentPort and FreePort permitted (R56)
- [ ] Root is not a wire and not counted in wire count (R57)
- [ ] Self-loop wires (`wire a.left a.left`) are rejected with parse error: `"port cannot be connected to itself at line {line}"` (R58)
- [ ] Free-to-free wires (`wire free(0) free(1)`) are rejected with parse error: `"free-to-free wires are not supported; at least one endpoint must be an agent port, at line {line}"` (R59)

## Notes

- The validation step after construction uses the same invariant checks that `debug_assert` guards in the `net` module (TASK-0015), now including I6 (ERA auxiliary slot cleanliness) and I7 (root port consistency) per SPEC-01 Revised v3. However, the parser validates unconditionally (not just in debug mode) because user input is untrusted.
- The `root` declaration is optional. If absent, `net.root` remains `None` (R55).
- Multiple `root` declarations in the same file MUST produce an error: "duplicate root declaration at line {line}" (R54).
- Root port reference can be either AgentPort or FreePort (R56). FreePort roots count toward free port count (R57).
- Self-loop detection (R58): both port references in a `wire` declaration are compared; if identical, reject.
- Free-to-free rejection (R59): if both port references are `FreePort`, reject. The port array has no slots for free ports.
- The `parse_ic` function is the public API (R51); internal functions (`parse_lines`, `collect_declarations`, `build_net`, `resolve_port_ref`) may be `pub(crate)` or private.
