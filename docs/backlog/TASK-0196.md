## Context

SPEC-09 Section 3.7 (R39a-R40, R42) defines the CSV output format for benchmark results. Three CSV files are produced (Revised v3, SC-005):

1. **detail.csv** (R39a): One row per (benchmark, input_size, workers, mode, repetition). Scalar aggregates per execution.
2. **rounds.csv** (R39b): One row per (benchmark, input_size, workers, mode, repetition, round). Per-round phase breakdown. Only populated for distributed modes.
3. **summary.csv** (R40): One row per configuration. Aggregated statistics with median fields and optional bootstrap CI columns.

All CSVs must be compatible with Python (pandas), R, or spreadsheets: comma delimiter, UTF-8 encoding, dot for decimals.

use std::fs::File;
use std::io::{self, Write, BufWriter};
use std::path::Path;
use super::{BenchmarkResult, AggregatedStats};

/// Write detail CSV (R39a): one row per BenchmarkResult (one per repetition).
///
/// Columns (SPEC-09 R39a):
/// benchmark,input_size,mode,workers,repetition,correct,
/// wall_clock_secs,total_interactions,mips,
/// rounds,speedup,efficiency,overhead_ratio,
/// peak_memory_bytes,bytes_sent,bytes_received,
/// con_con,dup_dup,era_era,con_dup,con_era,dup_era
pub fn write_csv_detail(path: &Path, results: &[BenchmarkResult]) -> io::Result<()> {
    let file = File::create(path)?;
    let mut w = BufWriter::new(file);

    // Header
    writeln!(w, "benchmark,input_size,mode,workers,repetition,correct,\
        wall_clock_secs,total_interactions,mips,\
        rounds,speedup,efficiency,overhead_ratio,\
        peak_memory_bytes,bytes_sent,bytes_received,\
        con_con,dup_dup,era_era,con_dup,con_era,dup_era")?;

    // Data rows
    for r in results {
        writeln!(w, "{},{},{},{},{},{},{:.6},{},{:.6},{},{:.6},{:.6},{:.6},\
            {},{},{},{},{},{},{},{},{}",
            r.benchmark, r.input_size, r.mode, r.workers, r.repetition,
            r.correct, r.wall_clock_secs, r.total_interactions, r.mips,
            r.rounds, r.speedup, r.efficiency, r.overhead_ratio,
            r.peak_memory_bytes, r.bytes_sent, r.bytes_received,
            r.interactions_by_rule.con_con, r.interactions_by_rule.dup_dup,
            r.interactions_by_rule.era_era, r.interactions_by_rule.con_dup,
            r.interactions_by_rule.con_era, r.interactions_by_rule.dup_era,
        )?;
    }

    w.flush()?;
    Ok(())
}

/// Write rounds CSV (R39b): one row per round per execution.
/// Only populated for distributed modes (Local, TcpLocalhost, TcpNetwork).
///
/// Columns (SPEC-09 R39b):
/// benchmark,input_size,workers,mode,repetition,round,
/// t_partition,t_compute,t_merge,t_network,
/// border_redexes,border_ratio,agents_at_start
pub fn write_csv_rounds(path: &Path, results: &[BenchmarkResult]) -> io::Result<()> {
    let file = File::create(path)?;
    let mut w = BufWriter::new(file);

    writeln!(w, "benchmark,input_size,workers,mode,repetition,round,\
        t_partition,t_compute,t_merge,t_network,\
        border_redexes,border_ratio,agents_at_start")?;

    for r in results {
        // Skip sequential mode (no per-round data)
        if r.mode == Mode::Sequential {
            continue;
        }
        for round in 0..r.rounds as usize {
            let t_part = r.partition_time_per_round.get(round).unwrap_or(&0.0);
            let t_comp = r.compute_time_per_round.get(round).unwrap_or(&0.0);
            let t_merge = r.merge_time_per_round.get(round).unwrap_or(&0.0);
            let t_net = r.network_time_per_round.get(round).unwrap_or(&0.0);
            let border = r.border_redexes_per_round.get(round).unwrap_or(&0);
            let bratio = r.border_ratio_per_round.get(round).unwrap_or(&0.0);
            let agents = r.agents_per_round.get(round).unwrap_or(&0);

            writeln!(w, "{},{},{},{},{},{},{:.6},{:.6},{:.6},{:.6},{},{:.6},{}",
                r.benchmark, r.input_size, r.workers, r.mode, r.repetition,
                round, t_part, t_comp, t_merge, t_net, border, bratio, agents,
            )?;
        }
    }

    w.flush()?;
    Ok(())
}

## Notes

- NOT using the `csv` crate to avoid adding a dependency for a simple writer. The format is straightforward: comma-separated, one row per result.
- In SPEC-09 v3, the per-round timing columns (`t_partition`, `t_compute`, `t_merge`, `t_network`) were moved from `detail.csv` to `rounds.csv` (R39b). The detail CSV now contains only scalar aggregates per execution.
- `correct` is written as `true`/`false` (Rust's `Display` for `bool`), which pandas reads as boolean.
- BufWriter is used for performance: with ~4200 datapoints, unbuffered writes would be slow.
- The three-file schema aligns with DATA-COLLECTION-PLAN Section 4, ensuring Python analysis scripts find expected schemas.
