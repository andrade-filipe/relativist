## Context

SPEC-09 Section 3.7 (R39-R42) defines the CSV output format for benchmark results. Two CSV files are produced: a **detail CSV** (one row per repetition, all raw datapoints) and a **summary CSV** (one row per configuration, aggregated statistics).

The detail CSV schema is specified in R39 with a unified format that works across all modes (sequential fields zeroed for non-grid executions). The summary CSV contains aggregated statistics from `AggregatedStats`.

Both CSVs must be compatible with Python (pandas), R, or spreadsheets: comma delimiter, UTF-8 encoding, dot for decimals.

use std::fs::File;
use std::io::{self, Write, BufWriter};
use std::path::Path;
use super::{BenchmarkResult, AggregatedStats};

/// Write detail CSV: one row per BenchmarkResult (one per repetition).
///
/// Columns (SPEC-09 R39):
/// benchmark,input_size,mode,workers,repetition,correct,
/// wall_clock_secs,total_interactions,mips,
/// rounds,speedup,efficiency,overhead_ratio,
/// peak_memory_bytes,bytes_sent,bytes_received,
/// con_con,dup_dup,era_era,con_dup,con_era,dup_era,
/// t_partition,t_compute,t_merge,t_network
pub fn write_csv_detail(path: &Path, results: &[BenchmarkResult]) -> io::Result<()> {
    let file = File::create(path)?;
    let mut w = BufWriter::new(file);

    // Header
    writeln!(w, "benchmark,input_size,mode,workers,repetition,correct,\
        wall_clock_secs,total_interactions,mips,\
        rounds,speedup,efficiency,overhead_ratio,\
        peak_memory_bytes,bytes_sent,bytes_received,\
        con_con,dup_dup,era_era,con_dup,con_era,dup_era,\
        t_partition,t_compute,t_merge,t_network")?;

    // Data rows
    for r in results {
        let t_partition: f64 = r.partition_time_per_round.iter().sum();
        let t_compute: f64 = r.compute_time_per_round.iter().sum();
        let t_merge: f64 = r.merge_time_per_round.iter().sum();
        let t_network: f64 = r.network_time_per_round.iter().sum();

        writeln!(w, "{},{},{},{},{},{},{:.6},{},{:.6},{},{:.6},{:.6},{:.6},\
            {},{},{},{},{},{},{},{},{},{:.6},{:.6},{:.6},{:.6}",
            r.benchmark, r.input_size, r.mode, r.workers, r.repetition,
            r.correct, r.wall_clock_secs, r.total_interactions, r.mips,
            r.rounds, r.speedup, r.efficiency, r.overhead_ratio,
            r.peak_memory_bytes, r.bytes_sent, r.bytes_received,
            r.interactions_by_rule.con_con, r.interactions_by_rule.dup_dup,
            r.interactions_by_rule.era_era, r.interactions_by_rule.con_dup,
            r.interactions_by_rule.con_era, r.interactions_by_rule.dup_era,
            t_partition, t_compute, t_merge, t_network,
        )?;
    }

    w.flush()?;
    Ok(())
}

## Notes

- NOT using the `csv` crate to avoid adding a dependency for a simple writer. The format is straightforward: comma-separated, one row per result.
- The `t_partition`, `t_compute`, `t_merge`, `t_network` columns are the SUM of per-round values (total time in each phase across all rounds). Per-round breakdown is available in the raw `BenchmarkResult` struct for detailed analysis.
- `correct` is written as `true`/`false` (Rust's `Display` for `bool`), which pandas reads as boolean.
- BufWriter is used for performance: with ~4200 datapoints, unbuffered writes would be slow.
