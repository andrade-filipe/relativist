## Context

`con_dup_expansion(n)` produces N CON-DUP pairs connected at principal ports, with auxiliary ports connected to free ports (R40). This is the Profile B (expansion+collapse) benchmark, which is the primary evaluation target for the TCC.

After commutation reduction, each CON-DUP pair expands into 4 new agents (2 CON + 2 DUP) cross-connected. This is the most important generator for the research because Profile B demonstrates the net-size expansion that creates parallelism opportunities for grid computing.

## Key Types / Signatures

```rust
/// Generate N CON-DUP expansion pairs (SPEC-12, R40; SPEC-09 R12).
///
/// Each pair: one CON + one DUP agent, principals connected,
/// auxiliaries to free ports.
/// After reduction: 4N agents (2N CON + 2N DUP), cross-connected.
pub fn con_dup_expansion(n: u32) -> Net {
    let mut net = Net::new();
    let mut free_id: u32 = 0;
    for _ in 0..n {
        let c = net.create_agent(Symbol::Con);
        let d = net.create_agent(Symbol::Dup);
        net.connect(PortRef::AgentPort(c, 0), PortRef::AgentPort(d, 0));
        net.connect(PortRef::AgentPort(c, 1), PortRef::FreePort(free_id));
        free_id += 1;
        net.connect(PortRef::AgentPort(c, 2), PortRef::FreePort(free_id));
        free_id += 1;
        net.connect(PortRef::AgentPort(d, 1), PortRef::FreePort(free_id));
        free_id += 1;
        net.connect(PortRef::AgentPort(d, 2), PortRef::FreePort(free_id));
        free_id += 1;
    }
    net
}
```

## Notes

- This generator mirrors the text DSL example in R14. The test DSL version has `n=1`; the generator accepts arbitrary `n`.
- Profile B is the focus of the TCC evaluation (see `project_profile_b_focus.md` in memory). The `con_dup_expansion` generator is therefore the most important for the research.
- After one round of commutation, each pair creates 4 new agents. This means the net grows: an input of size `n` produces roughly `4n` agents after reduction. The generated net (before reduction) has `2n` agents.
