## Context

SPEC-09 Section 4.3 defines the execution protocol for the benchmark suite: `run_benchmark_suite`. This is the top-level orchestrator that executes all benchmarks according to the suite configuration.

The protocol for each (benchmark, size, workers, mode) combination is:
1. Generate the input net
2. Execute sequential baseline (reduce_all) with warmup + repetitions
3. For each distributed configuration: warmup + repetitions
4. Verify correctness on EVERY repetition (R36)
5. Aggregate statistics
6. Write CSV and terminal output

A correctness failure halts the suite (R38) with detailed diagnostics. The 0-worker case (sequential baseline) is always included (R23). The 1-worker case runs in grid mode to measure protocol overhead in isolation (R22).

/// Result of a full benchmark suite execution.
#[derive(Debug)]
pub struct SuiteResult {
    /// All raw datapoints.
    pub detail: Vec<BenchmarkResult>,
    /// Aggregated statistics per configuration.
    pub summary: Vec<AggregatedStats>,
    /// Number of correctness failures (should be 0).
    pub correctness_failures: u32,
    /// Total wall-clock time for the entire suite.
    pub total_time_secs: f64,
}

/// Run the complete benchmark suite according to the given configuration.
///
/// Execution protocol (SPEC-09 Section 4.3):
/// 1. For each benchmark:
///    2. For each size:
///       3. Generate net, run sequential baseline with warmup + reps
///       4. For each mode:
///          5. For each worker count:
///             6. Warmup + measurement + verification + aggregation
///
/// A correctness failure (verify returns false) halts the suite
/// and reports full diagnostics (R38).
pub fn run_benchmark_suite(config: &BenchmarkSuiteConfig) -> SuiteResult {
    let suite_start = Instant::now();
    let benchmarks = build_benchmark_list(&config.benchmarks);
    let mut all_detail: Vec<BenchmarkResult> = Vec::new();
    let mut all_summary: Vec<AggregatedStats> = Vec::new();
    let mut failures = 0u32;

    for bench in &benchmarks {
        let sizes = /* config sizes or bench.default_sizes() */;
        for &size in &sizes {
            let net = bench.make_net(size);

            // --- Sequential baseline ---
            let (seq_net, seq_baseline_secs) = run_sequential_baseline(
                bench.as_ref(), &net, config.warmup, config.repetitions,
            );

            // --- Distributed configurations ---
            for &mode in &config.modes {
                for &w in &config.workers {
                    let mut rep_results = Vec::with_capacity(
                        config.repetitions as usize
                    );

                    // Warmup
                    for _ in 0..config.warmup {
                        execute_grid(net.clone(), w, mode, config.max_rounds);
                    }

                    // Measurement
                    for rep in 0..config.repetitions {
                        let mut result = measure_grid_execution(
                            bench.as_ref(), &net, w, mode,
                            &seq_net, seq_baseline_secs, rep,
                            config.max_rounds,
                        );

                        if !result.correct {
                            failures += 1;
                            eprintln!(
                                "CORRECTNESS FAILURE: bench={}, size={}, \
                                 workers={}, mode={:?}, rep={}",
                                bench.id(), size, w, mode, rep
                            );
                            // R38: halt on failure
                            // (could be configurable, but spec says MUST halt)
                        }

                        rep_results.push(result);
                    }

                    all_detail.extend(rep_results.iter().cloned());
                    let stats = aggregate_results(&rep_results);
                    print_progress(&stats);  // R41: terminal output
                    all_summary.push(stats);
                }
            }
        }
    }

    // Write CSVs
    if let Some(ref path) = config.csv_detail {
        write_csv_detail(path, &all_detail).expect("Failed to write detail CSV");
    }
    if let Some(ref path) = config.csv_summary {
        write_csv_summary(path, &all_summary).expect("Failed to write summary CSV");
    }

    SuiteResult {
        detail: all_detail,
        summary: all_summary,
        correctness_failures: failures,
        total_time_secs: suite_start.elapsed().as_secs_f64(),
    }
}

## Notes

- This is the capstone task of Phase 10: it ties together all benchmarks, metrics, CSV output, and the execution protocol.
- The `build_benchmark_list` function creates trait objects for all 9 benchmarks (or a filtered subset based on config). When `config.benchmarks` is empty, all benchmarks are included.
- The sequential baseline uses the median time (not mean) to reduce sensitivity to outliers (R30).
- Terminal output (R41) is a SHOULD requirement but is important for real-time monitoring during long benchmark runs (~4200 datapoints).
- The `execute_grid` function used during warmup does not collect metrics (discarded runs).
