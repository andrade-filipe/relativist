## Context

SPEC-07 Section 4.4 defines the worker lifecycle: parse config, connect to coordinator with retry, execute the worker loop (SPEC-06 `run_worker`), cleanup. This task wires the worker subcommand to the async `run_worker` function from Phase 5. The worker uses `tokio::task::spawn_blocking` for the CPU-bound `reduce_all` call (SPEC-13 R26).

The worker entry point is the simplest async command: parse the coordinator address, build NodeConfig, and delegate to `run_worker`.

## Acceptance Criteria

- [ ] `run_worker_command(args: WorkerArgs) -> Result<(), RelativistError>` is implemented as `async fn`
- [ ] Builds `NodeConfig` from args via `build_node_config_worker` (TASK-0102)
- [ ] If coordinator address is malformed, returns error with exit code 1
- [ ] Calls `run_worker(&node_config)` from Phase 5 (TASK-0093)
- [ ] If connection fails after all retries (SPEC-07 R17), prints error to stderr and returns error with exit code 2
- [ ] Logs "Worker shutdown complete." at INFO level on successful exit
- [ ] Function is wired into `main.rs` match arm for `Command::Worker`

## Key Types / Signatures

```rust
use crate::config::{WorkerArgs, build_node_config_worker};
use crate::protocol::run_worker;
use crate::error::RelativistError;

/// Execute worker mode: connect to coordinator and reduce partitions.
pub async fn run_worker_command(args: WorkerArgs) -> Result<(), RelativistError> {
    // 1. Parse config
    let node_config = build_node_config_worker(&args)?;

    // 2-3. Connect and execute worker loop (SPEC-06 Section 4.7)
    tracing::info!(
        host = %node_config.host,
        port = node_config.port,
        "connecting to coordinator"
    );
    run_worker(&node_config).await?;

    tracing::info!("worker shutdown complete");
    Ok(())
}
```
