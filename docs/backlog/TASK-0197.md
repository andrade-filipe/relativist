## Context

SPEC-09 Section 3.3 (R20) requires that derived metrics (MIPS, speedup, efficiency, overhead_ratio, border_ratio, worker idle time) are computed by the framework after each execution, not by the user. Section 3.5 (R32-R34) requires statistical aggregation over repetitions.

This task implements two key functions:

1. `compute_derived_metrics`: populates the derived fields of a `BenchmarkResult` after measurement.
2. `aggregate_results`: takes a `Vec<BenchmarkResult>` for one configuration and produces an `AggregatedStats`.

The aggregation computes mean, std, median, min, max for wall_clock_secs, mips, speedup, efficiency, and overhead_ratio (R33). It also computes CV for wall_clock and emits a high-variance warning if CV > 0.10 (R34). The break-even indicator (R46) is set when speedup_mean > 1.0.

use super::{BenchmarkResult, AggregatedStats, BenchmarkId, Mode};
use super::stats::{mean, std_dev, median, min_f64, max_f64, coeff_of_variation};

/// Compute derived metrics for a BenchmarkResult after measurement.
///
/// Called after timing and raw metrics are collected. Populates:
/// mips, speedup, efficiency, overhead_ratio, border_ratio_per_round,
/// and worker idle times.
///
/// SPEC-09 R20: derived metrics computed by the framework, not the user.
pub fn compute_derived_metrics(
    result: &mut BenchmarkResult,
    seq_baseline_secs: f64,
) {
    // MIPS
    result.mips = if result.wall_clock_secs > 0.0 {
        result.total_interactions as f64 / result.wall_clock_secs / 1_000_000.0
    } else {
        0.0
    };

    // Speedup
    result.speedup = if result.workers > 0 && result.wall_clock_secs > 0.0 {
        seq_baseline_secs / result.wall_clock_secs
    } else {
        1.0
    };

    // Efficiency
    result.efficiency = if result.workers > 0 {
        result.speedup / result.workers as f64
    } else {
        1.0
    };

    // Overhead ratio
    let total_compute: f64 = result.compute_time_per_round.iter().sum();
    result.overhead_ratio = if result.wall_clock_secs > 0.0 {
        1.0 - (total_compute / result.wall_clock_secs)
    } else {
        0.0
    };

    // Worker idle times (per round)
    for round_stats in &mut result.worker_stats {
        let max_compute = round_stats.iter()
            .map(|w| w.compute_time_secs)
            .fold(0.0f64, f64::max);
        for worker in round_stats.iter_mut() {
            worker.idle_time_secs = max_compute - worker.compute_time_secs;
        }
    }
}

/// Aggregate a set of BenchmarkResults (from repeated runs of the same
/// configuration) into a single AggregatedStats.
///
/// SPEC-09 R32-R34, R46.
pub fn aggregate_results(results: &[BenchmarkResult]) -> AggregatedStats {
    assert!(!results.is_empty(), "Cannot aggregate empty results");

    let times: Vec<f64> = results.iter().map(|r| r.wall_clock_secs).collect();
    let mips_vals: Vec<f64> = results.iter().map(|r| r.mips).collect();
    let speedups: Vec<f64> = results.iter().map(|r| r.speedup).collect();
    let efficiencies: Vec<f64> = results.iter().map(|r| r.efficiency).collect();
    let overheads: Vec<f64> = results.iter().map(|r| r.overhead_ratio).collect();

    let time_cv_val = coeff_of_variation(&times);

    AggregatedStats {
        benchmark: results[0].benchmark,
        input_size: results[0].input_size,
        mode: results[0].mode,
        workers: results[0].workers,
        num_repetitions: results.len() as u32,
        correct_count: results.iter().filter(|r| r.correct).count() as u32,
        time_mean: mean(&times),
        time_std: std_dev(&times),
        time_median: median(&times),
        time_min: min_f64(&times),
        time_max: max_f64(&times),
        time_cv: time_cv_val,
        mips_mean: mean(&mips_vals),
        mips_std: std_dev(&mips_vals),
        speedup_mean: mean(&speedups),
        speedup_std: std_dev(&speedups),
        efficiency_mean: mean(&efficiencies),
        efficiency_std: std_dev(&efficiencies),
        overhead_ratio_mean: mean(&overheads),
        overhead_ratio_std: std_dev(&overheads),
        peak_memory_mean: mean(&results.iter()
            .map(|r| r.peak_memory_bytes as f64).collect::<Vec<_>>()),
        bytes_sent_mean: mean(&results.iter()
            .map(|r| r.bytes_sent as f64).collect::<Vec<_>>()),
        bytes_received_mean: mean(&results.iter()
            .map(|r| r.bytes_received as f64).collect::<Vec<_>>()),
        speedup_above_one: mean(&speedups) > 1.0,
        high_variance_warning: time_cv_val > 0.10,
    }
}

## Notes

- `compute_derived_metrics` takes a mutable reference to `BenchmarkResult` and populates in-place. This avoids creating a new struct and allows the measurement code to fill raw fields first, then call this function to populate derived fields.
- The overhead_ratio formula: `1.0 - (compute_time / total_time)`. When compute_time == total_time (no overhead), overhead_ratio == 0.0. When compute_time == 0 (all overhead), overhead_ratio == 1.0. DISC-006 v2 Section 2.3: if overhead_ratio > 0.5, distribution likely does not pay off.
- Edge case: for sequential mode (workers == 0), speedup and efficiency are both 1.0 by definition.
