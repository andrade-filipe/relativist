## Context

This task implements Step 3 of the merge algorithm (SPEC-05, Section 4.2): restoring boundary connections by iterating over the border map, consulting the `free_port_index` of each partition, and calling `Net::connect` to re-establish the original wires. Border redex detection is automatic and incremental: `Net::connect` inserts into the redex queue when both endpoints are principal ports (SPEC-02, R13). The `is_principal_pair` helper (TASK-0064) is used to count border redexes for metrics (R13).

This completes the `merge` function whose skeleton was created in TASK-0065.

```rust
// Step 3: Restore boundary connections
let mut border_redex_count: u32 = 0;

for (border_id, (_orig_a, _orig_b)) in borders {
    let mut current_a: Option<PortRef> = None;
    let mut current_b: Option<PortRef> = None;

    for partition in &partitions {
        if let Some(port_ref) = partition.free_port_index.get(border_id) {
            if current_a.is_none() {
                current_a = Some(*port_ref);
            } else {
                current_b = Some(*port_ref);
            }
        }
    }

    match (current_a, current_b) {
        (Some(port_a), Some(port_b)) => {
            result.connect(port_a, port_b);
            if is_principal_pair(port_a, port_b) {
                border_redex_count += 1;
            }
        }
        // One side removed by erasure (R6)
        (Some(_), None) | (None, Some(_)) => {}
        // Both sides removed by erasure (R7)
        (None, None) => {}
    }
}
```

## Notes

- The `_orig_a` and `_orig_b` values from the border map are the ORIGINAL endpoints at split time. They are not used during merge because local reduction may have changed the endpoints. The CURRENT endpoints are found via the `free_port_index`.
- `Net::connect` handles redex queue insertion automatically. The `is_principal_pair` check is ONLY for the metrics counter, NOT for triggering reduction.
- Each border_id appears in exactly two partitions (SPEC-04, R8/C3). The loop over partitions finds at most two entries. If it finds more than two, that is a bug in the split operation.
