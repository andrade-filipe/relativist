## Context

SPEC-05 R10 and SPEC-01 D1 require that `merge(split(net, plan)) ~ net` (structural isomorphism) when no local reduction has occurred. This is the foundational correctness guarantee of the partition/merge protocol, derived from ARG-002 Part I (Steps 1-4). This task creates the integration test that validates this identity for various net topologies.

This is a TEST task. It defines what to test, not the implementation of merge itself.

## Acceptance Criteria

- [ ] Test file `tests/merge_split_identity.rs` (or equivalent) is created
- [ ] Tests `merge(split(net))` produces a net isomorphic to the original for:
  - A simple 2-agent net with 1 internal wire
  - A 4-agent chain with mixed CON/DUP/ERA agents
  - A net with pre-existing FreePort (Lafont) interface ports
  - A net with 2 workers (1 border wire)
  - A net with 3+ workers (multiple border wires)
- [ ] Isomorphism check verifies: same agent count, same symbols, same wire topology
- [ ] Edge case: 1 worker -> net returned unchanged
- [ ] Edge case: empty net -> empty net returned
- [ ] All tests pass with `cargo test`

## Notes

- "Isomorphic" means structural equality modulo agent ID renaming (SPEC-01, D1). Since Relativist uses static ID space partitioning, the IDs after split+merge may differ from the original. The isomorphism check must account for this.
- Alternatively, if static ID partitioning assigns the SAME IDs (which it does when no reduction occurs and `next_id` is preserved), direct equality may suffice for this test. The engineer should verify.
- This test validates a MUST requirement and is critical for the correctness argument of the TCC.
