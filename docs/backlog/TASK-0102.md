## Context

SPEC-07 Section 4.2 defines the mapping from CLI arguments to internal configuration structures: `GridConfig` (SPEC-05) and `NodeConfig` (SPEC-06). This task implements `build_grid_config`, `build_node_config_coordinator`, `build_node_config_worker`, and `parse_strategy`. These functions translate the raw CLI Args structs into the typed config structs used by the rest of the system. The `parse_host_port` helper parses the worker's `--coordinator HOST:PORT` argument.

## Acceptance Criteria

- [ ] `build_grid_config(args: &CoordinatorArgs) -> GridConfig` correctly maps `workers` and `max_rounds`
- [ ] `build_grid_config_from_local(args: &LocalArgs) -> GridConfig` does the same for local mode
- [ ] `build_node_config_coordinator(args: &CoordinatorArgs) -> NodeConfig` sets role=Coordinator with all defaults from SPEC-07 R12
- [ ] `build_node_config_worker(args: &WorkerArgs) -> Result<NodeConfig, RelError>` parses HOST:PORT and sets role=Worker
- [ ] `parse_host_port(addr: &str) -> Result<(String, u16), RelError>` correctly splits "host:port"
- [ ] `parse_strategy(name: &str) -> Result<Box<dyn PartitionStrategy>, RelError>` maps "round-robin" to `ContiguousIdStrategy` (only strategy in v1)
- [ ] Sensible defaults are applied: host=0.0.0.0, max_payload_size=256MiB, worker_connect_timeout=120s, distribute_timeout=60s, collect_timeout=600s
- [ ] Invalid strategy name returns `RelError::Config`
- [ ] Invalid host:port format returns `RelError::Config`
