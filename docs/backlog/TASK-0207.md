## Context

This task implements the unit tests for Church numeral encoding and decoding that do not require the reduction engine. These tests verify structural correctness (ET-1 through ET-3), Normal Form property (ET-4), roundtrip encoding/decoding (ET-5), invariant preservation (ET-9 for encoding-only nets), and decode rejection (ET-12).

Test labels use the `ET-` prefix (Encoding Tests) to avoid collision with SPEC-01 invariant labels T1-T7, as specified in SPEC-14 Section 7. In SPEC-08 v3's global test label namespace, ET-1 through ET-12 are mapped to `ENC-1` through `ENC-12` (see SPEC-08 Section 3.18, R39).

## Acceptance Criteria

- [ ] ET-1: `encode_nat(0)` produces exactly 2 CON + 1 ERA with correct port connections per Section 4.2
- [ ] ET-2: `encode_nat(1)` produces exactly 3 CON + 0 DUP + 0 ERA with correct port connections
- [ ] ET-3: `encode_nat(2)` produces exactly 4 CON + 1 DUP with correct port connections
- [ ] ET-4: for n in {0, 1, 2, 5, 10, 100}, `encode_nat(n)` produces a net with zero redexes
- [ ] ET-5: for n in {0, 1, 2, 3, 5, 10, 50, 100}, `decode_nat(&encode_nat(n))` returns `Some(n)`
- [ ] ET-9: for all encodings in ET-1 through ET-5, generated nets satisfy invariants T1-T7 (via `net.assert_all_invariants()` or equivalent debug check)
- [ ] ET-12: `decode_nat` returns `None` for: empty net, `ep_annihilation(5)` net, net with non-zero redexes
