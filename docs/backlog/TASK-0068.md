## Context

The grid loop (SPEC-05, Section 4.5) uses a termination check: if the redex queue is empty, the net may be in Normal Form. However, the queue may contain stale entries (SPEC-02, Section 2; SPEC-01, I4) -- redexes whose agents have already been consumed. The `drain_stale_redexes` function (SPEC-05, Section 4.4) traverses the queue, discards all stale entries, and retains only valid redexes. After draining, if the queue is truly empty, the net is in Normal Form.

## Key Types / Signatures

```rust
use std::collections::VecDeque;

/// Traverses the redex queue and discards all stale entries (SPEC-05, Section 4.4).
///
/// A stale redex is one where either agent has been consumed, or the
/// principal port connection has changed since the redex was inserted.
/// After this function, the queue contains only valid redexes.
///
/// Complexity: O(Q) where Q is the size of the redex queue.
pub fn drain_stale_redexes(net: &mut Net) {
    let mut valid = VecDeque::new();
    while let Some((a, b)) = net.redex_queue.pop_front() {
        if net.is_valid_redex(a, b) {
            valid.push_back((a, b));
        }
    }
    net.redex_queue = valid;
}
```

## Test Expectations

- Unit test: empty queue -> still empty after drain
- Unit test: queue with only valid redexes -> all retained
- Unit test: queue with only stale redexes (agents removed) -> empty after drain
- Unit test: mixed valid and stale -> only valid retained, in original order

## Notes

- This function modifies the net in place (takes `&mut Net`).
- An alternative approach mentioned in SPEC-05 Section 4.4 is a full scan of the port array to detect ALL redexes (not just queued ones). This is more robust but more expensive: O(A * PORTS_PER_SLOT). The drain approach is sufficient because `connect` (SPEC-02, R13) guarantees that every created redex is inserted into the queue.
- This function is called by `run_grid` at two points (SPEC-05 v3, SC-002/SC-011): (1) step [0], the initial Normal Form check before the loop, and (2) step [7], the termination check after merge+reduce_all at the end of each round.
