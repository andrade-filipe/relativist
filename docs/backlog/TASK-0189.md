## Context

SPEC-09 Section 3.2.1 (R10) defines the **EP-Annihilation-CON** benchmark: N pairs of CON agents connected by principal ports, with auxiliary ports connected to FreePorts. This is Profile A (embarrassingly parallel) and exercises the CON-CON annihilation rule, which produces a cross-reconnection pattern.

This benchmark fills AC-005 Gap L6: the Haskell prototype only uses ERA in the primary EP benchmark. CON-CON pairs exercise the cross-reconnection that is distinct from trivial ERA-ERA annihilation.

After reduction: 0 CON agents remain. FreePorts are reconnected in the cross pattern (a.1<->b.2, a.2<->b.1 for each pair).

## Key Types / Signatures

```rust
/// EP-Annihilation-CON: N pairs of CON agents connected by principal ports.
/// Auxiliary ports connected to FreePorts.
/// Profile A (embarrassingly parallel). All redexes independent.
/// Rules exercised: CON-CON (annihilation cross-reconnect).
/// Expected result: 0 CON agents, FreePorts cross-connected.
///
/// SPEC-09 R10. Fills AC-005 Gap L6.
pub struct EPAnnihilationCon;

impl Benchmark for EPAnnihilationCon {
    fn id(&self) -> BenchmarkId {
        BenchmarkId::EPAnnihilationCon
    }

    fn describe(&self, size: u32) -> String {
        format!("EP-Annihilation (CON): {} pairs, {} agents, {} redexes",
                size, size * 2, size)
    }

    fn make_net(&self, n: u32) -> Net {
        let mut net = Net::with_capacity((n * 2) as usize);
        for i in 0..n {
            let a = net.create_agent(Symbol::Con);
            let b = net.create_agent(Symbol::Con);
            // Principal ports: CON.0 <-> CON.0 (forms a redex)
            net.connect(PortRef::AgentPort(a, 0), PortRef::AgentPort(b, 0));
            // Auxiliary ports -> FreePorts
            net.connect(PortRef::AgentPort(a, 1), PortRef::FreePort(4 * i));
            net.connect(PortRef::AgentPort(a, 2), PortRef::FreePort(4 * i + 1));
            net.connect(PortRef::AgentPort(b, 1), PortRef::FreePort(4 * i + 2));
            net.connect(PortRef::AgentPort(b, 2), PortRef::FreePort(4 * i + 3));
        }
        net
    }

    fn default_sizes(&self) -> Vec<u32> {
        vec![100, 500, 1_000, 5_000, 10_000, 50_000]
    }

    fn verify(&self, _sequential_result: &Net, distributed_result: &Net) -> bool {
        count_live_agents(distributed_result) == 0
    }
}
```

## Notes

- CON-CON annihilation produces the "cross" reconnection: a.aux1<->b.aux2, a.aux2<->b.aux1. This is the characteristic behavior that distinguishes CON from DUP (which produces "parallel" reconnection).
- This benchmark is placed in the same file as EP-Annihilation-ERA (`ep_annihilation.rs`) since all three EP variants share the same structure with different symbols.
