# TASK-0055: FreePort index lazy reconstruction

**Spec:** SPEC-04
**Requirements:** R13 (index consistency after reduction), R15a (border ID range discrimination)
**Priority:** P1 (important)
**Status:** TODO
**Depends on:** TASK-0041 (Partition), SPEC-02 types (Net, PortRef, PORTS_PER_SLOT)
**Blocked by:** TASK-0041
**Estimated complexity:** S (< 50 LoC)

## Context

During local reduction by a worker, the reduction rules may alter what is connected to a boundary FreePort. An agent connected to `FreePort(bid)` may be consumed, reconnected, or replaced. The `free_port_index` built at split time (TASK-0052) can become stale. Before merge (SPEC-05), the index must reflect the actual port array state. SPEC-04, Section 4.6 recommends lazy reconstruction (approach 2) as the baseline: scan the port array to rebuild the index. This avoids coupling the reduction engine to the partition index, at the cost of O(A_i * PORTS_PER_SLOT) per partition.

## Acceptance Criteria

- [ ] Function `rebuild_free_port_index` scans a partition's subnet port array
- [ ] Uses `border_id_start` and `border_id_end` from the Partition to discriminate boundary FreePorts from Lafont FreePorts and DISCONNECTED sentinels (R15a)
- [ ] A `FreePort(id)` is included if and only if `border_id_start <= id && id < border_id_end && id != u32::MAX`
- [ ] `FreePort(id)` with `id < border_id_start` is a Lafont FreePort (ignored)
- [ ] `FreePort(u32::MAX)` is DISCONNECTED (ignored)
- [ ] Returns a fresh `HashMap<u32, PortRef>` mapping `bid -> AgentPort(a, p)`
- [ ] Correctly handles all three scenarios from Section 4.6: reconnection, erasure (FreePort transfer, not deletion), propagation via CON-DUP
- [ ] The function replaces (not merges with) the existing `free_port_index`

## Notes

- This function is called at a well-defined point: after local reduction completes and before merge begins. It is NOT called during reduction.
- The spec explicitly recommends this approach as baseline for simplicity (Section 4.6). Migration to active notification (approach 1) is a future optimization guided by benchmarks.
- DISCONNECTED entries (FreePort(u32::MAX)) must be excluded from the index. They are transient states, not real boundary markers.
- Lafont FreePort entries (id < border_id_start) must also be excluded. The border_id_start/end range (R15a) provides the discrimination mechanism, eliminating the need for a HashSet of known border IDs.
- FreePort connections are NEVER simply deleted during local reduction. They are always transferred to replacement agents (Scenario 2 rewrite in SPEC-04 v3). The `free_port_index` entry is UPDATED, not removed.
