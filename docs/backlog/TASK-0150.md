## Context

The `CoordinatorMetrics` struct holds all Prometheus metrics for the coordinator process. It uses the `prometheus-client` crate (R11) and is entirely feature-gated under the `metrics` Cargo feature (R10). The struct contains counters, gauges, and histograms as specified in R12, all prefixed with `relativist_` (R17) and using custom histogram bucket boundaries tuned to IC reduction latencies (R15).

This task defines the struct, its constructor, and the `register()` method that registers all metrics with a `prometheus_client::registry::Registry` (R18).

## Notes

- `prometheus-client` 0.23 uses `Counter<f64, AtomicU64>` as the default Counter type. Check exact API at implementation time.
- The `register()` method follows the builder pattern: it takes `self`, registers metrics, and returns `self`. This allows `let metrics = CoordinatorMetrics::new().register(&mut registry);`.
- Gauge uses `i64` / `AtomicI64` because active_workers can conceptually decrease (worker disconnect).
- `partition_id` and `round` are explicitly NOT used as labels (R16) to avoid cardinality explosion.
