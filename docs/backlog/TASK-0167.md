## Context

The JSON format uses serde_json serialization of the `Net` struct for interoperability with external tools, visualization pipelines, and web-based consumers (R16). However, per R17, JSON support MAY be deferred to a post-v1 release. If deferred, the CLI MUST print a clear error message.

This task implements the JSON module. The implementation is straightforward since `Net` already has serde derives -- it is just a wrapper around `serde_json::to_string_pretty` and `serde_json::from_str`. If development time is constrained, this can ship as a stub that returns `IoError::UnsupportedFormat`.

## Acceptance Criteria

- [ ] `save_json(net: &Net, path: &Path) -> Result<(), IoError>` serializes net to JSON and writes to file
- [ ] `load_json(path: &Path) -> Result<Net, IoError>` reads file and deserializes from JSON
- [ ] `serialize_json(net: &Net) -> Result<String, IoError>` serializes net to a JSON string
- [ ] `deserialize_json(s: &str) -> Result<Net, IoError>` deserializes net from a JSON string
- [ ] JSON output is pretty-printed (indented) for readability
- [ ] Serialization errors are mapped to `IoError::Serialization`
- [ ] If implemented as a stub: all functions return `IoError::UnsupportedFormat("JSON format not yet supported; use .bin or .ic")`
- [ ] `serde_json` dependency is added to `Cargo.toml`

## Key Types / Signatures

```rust
use std::path::Path;
use std::fs;
use crate::net::Net;
use super::types::IoError;

/// Serialize a Net to a pretty-printed JSON string (SPEC-12, R16).
pub fn serialize_json(net: &Net) -> Result<String, IoError> {
    serde_json::to_string_pretty(net)
        .map_err(|e| IoError::Serialization(e.to_string()))
}

/// Deserialize a Net from a JSON string (SPEC-12, R16).
pub fn deserialize_json(s: &str) -> Result<Net, IoError> {
    serde_json::from_str(s)
        .map_err(|e| IoError::Serialization(e.to_string()))
}

/// Save a Net to a .json file (SPEC-12, R16).
pub fn save_json(net: &Net, path: &Path) -> Result<(), IoError> {
    let json = serialize_json(net)?;
    fs::write(path, json)?;
    Ok(())
}

/// Load a Net from a .json file (SPEC-12, R16).
pub fn load_json(path: &Path) -> Result<Net, IoError> {
    let contents = fs::read_to_string(path)?;
    deserialize_json(&contents)
}
```

## Notes

- This is marked P2 because R17 explicitly allows deferral. The stub implementation is acceptable for v1.
- If implementing the full version, `serde_json` is the only new dependency. The `Net` struct already derives `Serialize` and `Deserialize`.
- JSON is the most verbose format (agent arena with `Some`/`None` slots, full port array). For large nets, JSON files can be very large. This is acceptable because JSON is the interop format, not the performance format.
- A feature flag (`json` feature in `Cargo.toml`) could gate the `serde_json` dependency, but this is over-engineering for the TCC scope.
