## Context

The binary format is the primary serialization format for IC nets: serde + bincode encoding of the `Net` struct (SPEC-02). It is the fastest to parse, most compact, and the default format for benchmarks and programmatic workflows. File extension: `.bin`.

This task implements the `binary.rs` sub-module with functions to serialize a `Net` to bytes and deserialize bytes back to a `Net`, plus file-level convenience functions. The `Net` struct already has serde derives (from TASK-0008/TASK-0017), so this task wraps bincode calls with proper error mapping to `FileIoError`.

Note: `Net::to_bytes` / `Net::from_bytes` already exist from TASK-0017 on the `Net` struct itself. This module provides file-oriented wrappers that read/write files on disk, mapping through `FileIoError`.

## Acceptance Criteria

- [ ] `save_binary(net: &Net, path: &Path) -> Result<(), FileIoError>` serializes net to bincode and writes to file
- [ ] `load_binary(path: &Path) -> Result<Net, FileIoError>` reads file and deserializes from bincode
- [ ] `serialize_binary(net: &Net) -> Result<Vec<u8>, FileIoError>` serializes net to bincode bytes in memory
- [ ] `deserialize_binary(bytes: &[u8]) -> Result<Net, FileIoError>` deserializes net from bincode bytes in memory
- [ ] Bincode errors are mapped to `FileIoError::Serialization`
- [ ] File I/O errors are mapped via the `FileIoError::Io` variant (automatic via `#[from]`)
- [ ] Roundtrip identity holds: `deserialize_binary(serialize_binary(net)?) == net`

## Key Types / Signatures

```rust
use std::path::Path;
use std::fs;
use crate::net::Net;
use super::types::FileIoError;

/// Serialize a Net to bincode bytes (SPEC-12, R2).
pub fn serialize_binary(net: &Net) -> Result<Vec<u8>, FileIoError> {
    bincode::serialize(net)
        .map_err(|e| FileIoError::Serialization(e.to_string()))
}

/// Deserialize a Net from bincode bytes (SPEC-12, R2).
pub fn deserialize_binary(bytes: &[u8]) -> Result<Net, FileIoError> {
    bincode::deserialize(bytes)
        .map_err(|e| FileIoError::Serialization(e.to_string()))
}

/// Save a Net to a .bin file (SPEC-12, R3).
pub fn save_binary(net: &Net, path: &Path) -> Result<(), FileIoError> {
    let bytes = serialize_binary(net)?;
    fs::write(path, bytes)?;
    Ok(())
}

/// Load a Net from a .bin file (SPEC-12, R3).
pub fn load_binary(path: &Path) -> Result<Net, FileIoError> {
    let bytes = fs::read(path)?;
    deserialize_binary(&bytes)
}
```

## Notes

- This module coexists with `Net::to_bytes`/`Net::from_bytes` from TASK-0017. The difference is that this module uses `FileIoError` (the I/O subsystem's error type) while the methods on `Net` use `RelError`. Both delegate to bincode internally.
- The binary format uses bincode v1 (same as TASK-0017). If the project later moves to bincode v2, only this module and TASK-0017 need updating.
- The `serialize_binary`/`deserialize_binary` in-memory functions are useful for the wire protocol (SPEC-06) which may send nets as binary payloads.
