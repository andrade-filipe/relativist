## Context

`ContiguousIdStrategy` is the baseline (and initially only) implementation of `PartitionStrategy`. It sorts live agents by AgentId in ascending order and divides them into approximately equal contiguous chunks, assigning each chunk to a worker sequentially. This mirrors the Haskell prototype's `partitionNet` (AC-002), enabling direct comparison. It is O(A) in time and fully deterministic. It ignores graph topology, which may create many border redexes, but correctness is guaranteed by strong confluence regardless (DISC-004 v2, Section 1.6).

## Acceptance Criteria

- [ ] `ContiguousIdStrategy` is a unit struct implementing `PartitionStrategy`
- [ ] `allocate()` collects all live AgentIds from the net, sorts them ascending
- [ ] Divides into `ceil(|A| / num_workers)` sized chunks, assigns chunk `i` to worker `i`
- [ ] If `num_workers > |A|`, excess workers get no agents (empty set) -- this is valid
- [ ] The returned HashMap contains exactly one entry per live agent
- [ ] Every WorkerId is in range `[0, num_workers)`
- [ ] Deterministic: same net + same num_workers = identical output

## Notes

- The sort is O(A log A) in general, but since AgentIds are monotonically increasing and never reused (SPEC-01, I3), the live agent IDs collected from a `Vec<Option<Agent>>` are already in ascending order if iterated by index. The implementation can exploit this to avoid an explicit sort, reducing to O(A).
- This strategy is intentionally naive. The `PartitionStrategy` trait allows adding smarter strategies later without touching the split pipeline.
