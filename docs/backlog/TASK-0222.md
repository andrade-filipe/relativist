## Context

SPEC-09 Section 3.2.6 (R17b, added in Revised v3, SC-004) defines the **ChurchMul** benchmark: a net encoding Church numeral multiplication using `build_mul(a, b)` from SPEC-14 where a and b are chosen to approximate sqrt(N). This is Profile B (expansion with collapse) with O(a*b) interactions, making it more computationally intensive than ChurchAdd.

ChurchMul is a SHOULD benchmark. It complements ChurchAdd (MUST) by demonstrating multiplication, which exercises deeper CON-DUP commutation cascades. The result is verified by BOTH `decode_nat` (checking the arithmetic value) AND graph isomorphism with the sequential result.

> **Added in SPEC-09 Revised v3.** This benchmark was part of the resolution of Open Question 6.

## Key Types / Signatures

```rust
use crate::bench::{Benchmark, BenchmarkId};
use crate::bench::isomorphism::nets_isomorphic;
use crate::encoding::{build_mul, decode_nat};
use crate::net::Net;

/// ChurchMul: Church numeral multiplication benchmark.
/// Generates build_mul(floor(sqrt(N)), floor(sqrt(N))) using SPEC-14 encoding.
/// Profile B (expansion with collapse). O(a*b) interactions.
/// Verified by decode_nat AND graph isomorphism.
///
/// SPEC-09 R17b (SHOULD). Added in Revised v3.
pub struct ChurchMul;

impl Benchmark for ChurchMul {
    fn id(&self) -> BenchmarkId {
        BenchmarkId::ChurchMul
    }

    fn describe(&self, size: u32) -> String {
        let a = (size as f64).sqrt().floor() as u64;
        format!("ChurchMul: church({}) * church({}) = church({}), Profile B",
                a, a, a * a)
    }

    fn make_net(&self, n: u32) -> Net {
        let a = (n as f64).sqrt().floor() as u64;
        build_mul(a, a)
    }

    fn default_sizes(&self) -> Vec<u32> {
        vec![5, 10, 20, 50]
    }

    fn verify(&self, sequential_result: &Net, distributed_result: &Net) -> bool {
        // Check 1: arithmetic value
        let seq_val = decode_nat(sequential_result);
        let dist_val = decode_nat(distributed_result);
        let value_ok = seq_val.is_some() && dist_val.is_some()
            && seq_val == dist_val;

        // Check 2: structural isomorphism
        let iso_ok = nets_isomorphic(sequential_result, distributed_result);

        value_ok && iso_ok
    }
}
```

## Test Expectations

- Unit test: sequential and distributed results for `make_net(9)` are isomorphic.
- Unit test: `default_sizes()` returns `[5, 10, 20, 50]`.

## Notes

- Church multiplication has O(a*b) interactions, which grows quadratically. For N=50, a=7, producing 49 interactions. For N=100, a=10, producing 100 interactions. Default sizes are kept small to avoid very long benchmark runs.
- The dual verification (decode_nat + isomorphism) mirrors ChurchAdd (TASK-0221).
- For non-perfect-square N, floor(sqrt(N))^2 != N. Example: N=10 -> a=3, result=9. This is consistent with the spec.
- This benchmark was added as SHOULD to complement ChurchAdd (MUST) in resolving OQ-6 (SPEC-09 v3).
- WARNING: Church multiplication with large N may exhaust the AgentId space. Recommended: N <= 1000 for ChurchMul.
