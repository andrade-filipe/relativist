## Context

The wire protocol `Message` enum (TASK-0082, SPEC-06) currently has 4 variants for the grid loop. SPEC-10 requires 3 additional variants for the registration handshake: `Register` (worker -> coordinator, carrying an optional auth token), `RegisterAck` (coordinator -> worker, confirming registration), and `RegisterNack` (coordinator -> worker, rejecting registration with a reason). These are the first messages exchanged on each connection, before the grid loop begins.

## Acceptance Criteria

- [ ] `Message::Register` variant is added with fields: `auth_token: Option<[u8; 32]>` (R19)
- [ ] `Message::RegisterAck` variant is added with field: `worker_id: WorkerId`
- [ ] `Message::RegisterNack` variant is added with field: `reason: String`
- [ ] All three variants derive `serde::Serialize` and `serde::Deserialize` (inherited from Message)
- [ ] `auth_token` is `Option<[u8; 32]>`: `None` in Tier 1, `Some(bytes)` in Tier 2/3 (R19)
- [ ] `RegisterNack::reason` is always `"authentication failed"` for auth failures (R16, R37)
- [ ] Doc comments specify direction (Worker->Coordinator or Coordinator->Worker)

## Key Types / Signatures

```rust
/// Messages in the communication protocol (SPEC-06 + SPEC-10).
#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum Message {
    // === Registration handshake (SPEC-10) ===

    /// Worker -> Coordinator: Request registration.
    /// Includes auth token when running in Tier 2 or 3 (R19).
    Register {
        /// Authentication token as raw bytes. None in Tier 1 (no auth).
        auth_token: Option<[u8; 32]>,
    },

    /// Coordinator -> Worker: Registration accepted.
    /// The coordinator assigns a WorkerId to the worker (R17).
    RegisterAck {
        /// Assigned worker identifier.
        worker_id: WorkerId,
    },

    /// Coordinator -> Worker: Registration rejected.
    /// The reason MUST NOT reveal internal state (R16, R37).
    RegisterNack {
        /// Human-readable reason (always "authentication failed" for auth errors).
        reason: String,
    },

    // === Grid loop (SPEC-06) ===
    // ... existing variants ...
}
```

## Notes

- The `auth_token` field uses `Option<[u8; 32]>` instead of `Option<AuthToken>` because the wire protocol transmits raw bytes (R14). The coordinator converts between `AuthToken` and `[u8; 32]` at the validation boundary.
- The `RegisterNack::reason` field is a `String` for extensibility, but R16/R37 require that authentication errors always use the generic `"authentication failed"` message. Protocol errors may use `"protocol error"`.
- This task modifies the `Message` enum defined in Phase 5 (TASK-0082). It is a non-breaking addition (new variants, no changes to existing ones).
