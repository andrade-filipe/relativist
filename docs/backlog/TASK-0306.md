# TASK-0306: Implement ChannelTransport

**Spec:** SPEC-17, Section 3.4 (R15-R16, R28), Section 4.4
**Phase:** v2 — Milestone M1
**Priority:** P0
**Complexity:** S (~60 LoC)
**Depends on:** TASK-0303

## Description

Create `src/protocol/channel.rs` implementing `ChannelTransport` for in-memory testing (SPEC-13 R29, R31):

1. **Struct**: holds `accept_streams: Vec<TransportStream>` and `connect_streams: Vec<TransportStream>`
2. **`pair(num_channels, buffer_size)` constructor** (R16): creates `num_channels` duplex pairs via `tokio::io::duplex()`, splits into accept/connect halves
3. **`Transport` impl**:
   - `listen()`: no-op
   - `accept()`: pop from accept_streams, error if exhausted
   - `connect()`: pop from connect_streams, error if exhausted
4. **Not CLI-selectable** (R28): only usable programmatically

## Acceptance Criteria

1. `ChannelTransport::pair(n, buf_size)` returns `(server, client)` transports
2. `accept()` and `connect()` return paired streams
3. Streams work with `send_frame`/`recv_frame`
4. Exhaustion returns error (not panic)
5. Tests: TR4 (round-trip), TR5 (exhaustion)
6. All 690 tests still pass
