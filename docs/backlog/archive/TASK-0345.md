# TASK-0345: Frame header v2 (R14-R19)

**Phase:** SPEC-18 §3.4 — Frame Header Update (item 2.23)
**Size:** S (~150 LoC inc. tests)
**Depends on:** TASK-0343 (bincode v2)
**Spec:** SPEC-18 R14, R15, R16, R17, R18, R19, R35 (partial: `UnknownFlags`)

**Scope:** Extend `FrameHeader` from 8 bytes to 9 bytes by adding a `flags`
byte. Initial flag bits are reserved for compression (TASK-0346) and rkyv
(deferred). Reject frames with unknown flag bits set (forward-compat).

---

## Deliverable

1. **`src/protocol/frame.rs`**:
   - `pub const FRAME_HEADER_SIZE: usize = 9;` (was 8).
   - `pub const FLAG_COMPRESSED: u8 = 0b0000_0001;`
   - `pub const FLAG_ARCHIVED:   u8 = 0b0000_0010;`
   - `pub const FLAG_RESERVED:   u8 = 0b1111_1100;`
   - `FrameHeader { length: u32, checksum: u32, flags: u8 }` — add `flags` field.
   - Methods: `is_compressed()`, `is_archived()`, `has_unknown_flags()`.
   - `send_frame` now writes 9 bytes (length LE u32 + checksum LE u32 +
     1 byte flags). `flags` is **always 0** in this task — compression
     is wired in TASK-0346.
   - `recv_frame` reads 9 bytes, extracts flags, returns
     `ProtocolError::UnknownFlags { flags }` if `header.has_unknown_flags()`.
   - `bytes_sent_per_round` / `bytes_received_per_round` accounting updates
     to use the new 9-byte header (R38 partial — full scope ships with
     TASK-0346 since metrics are tied to compression ratio).

2. **`src/protocol/error.rs`**:
   - Add `ProtocolError::UnknownFlags { flags: u8 }` variant
     (`#[error("unknown frame flags: {flags:#010b}")]`).

3. **Tests**:
   - **T-frame-roundtrip-v2**: send a `Message::Heartbeat` (or smallest
     variant) through `send_frame` + `recv_frame`, verify round-trip
     and that `header.flags == 0` on the received side.
   - **T-flags-unknown-rejection** (R19): construct a frame with raw
     `flags = 0b0000_0100` (reserved bit set) and feed it to `recv_frame`
     via an in-memory cursor; expect `ProtocolError::UnknownFlags`.
   - **T-header-size**: assert `FRAME_HEADER_SIZE == 9`.
   - **T-flag-bit-helpers**: `FrameHeader { flags: 0b11, .. }.is_compressed() && .is_archived()`.

## Acceptance

- `cargo test --workspace` count: 805 → 809+ (≥ +4).
- All existing 801 tests pass with the new 9-byte header; tests that
  asserted `FRAME_HEADER_SIZE == 8` are updated.
- `cargo clippy --workspace --all-targets -- -D warnings` clean.
- `cargo fmt --check` clean.

## Wire Compatibility

This task **alone** is a wire break (8-byte → 9-byte header). The
PROTOCOL_VERSION bump is in TASK-0347; TASK-0345 ships the new format
behind the old PROTOCOL_VERSION constant. This is intentional —
TASK-0343, 0344, 0345, 0346 all accumulate wire-incompatible changes
under the same v1 PROTOCOL_VERSION while the migration is in progress;
TASK-0347 commits the version bump as the final atomic step.

The implication is: between TASK-0345 merge and TASK-0347 merge,
**workers and coordinators built from intermediate commits cannot interop
with each other**. This is acceptable because v2-development is the only
consumer; v1-feature-complete remains frozen and wire-stable.

## Out of Scope

- Setting `FLAG_COMPRESSED` (TASK-0346).
- Setting `FLAG_ARCHIVED` (deferred to 2.24).
- PROTOCOL_VERSION bump (TASK-0347).
