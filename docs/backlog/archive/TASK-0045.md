## Context

When the split operation creates border IDs for cut wires, those IDs must not collide with pre-existing FreePort IDs already in the net (R12). The `max_freeport_id` helper scans the net's port array for all `FreePort(id)` entries and returns the maximum ID found (or `None` if no FreePort exists). New border IDs start from `max_existing + 1`. This is a small utility function, but it is critical for the correctness of border ID assignment.

## Notes

- The DISCONNECTED sentinel uses `FreePort(u32::MAX)`. This function MUST exclude `u32::MAX` from the scan, since it is not a real FreePort. If the only FreePort entries are DISCONNECTED, the function returns None.
