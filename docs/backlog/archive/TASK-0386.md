## Context

R4: "When ALL workers report both `has_border_activity: false` AND zero local
redexes (`local_redexes: 0` in `WorkerRoundStats`), the coordinator MUST
conclude that the net has reached Global Normal Form."

R21 phase 3: "When the coordinator determines Global Normal Form (R4), it
sends `FinalStateRequest` to each worker."

The delta-mode convergence predicate combines three signals:
1. Every worker reports `has_border_activity == false`.
2. Every worker reports `stats.local_redexes == 0` (no pending local work).
3. `BorderGraph::detect_border_redexes()` returns an empty vec (no pending
   border redexes).

**DC-C5 (bundle index):** the current draft uses a **two-predicate** version:
(a) all workers have no border activity AND (b) `BorderGraph` has zero
redexes. Per DC-C5, the `local_redexes == 0` check is redundant under the
assumption that a worker's `reduce_all` always reaches a local fixed point
— post-round, `local_redexes` in `WorkerRoundStats` records interactions
PERFORMED, not remaining. The post-reduction `partition.redex_queue` is
always empty by `reduce_all` postcondition. If spec-critic rules defensive
inclusion of the third predicate, we add it; it costs one `stats.iter()`
scan and is always `true` post-`reduce_all`.

This task ships the pure predicate function + its tests.

## Key Types / Signatures

```rust
// merge/grid.rs

/// SPEC-19 R4, R21.3 (TASK-0386, DC-C5 option: two-predicate): Global
/// Normal Form predicate for the delta BSP loop.
///
/// Returns `true` iff BOTH:
/// - every worker in `results` reported `has_border_activity == false`
///   (no local border endpoint is a principal port that could fire
///   under a future coordinator-side resolution), AND
/// - `border_graph.detect_border_redexes()` is empty (no pending
///   coordinator-side redex).
///
/// The `local_redexes == 0` check from R4 is omitted per DC-C5: the
/// invariant `reduce_all postcondition: redex_queue.is_empty()`
/// guarantees every worker reaches a local fixed point before
/// reporting; `stats.local_redexes` records interactions PERFORMED
/// during the round, which is not the same as "pending work". If
/// spec-critic rules defensive inclusion, this function grows a third
/// conjunct `&& results.iter().all(|r| r.stats.local_redexes == 0)`.
///
/// Pure / O(|results| + |graph|) — the `detect_border_redexes` call
/// uses the incremental redex-set (R18 SHOULD), so graph scan is
/// O(|active_redexes|) in practice.
pub(crate) fn check_delta_convergence(
    results: &[RoundResultPayload],
    border_graph: &BorderGraph,
) -> bool {
    let all_no_border_activity = results.iter().all(|r| !r.has_border_activity);
    let graph_has_no_redexes = border_graph.detect_border_redexes().is_empty();
    all_no_border_activity && graph_has_no_redexes
}
```

## Notes

- **DC-C5** rationale: a worker's per-round `reduce_all` invariant
  guarantees local redex queue is empty at the time `has_border_activity`
  is computed (compute order: `reduce_all` → `rebuild_free_port_index` →
  `compute_border_activity`, per TASK-0381). `stats.local_redexes`
  records the number of interactions that fired DURING the round, which
  is ≥ 0 even on a net that ends in Normal Form. Therefore
  `stats.local_redexes == 0` is a weaker signal than
  `has_border_activity == false` (which reflects post-reduction state),
  and predicating GNF on both is redundant.
- **Defense-in-depth:** a defensive assertion
  `debug_assert!(!graph_has_no_redexes || results.iter().all(|r| !r.has_border_activity))`
  may be added if spec-critic rules that the two predicates should be
  equivalent modulo a coordinator race — but under strict BSP the
  coordinator only inspects the graph AFTER applying every worker's
  delta, so the two predicates are indeed independent signals
  (activity = worker's local view; graph redex = coordinator's cross-
  partition view).
- **Integration with TASK-0385:** TASK-0385's round loop calls this
  predicate at step 3h, immediately after applying deltas to the graph
  and before the next iteration's `detect_border_redexes` call.
- **Integration with TASK-0387:** returning `true` from this predicate
  triggers TASK-0387's Final State Collection path.
- **Max-rounds interaction (R30, TASK-0388):** if the loop exits due
  to max_rounds, `check_delta_convergence` may still return `false`;
  the metrics record `delta_max_rounds_hit = Some(true)` and final
  collection still runs to produce the best-effort partial net.
