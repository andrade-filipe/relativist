## Context

`remove_agent(id)` removes an agent from the net. It disconnects all of the agent's ports (using `disconnect`), then marks the agent's slot as `None` in the arena. The `AgentId` is NOT reused -- the slot remains `None` for the rest of the execution. This is O(1) because each agent has at most 3 ports.

This operation is called by the reduction engine when consuming agents in an active pair (annihilation removes 2 agents, erasure removes the 2 original agents and creates new ones).

## Acceptance Criteria

- [ ] `remove_agent(id: AgentId)` is a `pub` method on `Net`
- [ ] After removal: `agents[id] == None`
- [ ] All ports of the removed agent are disconnected (their targets are also disconnected)
- [ ] The `AgentId` is NOT reused (`next_id` is unchanged)
- [ ] If `agents[id]` is already `None`, the operation is a no-op (no panic)
- [ ] O(1) complexity

## Key Types / Signatures

```rust
impl Net {
    /// Removes an agent from the net. The slot is marked as None.
    /// All of the agent's ports are disconnected in the port array.
    /// The AgentId is NOT reused.
    ///
    /// Complexity: O(1) (disconnects at most 3 ports).
    /// Precondition: agents[id].is_some() (soft -- handles None gracefully).
    pub fn remove_agent(&mut self, id: AgentId) {
        if let Some(agent) = self.agents[id as usize] {
            let num_ports = total_ports(agent.symbol);
            for p in 0..num_ports {
                let port = PortRef::AgentPort(id, p);
                self.disconnect(port);
            }
            self.agents[id as usize] = None;
        }
    }
}
```

## Notes

- The Haskell prototype's `removeAgent` filtered ALL wires to find those referencing the agent (O(w) -- AC-001, lines 119-123). Relativist's version is O(1) because it knows exactly which port array slots belong to the agent via `port_index`.
- `remove_agent` does NOT clean up the redex queue. Stale entries referencing the removed agent will be detected by `is_valid_redex` at dequeue time (R17).
