## Context

R20: "The delta protocol MUST be activated via a configuration flag
`GridConfig.delta_mode: bool`, defaulting to `false`. When `delta_mode` is
`false`, the v1 full-partition protocol is used unchanged. When `delta_mode`
is `true`, the delta BSP loop is used."

R21: the three-phase lifecycle (Round 0 â†’ Rounds 1+ â†’ Final State Collection).

This task ships (a) the public entry-point function `run_grid_delta`, (b) a
trait `WorkerDispatch` (DC-C2 option (c)) that abstracts the actual I/O path,
and (c) the dispatch logic so that `run_grid` (v1) or `run_grid_delta` (v2)
is selected by callers based on `config.delta_mode`. The actual round loop
is TASK-0385; convergence detection is TASK-0386; final collection is
TASK-0387; `max_rounds` cap is TASK-0388. This task ONLY wires the outer
shell.
