## Context

Before a locally-reduced partition can be merged back into the global net, its `free_port_index` must accurately reflect which `AgentPort` is currently connected to each `FreePort(border_id)`. Local reduction may have changed these connections through reconnection, erasure, or CON-DUP expansion (R22 scenarios 1-3). The baseline approach is lazy reconstruction: a complete scan of the sub-net's port array to build a fresh `HashMap<u32, PortRef>` (R21). This is the simplest and least error-prone approach recommended by SPEC-04 Section 4.6.

## Acceptance Criteria

- [ ] Function `rebuild_free_port_index` is implemented in `src/merge.rs`
- [ ] Scans all live agents in the subnet's port array
- [ ] Uses `border_id_start` and `border_id_end` from the Partition to discriminate boundary FreePorts from Lafont FreePorts and DISCONNECTED sentinels (SPEC-04, R15a)
- [ ] A `FreePort(id)` is included if and only if `border_id_start <= id && id < border_id_end && id != u32::MAX`
- [ ] `FreePort(id)` with `id < border_id_start` is a Lafont FreePort (excluded)
- [ ] `FreePort(u32::MAX)` is DISCONNECTED (excluded)
- [ ] Handles the reconnection scenario: returns the NEW endpoint after a local reduction rewired a FreePort
- [ ] Handles the erasure scenario: FreePort is transferred to replacement ERA agent (SPEC-04 v3 Scenario 2 -- FreePort connections are NEVER deleted during local reduction)
- [ ] Handles the CON-DUP scenario: returns whichever new agent inherited the FreePort connection
- [ ] Complexity is O(A_i * PORTS_PER_SLOT) per partition (R23)
- [ ] The module compiles with `cargo check`

## Notes

- PORTS_PER_SLOT is 3 (the port array uses 3 slots per agent: principal + 2 auxiliary). For ERA agents (arity 0), only port 0 is meaningful, but the port array still has 3 slots. The scan should iterate over `0..3` for all agents (or use `total_ports(symbol)` if available).
- This function is called after `reduce_all` on each partition, before the merge phase.
- The function signature takes `&Net` (immutable reference) and returns a new HashMap. It does NOT modify the partition.
