## Context

When the grid loop completes (either the net reaches normal form or the round limit is hit), the coordinator must send a `Message::Shutdown` to every connected worker and then close connections. This corresponds to the `ShuttingDown -> Done` transition in the coordinator FSM (R26). The shutdown is best-effort: if sending Shutdown to one worker fails, the coordinator logs the error and continues shutting down other workers. Workers close their connections upon receiving Shutdown.

## Notes

- The Haskell prototype uses `sendMessage(sock, [])` as the shutdown signal (an empty list). Relativist uses a typed `Message::Shutdown` variant, which is clearer and type-safe (SPEC-06 Section 5.4).
- Shutdown does NOT expect a response from workers. It is fire-and-forget. Workers close their connections upon receiving Shutdown.
- The function does NOT return a Result because shutdown failures are non-critical at this point -- the computation is already complete.
