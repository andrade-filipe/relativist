## Context

The observability initialization function needs to know (1) which log output format to use (human-readable text vs. machine-parseable JSON) and (2) whether the current process is a coordinator or worker (affects OTel resource attributes and default behaviors). These two enums are fundamental building blocks used by `ObservabilityConfig` and the `init_tracing` function.

`LogFormat` maps directly to R3: text format uses `tracing_subscriber::fmt::format::Full`, JSON format uses `tracing_subscriber::fmt::format::Json`. `ProcessRole` maps to R28, R31, and R33a: it determines the `service.name` OTel attribute and is carried in the config struct. The `Local` variant (R33a) represents the local reduction mode (`relativist reduce`) where logging is active but HTTP endpoints are NOT started.

## Key Types / Signatures

```rust
/// Log output format selection (R3).
///
/// `Text` produces human-readable output suitable for development.
/// `Json` produces machine-parseable JSON suitable for production and log aggregation.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum LogFormat {
    /// Human-readable output (tracing_subscriber::fmt::format::Full).
    Text,
    /// Machine-parseable JSON output (tracing_subscriber::fmt::format::Json).
    Json,
}

impl Default for LogFormat {
    fn default() -> Self {
        LogFormat::Text
    }
}

/// Role of the current process in the grid (R28, R31, R33a).
///
/// Determines `service.name` for OTel resource attributes and affects
/// default behaviors in the observability subsystem.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum ProcessRole {
    Coordinator,
    Worker,
    /// Local reduction mode (SPEC-13, R41: `relativist reduce`).
    /// Logging is initialized; HTTP endpoints are NOT started.
    Local,
}
```

## Notes

- `LogFormat` does not auto-detect TTY here. TTY detection logic belongs in the CLI/config layer (Phase 6) which maps `--log-format` to the enum. The default `Text` is a safe fallback.
- `ProcessRole` is distinct from `NodeRole` in the protocol module (TASK-0084). `NodeRole` is a network configuration concept; `ProcessRole` is an observability configuration concept. They may be unified later, but for now they are separate to avoid coupling observability to the protocol module.
- The `Local` variant was added in SPEC-11 Revised v2 (SC-013) to define observability behavior for `relativist reduce` local mode. See R33a for behavioral requirements.
