**Status:** TODO

## Context

R12 specifies the redex-detection primitive: return all border entries
where `is_redex == true`. R18 SHOULD prescribes incremental detection so
this is O(|active_redexes|), which TASK-0362 already maintains via the
internal `active_redexes: HashSet<u32>` field.

This task ships the read-only query methods that the coordinator (item
2.26) will call once per round to know which borders need resolution. Per
bundle DC-3, the return type is `Vec<(u32, &BorderState)>` (borrowed) per
spec §4.2 pseudocode, NOT `Vec<(u32, BorderState)>` (owned) per spec §3.2
R12 prose; spec-critic to confirm before Stage 2 dispatch (recommendation
already documented in the bundle index).

Three companion accessors land alongside: `len`, `is_empty` /
`has_no_redexes`, and an `iter_borders` for debugging / observability.

## Acceptance Criteria

- [ ] `BorderGraph` gets a new method:
      `pub fn detect_border_redexes(&self) -> Vec<(u32, &BorderState)>`
      that iterates `self.active_redexes`, looks up each `border_id` in
      `self.borders`, and collects the `(border_id, &BorderState)` pairs.
      Filter-map skips any `border_id` whose `borders` entry was removed
      (defensive — should not happen if `apply_deltas` invariant holds,
      but protects against a future bug).
- [ ] `BorderGraph` gets a new method:
      `pub fn len(&self) -> usize { self.borders.len() }` — total border
      count (alive only, since R17 removes dead borders eagerly).
- [ ] `BorderGraph` gets a new method:
      `pub fn has_no_redexes(&self) -> bool { self.active_redexes.is_empty() }`.
      (Per spec §4.2 pseudocode wording. Used by the coordinator to detect
      Global Normal Form's "no border redexes" precondition.)
- [ ] `BorderGraph` gets a new method:
      `pub fn is_empty(&self) -> bool { self.borders.is_empty() }` — for
      symmetry with `len`. (Different semantics from `has_no_redexes`:
      `is_empty` means "no borders at all"; `has_no_redexes` means "no
      ACTIVE pairs" but borders may still exist with non-principal
      endpoints.)
- [ ] `BorderGraph` gets a new method:
      `pub fn active_redex_count(&self) -> usize { self.active_redexes.len() }`
      — for observability metrics (item 2.26 will likely emit this as a
      `gauge!` per round).
- [ ] `detect_border_redexes` MUST be O(|active_redexes|) — no scan of
      `self.borders` (assert in implementation by iterating
      `active_redexes` directly, NOT `borders.iter().filter(...)`).
- [ ] Inline `#[cfg(test)] mod tests` extends the existing module:
      - `detect_returns_empty_when_no_redexes`: graph with 2 borders,

      - `is_empty_distinct_from_has_no_redexes`: graph with 1 border, no

## Key Types / Signatures

```rust
impl BorderGraph {
    /// Return all border entries currently in a redex state (SPEC-19 R12).
    ///
    /// O(|active_redexes|) per R18 SHOULD: iterates the incremental
    /// `active_redexes` set rather than scanning all borders.
    pub fn detect_border_redexes(&self) -> Vec<(u32, &BorderState)> {
        self.active_redexes
            .iter()
            .filter_map(|&bid| self.borders.get(&bid).map(|state| (bid, state)))
            .collect()
    }

    /// Number of border wires currently tracked (alive borders only).
    pub fn len(&self) -> usize {
        self.borders.len()
    }

    /// `true` iff no border wires are tracked (all consumed).
    pub fn is_empty(&self) -> bool {
        self.borders.is_empty()
    }

    /// `true` iff zero borders are in a redex state. Used by item 2.26 to
    /// detect Global Normal Form's "no border redexes" precondition. This
    /// is **not** the same as `is_empty` — borders can exist without
    /// being active pairs.
    pub fn has_no_redexes(&self) -> bool {
        self.active_redexes.is_empty()
    }

    /// Count of border wires currently in a redex state. For observability
    /// (item 2.26 emits this as a per-round metric).
    pub fn active_redex_count(&self) -> usize {
        self.active_redexes.len()
    }
}
```

## Notes

- **Why `Vec` and not an `Iterator` return:** the spec §4.2 pseudocode
  returns `Vec`. An iterator would let the coordinator stream over
  redexes lazily, but the coordinator loop in item 2.26 will materialise
  them anyway (it dispatches each one as a separate worker request) — and
  returning an iterator borrowing from `self` would prevent the
  coordinator from calling `apply_deltas` mid-iteration (because that's
  `&mut self`). Vec is the right ergonomic choice here.
- **Why `len` instead of just `is_empty`:** observability. The coordinator
  metrics record per-round border counts; `len` is the natural API.
- **`has_no_redexes` vs `active_redex_count() == 0`:** they're equivalent.
  Spec §4.2 names `has_no_redexes` explicitly so we ship it as a named
  method for spec-traceability. `active_redex_count` exists for the
  observability path.
- **Per DC-3 of `docs/spec-reviews/SPEC-19-section-3.2-design-choices-2026-04-17.md`:** owned `Vec<(u32, BorderState)>` per §3.2 R12 normative text. The §4.2 pseudocode is non-normative and contradicts the coordinator's mutable-borrow pattern (callers borrow `&mut self` while iterating, so owned return is required to satisfy the borrow checker).

**Updated `detect_border_redexes` body (mandated form):**

```rust
pub fn detect_border_redexes(&self) -> Vec<(u32, BorderState)> {
    self.active_redexes
        .iter()
        .filter_map(|&bid| self.borders.get(&bid).map(|state| (bid, state.clone())))
        .collect()
}
```

**Reason recorded:** spec-critic ruled the §3.2 R12 normative text wins over the §4.2 non-normative pseudocode. The borrowed form is impossible to use in the item 2.26 coordinator BSP loop (which needs `&mut border_graph` while iterating the redex list). `BorderState` is `Clone` and ~32 bytes; per-round clone cost is 32 KB at worst case (noise). Ship owned once, never revisit.

**LoC estimate:** unchanged (~40 LoC; one `.clone()` added per yielded entry — net ~+1 character per code-block line).

**R18 complexity bound preserved:** still O(|active_redexes|); the `.clone()` is per-yielded entry, not per-border.

**No spec amendment required.** §3.2 R12 already mandates the owned form; §4.2 pseudocode is left as-is (a documented illustrative reference, not a prescriptive signature).

**Cascades:** Bundle index already updated by spec-critic to record the DC-3 ruling. TEST-SPEC-0363 (Stage 2) will encode the owned signature in test assertions.
