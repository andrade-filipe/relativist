## Context

SPEC-13 R24 defines the worker's finite state machine with 6 states: Init, Idle, Reducing, Returning, Error, Done. Like the coordinator, the worker FSM follows the stimulus-response pattern. This task defines the state, event, and action enums. Workers have no knowledge of each other (R27) -- they only communicate with the coordinator via the star topology.

## Acceptance Criteria

- [ ] `WorkerState` enum is defined with 6 variants: Init, Idle, Reducing, Returning, Error, Done
- [ ] `WorkerState` derives `Debug`, `Clone`, `PartialEq`, `Eq`, `serde::Serialize`
- [ ] `WorkerEvent` enum is defined with variants: `Connected`, `ReceivePartition(Partition)`, `ReductionComplete(Partition)`, `SendComplete`, `Shutdown`, `ReductionError(String)`, `ConnectionLost`
- [ ] `WorkerAction` enum is defined with variants: `SendMessage(Message)`, `CloseConnection`, `ShutdownSelf`, `LogTransition { from, to }` (no `AttemptReconnect` -- worker does NOT reconnect per SPEC-06 R25)
- [ ] All types are public and in the `worker` module
- [ ] Worker module is declared in `lib.rs`

## Key Types / Signatures

```rust
//! Worker FSM for distributed IC reduction.
//!
//! Workers connect to the coordinator, receive partitions,
//! reduce them locally via reduce_all (SPEC-03), and return results.
//! Workers have no knowledge of each other (star topology, SPEC-13 R27).

use crate::net::Net;
use crate::partition::Partition;
use crate::protocol::Message;

/// Worker FSM states (SPEC-13 R24).
#[derive(Debug, Clone, PartialEq, Eq, serde::Serialize)]
pub enum WorkerState {
    Init,
    Idle,
    Reducing,
    Returning,
    Error,
    Done,
}

/// Events that drive the worker FSM (SPEC-13 R25).
#[derive(Debug, Clone)]
pub enum WorkerEvent {
    Connected,
    ReceivePartition(Partition),
    ReductionComplete(Partition),
    SendComplete,
    Shutdown,
    ReductionError(String),
    ConnectionLost,
}

/// Actions the worker runtime must execute (SPEC-13 R25).
#[derive(Debug)]
pub enum WorkerAction {
    SendMessage(Message),
    CloseConnection,
    /// Shut down the worker process. Used on ConnectionLost since the
    /// coordinator has already aborted per SPEC-06 R25. No reconnection.
    ShutdownSelf,
    LogTransition { from: WorkerState, to: WorkerState },
}
```

## Notes

- SPEC-13 R26 specifies that reduction must be synchronous (CPU-bound) and offloaded via `tokio::task::spawn_blocking`. That wiring happens in the worker startup task (TASK-0113), not here. This task only defines the FSM types.
- The `WorkerEvent::ReceivePartition` carries a full `Partition` struct. In practice, this is extracted from a `Message::AssignPartition`. The event mapping from messages to events happens in the async runtime layer.
