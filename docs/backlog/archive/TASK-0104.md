## Context

SPEC-07 Section 4.6 defines a `.bin` file format: the `Net` struct serialized with serde + bincode. The coordinator, local mode, and generate subcommand all need to read/write nets from/to files. This task provides `serialize_net`, `deserialize_net`, `load_net_from_file`, and `save_net_to_file` helper functions. The round-trip property (`deserialize(serialize(net)) == net`) must hold for every valid Net.

These functions belong in the `io` module since they handle user I/O (SPEC-12), but they are needed by Phase 6's CLI wiring.

## Acceptance Criteria

- [ ] `serialize_net(net: &Net) -> Result<Vec<u8>, RelativistError>` serializes with bincode
- [ ] `deserialize_net(bytes: &[u8]) -> Result<Net, RelativistError>` deserializes with bincode
- [ ] `load_net_from_file(path: &Path) -> Result<Net, RelativistError>` reads file and deserializes
- [ ] `save_net_to_file(net: &Net, path: &Path) -> Result<(), RelativistError>` serializes and writes
- [ ] File not found produces `RelativistError::Io` with clear message
- [ ] Corrupt data produces `RelativistError::Config` (or a new `Deserialize` variant) with the path in the message
- [ ] `load_net_from_file` logs at `info` level: number of agents and redexes after loading
- [ ] Functions use synchronous `std::fs` (not tokio), since local mode and generate are sync

## Key Types / Signatures

```rust
use std::path::Path;
use crate::net::Net;
use crate::error::RelativistError;

/// Serialize a Net to bytes (.bin format, bincode).
pub fn serialize_net(net: &Net) -> Result<Vec<u8>, RelativistError> {
    bincode::serialize(net)
        .map_err(|e| RelativistError::Config(format!("serialization failed: {}", e)))
}

/// Deserialize a Net from bytes (.bin format, bincode).
pub fn deserialize_net(bytes: &[u8]) -> Result<Net, RelativistError> {
    bincode::deserialize(bytes)
        .map_err(|e| RelativistError::Config(format!("deserialization failed: {}", e)))
}

/// Load a Net from a .bin file.
pub fn load_net_from_file(path: &Path) -> Result<Net, RelativistError> {
    let bytes = std::fs::read(path)?;
    let net = deserialize_net(&bytes)
        .map_err(|e| RelativistError::Config(
            format!("failed to deserialize {:?}: {}", path, e)
        ))?;
    tracing::info!(
        path = ?path,
        agents = net.agents.iter().filter(|a| a.is_some()).count(),
        redexes = net.redex_queue.len(),
        "loaded network"
    );
    Ok(net)
}

/// Save a Net to a .bin file.
pub fn save_net_to_file(net: &Net, path: &Path) -> Result<(), RelativistError> {
    let bytes = serialize_net(net)?;
    std::fs::write(path, &bytes)?;
    tracing::info!(path = ?path, "saved network");
    Ok(())
}
```

## Notes

- SPEC-07 R24 states the output format must match the input format, enabling chaining. These functions ensure that by using the same bincode configuration for both directions.
- Async variants (`load_net_from_file_async` using `tokio::fs`) may be needed for the coordinator. They can be added later or the coordinator can use `spawn_blocking` to call the sync versions.
- The bincode version in Cargo.toml is 1.x. SPEC-13 R11 mentions bincode 2.x. The implementer should verify compatibility or upgrade. This task uses whichever version is in Cargo.toml.
