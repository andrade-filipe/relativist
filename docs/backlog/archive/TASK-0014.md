## Context

`is_reduced` checks whether the net is in normal form by testing if the redex queue is empty. Note that the queue may still contain stale entries that haven't been drained -- for a rigorous check, the reduction engine must drain the queue first.

`is_valid_redex` validates whether a pair `(a, b)` from the redex queue is still a valid active pair. A redex is stale if either agent has been consumed by another reduction, or if the principal port connection has changed. The reduction engine MUST call this before applying a rule and silently discard invalid entries (R17, invariant I4).

## Acceptance Criteria

- [ ] `is_reduced() -> bool` is a `pub` method on `Net`
- [ ] Returns `true` if and only if `redex_queue.is_empty()`
- [ ] `is_valid_redex(a: AgentId, b: AgentId) -> bool` is a `pub` method on `Net`
- [ ] Returns `false` if `agents[a]` is `None` or out of bounds
- [ ] Returns `false` if `agents[b]` is `None` or out of bounds
- [ ] Returns `false` if `get_target(AgentPort(a, 0)) != AgentPort(b, 0)`
- [ ] Returns `true` only if both agents exist AND their principal ports are connected to each other

## Key Types / Signatures

```rust
impl Net {
    /// Returns true if the net is in normal form (no active pairs remain
    /// in the queue).
    ///
    /// Note: the queue may contain stale entries. For rigorous verification,
    /// drain stale entries first or simply run reduce_all (SPEC-03).
    pub fn is_reduced(&self) -> bool {
        self.redex_queue.is_empty()
    }

    /// Checks whether a pair (a, b) in the redex queue is valid (non-stale).
    ///
    /// A redex is valid if:
    /// 1. Both agents exist (checked via `get_agent`, R15a).
    /// 2. The principal ports of a and b are connected to each other.
    pub fn is_valid_redex(&self, a: AgentId, b: AgentId) -> bool {
        if self.get_agent(a).is_none() || self.get_agent(b).is_none() {
            return false;
        }
        self.get_target(PortRef::AgentPort(a, 0)) == PortRef::AgentPort(b, 0)
    }
}
```

## Notes

- `is_reduced` is a simple queue-empty check. It does NOT drain stale entries. A net with only stale redexes in the queue will return `false` from `is_reduced` but will produce no actual reductions when the engine processes them. **SPEC-03 v3 caveat (SC-005):** `is_reduced()` is a necessary but not sufficient condition for Normal Form when stale entries exist. The canonical way to verify Normal Form is `reduce_all`, not `is_reduced()` alone. After `reduce_all` returns, `is_reduced()` is guaranteed to return `true`.
- `is_valid_redex` is the "validate-or-skip" pattern described in SPEC-02 Section 4.7. The reduction engine calls this for every dequeued pair and skips stale ones.
