## Context

The coordinator and workers need runtime configuration for bind address, worker count, payload limits, and timeouts. SPEC-06 Revised v3 Section 4.5 defines a `NodeConfig` struct without `NodeRole` (removed because SPEC-13's separate `CoordinatorArgs`/`WorkerArgs` CLI structs make a role discriminant unnecessary). The `NodeConfig` uses `bind: SocketAddr` (default `127.0.0.1:9000`) instead of separate `host: String` + `port: u16`. A `Default` impl is provided.

- [ ] `NodeConfig` struct is defined with 6 fields from SPEC-06 Revised v3 Section 4.5: `bind`, `num_workers`, `max_payload_size`, `worker_connect_timeout`, `distribute_timeout`, `collect_timeout`
- [ ] `NodeConfig` derives `Debug`, `Clone`
- [ ] `impl Default for NodeConfig` provides: bind=127.0.0.1:9000, num_workers=1, max_payload_size=256 MiB, worker_connect_timeout=120s, distribute_timeout=60s, collect_timeout=600s
- [ ] `NodeRole` enum is NOT defined (removed in SPEC-06 v3; role is determined by CLI subcommand per SPEC-13 R43)

## Key Types / Signatures

```rust
use std::net::SocketAddr;
use std::time::Duration;

/// Configuration of the coordinator node (SPEC-06 Revised v3 Section 4.5).
///
/// SPEC-13 R44-R45 defines separate `CoordinatorArgs` and `WorkerArgs`
/// for the CLI layer. `NodeConfig` is an internal configuration struct
/// populated after CLI argument parsing. The coordinator and worker know
/// their role from the CLI subcommand (SPEC-13 R43), so a `role` field
/// is unnecessary.
#[derive(Debug, Clone)]
pub struct NodeConfig {
    /// Address and port for the TCP listener (coordinator) or
    /// coordinator address (worker).
    /// Default: 127.0.0.1:9000 (SPEC-10 R5, SPEC-13 R44).
    pub bind: SocketAddr,

    /// Expected number of workers (relevant only for coordinator).
    pub num_workers: u32,

    /// Maximum accepted payload size, in bytes.
    /// Default: DEFAULT_MAX_PAYLOAD_SIZE (256 MiB).
    pub max_payload_size: u32,

    /// Timeout for waiting for all workers to connect (coordinator).
    /// Default: 120 seconds.
    pub worker_connect_timeout: Duration,

    /// Timeout for distributing partitions in a round (SHOULD).
    /// Default: 60 seconds.
    pub distribute_timeout: Duration,

    /// Timeout for collecting results in a round (MUST).
    /// Default: 600 seconds.
    pub collect_timeout: Duration,
}

impl Default for NodeConfig {
    fn default() -> Self {
        Self {
            bind: "127.0.0.1:9000".parse().unwrap(),
            num_workers: 1,
            max_payload_size: DEFAULT_MAX_PAYLOAD_SIZE,
            worker_connect_timeout: Duration::from_secs(120),
            distribute_timeout: Duration::from_secs(60),
            collect_timeout: Duration::from_secs(600),
        }
    }
}
```

## Notes

- `NodeConfig` is separate from `GridConfig` (SPEC-05): `GridConfig` controls the grid loop behavior (max_rounds, partition strategy), while `NodeConfig` controls network parameters (bind address, timeouts).
- The `num_workers` field is only meaningful for the coordinator role. For workers, it is ignored. Consider documenting this clearly.
- The default timeout values come from R24 (120s worker connect), R30 (60s distribute, 600s collect). Note: `collect_timeout` is now MUST (upgraded from SHOULD in SPEC-06 v3, SC-014).
- `NodeRole` was removed in SPEC-06 v3 (SC-005). The coordinator and worker know their role from the CLI subcommand (SPEC-13 R43), so a role discriminant is unnecessary.
- CLI argument mapping (R37) is handled by the config module (SPEC-07/SPEC-13), not here. This task only defines the data structure.
