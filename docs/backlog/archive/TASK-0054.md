# TASK-0054: Debug assertions for C2 (wire coverage) and C3 (FreePort bijectivity)

**Spec:** SPEC-04
**Requirements:** R7, R8, R10

## Context

Conditions C2 and C3 are critical for the correctness of the split operation. C2 requires that every wire is classified as internal, interface, or border -- no wire is lost. C3 requires that each `borderId` appears in exactly two distinct partitions. These assertions catch subtle bugs in wire classification and FreePort generation. They run only in debug mode per R10. Combined into one task because they share iteration patterns over partitions and borders.

## Acceptance Criteria

- [ ] Function `assert_c2` verifies complete wire coverage: the total number of connections across all partitions (internal + interface + border) equals the original net's connection count
- [ ] Function `assert_c3` verifies FreePort bijectivity: each borderId in the border map appears in exactly 2 distinct partitions' free_port_index
- [ ] Both functions are gated by `#[cfg(debug_assertions)]`
- [ ] Both panic with descriptive messages on violation
- [ ] Do NOT run in release mode

## Notes

- The C3 assertion logic is provided as pseudocode in SPEC-04, Section 4.8 (`assert_border_consistency`). Follow it closely.
- C2 is more nuanced: counting "connections" requires care with DISCONNECTED slots, ERA's unused ports, and the bidirectionality (each wire is counted from both sides in the port array, so raw port counts must be halved or matched pairwise).
- These assertions are expensive (O(A + B * n) for C3, O(A) for C2) but only run in debug mode.
