## Context

Wire classification is the core of Step 4 of the split algorithm. Given the allocation function sigma (a `HashMap<AgentId, WorkerId>`), every wire in the net must be classified as exactly one of: internal (both endpoints in the same partition), interface (one endpoint is a pre-existing Lafont FreePort), or border (endpoints in different partitions). Border wires are replaced by pairs of FreePort(Boundary) sentinels with unique border IDs. This task implements the classification and border ID generation logic, producing the data structures consumed by the subnet builder (TASK-0050).

## Acceptance Criteria

- [ ] `WireClass` enum (or equivalent) distinguishes Internal, Interface, and Border
- [ ] Function classifies every wire by iterating the port array with the allocation map
- [ ] Border wires are detected only when `sigma(a) != sigma(b)` and `a.id < b.id` (avoids double-processing)
- [ ] Each border wire receives a unique `borderId` starting from `max_freeport_id + 1`
- [ ] `WireClassification` tracks `border_id_start` (= `max_freeport_id + 1`) for R15a
- [ ] Border map (`HashMap<u32, (PortRef, PortRef)>`) is populated: `bid -> (AgentPort(a, p), AgentPort(b, q))`
- [ ] Border entries per partition are collected: `Vec<Vec<(AgentId, PortId, u32)>>` indexed by worker
- [ ] Pre-existing Lafont FreePorts are classified as interface, not border (R15)
- [ ] DISCONNECTED entries are skipped

## Notes

- The `a.id < b.id` convention for border wire processing is critical: without it, each border wire would be processed twice, leading to duplicate border IDs and violating C3.
- This function does NOT build the sub-nets. It produces the data structures that TASK-0050 (build_subnet) consumes.
- Complexity: O(A * PORTS_PER_SLOT) which is O(A) since PORTS_PER_SLOT is constant.
