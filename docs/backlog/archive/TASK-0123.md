## Context

The `AuthToken` is a 256-bit (32-byte) cryptographically random value used for worker registration authentication. It is generated by the coordinator at startup using a CSPRNG (`rand::rngs::OsRng`), encoded as base64 for human display and file storage, and transmitted as raw bytes in the wire protocol. This task defines the `AuthToken` struct and its construction/serialization methods. Constant-time verification is in a separate task (TASK-0124).

## Acceptance Criteria

- [ ] `AuthToken` struct wraps `[u8; 32]` as a newtype
- [ ] `AuthToken` derives ONLY `Clone` -- MUST NOT derive `PartialEq` or `Eq` (SC-009: all comparisons MUST go through `verify()`)
- [ ] `AuthToken` implements `Debug` manually to print `AuthToken([REDACTED])` (R34: token must not appear in debug output after initial display)
- [ ] `AuthToken::generate()` creates a new token using `rand::rngs::OsRng` and `rand::RngCore::fill_bytes` (R9)
- [ ] `AuthToken::from_base64(s: &str)` decodes a base64 string, returns `Result<Self, TokenError>` (R10)
- [ ] `AuthToken::from_base64` returns `TokenError::InvalidBase64` for non-base64 input
- [ ] `AuthToken::from_base64` returns `TokenError::InvalidLength` if decoded bytes are not exactly 32
- [ ] `AuthToken::to_base64(&self)` encodes the token as standard base64 (44 chars) (R10)
- [ ] `AuthToken::as_bytes(&self)` returns `&[u8; 32]` for wire protocol serialization (R19)
- [ ] Roundtrip: `AuthToken::from_base64(token.to_base64()).unwrap()` followed by `verify()` returns `true` against the original (T2)

## Notes

- The `Debug` impl is critical for security: `AuthToken` will appear in structs like `SecurityConfig` that may be logged or printed. The redaction ensures the token value never leaks into logs after the one-time startup display (R11, R34).
- `AuthToken` MUST NOT derive `PartialEq` or `Eq`. All comparisons MUST go through the `verify()` method which uses constant-time comparison via `subtle::ConstantTimeEq` (SC-009, R15).
- Base64 encoding uses the STANDARD alphabet (not URL-safe), producing 44 characters for 32 bytes (`ceil(32/3) * 4 = 44`).
- The `base64` crate (version 0.22) MUST be added to the always-on dependencies (extending SPEC-13 R11), per R10.
- The `subtle` crate (version 2.x) MUST be added to the always-on dependencies (extending SPEC-13 R11), per R15.
- Wire protocol uses raw `[u8; 32]` in `RegisterPayload.auth_token` (not the `AuthToken` struct), per R14.
