## Context

The port array is a flat `Vec<PortRef>` where each agent occupies a fixed number of slots (`PORTS_PER_SLOT = 3`). The index for port `(agent_id, port_id)` is computed by `agent_id * 3 + port_id`. This uniform layout simplifies indexing at the cost of wasting 2 slots for ERA agents (which only have 1 port).

The `DISCONNECTED` sentinel (`FreePort(u32::MAX)`) marks slots that are not connected to anything. It is a transient state used during reduction operations. After a reduction rule completes, no port of a live agent may contain `DISCONNECTED`.

## Acceptance Criteria

- [ ] `PORTS_PER_SLOT` is a `pub const usize` equal to `3`
- [ ] `port_index(AgentId, PortId) -> usize` is a `pub` `#[inline]` function returning `(agent_id as usize) * PORTS_PER_SLOT + (port_id as usize)`
- [ ] `DISCONNECTED` is a `pub const PortRef` equal to `PortRef::FreePort(u32::MAX)`
- [ ] Doc comments explain that DISCONNECTED is transient and violates T1 if it persists after a rule

## Key Types / Signatures

```rust
/// Number of port slots per agent in the port array.
/// 3 = 1 (principal) + 2 (maximum auxiliary).
/// ERA wastes 2 slots, but uniformity simplifies indexing.
pub const PORTS_PER_SLOT: usize = 3;

/// Computes the index in the port array for port (agent_id, port_id).
#[inline]
pub fn port_index(agent_id: AgentId, port_id: PortId) -> usize {
    (agent_id as usize) * PORTS_PER_SLOT + (port_id as usize)
}

/// Sentinel for disconnected or invalid port slots.
/// Used internally during reduction operations.
/// A slot containing DISCONNECTED violates invariant T1 (linearity)
/// if it persists after a reduction rule completes.
pub const DISCONNECTED: PortRef = PortRef::FreePort(u32::MAX);
```

## Notes

- `port_index` is `#[inline]` because it is called on every port array access (hot path).
- `FreePort(u32::MAX)` is safe as a sentinel because valid `FreePort` IDs are assigned by the partitioner starting from 0 and incrementing. `u32::MAX` (~4 billion) will never be reached as a valid border ID.
