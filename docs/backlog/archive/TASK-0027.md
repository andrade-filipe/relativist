## Context

SPEC-03 Section 4.6.1 defines `reduce_step`, the core function of the reduction loop. It dequeues the next pair from the redex queue, validates it (discards stale redexes), normalizes the pair, dispatches to the correct interaction function, and optionally verifies invariants in debug mode. This is the function that `reduce_all` and `reduce_n` call repeatedly.

Key behaviors:
- Stale redex tolerance (R19, SPEC-01 I4): silently discard invalid pairs.
- Strong confluence preservation (R23): no ordering constraint on redex selection (FIFO is fine).
- In-place mutation (R24): mutates the Net directly.
- Debug assertions (R15): invoke `Net::assert_all_invariants()` after each rule in debug mode.

## Key Types / Signatures

```rust
use super::dispatch::{get_rule, normalize_pair, Rule};
use super::rules::{interact_anni, interact_comm, interact_eras, interact_void};
use crate::net::Net;

/// Result of a single reduction step (SPEC-03 Section 4.6.1).
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum StepResult {
    /// A redex was successfully reduced. Contains the applied rule.
    Reduced(Rule),
    /// The queue is empty: net is in Normal Form.
    NormalForm,
}

/// Executes a single reduction step (SPEC-03 Section 4.6.1, R12).
///
/// Dequeues pairs from the redex queue until a valid (non-stale) one is found,
/// applies the corresponding rule, and returns the result.
///
/// Stale redex handling (R19, SPEC-01 I4): pairs where either agent no longer
/// exists or the principal-port connection has changed are silently discarded.
///
/// Debug assertions (R15): in debug mode, `net.assert_all_invariants()` is
/// called after each rule to verify T1, I1, I2, I6, I7.
///
/// Complexity: O(1) amortized (each stale discard is O(1);
/// rule application is O(1)).
pub fn reduce_step(net: &mut Net) -> StepResult {
    loop {
        // 1. Dequeue next pair
        let (a_id, b_id) = match net.redex_queue.pop_front() {
            Some(pair) => pair,
            None => return StepResult::NormalForm,
        };

        // 2. Verify validity (discard stale)
        if !net.is_valid_redex(a_id, b_id) {
            continue;
        }

        // 3. Normalize pair for dispatch
        let (a, b) = normalize_pair(a_id, b_id, net);

        // 4. Determine rule
        let sym_a = net.agents[a as usize].unwrap().symbol;
        let sym_b = net.agents[b as usize].unwrap().symbol;
        let rule = get_rule(sym_a, sym_b);

        // 5. Apply rule
        match rule {
            Rule::Anni => interact_anni(net, a, b),
            Rule::Comm => interact_comm(net, a, b),
            Rule::Eras => interact_eras(net, a, b),
            Rule::Void => interact_void(net, a, b),
        }

        // 6. Verify invariants in debug mode (R15)
        #[cfg(debug_assertions)]
        net.assert_all_invariants();

        return StepResult::Reduced(rule);
    }
}
```

## Notes

- The inner `loop` handles stale redex discarding. Each iteration is O(1) (pop + validity check). The total cost of discarding K stale redexes is O(K), amortized over the interactions that created them.
- **SPEC-03 v3 change (SC-004/SC-007):** `reduce_step` does NOT increment counters. R12 step (5) was changed from "increment the interaction counter" to "return the applied rule so that callers can maintain interaction counts via ReductionStats." Counter management is the responsibility of `reduce_all` and `reduce_n`, which inspect the `StepResult::Reduced(Rule)` returned. This avoids adding mutable counter state to `Net`.
- The function operates by in-place mutation (R24). No new Net is created.
- Strong confluence (R23) is preserved by design: the function picks the FIFO-next valid redex, but correctness does not depend on this order.
