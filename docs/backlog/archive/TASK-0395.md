## Notes

- **Why these tests are integration-level, not unit-level:** they span `run_grid_delta` + `resolver` + `LocalDeltaDispatch` (which calls `handle_round_start` + `handle_final_state_request`). The whole round-trip is the unit of interest. Unit tests at the resolver level (inside `border_resolver.rs::tests`) and at the worker level (inside `worker.rs::tests`) already exercise the pieces in isolation; this is the composition test.
- **Why net canonicalization and not byte-equality:** `run_grid_delta` and `run_grid` may produce nets that differ in AgentId labeling (depending on which worker minted which agent) but are isomorphism-equivalent. The canonicalizer relabels AgentIds 0..N in a deterministic order based on net structure, yielding a representation that equates up to isomorphism.
- **On the `WorkerDispatch` trait exposure:** this task uncovers a general need (SF-001 / TASK-0396 will face the same issue when wiring `run_grid_entry`). Coordinating the test-support feature gate between TASK-0395 and TASK-0396 DEV stages is recommended.
- **MF-001 / MF-002 correlation:** if TASK-0394 DEV is incomplete when this task's DEV starts, CON-DUP / CON-ERA / DUP-ERA tests in UT-0385-08 will fail. That's the intended correctness gate — UT-0385-08 is effectively a regression test for MF-001 + MF-002 together.
- **Expected DEV runtime:** UT-0385-06/07 on minimal 2-CON fixtures should complete in <100ms each. UT-0385-08 runs 6 fixtures × 2 modes (lenient/strict) = 12 parameterized cases; total ~500ms-1s. No timeout concerns.

## Design Choices Flagged for Spec-Critic (Stage 1.5)

- **DC-0395-A — exposure gate for `run_grid_delta` and related types.** Options:
  - (a) Promote `run_grid_delta`, `WorkerDispatch`, `RoundStartDispatch`, `RoundResultPayload` from `pub(crate)` to `pub` — simplest but leaks internals.

  - **Default plan:** (b). **Spec-critic please confirm or choose otherwise.**
- **DC-0395-B — canonicalization algorithm.** Two viable approaches:
  - (a) Topological sort by symbol + sorted-port-targets, relabel AgentIds 0..N in traversal order, return the relabeled net's byte serialization.
  - (b) Graph isomorphism via weisfeiler-lehman hashing or similar.
  - **Default plan:** (a) — simpler, deterministic for the small test inputs here (at most 20 agents per fixture). If a fixture grows beyond the scope where (a) is deterministic (e.g., two agents of the same symbol with identical neighborhoods), flag for spec-critic to choose (b).
- **DC-0395-C — parametrization shape for UT-0385-08.** Options:
  - (a) One `#[test]` fn per rule (6 fns, 12 with lenient/strict variants).
  - (b) One `#[test]` fn with a loop over fixtures + modes.
  - **Default plan:** (a) — better per-case failure diagnostics. Each fn name embeds the rule + mode. `cargo test` output names the failing case explicitly.
