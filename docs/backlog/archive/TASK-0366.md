## Context

SPEC-19 R33 mandates the exact shape of `BorderDelta { border_id: u32,
new_target: PortRef }` with derives `Debug, Clone, PartialEq, Eq,
Serialize, Deserialize`. The earlier §3.2 bundle (TASK-0362) already
shipped this struct in `relativist-core/src/merge/border_graph.rs` lines
126-131, and re-exported it via `merge/mod.rs` (`pub use
border_graph::{AddBorderEntry, BorderDelta, BorderGraph, BorderState};`).

**The struct itself is re-used verbatim.** What this task adds is:
1. A protocol-layer re-export (`crate::protocol::BorderDelta`) so the
   subsequent `Message` variants (TASKs 0368, 0369) can name
   `BorderDelta` without reaching up into `merge/`.
2. A `//!` doc-comment update at the top of `protocol/types.rs` noting
   that SPEC-19 §3.4 appends five new variants after `RegisterNack` and
   that `BorderDelta` is the shared payload type for delta rounds.

No new types. No test-count change. Pure-core invariant: `BorderDelta`
originates in `merge/` (pure-core, per R19 of §3.2), so protocol-layer
re-export does NOT violate the dependency direction (`protocol/` → `merge/`
is an allowed edge per SPEC-13 R6-R8).

```rust
// In relativist-core/src/merge/border_graph.rs — new structs per R33
// (DC-B3 + DC-B5 amendment, 2026-04-17). Verbatim from SPEC-19 §3.4.

use crate::net::{AgentId, PortRef, Symbol};

/// DC-B3: internal port reconnection dispatched from coordinator to worker
/// after a CON-DUP border-redex resolution produces new agents whose
/// auxiliary ports are not themselves borders.
#[derive(Debug, Clone, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
pub struct LocalReconnection {
    /// Agent on the worker side whose port needs rewiring.
    pub agent_id: AgentId,
    /// Port of that agent (0 = principal, 1..arity = aux).
    pub port: u8,
    /// New endpoint this port connects to.
    pub new_target: PortRef,
}

/// DC-B5: AgentId allocation request sent coordinator → worker via RoundStart.
/// Workers allocate a fresh AgentId per request and echo it back in
/// RoundResult.minted_agents (SPEC-19 R48).
#[derive(Debug, Clone, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
pub struct PendingCommutation {
    /// Correlation ID, unique within the BSP run — coordinator allocates
    /// from a monotonically increasing counter (R48).
    pub request_id: u32,
    /// Symbol type for the new agent (CON, DUP, ERA).
    pub symbol_type: Symbol,
    /// Arity of the new agent (0 for ERA, 1 for CON/DUP principal, etc.).
    pub arity: u8,
}

/// DC-B5: Worker's response to a PendingCommutation request, carried on
/// RoundResult.minted_agents.
#[derive(Debug, Clone, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
pub struct MintedAgent {
    /// Matches the PendingCommutation.request_id from the RoundStart.
    pub request_id: u32,
    /// The AgentId the worker has allocated for this request. MUST NOT
    /// overlap the coordinator-reserved range `u32::MAX - 10_000 .. u32::MAX`
    /// (R48).
    pub minted_agent_id: AgentId,
}
```

## Notes

- **Why a re-export rather than moving the struct:** the struct's
  canonical home is `merge/` because `BorderGraph` (pure-core) consumes
  it directly. Moving it to `protocol/` would force `merge/` to depend
  on `protocol/`, violating R19 (§3.2) and the SPEC-13 dependency
  direction. Re-exporting preserves the clean layering while giving the
  protocol layer a local name.
- **Why augment rather than rewrite the `//!` doc:** the existing doc
  block is canonical SPEC-06 prose; rewriting would trigger cascades in
  `spec-reviews/` and `docs/progress.md` references. Appending a SPEC-19
  paragraph keeps history-friendly diffs.
- **R37 compliance:** the discriminant-stability comment (`IMPORTANT: ...
  MUST be appended at the end of this enum`) is explicitly preserved. The
  actual enforcement test ships in TASK-0371.
