## Context

The `Symbol` type represents the 3 universal symbols of Lafont's Interaction Combinators (REF-002, p.71-72): gamma (constructor), delta (duplicator), and epsilon (eraser). Every agent in the net has exactly one symbol. The symbol determines the agent's arity and which interaction rule applies when two agents form an active pair. This is the most fundamental type in the entire system.

## Acceptance Criteria

- [ ] `Symbol` enum has exactly 3 variants: `Con`, `Dup`, `Era`
- [ ] `Symbol` has `#[repr(u8)]` with `Con = 0`, `Dup = 1`, `Era = 2`
- [ ] `Symbol` derives: `Debug, Clone, Copy, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize`
- [ ] Doc comments explain the formal correspondence (Con = gamma/constructor, Dup = delta/duplicator, Era = epsilon/eraser)

## Key Types / Signatures

```rust
/// The 3 universal symbols of Lafont (REF-002, p.71-72).
/// Con = gamma (constructor, arity 2)
/// Dup = delta (duplicator, arity 2)
/// Era = epsilon (eraser, arity 0)
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize)]
#[repr(u8)]
pub enum Symbol {
    Con = 0,
    Dup = 1,
    Era = 2,
}
```
