## Context

SPEC-07 Section 4.3 defines the coordinator lifecycle: parse config, load network, run the distributed grid loop (via `run_coordinator` from Phase 5), output results. This task wires the coordinator subcommand to the async `run_coordinator` function from SPEC-06. It is the async counterpart of `run_local_command` (TASK-0111).

The coordinator entry point is an `async fn` because it needs the tokio runtime for TCP operations. It follows the lifecycle: parse config -> load net -> run distributed loop -> write output -> write metrics -> print summary.

## Key Types / Signatures

```rust
use crate::config::{CoordinatorArgs, build_grid_config, build_node_config_coordinator, parse_strategy};
use crate::io::{load_net_from_file, save_net_to_file, write_metrics, print_summary};
use crate::protocol::run_coordinator;
use crate::error::RelativistError;

/// Execute coordinator mode: distributed grid loop over TCP.
pub async fn run_coordinator_command(args: CoordinatorArgs) -> Result<(), RelativistError> {
    // 1. Parse config
    let grid_config = build_grid_config(&args);
    let node_config = build_node_config_coordinator(&args);
    let strategy = parse_strategy(&args.strategy)?;

    // 2. Load network (sync, wrapped in spawn_blocking if needed)
    let net_path = args.input.clone();
    let net = tokio::task::spawn_blocking(move || load_net_from_file(&net_path))
        .await
        .map_err(|e| RelativistError::Config(format!("task join error: {}", e)))??;

    // 3. Distributed grid loop (SPEC-06 Section 4.6)
    let (reduced_net, metrics) = run_coordinator(net, &node_config, &grid_config, &*strategy).await?;

    // 4. Output
    print_summary(&reduced_net, &metrics);

    if let Some(ref output_path) = args.output {
        save_net_to_file(&reduced_net, output_path)?;
    }

    if let Some(ref metrics_path) = args.metrics {
        write_metrics(&metrics, metrics_path)?;
    }

    Ok(())
}
```

## Notes

- The net loading could use `tokio::fs::read` directly instead of `spawn_blocking`. Either approach is acceptable. The key constraint is not blocking the tokio runtime with synchronous I/O.
- The coordinator command requires `#[tokio::main]` or equivalent in `main.rs`. See TASK-0116 for the main entrypoint wiring.
- Error messages should include the file path for diagnostics (SPEC-07 R14).
