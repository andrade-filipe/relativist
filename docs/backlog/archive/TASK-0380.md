# TASK-0380: Worker Round 0 handler — receive `InitialPartition`, seed state (R21.1, R22)

**Bundle:** SPEC-19 §3.3 — Stateful Worker Lifecycle (item 2.26-C)
**Spec:** SPEC-19 R21 phase 1 (Initial Dispatch), R22 (persistent partition state)
**Priority:** P0 (lifecycle entry point)
**Status:** TODO
**Depends on:** TASK-0379 (struct exists); 2.26-A TASK-0367 (`Message::InitialPartition` variant)
**Blocked by:** TASK-0379; 2.26-A wire variants
**Estimated complexity:** S (~100 LoC inc. tests)

## Context

R21 phase 1 says: "the coordinator sends `InitialPartition` containing the full
`Partition` (one-time). The worker stores this partition in its local state.
This is the only time the complete partition crosses the wire until convergence."

This task wires `Message::InitialPartition` (shipped by 2.26-A TASK-0367) into the
worker's runtime. It extends the worker's state so that `WorkerDeltaState` (from
TASK-0379) can live alongside the v1 FSM, and adds a pure handler that takes the
incoming `Message::InitialPartition { round, partition }` payload and returns a
fresh `WorkerDeltaState`.

**DC-C1 (bundle index):** this task assumes **no explicit ack** is sent back to the
coordinator after `InitialPartition` — the transition to Round 1 is triggered by
arrival of the first `Message::RoundStart` (option (b) in DC-C1). If spec-critic
rules that an explicit ack is required, a new `Message::InitialPartitionAck`
variant must be added to 2.26-A and this task gains a 5-10 LoC `SendMessage`
action in the return value.

      `Option<WorkerDeltaState>` field named `delta_state`, defaulting to `None`.
      `Default::default()` continues to produce a `WorkerContext` with
      `state = WorkerState::Init`, `round = 0`, `delta_state = None`.
- [ ] A new pure function is added to `worker.rs`:
      ```rust
      pub fn handle_initial_partition(
          ctx: &mut WorkerContext,
          round: u32,
          partition: Partition,
      ) -> Vec<WorkerAction>
      ```
      with semantics:
      1. Assert (debug_assert in release, explicit `ReductionError` in FSM in
         test mode) that `round == 0` (R21 phase 1 guarantees Round 0 only).
      2. Build `WorkerDeltaState::from_initial_partition(partition)` (via
         TASK-0379's constructor) and store it in `ctx.delta_state = Some(...)`.
      3. Transition `ctx.state` from `WorkerState::Idle` to a new variant
         `WorkerState::DeltaIdle` (added in this task) indicating the worker
         is in delta mode and waiting for the first `RoundStart`.
      4. Return `vec![WorkerAction::LogTransition { from, to }]` — **no
         SendMessage** per DC-C1 option (b).

      removed or renumbered.
- [ ] The v1 `transition(ctx, event)` function is NOT called by

      Existing v1 transitions are unchanged.

## Key Types / Signatures

```rust
// relativist-core/src/worker.rs

pub enum WorkerState {
    // existing v1 variants unchanged
    Init,
    Idle,
    Reducing,
    Returning,
    Error,
    Done,
    /// SPEC-19 R21.1 (TASK-0380): delta mode, partition stored, waiting
    /// for first `Message::RoundStart`.
    DeltaIdle,
}

pub struct WorkerContext {
    pub state: WorkerState,
    pub round: u32,
    /// SPEC-19 R22 (TASK-0380): delta-mode persistent state. `None` in
    /// v1 full-partition mode; `Some(_)` after Round 0 `InitialPartition`.
    pub delta_state: Option<WorkerDeltaState>,
}

/// SPEC-19 R21.1, R22 (TASK-0380): handle the Round 0
/// `Message::InitialPartition { round: 0, partition }` message. Stores
/// the partition in `ctx.delta_state`, transitions to `DeltaIdle`, and
/// returns log actions. **Does NOT send an ack** (DC-C1 option (b) —
/// the coordinator treats `InitialPartition` dispatch as fire-and-forget;
/// the worker's first `RoundResult` at Round 1 is the implicit ack.)
pub fn handle_initial_partition(
    ctx: &mut WorkerContext,
    round: u32,
    partition: Partition,
) -> Vec<WorkerAction> {
    debug_assert!(round == 0, "InitialPartition MUST arrive at round 0 (R21.1)");
    let from = ctx.state.clone();
    ctx.delta_state = Some(WorkerDeltaState::from_initial_partition(partition));
    ctx.state = WorkerState::DeltaIdle;
    ctx.round = 0;
    vec![WorkerAction::LogTransition {
        from,
        to: ctx.state.clone(),
    }]
}
```

## Notes

- **DC-C1:** this task implements option (b) — no explicit ack. The Round 0
  → Round 1 transition is arrival-triggered. If spec-critic rules option (a)
  (explicit `InitialPartitionAck` variant), 2.26-A gains a 6th new variant
  and this task's return value changes to include
  `WorkerAction::SendMessage(Box::new(Message::InitialPartitionAck { ... }))`.
- **Retry semantics:** not addressed. Coordinator retries are a 2.26-C-wire
  concern.
- **v1 untouched:** the v1 `transition` function, existing `WorkerEvent`
  variants, and existing tests all keep byte-identical behavior.
- **Wire-level integration (DC-C2):** the call site that pattern-matches
  `Message::InitialPartition` and invokes `handle_initial_partition` lives
  in `protocol/worker.rs` (the async read-loop). Updating that call site is
  OUT of scope for this bundle — it belongs to either a 2.26-C-wire extension
  or 2.26-D. This task ships the pure handler; the integration stub is
  provided via an inline unit test that simulates the call.

## Spec-Critic Amendments (2026-04-17)

| DC | Verdict | Change |
|---|---|---|
| DC-C1 | B (locked in) | Arrival-triggered Round 0 → Round 1 transition; NO ack/nack from worker after `InitialPartition` |

**Details:**
- Per DC-C1 of `docs/spec-reviews/SPEC-19-section-3.3-2.26C-design-choices-2026-04-17.md`: Round 0 is fire-and-forget. The worker stores the partition and transitions to `DeltaIdle` on arrival of `Message::InitialPartition`, then waits for the first `Message::RoundStart` or `Message::FinalStateRequest`.
- No ack message is sent after `InitialPartition` — the coordinator learns storage succeeded only when the worker's first `RoundResult` (round=1) arrives. This mirrors the v1 `AssignPartition` → `PartitionResult` fire-and-forget contract (`worker.rs` L108-117).
- R21.1 literal ("The worker stores this partition in its local state") imposes a storage obligation, NOT a protocol obligation to confirm storage.
- Option A (`RoundStartAck` variant) rejected: adds a 6th coordinator-bound wire variant mid-flight, cascades through 2.26-A TASK-0371 byte-stability test for zero benefit (TCP FIN / timeout via SPEC-06 R18-R19 already detects failure).
- Option C (empty `RoundResult` as ack) rejected: would require a SPEC-19 §3.3 amendment to R26's `RoundResult` shape.
- No `Message::InitialPartitionAck` added; 2.26-A's 5-variant shape (discriminants 7..=11) stands.
