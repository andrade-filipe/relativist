## Context

SPEC-09 Section 3.2.5 (R16) defines the **MixedNet** benchmark: a net containing redexes of ALL 6 interaction rule types in controlled proportions. No benchmark in the Haskell prototype exercises all 6 rules simultaneously in a distributed context.

The net generates N pairs of each rule type (6N total pairs, ~12N agents):
- CON-CON pairs (annihilation cross)
- DUP-DUP pairs (annihilation parallel)
- ERA-ERA pairs (void)
- CON-DUP pairs (commutation)
- CON-ERA pairs (erasure)
- DUP-ERA pairs (erasure)

This is Profile B due to the presence of CON-DUP pairs, which trigger expansion cascades. The CON-DUP interactions create additional agents that may interact with other segments of the net.

```rust
use crate::bench::{Benchmark, BenchmarkId};
use crate::bench::isomorphism::nets_isomorphic;
use crate::net::{Net, Symbol, PortRef};

/// MixedNet: N pairs of each of the 6 rule types.
/// Profile B (due to CON-DUP commutation). Exercises ALL 6 rules.
/// Expected: complex cascading reduction ending in a normal form.
///
/// SPEC-09 R16. Addresses DISC-003 v2 Section 5.2 point 2
/// (insufficient coverage of the 6 rules in distributed context).
pub struct MixedNet;

impl Benchmark for MixedNet {
    fn id(&self) -> BenchmarkId {
        BenchmarkId::MixedNet
    }

    fn describe(&self, size: u32) -> String {
        format!("MixedNet: {} pairs per rule, {} total pairs, ~{} agents",
                size, size * 6, size * 12)
    }

    fn make_net(&self, n: u32) -> Net {
        // Approximate capacity: 12N agents (2 per pair, 6 types)
        // Some types (ERA) have 1 port, others (CON, DUP) have 3 ports.
        let mut net = Net::with_capacity((n * 12) as usize);
        let mut fp = 0u32; // FreePort counter

        // 1. ERA-ERA pairs (void): 2 agents each, no aux ports
        for _ in 0..n {
            let a = net.create_agent(Symbol::Era);
            let b = net.create_agent(Symbol::Era);
            net.connect(PortRef::AgentPort(a, 0), PortRef::AgentPort(b, 0));
        }

        // 2. CON-CON pairs (annihilation cross): 2 agents, 4 aux FreePorts
        for _ in 0..n {
            let a = net.create_agent(Symbol::Con);
            let b = net.create_agent(Symbol::Con);
            net.connect(PortRef::AgentPort(a, 0), PortRef::AgentPort(b, 0));
            net.connect(PortRef::AgentPort(a, 1), PortRef::FreePort(fp)); fp += 1;
            net.connect(PortRef::AgentPort(a, 2), PortRef::FreePort(fp)); fp += 1;
            net.connect(PortRef::AgentPort(b, 1), PortRef::FreePort(fp)); fp += 1;
            net.connect(PortRef::AgentPort(b, 2), PortRef::FreePort(fp)); fp += 1;
        }

        // 3. DUP-DUP pairs (annihilation parallel): 2 agents, 4 aux FreePorts
        for _ in 0..n {
            let a = net.create_agent(Symbol::Dup);
            let b = net.create_agent(Symbol::Dup);
            net.connect(PortRef::AgentPort(a, 0), PortRef::AgentPort(b, 0));
            net.connect(PortRef::AgentPort(a, 1), PortRef::FreePort(fp)); fp += 1;
            net.connect(PortRef::AgentPort(a, 2), PortRef::FreePort(fp)); fp += 1;
            net.connect(PortRef::AgentPort(b, 1), PortRef::FreePort(fp)); fp += 1;
            net.connect(PortRef::AgentPort(b, 2), PortRef::FreePort(fp)); fp += 1;
        }

        // 4. CON-DUP pairs (commutation): 2 agents, 4 aux FreePorts
        for _ in 0..n {
            let con = net.create_agent(Symbol::Con);
            let dup = net.create_agent(Symbol::Dup);
            net.connect(PortRef::AgentPort(con, 0), PortRef::AgentPort(dup, 0));
            net.connect(PortRef::AgentPort(con, 1), PortRef::FreePort(fp)); fp += 1;
            net.connect(PortRef::AgentPort(con, 2), PortRef::FreePort(fp)); fp += 1;
            net.connect(PortRef::AgentPort(dup, 1), PortRef::FreePort(fp)); fp += 1;
            net.connect(PortRef::AgentPort(dup, 2), PortRef::FreePort(fp)); fp += 1;
        }

        // 5. CON-ERA pairs (erasure): CON has 2 aux FreePorts
        for _ in 0..n {
            let con = net.create_agent(Symbol::Con);
            let era = net.create_agent(Symbol::Era);
            net.connect(PortRef::AgentPort(con, 0), PortRef::AgentPort(era, 0));
            net.connect(PortRef::AgentPort(con, 1), PortRef::FreePort(fp)); fp += 1;
            net.connect(PortRef::AgentPort(con, 2), PortRef::FreePort(fp)); fp += 1;
        }

        // 6. DUP-ERA pairs (erasure): DUP has 2 aux FreePorts
        for _ in 0..n {
            let dup = net.create_agent(Symbol::Dup);
            let era = net.create_agent(Symbol::Era);
            net.connect(PortRef::AgentPort(dup, 0), PortRef::AgentPort(era, 0));
            net.connect(PortRef::AgentPort(dup, 1), PortRef::FreePort(fp)); fp += 1;
            net.connect(PortRef::AgentPort(dup, 2), PortRef::FreePort(fp)); fp += 1;
        }

        net
    }

    fn default_sizes(&self) -> Vec<u32> {
        vec![10, 50, 100, 500, 1_000]
    }

    fn verify(&self, sequential_result: &Net, distributed_result: &Net) -> bool {
        nets_isomorphic(sequential_result, distributed_result)
    }
}
```

- Unit test: `make_net(1)` has agents of all 3 symbols: Con, Dup, Era.

- Unit test: after `reduce_all(make_net(10))`, net reaches a normal form with exactly 40 ERA agents (4N), 0 redexes.
- Unit test: all remaining agents after reduction are ERA, each connected to a FreePort.
- Unit test: sequential and distributed results for `make_net(5)` are isomorphic.

## Notes

- The actual agent count is slightly less than 12N because ERA agents only have 1 port (no auxiliary ports). ERA-ERA pairs contribute 2 agents; CON-ERA and DUP-ERA pairs contribute 2 agents each. The total is: ERA-ERA: 2N, CON-CON: 2N, DUP-DUP: 2N, CON-DUP: 2N, CON-ERA: 2N, DUP-ERA: 2N = 12N agents.
- The CON-DUP pairs will expand during reduction, potentially creating interactions between expanded agents and other segments. This creates Profile B behavior for the entire net.
- The `verify` method uses `nets_isomorphic` because the normal form after reducing all 6 rule types simultaneously is non-trivial (remaining FreePorts reconnected in complex patterns).
