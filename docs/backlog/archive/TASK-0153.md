## Context

When the coordinator receives `PartitionResult` messages containing `WorkerRoundStats` data (extended by SPEC-11 with `reduce_duration_secs` and `interactions_by_rule`), it must aggregate the worker-reported metrics into its own Prometheus registry (R13). This is the mechanism by which per-worker reduction statistics become visible through the single `/metrics` scrape endpoint.

Aggregation includes: observing worker reduction durations in a histogram, incrementing per-rule interaction counters (using `interactions_by_rule_total` Family metric), and tracking bytes received. This task implements the aggregation function that bridges `WorkerRoundStats` to `CoordinatorMetrics`.

## Key Types / Signatures

```rust
/// Aggregate a worker's round stats into the coordinator's Prometheus registry (R13).
///
/// Called by the coordinator after receiving each `PartitionResult` message.
/// Multiple calls per round (one per worker) are expected and safe.
///
/// The `WorkerRoundStats` type is defined in the merge module (SPEC-05 R37,
/// extended by SPEC-11 with reduce_duration_secs and interactions_by_rule).
#[cfg(feature = "metrics")]
pub fn aggregate_worker_stats(
    metrics: &CoordinatorMetrics,
    stats: &WorkerRoundStats,
    payload_bytes: u64,
) {
    // Observe worker reduction duration
    // metrics.round_duration_seconds or a dedicated histogram

    // Increment return_bytes_total by the payload size.
    metrics.return_bytes_total.inc_by(payload_bytes);

    // Aggregate per-rule interaction counts into the Family metric.
    // Rule index mapping: 0=CON-CON, 1=CON-DUP, 2=CON-ERA, 3=DUP-DUP, 4=DUP-ERA, 5=ERA-ERA
    let rule_names = ["CON-CON", "CON-DUP", "CON-ERA", "DUP-DUP", "DUP-ERA", "ERA-ERA"];
    for (i, &count) in stats.interactions_by_rule.iter().enumerate() {
        if count > 0 {
            let label = vec![("rule".to_string(), rule_names[i].to_string())];
            metrics.interactions_by_rule_total
                .get_or_create(&label)
                .inc_by(count);
        }
    }

    tracing::debug!(
        worker_id = stats.worker_id,
        reduce_duration_secs = stats.reduce_duration_secs,
        local_redexes = stats.local_redexes,
        "aggregated worker round stats"
    );
}
```

## Notes

- The aggregation now uses `WorkerRoundStats` directly instead of the eliminated `WorkerMetricsReport`. The extended `WorkerRoundStats` (SPEC-11 Section 4.4) carries both the canonical SPEC-05 fields and the observability fields.
- The `payload_bytes` parameter is the serialized size of the `PartitionResult` message, not computed from the stats. It is provided by the collect phase which knows the frame size.
- This function is called on the coordinator's main event loop. It must be fast (no allocations beyond label creation, no I/O). All Prometheus counter/histogram operations are lock-free atomic operations.
- **Change history:** Originally this task referenced `WorkerMetricsReport`. SPEC-11 Revised v2 (SC-002) eliminated that type; stats are now part of `WorkerRoundStats`.
