# TASK-0161: Define FileIoError, NetFormat, and InspectOutputFormat types

**Spec:** SPEC-12
**Requirements:** R48, R49, R50, R52
**Priority:** P0 (critical path)
**Status:** TODO
**Depends on:** TASK-0160
**Blocked by:** none
**Estimated complexity:** S (< 50 LoC)

## Context

The I/O subsystem needs a unified error type (`FileIoError`) that distinguishes between file I/O errors, parse errors, serialization errors, and unrecognized/unsupported format errors (R52). It also needs a `NetFormat` enum representing the three supported file formats (binary, text DSL, JSON) used for format detection and override (R48, R50). The `InspectOutputFormat` enum controls the output format of the `inspect` subcommand (R28).

These types are foundational to every other I/O task and must be defined first.

> **Revision note (SPEC-12 v2, SC-004):** The error type was renamed from `IoError` to `FileIoError` to avoid name collision with `std::io::Error` and to prevent duplicate `#[from] std::io::Error` conversions when composed into SPEC-13 R17's `RelativistError`. SPEC-13 R17's `Io(#[from] std::io::Error)` variant MUST be replaced by `FileIo(#[from] FileIoError)` when the `io` module is added.

## Acceptance Criteria

- [ ] `FileIoError` enum is defined with variants: `Io`, `Parse`, `Serialization`, `UnrecognizedFormat`, `UnsupportedFormat`
- [ ] `FileIoError` derives `Debug` and uses `thiserror::Error` for `Display`
- [ ] `FileIoError` has `#[from] std::io::Error` on the `Io` variant for automatic conversion
- [ ] `Parse` variant includes `line: usize` and `message: String` fields
- [ ] `NetFormat` enum is defined with variants: `Bin`, `Ic`, `Json`
- [ ] `NetFormat` derives `Debug`, `Clone`, `Copy`, `clap::ValueEnum`
- [ ] `InspectOutputFormat` enum is defined with variants: `Text`, `Json`
- [ ] `InspectOutputFormat` derives `Debug`, `Clone`, `Copy`, `clap::ValueEnum`
- [ ] `thiserror` dependency is added to `Cargo.toml` if not already present

## Key Types / Signatures

use std::path::Path;

/// Errors from the file I/O subsystem (SPEC-12, R52).
///
/// Named `FileIoError` (not `IoError`) to avoid collision with `std::io::Error`
/// and to allow clean composition into `RelativistError` via `#[from]`.
#[derive(Debug, thiserror::Error)]
pub enum FileIoError {
    #[error("file I/O error: {0}")]
    Io(#[from] std::io::Error),

    #[error("parse error at line {line}: {message}")]
    Parse { line: usize, message: String },

    #[error("serialization error: {0}")]
    Serialization(String),

    #[error("unrecognized file extension: {0}")]
    UnrecognizedFormat(String),

    #[error("format not supported in this build: {0}")]
    UnsupportedFormat(String),
}

/// Supported net file formats (SPEC-12, R48).
#[derive(Debug, Clone, Copy, clap::ValueEnum)]
pub enum NetFormat {
    /// Binary: serde + bincode (fastest, most compact).
    Bin,
    /// Text DSL: human-readable (.ic files).
    Ic,
    /// JSON: serde_json (interop).
    Json,
}

pub enum InspectOutputFormat {
    /// Human-readable text (default).
    Text,
    /// Machine-readable JSON.
    Json,
}

## Test Expectations

- Unit test: `detect_format("foo.bin")` returns `Ok(NetFormat::Bin)`
- Unit test: `detect_format("foo.ic")` returns `Ok(NetFormat::Ic)`
- Unit test: `detect_format("foo.json")` returns `Ok(NetFormat::Json)`
- Unit test: `detect_format("foo.xyz")` returns `Err(FileIoError::UnrecognizedFormat(...))`
- Unit test: `detect_format("foo")` (no extension) returns an error
- Unit test: `FileIoError::Io` variant can be created from `std::io::Error`

## Notes

- The `detect_format` function is placed here because it depends only on the `NetFormat` and `FileIoError` types. It will be used by `load_net` and `save_net` in TASK-0168.
- The `Parse` variant carries a line number for user-friendly error messages from the text DSL parser (R11).
- The `UnsupportedFormat` variant is used for JSON if it is deferred (R17): the format is recognized but not implemented.
- When integrating with SPEC-13 R17, `RelativistError::Io(#[from] std::io::Error)` MUST be replaced by `RelativistError::FileIo(#[from] FileIoError)` to avoid duplicate `#[from]` conversions.
