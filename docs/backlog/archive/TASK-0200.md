## Context

SPEC-14 R1-R3 define the encoding module as a Core Layer module residing in `src/encoding/` with three files: `mod.rs`, `church.rs`, and `arithmetic.rs`. The module must be pure synchronous Rust (no async, no tokio, no I/O) and depend only on types from the `net` module (SPEC-02).

This task creates the directory structure and stub files with the correct imports and re-exports, matching the module organization in R3.

- [ ] `src/encoding/mod.rs` exists with `pub mod church;` and `pub mod arithmetic;`
- [ ] `src/encoding/mod.rs` re-exports: `encode_nat`, `encode_church_into`, `decode_nat`, `build_add`, `build_mul`, `build_exp`
- [ ] `src/encoding/church.rs` exists with placeholder functions and correct signatures
- [ ] `src/encoding/arithmetic.rs` exists with placeholder functions and correct signatures
- [ ] `src/lib.rs` includes `pub mod encoding;` in the Core Layer section
- [ ] No `use tokio::` or `async fn` anywhere in the module (R1)
- [ ] Only imports from `crate::net` (R2)
