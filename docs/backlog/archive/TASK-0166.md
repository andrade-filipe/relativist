## Context

The text DSL serializer converts a `Net` into a human-readable `.ic` string. SPEC-12 Section 4.2 defines the algorithm: assign names to agents (`a0`, `a1`, ...), emit `agent` declarations, emit `wire` declarations (each bidirectional connection once, canonical order), and optionally emit a `root` declaration.

The key correctness requirement (R15) is that the serialized text, when re-parsed, MUST yield a structurally equivalent `Net`. This ensures roundtrip identity: `parse_ic(format_ic(net)) == net`.

## Acceptance Criteria

- [ ] `format_ic(net: &Net) -> String` produces a valid text DSL string
- [ ] Each live agent gets a name `a<id>` (e.g., `a0`, `a1`, `a2`)
- [ ] Agent declarations are emitted in AgentId order: `agent a0 CON`, `agent a1 DUP`, etc.
- [ ] Wire declarations are emitted for each connection, each bidirectional pair emitted exactly once
- [ ] Canonical ordering: for each wire, the endpoint with lower `(AgentId, PortId)` tuple comes first; FreePort endpoints sort after all AgentPort endpoints
- [ ] Port names use the human-readable form: `principal`, `left`, `right` (not `p0`, `p1`, `p2`)
- [ ] ERA agents emit only `principal` port references (never `left`/`right`)
- [ ] Free ports are emitted as `free(<id>)`
- [ ] If `net.root` is `Some(port_ref)`, a `root` line is emitted
- [ ] The output always ends with a trailing newline
- [ ] Roundtrip: `parse_ic(format_ic(net))` produces a net structurally equivalent to `net` (R15)

## Notes

- The canonical ordering of wire endpoints is important to ensure deterministic output. Without it, the same net could produce different text representations depending on iteration order, making roundtrip tests flaky.
- The serializer must handle "holes" in the agent arena (slots where `agents[id] == None`). These are dead agents that have been removed during reduction. The serializer skips them.
- Agent names use the raw `AgentId` index (`a0`, `a1`, ...) rather than sequential renumbering. This means holes may appear (e.g., `a0`, `a2` if agent 1 was removed). This is intentional: it preserves the relationship between names and IDs.
