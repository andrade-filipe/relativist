## Context

SPEC-11 R6 mandates that `reduce_all()` in the reduction module is annotated with `#[tracing::instrument]` to create a span with structured fields. The reduction engine is a hot path (default log level WARN per R5), so the instrument attribute must be lightweight: skip large arguments, and let the span be filtered out entirely when the log level is below the threshold.

The span fields `partition_index` and `initial_redexes` provide the essential context for diagnosing reduction performance without requiring per-interaction logging.

## Acceptance Criteria

- [ ] `reduce_all()` function in `src/reduction/` has `#[tracing::instrument]` attribute
- [ ] Span includes field `partition_index` (optional, passed as parameter or 0 for non-partitioned use)
- [ ] Span includes field `initial_redexes` (size of the redex queue before reduction starts)
- [ ] The `skip` parameter excludes the `Net` argument from span fields
- [ ] Span level is compatible with the default WARN filter (span is created at INFO or DEBUG level)
- [ ] Function compiles and existing behavior is unchanged

## Notes

- The `level = "info"` on the instrument macro means this span is only created when the `relativist::reduction` target is at INFO or more verbose. Since the default level for reduction is WARN (R5), this span is **not created by default** -- it only appears when operators explicitly lower the level. This avoids hot-path overhead in production.
- If `reduce_all()` does not currently take a `partition_index` parameter, one may need to be added (or the field can be set to `0` for standalone reduction). Coordinate with Phase 2 API.
- The `initial_redexes` field is computed from `net.redex_queue.len()` at span entry time, before any reduction occurs.
