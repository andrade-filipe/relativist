**Bundle:** SPEC-19 §3.3 — Stateful Worker Lifecycle (item 2.26-C)
**Spec:** SPEC-19 R25 (maintain `previous_border_state`; emit only changed entries)
**Priority:** P0 (required by TASK-0381)
**Status:** TODO
**Depends on:** TASK-0379 (for `previous_border_state` type)
**Blocked by:** none at DEV time (pure function, no external deps beyond `HashMap`,
  `PortRef`, `BorderDelta`).
**Estimated complexity:** S (~120 LoC inc. tests)

## Context

R25 mandates: "the worker MUST maintain a `previous_border_state:
HashMap<u32, PortRef>` that records the last-reported endpoint for each
border ID. After `reduce_all` and `rebuild_free_port_index`, the worker
compares the current `free_port_index` against `previous_border_state` to
identify changes. Only changed entries are emitted as deltas."

This task ships the pure diff function. TASK-0381 calls it after each
`reduce_all` + `rebuild_free_port_index`.

The semantics of "changed" include three cases:
1. A border ID present in both maps but with different `PortRef` values
   → emit delta `(border_id, new_value)`.
2. A border ID present in the new map but NOT in the previous map
   → emit delta `(border_id, new_value)` (a border entry was added; this
   happens via `apply_border_deltas_to_partition`'s `new_borders` case
   in TASK-0381 — the FIRST report after arrival).
3. A border ID present in the previous map but NOT in the new map
   → emit a "disconnection" delta. **DC-C6 (new)**: how is disconnection
   represented? R17 of SPEC-19 §3.2 says: "the worker reports a delta
   `(border_id, DISCONNECTED)` or `(border_id, None)`." The
   `BorderDelta` struct (R33) has `new_target: PortRef`, not
   `Option<PortRef>`. Two candidates:
   (a) `PortRef` has (or gains) a `Disconnected` variant — this is a
       CROSS-BUNDLE change touching `net/types.rs`, OUT of this bundle's
       allowed files;
   (b) the worker reports disconnection as a distinct `resolved_borders`
       style set — but `RoundResult` does not carry a `resolved_borders`
       field (only `border_deltas`);
   (c) the worker reports `(border_id, FreePort(border_id))` as a
       sentinel meaning "I no longer have a live endpoint; coordinator,
       remove your side-tracking for me."
   Current bundle assumes **(c)** — sentinel encoding via
   `PortRef::FreePort(bid)`. Spec-critic must rule; flagged as
   **DC-C6** to the bundle index (append if kept).

## Acceptance Criteria

- [ ] New pure function in `relativist-core/src/merge/helpers.rs`:
      ```rust

          previous: &HashMap<u32, PortRef>,

      ```
- [ ] Semantics (per R25 + DC-C6 option (c)):
  - For every key K present in `current`:
    - If K not in `previous`, or `previous[K] != current[K]`, emit
      `BorderDelta { border_id: K, new_target: current[K] }`.
    - Else (unchanged) emit nothing.
  - For every key K present in `previous` but NOT in `current`:
    - Emit `BorderDelta { border_id: K, new_target: PortRef::FreePort(K) }`
      (DC-C6 option (c) sentinel — "I have disconnected from this
      border"). Document in a code comment that spec-critic may
      override this encoding.
  - Return the vector with no particular ordering guarantee; Stage 2 TESTS
    will specify whether a deterministic sort by `border_id` is required.
- [ ] Unit tests (inline `#[cfg(test)]`):
  - `compute_outgoing_deltas_empty_both`: both maps empty → `Vec::new()`.
  - `compute_outgoing_deltas_unchanged_entry_emits_nothing`: one entry
    in both with identical `PortRef` → `Vec::new()`.
  - `compute_outgoing_deltas_changed_entry_emits_delta`: one entry with
    `PortRef::AgentPort(AgentId(1), 0)` in previous and
    `PortRef::AgentPort(AgentId(2), 1)` in current → emits

  - `compute_outgoing_deltas_mixed`: a map with 5 entries, 2 unchanged,
    1 changed, 1 new, 1 removed → emits exactly 3 deltas. Assert
    membership by deterministic sort.
  - `compute_outgoing_deltas_large_unchanged`: 1000 identical entries →

## Key Types / Signatures

```rust
// relativist-core/src/merge/helpers.rs

use std::collections::HashMap;
use crate::merge::BorderDelta;
use crate::net::PortRef;

/// SPEC-19 R25 (TASK-0382): diff the previously-reported border-port
/// state against the freshly rebuilt `free_port_index` and emit only
/// the changed entries as `BorderDelta`s for inclusion in the round's
/// `Message::RoundResult`.
///
/// The three cases handled:
/// - **Changed endpoint:** `previous[K] != current[K]` → delta with
///   the new target.
/// - **Newly-created border:** `K ∈ current ∧ K ∉ previous` → delta
///   with the new target (first report after CON-DUP expansion gave
///   the worker a new `FreePort(K)` in TASK-0381's
///   `apply_border_deltas_to_partition`).
/// - **Disconnected border:** `K ∈ previous ∧ K ∉ current` → delta
///   with sentinel `PortRef::FreePort(K)` (DC-C6 option (c) — subject
///   to spec-critic ruling).
///
/// Pure / O(|previous| + |current|). No allocation beyond the result
/// vector.
pub(crate) fn compute_outgoing_deltas(
    previous: &HashMap<u32, PortRef>,
    current: &HashMap<u32, PortRef>,
) -> Vec<BorderDelta> {
    let mut out = Vec::new();

    // Entries that are new or changed in `current`.
    for (&bid, &new_target) in current {
        match previous.get(&bid) {
            Some(prev) if *prev == new_target => {}  // unchanged
            _ => out.push(BorderDelta { border_id: bid, new_target }),
        }
    }

    // Entries that disappeared from `current` (present only in `previous`).
    // DC-C6 option (c): emit sentinel `FreePort(K)` meaning "disconnected".
    // If spec-critic rules option (a) (add `PortRef::Disconnected` variant),
    // this branch changes to `PortRef::Disconnected`. If (b) is chosen
    // (separate `resolved_borders` set), the signature of this function
    // gains a `&mut Vec<u32>` out-parameter and TASK-0381 plumbs it into
    // `Message::RoundResult` (which would require a 2.26-A amendment).
    for &bid in previous.keys() {
        if !current.contains_key(&bid) {
            out.push(BorderDelta {
                border_id: bid,
                new_target: PortRef::FreePort(bid),
            });
        }
    }

    out
}
```

## Notes

- **DC-C6** (flagged in this task): disconnection encoding. Current choice
  (c) — sentinel `FreePort(K)` — is type-preserving and requires no
  cross-bundle changes. The coordinator's `BorderGraph::apply_deltas`
  (TASK-0363 from §3.2 bundle) must agree on this interpretation; if not,
  a follow-up amendment to 2.26-B or to the BorderGraph semantics is
  required. Spec-critic MUST rule before Stage 2 TESTS pins the test.
- **Hot path:** `compute_outgoing_deltas` runs O(|current|) per worker
  per round. For partitions with ~1000 borders, that's 1000 HashMap
  lookups — negligible. No SmallVec optimization planned.
- **Invariance:** if `previous == current` exactly, the function returns
  an empty vector. This is the coordinator-free round case (R3 of §3.1):
  zero delta traffic in both directions for that round.
- **Ordering guarantee:** none. Stage 2 TESTS may require
  `out.sort_by_key(|d| d.border_id)` if downstream hashing / byte-level
  comparisons are required. Current draft leaves the order unspecified
  to match `HashMap` iteration semantics.

## Spec-Critic Amendments (2026-04-17)

| DC | Verdict | Change |
|---|---|---|
| DC-C6 | C (locked in) | Reuse `PortRef::FreePort(u32::MAX)` sentinel (via `crate::net::DISCONNECTED`); do NOT change `BorderDelta.new_target` to `Option<PortRef>` |

**Details:**
- Per DC-C6 of `docs/spec-reviews/SPEC-19-section-3.3-2.26C-design-choices-2026-04-17.md`: disconnected borders (freed by local reduction erasing the agent connected to the border) are emitted as `BorderDelta { border_id, new_target: DISCONNECTED }` where `DISCONNECTED = PortRef::FreePort(u32::MAX)` (import from `crate::net::DISCONNECTED`).
- This matches DC-1 of SPEC-19 §3.2 (BorderGraph) verdict — the in-memory `BorderDelta` IS the same struct that 2.26-A serializes over the wire.
- R33 is LITERAL: `new_target: PortRef`, not `Option<PortRef>`. Changing to `Option<PortRef>` would be a spec deviation requiring SPEC-19 amendment.
- SPEC-18 §4.3 compact wire encoding collapses the sentinel to 1 byte (`PORTREF_TAG_DISCONNECTED = 0xFF`, verified at `net/types.rs` L509 `test_portref_compact_disconnected_is_one_byte`) — Option A (`Option<PortRef>`) would add 1 wire byte per reconnect delta (the hot path) for zero information gain.
- Option B (separate `disconnected: bool` field) rejected: redundant with `new_target == DISCONNECTED`; creates a drift vulnerability.
- **Correction to current task text:** the draft semantics at acceptance criteria line 64 incorrectly emit `PortRef::FreePort(K)` (border-id-indexed) for disconnection. Per DC-C6, the correct sentinel is `PortRef::FreePort(u32::MAX)` (aliased as `crate::net::DISCONNECTED`), NOT `FreePort(K)`. The "Key Types / Signatures" block and tests must use `crate::net::DISCONNECTED` verbatim.
- Two new tests added (replacing and tightening the existing sentinel-oriented tests):
  - `compute_outgoing_deltas_encodes_disconnect_as_sentinel`: construct a partition where local reduction erases an agent connected to a `FreePort(border_id)`; run `compute_outgoing_deltas`; assert the emitted `BorderDelta` has `new_target == crate::net::DISCONNECTED`.
  - `compute_outgoing_deltas_encodes_reconnect_as_agentport`: construct a partition where local reduction connects a new agent's principal port to a `FreePort(border_id)`; assert the emitted `BorderDelta.new_target` is a `PortRef::AgentPort(_, _)` matching the new endpoint.

**Test count delta:** +2 (the existing `compute_outgoing_deltas_removed_entry_emits_sentinel` test updates its expected sentinel from `FreePort(K)` to `DISCONNECTED`; adds two DC-C6-specific tests for disconnect / reconnect paths).

**No spec amendment required.**
