# TASK-0157: Implement axum HTTP server spawn as background tokio task

**Spec:** SPEC-11
**Requirements:** R20, R23, R24, R24a, R33a
**Priority:** P0 (critical path)
**Status:** TODO
**Depends on:** TASK-0154, TASK-0155, TASK-0156
**Blocked by:** none
**Estimated complexity:** M (50-200 LoC)

## Context

The metrics/health/ready HTTP server must run as a background tokio task within the coordinator's async runtime (R23). It binds to the same address as the main coordinator listener but on a separate port (default 9090, configurable via `--metrics-port` per R20). The server must NOT block the grid protocol event loop.

R24a requires that the HTTP server is shut down when the coordinator FSM enters `Done` or `Error` state. The coordinator SHOULD use a `tokio_util::sync::CancellationToken` (or equivalent mechanism) to signal the background HTTP task to terminate gracefully.

R33a specifies that HTTP endpoints MUST NOT be started in local mode (`ProcessRole::Local`). The caller (coordinator startup) is responsible for checking the role, but this function should accept and document this constraint.

This task implements the `spawn_metrics_server()` function that starts the axum HTTP server in a detached tokio task, with graceful shutdown support.

## Key Types / Signatures

```rust
use tokio::task::JoinHandle;
use std::net::SocketAddr;
use tokio_util::sync::CancellationToken;

/// Spawn the metrics/health/ready HTTP server as a background task (R20, R23, R24a).
///
/// The server runs on a dedicated port, separate from the grid protocol port.
/// Returns a JoinHandle that can be used to await server shutdown.
///
/// The server shuts down gracefully when `shutdown_token` is cancelled (R24a).
/// The coordinator FSM SHOULD cancel the token when entering `Done` or `Error` state.
///
/// MUST NOT be called in local mode (R33a) -- the caller is responsible for
/// checking `ProcessRole != Local` before calling this function.
///
/// # Errors
///
/// Does not return an error. If binding fails, an ERROR event is logged
/// and the returned JoinHandle completes immediately.
#[cfg(feature = "metrics")]
pub async fn spawn_metrics_server(
    bind_addr: &str,
    port: u16,
    registry: std::sync::Arc<prometheus_client::registry::Registry>,
    coordinator_is_ready: std::sync::Arc<std::sync::atomic::AtomicBool>,
    shutdown_token: CancellationToken,
) -> JoinHandle<()> {
    let router = metrics_router(registry, coordinator_is_ready);
    let addr: SocketAddr = format!("{}:{}", bind_addr, port)
        .parse()
        .expect("valid bind address");

    tracing::info!(
        bind = %addr,
        "metrics HTTP server starting"
    );

    tokio::spawn(async move {
        let listener = match tokio::net::TcpListener::bind(addr).await {
            Ok(l) => l,
            Err(e) => {
                tracing::error!(
                    error = %e,
                    bind = %addr,
                    "failed to bind metrics HTTP server"
                );
                return;
            }
        };
        // Graceful shutdown: axum::serve with signal (R24a)
        axum::serve(listener, router)
            .with_graceful_shutdown(async move {
                shutdown_token.cancelled().await;
                tracing::info!("metrics HTTP server shutting down");
            })
            .await
            .unwrap_or_else(|e| {
                tracing::error!(error = %e, "metrics HTTP server error");
            });
    })
}
```

## Notes

- `axum::serve` in axum 0.8 supports `with_graceful_shutdown()` which takes a Future. The `CancellationToken` from `tokio_util::sync` provides a clean cancellation mechanism.
- The coordinator FSM (Phase 6, TASK-0108) should cancel the shutdown token when transitioning to `Done` or `Error` state, as required by R24a.
- The server binds to the same address as the main coordinator (respecting `--bind`, R24) but on the metrics port. In practice this is usually `0.0.0.0:9090`.
- If the port is already in use, the server fails gracefully with a log message. The coordinator continues to operate without metrics exposition.
- `tokio_util` may already be a transitive dependency. If not, it should be added as an optional dependency gated under `metrics`.
- **Change history:** Originally used `Arc<AtomicU8>` and had no graceful shutdown. SPEC-11 Revised v2 added R22a (AtomicBool) and R24a (graceful shutdown).
