## Context

`encode_church_into` is the core builder function that constructs a Church numeral inside an existing `Net`. It handles three cases:

- **n = 0** (`lambda f. lambda x. x`): 2 CON + 1 ERA, with self-loop on inner lambda auxiliaries (R5)
- **n = 1** (`lambda f. lambda x. f x`): 3 CON, single application (R6)
- **n >= 2** (`lambda f. lambda x. f^n(x)`): (n+2) CON + (n-1) DUP, application chain with DUP fan-out (R7)

The function returns the `AgentId` of the outer lambda agent (`lambda f`). It does NOT set `net.root` -- the caller is responsible. It panics if n > 10_000 (R4b).

This is the foundation for both `encode_nat` (TASK-0202) and the arithmetic combinators (TASK-0204-0206).

## Key Types / Signatures

```rust
/// Encode a natural number as a Church numeral inside an existing net.
///
/// Returns the AgentId of the outer lambda agent (lambda f).
/// Does NOT set net.root -- the caller is responsible for wiring
/// the returned agent's principal port into the surrounding net.
///
/// # Panics
/// Panics if n > 10_000.
pub fn encode_church_into(net: &mut Net, n: u64) -> AgentId {
    assert!(n <= 10_000, "encode_church_into: n = {n} exceeds maximum supported value 10_000");

    // Step 1: Create the two lambda abstractions
    let lam_f = net.create_agent(Symbol::Con);
    let lam_x = net.create_agent(Symbol::Con);
    net.connect(
        PortRef::AgentPort(lam_f, 2),
        PortRef::AgentPort(lam_x, 0),
    );

    match n {
        0 => { /* ERA on f, self-loop on x */ }
        1 => { /* single application CON */ }
        n => { /* n applications + (n-1) DUPs */ }
    }

    lam_f
}
```
