# TASK-0388: `max_rounds` cap + non-convergence indicator in `GridMetrics` (R30)

**Bundle:** SPEC-19 §3.3 — Stateful Worker Lifecycle (item 2.26-C)
**Spec:** SPEC-19 R30 (preserve `max_rounds` cap; return partial net with
  non-convergence indicator).
**Priority:** P1 (safety — without this, a non-converging input loops forever)
**Status:** TODO
**Depends on:** TASK-0385 (loop body), TASK-0386 (convergence predicate),
  TASK-0387 (final collection still runs on cap).
**Blocked by:** TASK-0385, TASK-0386, TASK-0387.
**Estimated complexity:** S (~80 LoC inc. tests)

## Context

R30: "The delta protocol MUST preserve the v1 capability of `max_rounds`
(SPEC-05 R29). If `max_rounds` is reached without convergence, the
coordinator MUST initiate Final State Collection (R27-R29) and return the
partially reduced net with a non-convergence indicator in metrics."

This task ships:
1. The `check_max_rounds_cap` helper that TASK-0385 calls at the top of
   each loop iteration.
2. The wiring so that when the cap fires, `metrics.delta_max_rounds_hit =
   Some(true)` is set, the loop breaks, Final Collection still runs (per
   R30), and `metrics.converged = false` is explicit.
3. Tests covering (i) `max_rounds = None` → infinite loop allowed, (ii)
   `max_rounds = Some(0)` → immediate exit, (iii) `max_rounds = Some(N)`
   on a net requiring >N rounds → cap hit, partial net returned.

## Key Types / Signatures

```rust
// merge/grid.rs

/// SPEC-19 R30 (TASK-0388): check whether `run_grid_delta`'s round
/// loop has reached the `max_rounds` cap. Called at the top of each
/// iteration BEFORE any resolver/dispatch work for the round.
///
/// Preserves v1 `run_grid` semantics (SPEC-05 R29): `max_rounds ==
/// None` means unbounded (caller must rely on `check_delta_convergence`
/// for termination); `max_rounds == Some(m)` caps the round count at
/// `m` (inclusive: `rounds >= m` → cap hit).
///
/// On cap hit, TASK-0385's caller sets `metrics.delta_max_rounds_hit
/// = Some(true)` and `metrics.converged = false`, then breaks to
/// TASK-0387's Final Collection. The returned net is "best effort" —
/// partially reduced, with any remaining border redexes unresolved.
pub(crate) fn check_max_rounds_cap(
    config: &GridConfig,
    metrics: &GridMetrics,
) -> bool {
    match config.max_rounds {
        None => false,
        Some(m) => metrics.rounds >= m,
    }
}
```

```rust
// merge/grid.rs — TASK-0385's run_grid_delta_inner, relevant excerpt:

loop {
    if check_max_rounds_cap(config, metrics) {
        metrics.delta_max_rounds_hit = Some(true);
        metrics.converged = false;
        break;
    }
    // ... normal round body ...
    if check_delta_convergence(&results, &border_graph) {
        metrics.converged = true;
        // delta_max_rounds_hit stays None — distinguishes this exit.
        break;
    }
}

// R30: Final Collection runs REGARDLESS of convergence vs cap.
let final_net = run_grid_delta_final_collect(...)?;
```

## Notes

- **R30 mandates Final Collection on cap:** the partial-net return is
  NOT a direct return of the `current_net`; it is the result of running
  `merge()` on the collected worker partitions + remaining
  `BorderGraph`. Workers respond with their current partial state;
  coordinator assembles it; returned net may have unresolved border
  redexes (the caller's `metrics.delta_max_rounds_hit = Some(true)`
  flags this).
- **v1 `run_grid` non-regression:** the existing `if let Some(max) =
  config.max_rounds { if metrics.rounds >= max { ... break; } }` block
  in `run_grid` is unchanged. No `delta_max_rounds_hit` write in v1.
  Tests in this task verify the field stays `None` in v1 runs.
- **Zero-max-rounds edge case:** `Some(0)` means "do no rounds at all";
  TASK-0385's loop enters, immediately caps, breaks. Final Collection
  still runs on the initially dispatched partitions (every worker
  responds with its Round-0 partition unchanged). Output net is the
  trivial union of input partitions (bit-identical to input net modulo
  partition ordering). This matches v1's `max_rounds = Some(0)`
  semantics.
- **DC-C3 interaction:** this task assumes `strict_bsp = true` (asserted
  in TASK-0384). Lenient-mode max-rounds semantics are out of scope.
- **Spec-critic review opportunity:** the current design sets
  `metrics.converged = false` AND `metrics.delta_max_rounds_hit =
  Some(true)` on cap, then STILL calls `run_grid_delta_final_collect`.
  An alternative would be to skip Final Collection on cap and return
  the original pre-split net. Spec-critic should rule: R30 literally
  says "MUST initiate Final State Collection", so the current path is
  correct — but the partial net may contain inconsistent state (e.g.,
  `BorderGraph` redexes that the workers' own partitions don't
  reflect). Documented risk.
