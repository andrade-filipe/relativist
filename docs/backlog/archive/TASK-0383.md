## Context

R21 phase 3: "When the coordinator determines Global Normal Form (R4), it
sends `FinalStateRequest` to each worker. Each worker responds with
`FinalStateResult` containing its full current partition. The coordinator
performs a final `merge()` (SPEC-05 R1-R11) and produces the output net."

R28: "Upon receiving `FinalStateRequest`, each worker MUST respond with
`FinalStateResult` containing its current full partition (the same
`Partition` type used in v1's `PartitionResult`)."

This task ships the pure handler that reads `ctx.delta_state`, extracts the
stored `Partition`, and returns a `SendMessage(Message::FinalStateResult {
round, partition })` action. Transition: `DeltaActive` → `Returning` →
(implicit) `Done` once the coordinator closes the connection (or explicit
`Shutdown` message — v1 behavior).

## Key Types / Signatures

```rust
// relativist-core/src/worker.rs

/// SPEC-19 R21 phase 3, R28 (TASK-0383): handle the coordinator's
/// `Message::FinalStateRequest { round }` at convergence. Extracts the
/// stored partition from `ctx.delta_state` and returns a SendMessage
/// action carrying `Message::FinalStateResult { round, partition }`.
///
/// After this call, `ctx.delta_state == None` and `ctx.state ==
/// WorkerState::Returning`. The next coordinator message is typically
/// a connection close or `Message::Shutdown`, handled by the existing
/// v1 FSM via `transition(ctx, WorkerEvent::Shutdown)`.
pub fn handle_final_state_request(
    ctx: &mut WorkerContext,
    round: u32,
) -> Vec<WorkerAction> {
    let from = ctx.state.clone();
    let partition = ctx.delta_state
        .take()
        .expect("handle_final_state_request requires prior handle_initial_partition (R21.1)")
        .partition;
    ctx.round = round;
    ctx.state = WorkerState::Returning;
    vec![
        WorkerAction::LogTransition { from, to: ctx.state.clone() },
        WorkerAction::SendMessage(Box::new(Message::FinalStateResult {
            round,
            partition,
        })),
    ]
}
```

  (async read-loop, OUT of this bundle per DC-C2) destructures and passes

## Notes

- **Round-0-only convergence:** if the input net already had zero redexes,
  the coordinator's `run_grid_delta` (TASK-0386) will detect Global Normal
  Form after Round 0 alone (every worker reports `has_border_activity ==
  false` from the seeded state, and the coordinator's `BorderGraph` has
  zero redexes). The coordinator then sends `FinalStateRequest` before
  any `RoundStart` ever leaves — so this handler MUST accept `DeltaIdle`
  as well as `DeltaActive`. Acceptance criteria #2 covers this.
- **`.take()` semantics:** after final-state response, the worker has no
  reason to keep the partition (the coordinator owns the authoritative
  merge result). `.take()` immediately frees the memory; if the coordinator
  needs a retransmit due to packet loss, the worker sends `Message::Error`
  (v1 FSM) and the coordinator aborts the round (SPEC-06 R25 semantics).
- **Wire integration (DC-C2):** the async caller in `protocol/worker.rs`
  updates OUT of this bundle. This task ships the pure handler + tests.
