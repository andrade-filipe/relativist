## Context

This task implements the remaining three generators plus the three Church encoding generators from SPEC-14.

`erasure_propagation(n)` creates an ERA propagation chain of length N (SPEC-09, R17). An ERA agent connected to a chain of CON/DUP agents, where the erasure propagates through the chain during reduction. This is Profile C: nets with cascading boundary effects.

The Church encoding generators (`church_nat`, `church_add`, `church_mul`) encode natural numbers and arithmetic operations as IC nets, per SPEC-14 R4/R15/R16 respectively. These are Profile B benchmarks demonstrating that IC nets can encode computation.

> **Revision note (SPEC-12 v2, SC-016):** Church numeral cross-references were corrected from the non-existent "SPEC-14 R26" to the actual requirement numbers: ChurchNat -> SPEC-14 R4, ChurchAdd -> SPEC-14 R15, ChurchMul -> SPEC-14 R16. R42a was added with full pseudocode for `erasure_propagation`. R60 requires all generators to accept `size = 0`.

## Notes

- The Church encoding generators depend on SPEC-14 (Revised v2). The encoding module (`src/encoding/`) provides `encode_nat`, `build_add`, `build_mul` as the canonical implementations. These generators in `src/io/examples.rs` are thin wrappers that delegate to the encoding module functions, adapting the `u32` size parameter to the `u64` interface.
- `church_add` uses `n/2` and `n - n/2` to split the parameter into two addends. For odd `n`, one addend is `floor(n/2)` and the other is `ceil(n/2)`.
- `church_mul` uses `floor(sqrt(n))` for both factors. This means the actual product may not equal `n` exactly (e.g., `church_mul(10)` uses `3 * 3 = 9`). This is consistent with the spec.
- The erasure propagation chain structure: `ERA -> CON_1 -> CON_2 -> ... -> CON_N -> free`. The ERA connects to `CON_1.principal`. Each `CON_i.left` connects to `CON_{i+1}.principal`. Right auxiliary ports and the last CON's left port connect to free ports.
- `build_exp` is now MUST (R17 promoted from SHOULD in Revised v2). A `ChurchExp` variant MAY be added to `ExampleNet` in a follow-up task, but R26 only mandates ChurchNat, ChurchAdd, ChurchMul.
- The encoding module (TASK-0200 through TASK-0206) must be implemented before the Church generators in this task, since they delegate to `encode_nat`, `build_add`, `build_mul`.
