## Context

SPEC-09 Section 3.3 (R18) defines the `BenchmarkResult` structure containing all metrics for a single benchmark execution (one datapoint). This is the primary data type flowing through the benchmark pipeline: each execution produces a `BenchmarkResult`, which is then written to CSV, aggregated into statistics, and displayed in the terminal.

The struct contains 5 categories of fields: identification, correctness, timing, grid metrics, and derived metrics. Two supporting structs are also defined: `InteractionsByRule` (6-field breakdown) and `WorkerBenchStats` (per-worker per-round statistics).

## Key Types / Signatures

```rust
/// Complete result of a single benchmark execution (SPEC-09 R18).
/// One instance per datapoint: (benchmark, size, workers, mode, repetition).
#[derive(Debug, Clone, serde::Serialize)]
pub struct BenchmarkResult {
    // --- Identification ---
    pub benchmark: BenchmarkId,
    pub input_size: u32,
    pub mode: Mode,
    pub workers: u32,
    pub repetition: u32,

    // --- Correctness ---
    pub correct: bool,

    // --- Timing ---
    pub wall_clock_secs: f64,

    // --- Interactions ---
    pub total_interactions: u64,
    pub mips: f64,
    pub interactions_by_rule: InteractionsByRule,

    // --- Grid metrics ---
    pub rounds: u32,
    pub border_redexes_per_round: Vec<u32>,
    pub border_ratio_per_round: Vec<f64>,

    // --- Memory ---
    pub peak_memory_bytes: u64,
    pub agents_per_round: Vec<usize>,

    // --- Communication ---
    pub bytes_sent: u64,
    pub bytes_received: u64,
    pub bytes_sent_per_round: Vec<u64>,
    pub bytes_received_per_round: Vec<u64>,

    // --- Time breakdown per phase ---
    pub partition_time_per_round: Vec<f64>,
    pub compute_time_per_round: Vec<f64>,
    pub merge_time_per_round: Vec<f64>,
    pub network_time_per_round: Vec<f64>,

    // --- Per-worker metrics ---
    pub worker_stats: Vec<Vec<WorkerBenchStats>>,

    // --- Derived metrics ---
    pub speedup: f64,
    pub efficiency: f64,
    pub overhead_ratio: f64,
}

/// Interactions broken down by rule type (SPEC-09 R19).
#[derive(Debug, Clone, Default, serde::Serialize)]
pub struct InteractionsByRule {
    pub con_con: u64,
    pub dup_dup: u64,
    pub era_era: u64,
    pub con_dup: u64,
    pub con_era: u64,
    pub dup_era: u64,
}

/// Per-worker statistics for a single round (SPEC-09 R19).
#[derive(Debug, Clone, serde::Serialize)]
pub struct WorkerBenchStats {
    pub worker_id: u32,
    pub agents_before: usize,
    pub agents_after: usize,
    pub local_interactions: u64,
    pub compute_time_secs: f64,
    pub idle_time_secs: f64,
}
```

## Notes

- `BenchmarkResult` does NOT derive `Deserialize` intentionally -- results are write-only (serialized to CSV/JSON, never read back by the framework). This can be added later if needed.
- Fields for communication and network time are zeroed for `Sequential` and `Local` modes. The struct uses a unified schema (SPEC-09 R39a/R39b) rather than mode-specific variants. Per-round data is written to rounds.csv (R39b), scalar aggregates to detail.csv (R39a).
- The `worker_stats` field is `Vec<Vec<WorkerBenchStats>>`: outer Vec is per-round, inner Vec is per-worker within that round.
