## Notes

- **Why the loop sequence matters (register_minted_agents BEFORE apply_deltas):** the round N+2 finalizer materializes pending borders via `add_border_states`; the same round's `apply_deltas` on the received BorderDeltas may reference those newly-materialized borders. Reversing order would panic on missing border_id.
- **Why D-003 closes fully only after UT-0385-08's 12 cases pass:** the DEFERRED-WORK.md D-003 acceptance signal was "final merge() reconstructs the same output net that v1 produces on identical input" — canonical-equality across all 6 IC rules. This task's gate IS that acceptance signal.
- **Trace on the duplicate-request_id path:** if any asymmetric fixture triggers a duplicate request_id scenario (unlikely given the fixture design), UT-0398-06 pins the LENIENT behavior; UT-0385-08 tests should still pass because the coordinator uses the first-wins canonical mint, and both sides of the border see consistent AgentIds.
- **If SKIP_ASYMMETRIC=false reveals a latent bug in the resolver (2.26-B):** flag but do NOT fix in this task. That's a SEPARATE bundle (and likely opens D-005). The current hypothesis based on TASK-0395's analysis is that the resolver is correct and only the coordinator-side finalizer was missing.

## Design Choices Flagged for Spec-Critic (Stage 1.5)

- **DC-0399-A — register_minted_agents call site vs batched.** Options:
  - (a) Call per-result in the loop (current default plan): 1 call per worker per round.
  - (b) Collect all `minted_agents` across all workers first, then call once with the aggregated slice.
  - **Default:** (a). Rationale: per-worker error diagnostics (the WorkerId parameter of register_minted_agents carries the guilty worker in the R48 error message). Option (b) loses that attribution.
- **DC-0399-B — error propagation policy on ProtocolViolation.** Options:
  - (a) Propagate via `?` to the loop's existing Err branch → metrics.converged=false + degenerate net return (current default plan).
  - (b) Abort the entire BSP run with a panic (treats R48 as unrecoverable).
  - **Default:** (a). Rationale: R48 is a protocol violation at one worker; the grid can report divergence via metrics rather than crash. Matches existing error patterns.
- **DC-0399-C — delete or keep `package_resolutions` after migration.** Options:
  - (a) KEEP with docblock note (backward-compat, zero disruption).
  - (b) DELETE (clean break; callers must migrate).
  - (c) `#[deprecated]` annotation; flag for removal in next refactor.
  - **Default:** (a) — keep for now; only `LocalDeltaDispatch` consumes it today and this task migrates that caller anyway, but the function remains callable for future needs (e.g., tests that want dispatch-only without pending borders).
