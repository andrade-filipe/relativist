## Context

SPEC-08 v3 (SC-013) identified that the workload profile coverage matrix had no property-based tests targeting Profile B (CON-DUP expansion) or Profile C (sequential dependency). Two new property tests were added with dedicated generators:

- **PB15:** `prop_condup_fundamental_property` using `arb_condup_net(max_pairs)` -- generates nets with predominantly CON-DUP pairs (guaranteeing Profile B behavior). Verifies `reduce_all(net) ~ run_grid(net, n)` for random n.
- **PB16:** `prop_chain_fundamental_property` using `arb_chain_net(max_depth)` -- generates tree/chain topologies with level-dependent reduction (approximating Profile C behavior). Verifies `reduce_all(net) ~ run_grid(net, n)` for random n.

Both generators were added to the generator signatures in SPEC-08 Section 4.3. Both tests verify the Fundamental Property (G1) and are added to the G1 row in the invariant coverage matrix.

## Notes

- These generators are intentionally simpler than `arb_net` (the general random net generator). They target specific workload profiles rather than generating arbitrary topologies.
- `arb_condup_net` creates nets where every redex is a CON-DUP commutation, maximizing agent expansion. This is the hardest case for the merge protocol because it generates emergent border redexes.
- `arb_chain_net` creates nets with sequential dependency chains, where reduction at one level enables reduction at the next. This tests multi-round convergence.
- Both tests require Phase 4 (grid cycle) to be functional. If Phase 4 is not yet implemented, these tests should be created with `#[ignore]` annotations.
- Added to G1 row in SPEC-08 v3 invariant coverage matrix.
