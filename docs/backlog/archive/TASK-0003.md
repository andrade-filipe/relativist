## Context

`AgentId` and `PortId` are the fundamental identifier types used throughout the system. `AgentId` is a `u32` that uniquely identifies an agent in the net -- it is monotonically increasing and never reused (SPEC-01, I3). `PortId` is a `u8` with values in `{0, 1, 2}`, where 0 is the principal port and 1-2 are auxiliary ports.

These are type aliases (not newtypes) to keep the implementation simple and match the spec's design section exactly.

## Acceptance Criteria

- [ ] `AgentId` is defined as `pub type AgentId = u32;`
- [ ] `PortId` is defined as `pub type PortId = u8;`
- [ ] Doc comments explain: AgentId is monotonically increasing, never reused; PortId values are 0 (principal), 1-2 (auxiliary)
- [ ] Both types are publicly exported from the `net` module

## Files to Create/Modify

- `src/net/types.rs` -- **modify**: add `AgentId` and `PortId` type aliases

## Key Types / Signatures

```rust
/// Unique identifier for an agent. Monotonically increasing, never reused.
/// u32 allows ~4 billion agents over the lifetime of an execution.
pub type AgentId = u32;

/// Port index within an agent. 0 = principal, 1-2 = auxiliary.
pub type PortId = u8;
```

## Test Expectations

- Compile-time: `AgentId` is `u32` and `PortId` is `u8` (verified by usage in subsequent tasks)

## Dependencies Context

- None. These are leaf type aliases.

## Notes

- The spec explicitly uses type aliases, not newtypes. This is a deliberate simplicity choice for the academic prototype. Newtypes could be added later if type safety issues arise.
- `PortId` values beyond 2 are invalid for any agent in this system (CON/DUP have arity 2, ERA has arity 0). Validation is done at the call site, not in the type itself.
