## Context

The security module requires several new crate dependencies that are not yet in Cargo.toml: `rand` (for CSPRNG token generation), `base64` (for token encoding/display), `subtle` (for constant-time comparison), and `rustls-pemfile` (for PEM certificate parsing, feature-gated with `tls`). The `rustls` and `tokio-rustls` dependencies already exist in Cargo.toml as optional deps under the `tls` feature. This task adds the missing dependencies.

- [ ] `rand` is added to `[dependencies]` (for `OsRng` and `RngCore`, R9)
- [ ] `base64` version 0.22 is added to `[dependencies]` as always-on (extending SPEC-13 R11, per R10)
- [ ] `subtle` version 2.x is added to `[dependencies]` as always-on (extending SPEC-13 R11, per R15; constant-time via `ConstantTimeEq`)
- [ ] `rustls-pemfile` is added to `[dependencies]` as optional, gated under `tls` feature (for PEM parsing, R25/R26)
- [ ] The `tls` feature in `[features]` is updated to include `rustls-pemfile`

## Notes

- `rand` version 0.8 is the current stable release. The `OsRng` type is in `rand::rngs::OsRng` and `RngCore` trait provides `fill_bytes()`.
- `base64` version 0.22 uses the engine-based API: `base64::engine::general_purpose::STANDARD.encode(bytes)`. Older versions had a different API.
- `subtle` version 2 provides `ConstantTimeEq` which is stable and widely used.
- `rustls-pemfile` version 2 is compatible with `rustls` 0.23.
- All non-TLS dependencies (`rand`, `base64`, `subtle`) are always-on (not feature-gated) because token generation and validation are needed in Tier 2 even without TLS.
