# TASK-0149: Add FSM state transition logging

**Spec:** SPEC-11

**Status:** TODO

**Blocked by:** none
**Estimated complexity:** M (50-200 LoC)

## Context

SPEC-11 R7 requires that every coordinator and worker FSM state transition (defined in SPEC-13, R19-R25) is logged at INFO level with structured fields `from_state`, `to_state`, and `event`. This provides operators with a clear timeline of the grid cycle lifecycle without requiring verbose debug logging.

The coordinator FSM states are: `Init`, `WaitingForWorkers`, `Partitioning`, `Dispatching`, `WaitingForResults`, `Merging`, `CheckTermination`, `Done`, `Error`. The worker FSM states are: `Init`, `Idle`, `Reducing`, `Returning`, `Error`, `Done`. Coordinator events use `WorkerConnected` (not `WorkerRegistered`) and `PhaseTimeout` (not `HeartbeatTimeout`) per SPEC-13 Revised v2.

This task adds `tracing::info!` calls at every state transition point in the coordinator and worker state machines. It also adds the recommended contextual fields from R8 (e.g., `round` for grid round context).

## Acceptance Criteria

- [ ] Every coordinator FSM state transition emits an INFO log event
- [ ] Every worker FSM state transition emits an INFO log event
- [ ] Each event includes `from_state`, `to_state`, and `event` fields (R7)
- [ ] Coordinator events include `round` field where applicable (R8)
- [ ] Worker events include `worker_id` field where applicable (R8)
- [ ] Log format matches the SPEC-11 R7 example: `INFO relativist::coordinator: state transition from_state="..." to_state="..." event="..." round=N`
- [ ] No behavioral changes to the FSMs -- logging only

## Files to Create/Modify

- `src/protocol/*.rs` or `src/coordinator.rs` -- **modify**: add `tracing::info!` at each coordinator state transition
- `src/protocol/*.rs` or `src/worker.rs` -- **modify**: add `tracing::info!` at each worker state transition

## Key Types / Signatures

```rust
// Helper macro or function for consistent state transition logging.
// Can be a simple inline tracing::info! call at each transition point.

// Coordinator example (at each transition point in the FSM):
tracing::info!(
    from_state = "WaitingForResults",
    to_state = "Merging",
    event = "AllPartitionsReturned",
    round = round_number,
    "state transition"
);

// Worker example:
tracing::info!(
    from_state = "WaitingForPartition",
    to_state = "Reducing",
    event = "PartitionReceived",
    worker_id = %worker_id,
    "state transition"
);
```
