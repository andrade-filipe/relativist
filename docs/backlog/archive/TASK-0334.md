# TASK-0334: LambdaCodec decoder (port-directed readback)

**Spec:** SPEC-27 R14-R15
**Phase:** v2 — Layer 1 (Encoder/Decoder API), Phase 3 (LambdaCodec)
**Priority:** P0
**Complexity:** M (~150 LoC)
**Depends on:** TASK-0333 (encoder)

## Description

Implement the decoder side of `LambdaCodec` in the same module
`relativist-core/src/encoding/codec_lambda.rs`. Three sub-deliverables:

1. **Entry-point resolution** (~20 LoC)
   - Find the readback start port. The encoder wires the top-level result to
     `FreePort(0)` (TASK-0333). Locate the agent port whose target is
     `FreePort(0)` and use it as the readback entry. Fallback: a unique
     CON whose principal port is `DISCONNECTED` (mirrors `discover_root`
     for arithmetic).
   - If neither is found, return
     `DecodeError::UnrecognizedStructure("no readback entry port")`.

2. **Port-directed readback** (~100 LoC)
   - Maintain `var_map: HashMap<(AgentId, PortId), String>` of variable
     names introduced by Lambda nodes we have entered (binding port = p1).
   - `readback(port_we_arrived_at) -> Term`:
     - If `(id, p)` is in `var_map`, emit `Var(name)`.
     - On CON via p0 → Lambda. Generate fresh name `x_N`, insert
       `(id, 1) -> name` in `var_map`, recurse on
       `get_target(AgentPort(id, 2))` for the body, return `Lam(name, body)`.
     - On CON via p1 → Application result. Recurse on
       `get_target(AgentPort(id, 0))` for fn, `get_target(AgentPort(id, 2))`
       for arg, return `App(fn, arg)`.
     - On CON via p2 → entering body of an outer Lambda from inside.
       Recurse on `get_target(AgentPort(id, 0))` (the lambda's "up" port).
     - On ERA → return a synthetic `Var("_")` placeholder (erased). For
       cleaner output, the pretty-printer detects this name and prints `_`.
     - On DUP via p1/p2 → follow `get_target(AgentPort(id, 0))` (the source
       being shared). For our minimal lambda calculus, both DUP outputs
       readback to the same Term; the recursive walk is structurally safe
       because reduction has already eliminated the DUP-CON commutation
       redexes when the net is in normal form.
     - On DUP via p0 → return readback of `get_target(AgentPort(id, 1))`
       (one copy is enough; the other side has been consumed in normal form).

3. **Pretty-printer + JSON output** (~30 LoC)
   - `fn print_term(&Term) -> String` produces the canonical string form:
     `λx. body`, `(f x)` with parens only when needed (App always
     parenthesised when nested under another App-arg or under a Lam body
     that ends in App). Free variables print as their name.
   - Output schema (R15):
     ```json
     { "term": "<string>", "agents": <usize>, "interactions": null }
     ```
     `interactions` is filled in by the pipeline driver (Phase 5),
     not here — emit `serde_json::Value::Null` for now.

## Acceptance Criteria

1. Round-trip: for each T5-T9 input string, `encode → reduce → decode →
   pretty-print` yields the expected normal form (alpha-equivalence: rename
   bound vars deterministically before comparison).
2. `validate_encoded_net` post-encoding accepts T5 (identity) net even
   though it has no redexes — special handling: identity is already in
   normal form, so the decoder may be invoked directly without reduction.
   (Implication: validate_encoded_net's E2 check must be relaxed for
   decode-only nets — alternatively, the LambdaCodec encoder MAY skip
   validation when the term has no beta-redex. Decision: skip validation
   for terms with zero applications. Document inline.)
3. Decoder is robust: malformed nets return
   `DecodeError::UnrecognizedStructure(...)` rather than panicking.
4. `Box<dyn Codec>` works for `LambdaCodec` (object safety, R3).
