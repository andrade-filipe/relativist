## Context

SPEC-03 Section 4.6.3 defines `reduce_n`, which applies at most `budget` interactions and returns the number actually performed. If Normal Form is reached before the budget is exhausted, it stops immediately. This function serves two purposes:

1. **Granularity control in the grid:** Workers execute a budget of reductions and return a partial result, enabling the coordinator to re-partition and redistribute work.
2. **Safety valve for non-terminating nets:** Prevents infinite loops by bounding the number of steps (SPEC-01, I5).

Complexity: O(min(budget, S)) where S is the total steps to Normal Form.

## Key Types / Signatures

```rust
/// Reduces the net by at most `budget` interactions (SPEC-03 Section 4.6.3, R14).
///
/// Useful for:
/// - Granularity control in the grid (workers execute a budget and return
///   a partial result).
/// - Safeguard against non-terminating nets (SPEC-01, I5).
///
/// Returns statistics of the interactions performed (may be < budget
/// if Normal Form is reached before the budget is exhausted).
///
/// Complexity: O(min(budget, S)) where S = steps to Normal Form.
pub fn reduce_n(net: &mut Net, budget: usize) -> ReductionStats {
    let mut stats = ReductionStats::default();

    for _ in 0..budget {
        match reduce_step(net) {
            StepResult::NormalForm => return stats,
            StepResult::Reduced(rule) => stats.record(rule),
        }
    }

    stats
}
```

## Notes

- `reduce_n` is structurally identical to `reduce_all` except for the bounded loop. The `for _ in 0..budget` pattern naturally stops after `budget` iterations.
- This function is essential for the distributed grid protocol (SPEC-04, SPEC-05): workers receive a partition and a reduction budget, reduce locally, then return results. The coordinator merges and re-distributes.
- A budget of 0 is valid and means "do nothing" -- useful as a no-op baseline in benchmarks.
