# TASK-0115: Align Cargo.toml with SPEC-13 dependency map

**Spec:** SPEC-13
**Requirements:** R11, R12, R13, R14, R37, R38 (SPEC-13)
**Priority:** P1 (important but not blocking)
**Status:** TODO
**Depends on:** Phase 5
**Blocked by:** none
**Estimated complexity:** S (< 50 LoC)

## Context

SPEC-13 R11 specifies the exact always-on dependencies and their versions. SPEC-13 R12 specifies feature-gated dependencies with `dep:` syntax. SPEC-13 R37 specifies the exact feature flags. The current `Cargo.toml` is close but has minor discrepancies: bincode is v1 (spec says v2), feature flags don't use `dep:` syntax, `rand` is missing, `thiserror` version may differ, and dev-dependencies are incomplete.

This task audits and aligns `Cargo.toml` with the spec, ensuring the dependency map is correct for Phase 6 wiring.

## Acceptance Criteria

- [ ] Always-on dependencies match SPEC-13 R11: tokio 1.x, serde 1.x, bincode (version reconciled), clap 4.x, tracing 0.1, tracing-subscriber 0.3, thiserror 2.x, rand 0.8
- [ ] `rand` dependency is added (needed for token generation, test data)
- [ ] Feature-gated dependencies use `dep:` syntax (R12): `tls = ["dep:rustls", "dep:tokio-rustls"]`, etc.
- [ ] Features section matches SPEC-13 R37: `default = []`, `tls`, `metrics`, `otel`, `full = ["tls", "metrics", "otel"]`
- [ ] The `full` feature is added
- [ ] Dev-dependencies include `proptest`, `criterion`, `tokio-test` (R13)
- [ ] `serde_json` remains (needed for metrics JSON output)
- [ ] `rayon` remains (needed for local mode parallelism)
- [ ] `crc32fast` remains (needed for SPEC-06 framing)
- [ ] No unnecessary dependencies are present (R14)

## Files to Create/Modify

- `Cargo.toml` -- **modify**: update dependencies, features, dev-dependencies

## Dependencies Context

- This task modifies `Cargo.toml` only. No Rust code changes.
- The bincode version (1.x vs 2.x) is a deliberate decision: SPEC-13 says 2.x but the current code uses 1.x API. The implementer should evaluate whether to upgrade. If upgrading, `serialize`/`deserialize` calls in other tasks must adapt.

## Notes

- SPEC-13 R12 mentions `rustls-pemfile` for TLS but the current Cargo.toml does not include it. It should be added under the `tls` feature.
- The `async-trait` crate is not listed in SPEC-13 R11 but may be needed for the `Transport` trait (SPEC-13 R28). See SPEC-13 Open Question 1. If native async traits suffice, `async-trait` is not needed.
- `tokio-test` version should match the tokio version. Current tokio is 1.x, so `tokio-test = "0.4"` is appropriate.
