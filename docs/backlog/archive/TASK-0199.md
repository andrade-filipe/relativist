## Context

SPEC-09 Section 3.1 (R1) requires a benchmark binary (`cargo bench` or `cargo run --bin bench`). Section 3.1 (R6) defines CLI parameters via `clap`. Section 3.5 (R7, R35) requires Criterion integration for micro-benchmarks of the local reduction engine.

This task implements two components:

1. **CLI binary** (`src/bin/bench.rs`): parses command-line arguments into a `BenchmarkSuiteConfig` and calls `run_benchmark_suite`. Supports `--benchmark`, `--sizes`, `--workers`, `--mode`, `--warmup`, `--repetitions`, `--csv`, `--max-rounds`.

2. **Criterion micro-benchmarks** (`benches/benchmarks.rs`): replaces the existing placeholder with criterion benchmarks for the local reduction engine. These measure `reduce_all` performance for each benchmark at small sizes, providing micro-level performance data with automatic iteration control.

```rust
// src/bin/bench.rs
use clap::Parser;
use relativist::bench::*;

/// Relativist Benchmark Suite (SPEC-09).
/// Executes parametric benchmarks for distributed IC reduction.
#[derive(Parser, Debug)]
#[command(name = "relativist-bench", about = "Benchmark suite for Relativist")]
struct Cli {
    /// Which benchmark to execute (default: all).
    #[arg(long, default_value = "all")]
    benchmark: String,

    /// Problem sizes (comma-separated). If omitted, uses benchmark defaults.
    #[arg(long, value_delimiter = ',')]
    sizes: Option<Vec<u32>>,

    /// Number of workers (comma-separated, default: 1,2,4,8).
    #[arg(long, value_delimiter = ',', default_values_t = vec![1, 2, 4, 8])]
    workers: Vec<u32>,

    /// Execution mode.
    #[arg(long, default_value = "local")]
    mode: String,

    /// Warmup runs discarded before measurement.
    #[arg(long, default_value_t = 2)]
    warmup: u32,

    /// Timed repetitions per configuration.
    #[arg(long, default_value_t = 5)]
    repetitions: u32,

    /// Path for detail CSV output (one row per repetition).
    #[arg(long)]
    csv: Option<String>,

    /// Path for rounds CSV output (one row per round per execution, R39b).
    #[arg(long)]
    csv_rounds: Option<String>,

    /// Path for summary CSV output (one row per configuration, R40).
    #[arg(long)]
    csv_summary: Option<String>,

    /// Maximum grid loop rounds.
    #[arg(long)]
    max_rounds: Option<u32>,
}

fn main() {
    let cli = Cli::parse();

    let config = BenchmarkSuiteConfig {
        benchmarks: parse_benchmark_ids(&cli.benchmark),
        workers: cli.workers,
        modes: vec![parse_mode(&cli.mode)],
        warmup: cli.warmup,
        repetitions: cli.repetitions,
        csv_detail: cli.csv.map(|s| s.into()),
        csv_rounds: cli.csv_rounds.map(|s| s.into()),
        csv_summary: cli.csv_summary.map(|s| s.into()),
        max_rounds: cli.max_rounds,
    };

    let result = run_benchmark_suite(&config);

    if result.correctness_failures > 0 {
        eprintln!("SUITE FAILED: {} correctness failures", result.correctness_failures);
        std::process::exit(1);
    }

    println!("Suite completed in {:.2}s. {} datapoints, 0 failures.",
             result.total_time_secs, result.detail.len());
}
```

```rust
// benches/benchmarks.rs
use criterion::{criterion_group, criterion_main, Criterion, BenchmarkId};
use relativist::bench::benchmarks::*;
use relativist::bench::Benchmark;
use relativist::reduction::reduce_all;

fn bench_reduce_all_ep(c: &mut Criterion) {
    let bench = EPAnnihilation;
    let mut group = c.benchmark_group("reduce_all_ep_annihilation");
    for &size in &[100, 500, 1000, 5000] {
        group.bench_with_input(
            BenchmarkId::from_parameter(size),
            &size,
            |b, &size| {
                let net = bench.make_net(size);
                b.iter(|| reduce_all(net.clone()));
            },
        );
    }
    group.finish();
}

fn bench_reduce_all_condup(c: &mut Criterion) {
    let bench = ConDupExpansion;
    let mut group = c.benchmark_group("reduce_all_condup_expansion");
    for &size in &[10, 50, 100] {
        group.bench_with_input(
            BenchmarkId::from_parameter(size),
            &size,
            |b, &size| {
                let net = bench.make_net(size);
                b.iter(|| reduce_all(net.clone()));
            },
        );
    }
    group.finish();
}

fn bench_reduce_all_dualtree(c: &mut Criterion) {
    let bench = DualTree;
    let mut group = c.benchmark_group("reduce_all_dual_tree");
    for &depth in &[4, 6, 8, 10] {
        group.bench_with_input(
            BenchmarkId::from_parameter(depth),
            &depth,
            |b, &depth| {
                let net = bench.make_net(depth);
                b.iter(|| reduce_all(net.clone()));
            },
        );
    }
    group.finish();
}

criterion_group!(benches,
    bench_reduce_all_ep,
    bench_reduce_all_condup,
    bench_reduce_all_dualtree,
);
criterion_main!(benches);
```

## Notes

- The CLI binary is at `src/bin/bench.rs`, not `src/main.rs`. This allows the project to have both the main Relativist binary and the benchmark binary.
- Criterion benchmarks are in `benches/benchmarks.rs` (replacing the existing placeholder). They are separate from the CLI binary: `cargo bench` runs criterion, `cargo run --bin bench` runs the full suite.
- Criterion is appropriate for micro-benchmarks (< 1ms executions) where automatic iteration control is needed (R7, R35). For macro-benchmarks (distributed execution, multi-second), the suite runner uses `Instant` directly.
- The `--mode` flag currently accepts a single mode. For running multiple modes, the user runs the binary multiple times. This could be extended to accept comma-separated modes in a future improvement.
