# TASK-0140: Convert observability module to directory structure and add dependencies

**Spec:** SPEC-11
**Requirements:** R1, R2, R10, R11, R25
**Priority:** P0 (critical path)
**Status:** TODO
**Depends on:** none
**Blocked by:** none
**Estimated complexity:** S (< 50 LoC)

## Context

The `observability` module is currently a single file (`src/observability.rs`) with only a doc comment. To support the full observability subsystem (tracing init, metrics, HTTP endpoints, OTel), it needs to become a directory module (`src/observability/mod.rs`) with sub-modules. This task also ensures all required Cargo dependencies are present and correctly feature-gated.

The always-on dependencies (`tracing`, `tracing-subscriber` with `env-filter` and `json` features) are already partially in `Cargo.toml`. This task verifies completeness and adds any missing features (e.g., `json` feature for `tracing-subscriber`, `axum` for the `metrics` feature gate, and OTel export crates for the `otel` feature gate).

## Acceptance Criteria

- [ ] `src/observability.rs` is replaced by `src/observability/mod.rs`
- [ ] Sub-module declarations exist in `mod.rs`: `config`, `tracing_init`, `metrics` (cfg metrics), `http` (cfg metrics), `otel` (cfg otel)
- [ ] `tracing-subscriber` dependency has `json` feature enabled (needed for R3 JSON output)
- [ ] `axum` dependency is added, optional, gated under the `metrics` feature
- [ ] `opentelemetry-otlp` and `tracing-opentelemetry` are added as optional dependencies gated under `otel`
- [ ] Feature gates in `Cargo.toml` are updated: `metrics = ["prometheus-client", "dep:axum"]`, `otel = ["opentelemetry", "opentelemetry_sdk", "opentelemetry-otlp", "tracing-opentelemetry"]`
- [ ] `lib.rs` continues to compile with `pub mod observability;`
- [ ] `cargo check` passes with no features, with `--features metrics`, and with `--features otel`

## Files to Create/Modify

- `src/observability.rs` -- **delete** (replaced by directory)
- `src/observability/mod.rs` -- **create**: module declarations and re-exports
