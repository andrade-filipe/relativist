## Context

ET-11 verifies the Fundamental Property (G1 from SPEC-01) for arithmetic workloads: the result of distributed reduction must equal the result of local reduction. This is tested with `build_add(50, 50)` and k in {1, 2, 4} workers.

The test uses explicit multi-step Rust code as specified in the revised SPEC-14 (per SC-015):
1. Local reduction: `reduce_all(&mut local_net)` then `decode_nat(&local_net)`
2. Distributed reduction: `run_grid(build_add(50, 50), k)` then `decode_nat(&distributed_net)`
3. Assert equality

## Acceptance Criteria

  let mut local_net = build_add(50, 50);

use relativist::encoding::{build_add, decode_nat};
use relativist::reduction::reduce_all;
use relativist::merge::run_grid;

#[test]
fn et_11_distributed_correctness() {
    for k in [1, 2, 4] {
        let mut local_net = build_add(50, 50);
        reduce_all(&mut local_net);
        let local_result = decode_nat(&local_net);

        let distributed_net = run_grid(build_add(50, 50), k);
        let distributed_result = decode_nat(&distributed_net);

        assert_eq!(local_result, distributed_result,
            "Fundamental Property violated for k={k} workers");
    }
}
