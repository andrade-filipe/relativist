## Context

`build_exp(base: u64, exp: u64) -> Net` constructs an IC net that, when reduced to Normal Form via `reduce_all`, yields `church(base ^ exp)`. The function encodes `exp = lambda m. lambda n. n m`, which is the simplest arithmetic combinator -- just application.

R17 was promoted from SHOULD to MUST in Revised v2, making exponentiation mandatory. This is justified because the ARITH-EXP benchmarks are primary evaluation scenarios for the TCC, and exponentiation demonstrates Profile B/C behavior essential for the experimental analysis.

Per SPEC-14 Section 4.3.3, the construction uses 4 steps with one application CON agent.

## Notes

- R17 was promoted from SHOULD to MUST during the adversarial review (SC-017). `build_exp` is mandatory.
- SPEC-14 Section 4.3.3 notes a design choice: the function can produce `n m` (partially applied) or wrap in an additional lambda for the standard Church numeral interface. The implementer SHOULD use whichever approach correctly produces `church(base^exp)` after `reduce_all`.
- Exponentiation has deep sequential dependency chains, limiting parallelism (Profile B/C).
- WARNING: O(base^exp) interactions. Use small test values (e.g., 2^8 = 256 interactions).
