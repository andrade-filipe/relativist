# TASK-0305: Implement UnixTransport

**Spec:** SPEC-17, Section 3.3 (R10-R14, R44)
**Phase:** v2 — Milestone M1
**Priority:** P0
**Complexity:** S (~70 LoC)
**Depends on:** TASK-0303
**Blocks:** TASK-0307

## Description

Create `src/protocol/unix.rs` implementing `UnixTransport` (cfg(unix) only):

1. **Struct**: holds `socket_path: PathBuf`, `listener: Option<UnixListener>` (R10)
2. **`Transport` impl**:
   - `listen()`: delete stale socket file + log warn (R12, R44), bind UnixListener
   - `accept()`: accept connection, return boxed stream
   - `connect()`: connect to socket path, return boxed stream
3. **Conditional compilation** (R14): entire module gated with `#[cfg(unix)]`
4. **No TLS** (R13): even when `tls` feature is enabled
5. Default socket path: `/tmp/relativist.sock` (R11) — handled in factory, not here

## Platform Note

This task's code will NOT compile or be tested on Windows (user's primary platform). It will be validated via CI (Linux) or manual cross-compile. On Windows, `#[cfg(unix)]` ensures the module is not compiled.

## Acceptance Criteria

1. `UnixTransport::new(path)` constructs correctly
2. Stale socket file is removed before bind (R12)
3. Conditional compilation works: module absent on `cfg(not(unix))`
4. Tests (cfg(unix) only): TR2 (round-trip), TR3 (stale socket cleanup)
5. All 690 tests still pass on Windows
