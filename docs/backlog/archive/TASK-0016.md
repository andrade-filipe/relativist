## Context

The `BorderMap` is a `HashMap<u32, PortRef>` that maps `FreePort(bid)` identifiers to the `AgentPort` they are connected to. This is needed because `FreePort` references have no slot in the port array -- the port array only indexes by `(AgentId, PortId)`. The `BorderMap` provides the reverse lookup for boundary connections.

The `BorderMap` is NOT a field of `Net`. It is maintained externally by the partitioner (SPEC-04) and merger (SPEC-05).

**SPEC-02 v3 change:** The `BorderMap` type alias code block was removed from SPEC-02 Section 4.10. SPEC-02 now retains only the conceptual description, deferring the type definition to SPEC-04 for proximity to its usage. This task should define the type alias in the partition module (`src/partition/`) rather than `src/net/types.rs`. If downstream modules need it earlier, a re-export from `net::types` is acceptable.

R23 requires the representation to support the distinction between internal wires (both endpoints are `AgentPort`) and border wires (one endpoint is `AgentPort`, the other is `FreePort(bid)`). The `BorderMap` is the mechanism that makes this distinction operational.
