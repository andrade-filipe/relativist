## Context

Lands the two **coordinator → worker small-payload** delta-protocol
variants at discriminants 8 and 10:

- `RoundStart { round: u32, border_deltas: Vec<BorderDelta>,
  resolved_borders: Vec<u32>, new_borders: Vec<(u32, PortRef)>,
  local_reconnections: Vec<LocalReconnection>,
  pending_commutations: Vec<PendingCommutation> }` at discriminant **8**
  — per-round kickoff. Carries:
  - `border_deltas` — resolutions the coordinator computed for this
    worker's sides of border wires (from running `interact_*` on
    principal-pair borders, SPEC-19 §3.2 R13-R15).
  - `resolved_borders` — border IDs that were annihilated (CON-CON,
    DUP-DUP, ERA-ERA) and should be removed from the worker's index.
  - `new_borders` — fresh border IDs introduced by CON-DUP commutation
    (SPEC-19 R15 part 3), each paired with the worker's local endpoint
    (`PortRef::FreePort(border_id)` view).
  - `local_reconnections` (DC-B3, SPEC-19 R33) — internal port
    reconnections the worker must apply after a CON-DUP border-redex
    resolution produced new agents whose auxiliary ports are not
    themselves borders. Each `LocalReconnection` targets a specific
    (agent_id, port) on this worker's side.
  - `pending_commutations` (DC-B5, SPEC-19 R33 + R48) — AgentId
    allocation requests the worker must fulfil. Each carries a
    coordinator-allocated `request_id` (R48: monotonic, BSP-scoped) and
    the (symbol_type, arity) shape of the agent to mint. The worker
    allocates a fresh AgentId from its own reserved range and echoes it
    back via `RoundResult.minted_agents` next round (TASK-0369).
- `FinalStateRequest { round: u32 }` at discriminant **10** —
  broadcast at convergence (Global Normal Form reached, R27) to tell
  each worker to respond with `FinalStateResult` (disc 11).

Both variants carry small payloads (single-digit percent of partition
size per R36). The wire-level SHOULD-skip-compression verification
lands in TASK-0370; this task only ships the enum variants + in-memory
bincode round-trip tests.

## Notes

- **R36 (compression SHOULD skip below threshold):** the wire-level
  assertion lives in TASK-0370. This task's bincode round-trip tests
  exercise ONLY the serde layer, which is threshold-agnostic. A
  compression sanity check here would require a live `send_frame_v2`
  call, which is TASK-0370's territory.
- **Empty-Vec test rationale:** bincode v2 varint encodes a zero-length
  `Vec` as a single zero byte. If a future macro or attribute accidentally
  switches to `Option<Vec<...>>` or similar, the test catches it via a
  post-decode length assertion.
- **`new_borders: Vec<(u32, PortRef)>`:** tuple-typed fields serialise
  cleanly under bincode v2 without any adapter (plain product type).
  No custom serde impl needed. If a future refactor wants a named
  `NewBorder { border_id: u32, endpoint: PortRef }` struct, that's a
  separate task.
- **`PortRef` in production scope:** `relativist-core/src/protocol/types.rs`
  currently imports `PortRef` only inside `#[cfg(test)] mod tests`. This
  task promotes it to a production-level `use crate::net::PortRef;` at
  the top of the file. Grep precedent: `use crate::partition::{Partition,
  WorkerId};` already sits in production scope.
- **Order-of-landing coordination with TASK-0367/0369:** the simplest
  execution is for the developer to batch all three tasks' variant-
  addition diffs into a single Stage 3 DEV commit, preserving the enum
  source order 7 / 8 / 9 / 10 / 11 at every intermediate compile step.
  The LoC estimates for each task remain individually measurable (~150,
  ~130, ~130 for TASK-0367, TASK-0368, TASK-0369 respectively).
