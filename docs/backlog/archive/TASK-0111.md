## Context

SPEC-07 Section 4.5 defines the local mode lifecycle: parse config, load network, execute the grid loop (SPEC-05 `run_grid`), print summary, optionally write output and metrics. Local mode uses synchronous I/O and does NOT open TCP sockets (R18). It calls `run_grid` directly from Phase 4. The result must be identical to distributed mode for the same input (R19, Fundamental Property).

This is the simplest execution mode and the first one to be fully wirable, since it only depends on Core Layer modules (no protocol, no coordinator/worker).

- [ ] `run_local_command(args: LocalArgs) -> Result<(), RelativistError>` is implemented
- [ ] Parses `GridConfig` and strategy from args (via TASK-0102 functions)
- [ ] Loads network from `args.input` using `load_net_from_file` (TASK-0104)

- [ ] Function uses only synchronous I/O (no tokio)
- [ ] Function is wired into `main.rs` match arm for `Command::Local`

## Key Types / Signatures

```rust
use crate::config::{LocalArgs, build_grid_config_from_local, parse_strategy};
use crate::io::{load_net_from_file, save_net_to_file, write_metrics, print_summary};
use crate::merge::run_grid;
use crate::error::RelativistError;

/// Execute local mode: grid loop in-process, no TCP.
pub fn run_local_command(args: LocalArgs) -> Result<(), RelativistError> {
    // 1. Parse config
    let grid_config = build_grid_config_from_local(&args);
    let strategy = parse_strategy(&args.strategy)?;

    // 2. Load network
    let net = load_net_from_file(&args.input)?;

    // 3. Run grid loop (SPEC-05)
    let (reduced_net, metrics) = run_grid(net, &grid_config, &*strategy);

    // 4. Output
    print_summary(&reduced_net, &metrics);

    if let Some(ref output_path) = args.output {
        save_net_to_file(&reduced_net, output_path)?;
    }

    if let Some(ref metrics_path) = args.metrics {
        write_metrics(&metrics, metrics_path)?;
    }

    Ok(())
}
```

- SPEC-07 R20 (SHOULD use rayon for parallel partition reduction) is handled inside `run_grid` itself, not in this wiring function.
- This is the first fully functional end-to-end command. It validates the entire Core Layer pipeline.
