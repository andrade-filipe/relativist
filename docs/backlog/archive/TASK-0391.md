# TASK-0391: R42 regression — `delta_mode = false` preserves v1 baseline exactly

**Bundle:** SPEC-19 §3.6 — Config flag (item 2.26-D)
**Spec:** SPEC-19 R42 (default-disabled MUST preserve v1 behaviour exactly;
  zero behavioral regression)
**Priority:** P1 (required to close the sub-bundle — R42 is the load-bearing
  MUST of §3.6)
**Status:** TODO
**Depends on:** TASK-0389 (the field must exist so this regression can target
  it).
**Blocked by:** None (after TASK-0389). Independent of TASK-0390.
**Estimated complexity:** S (~40 LoC inc. tests)

## Context

SPEC-19 R42 states, verbatim: "`delta_mode` MUST default to `false` to preserve
backwards compatibility. No existing caller, test, or benchmark MUST change
behavior when `delta_mode` is not explicitly set."

This task operationalises R42 as a concrete regression test and a bundle-level
checklist item. It does NOT change any source other than adding tests. The test
is deliberately meta — it asserts invariants about the state of the tree after
TASK-0389 + TASK-0390 land.

The regression has three parts (all in one task — they are one logical
assertion):

1. **Default field value:** `GridConfig::default().delta_mode == false`.
   Already covered by TASK-0389's `grid_config_default_disables_delta_mode`,
   but reasserted here at a higher level (bundle-level invariant, not
   per-file).
2. **Default CLI parse:** every subcommand that accepts `--delta-mode`
   (coordinator, local) parses to `delta_mode = false` when the flag is
   omitted. Covered by TASK-0390 at the parse level; this task confirms it
   end-to-end through `build_grid_config(_from_local)`.
3. **No read-path change:** no production code outside tests reads
   `GridConfig.delta_mode` as of this sub-bundle. This is asserted
   structurally: a `grep -rn "\.delta_mode" relativist-core/src/` limited to
   non-test code MUST only return the struct declaration itself (R42
   regression-safe until 2.26-C lands the delta loop). This is NOT a
   #[test] — it is a documentation-and-grep check codified as a bundle
   acceptance criterion (see below).

The core regression test is a **characterisation test**:

```rust
/// TASK-0391 UT-01 (R42): a default GridConfig with delta_mode = false
/// produces bit-identical behaviour to a GridConfig constructed before
/// the delta_mode field existed. Concretely: run_grid(net, cfg) with
/// delta_mode = false MUST produce the same Net and the same
/// GridMetrics.total_interactions as the pre-bundle baseline for a
/// canonical smoke workload (church_add(2, 3) → 5, w=2).
#[test]
fn r42_default_delta_mode_preserves_v1_smoke_output() { ... }
```

## Notes

- **Why not use the full 968 lib tests as the regression?** Because that would
  be a tautology: `cargo test` already asserts that. R42 wants a *targeted*
  regression on the specific code path that could plausibly change behaviour —
  namely, a canonical `run_grid` smoke with the default `GridConfig`. This test
  closes the specific loophole "what if adding the field accidentally changed
  some struct-initialisation path".
- **Why a smoke, not a larger benchmark?** A smoke is fast (< 100 ms), runs in
  `cargo test`, and catches the failure mode R42 worries about (behavioural
  drift from the additive struct change). Larger validation is the job of the
  v1 frozen benchmark replay, not this unit test.
- **R42 interpretation:** "No existing caller, test, or benchmark MUST change
  behavior." The sub-bundle reads this as *behavioural* (not textual)
  invariance — see AMB-D-2 in the bundle index. If spec-critic rules for the
  strict textual reading, this task must additionally verify that
  `git diff --stat relativist-core/tests/` shows no modifications (only
  additions).
