## Context

SPEC-07 R35 requires Relativist to use the `tracing` crate with `tracing-subscriber` and `EnvFilter` for structured logging controlled by the `RUST_LOG` environment variable. This task creates the `init_tracing()` function in the `observability` module and calls it as the first action in `main()`. The function configures a `fmt` subscriber with env-filter and a default log level of `info` when `RUST_LOG` is not set.

## Acceptance Criteria

- [ ] `observability::init_tracing()` function is defined and public
- [ ] Subscriber uses `tracing_subscriber::fmt()` with `EnvFilter::from_default_env()`
- [ ] Default filter (when `RUST_LOG` is unset) is `info`
- [ ] `main()` calls `init_tracing()` before `Cli::parse()`
- [ ] Log output includes timestamps, level, target, and message
- [ ] Function does not panic if called when `RUST_LOG` has an invalid value (graceful fallback)

## Files to Create/Modify

- `src/observability.rs` -- **modify**: add `init_tracing()` function
- `src/main.rs` -- **modify**: call `observability::init_tracing()` at start of `main()`

## Key Types / Signatures

```rust
// In src/observability.rs

use tracing_subscriber::EnvFilter;

/// Initialize the tracing subscriber with env-filter.
///
/// Reads the `RUST_LOG` environment variable. If unset, defaults to `info`.
/// Must be called exactly once at program startup, before any tracing macros.
pub fn init_tracing() {
    let filter = EnvFilter::try_from_default_env()
        .unwrap_or_else(|_| EnvFilter::new("info"));

    tracing_subscriber::fmt()
        .with_env_filter(filter)
        .init();
}
```
