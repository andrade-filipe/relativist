**Bundle:** SPEC-19 §3.3 — Stateful Worker Lifecycle (item 2.26-C)
**Spec:** SPEC-19 R23 (`RoundStart` payload semantics), R24 (apply → reduce → rebuild
  → report pipeline), R26 (`RoundResult` payload shape)
**Priority:** P0 (the "round body" for every delta round)
**Status:** TODO
**Depends on:** TASK-0380 (`WorkerDeltaState` stored in `ctx.delta_state`,

**Blocked by:** TASK-0380, TASK-0382, 2.26-A wire variants
**Estimated complexity:** M (~180 LoC inc. tests)

## Context

R23 defines the `RoundStart` payload the coordinator sends at each delta round:
- `round: u32`,
- `border_deltas: Vec<BorderDelta>` — port reconnections to apply,
- `resolved_borders: Vec<u32>` — border IDs to remove from `free_port_index`,
- `new_borders: Vec<(u32, PortRef)>` — new border entries to add.

R24 defines the worker's per-round pipeline (5 steps):
1. Apply the received border deltas to the stored partition.
2. Run `reduce_all` on the stored partition.
3. Rebuild the `free_port_index` (SPEC-05 R20-R22).
4. Compute border deltas to emit (delegates to TASK-0382's helper).
5. Send a `RoundResult` with the computed deltas and stats.

R26 defines the `RoundResult` payload (already a wire variant from 2.26-A):
- `round: u32`,
- `border_deltas: Vec<BorderDelta>`,
- `stats: WorkerRoundStats`,
- `has_border_activity: bool`.

This task wires all five steps into a single pure handler
`handle_round_start(ctx, round, border_deltas, resolved_borders, new_borders)
-> Vec<WorkerAction>`. The handler reads `ctx.delta_state.as_mut().unwrap()`,
mutates the stored partition in place (apply deltas → `reduce_all` → rebuild
index), calls TASK-0382's `compute_outgoing_deltas` to diff against
`previous_border_state`, updates `previous_border_state` to the newly
reported snapshot, and returns a single `SendMessage(Message::RoundResult { ... })`
action plus a `LogTransition` action.

      with semantics (per R23):
      1. For each `BorderDelta { border_id, new_target }`: locate the port in
         `partition.subnet` currently connected to `FreePort(border_id)` and
         reconnect it to `new_target`. Update `partition.free_port_index`
         entry for `border_id` accordingly.
      2. For each `border_id` in `resolved_borders`: remove the `FreePort`
         sentinel from the subnet (the border wire no longer exists —
         annihilation / void resolution at the coordinator) and remove the
         `free_port_index` entry.
      3. For each `(border_id, port_ref)` in `new_borders`: insert a new
         `FreePort(border_id)` connection to `port_ref` (CON-DUP expansion
         at the coordinator created a new cross-partition wire) and update
         `free_port_index[border_id] = port_ref`.
- [ ] `handle_round_start` logic (sequenced per R24):
  1. Assert `ctx.state == WorkerState::DeltaIdle` (first round) OR
     `ctx.state == WorkerState::DeltaActive` (subsequent rounds). Emit
     `WorkerAction::LogTransition`.
  2. Extract `state = ctx.delta_state.as_mut().expect("delta_state set in
     TASK-0380")`.
  3. Apply deltas:
     `apply_border_deltas_to_partition(&mut state.partition, &border_deltas,
      &resolved_borders, &new_borders);`
  4. Run `reduce_all(&mut state.partition.subnet)` (from
     `crate::reduction`). Record stats.

     `state.partition.free_port_index = rebuild_free_port_index(&state.partition.subnet,

     `let outgoing = compute_outgoing_deltas(&state.previous_border_state,
      &state.partition.free_port_index);`
  7. Update `state.previous_border_state = state.partition.free_port_index.clone();`

  8. Compute `has_border_activity` via existing

  9. Build `stats: WorkerRoundStats` exactly as v1 does (worker_id, agents

     has_border_activity).
  10. `ctx.round = round;`  `ctx.state = WorkerState::DeltaActive;`
  11. Return:

      ```
          WorkerAction::SendMessage(Box::new(Message::RoundResult {
              round,
              border_deltas: outgoing,
      ```

```rust
// relativist-core/src/worker.rs

pub fn handle_round_start(
    ctx: &mut WorkerContext,
    round: u32,
    border_deltas: Vec<BorderDelta>,
    resolved_borders: Vec<u32>,
    new_borders: Vec<(u32, PortRef)>,
) -> Vec<WorkerAction> {
    use crate::merge::helpers::{
        apply_border_deltas_to_partition, compute_border_activity,
        rebuild_free_port_index,
    };
    use crate::reduction::{reduce_all, ReductionStats};
    use std::time::Instant;

    let from = ctx.state.clone();
    let state = ctx.delta_state.as_mut()
        .expect("handle_round_start requires prior handle_initial_partition (R21.1)");

    // R24.1 — apply coordinator-provided deltas.
    apply_border_deltas_to_partition(
        &mut state.partition,
        &border_deltas,
        &resolved_borders,
        &new_borders,
    );
    let agents_before = state.partition.subnet.count_live_agents();

    // R24.2 — local reduction.
    let t_reduce = Instant::now();
    let reduction_stats: ReductionStats = reduce_all(&mut state.partition.subnet);
    let reduce_duration = t_reduce.elapsed();

    // R24.3 — rebuild free_port_index against the post-reduction subnet.
    state.partition.free_port_index = rebuild_free_port_index(
        &state.partition.subnet,
        state.partition.border_id_start,
        state.partition.border_id_end,
    );

    // R24.4 — diff previous vs current to emit only changed entries.
    let outgoing = crate::merge::helpers::compute_outgoing_deltas(
        &state.previous_border_state,
        &state.partition.free_port_index,
    );
    state.previous_border_state = state.partition.free_port_index.clone();

    let has_border_activity = compute_border_activity(&state.partition);
    let agents_after = state.partition.subnet.count_live_agents();

    // R26 — build the RoundResult payload.
    let stats = WorkerRoundStats {
        worker_id: state.partition.worker_id,
        agents_before,
        agents_after,
        local_redexes: reduction_stats.total_interactions as usize,
        reduce_duration_secs: reduce_duration.as_secs_f64(),
        interactions_by_rule: reduction_stats.interactions_by_rule,
        has_border_activity,
    };

    ctx.round = round;
    ctx.state = WorkerState::DeltaActive;

    vec![
        WorkerAction::LogTransition { from, to: ctx.state.clone() },
        WorkerAction::SendMessage(Box::new(Message::RoundResult {
            round,
            border_deltas: outgoing,
            stats,
            has_border_activity,
        })),
    ]
}
```

```rust
// relativist-core/src/merge/helpers.rs — new pub(crate) helper

/// SPEC-19 R23 (TASK-0381): apply a `RoundStart`'s three-part payload to
/// a stored worker partition. All three sub-operations are idempotent
/// against a well-formed partition; the function performs no reduction
/// (that is the caller's step R24.2).
pub(crate) fn apply_border_deltas_to_partition(
    partition: &mut Partition,
    border_deltas: &[BorderDelta],
    resolved_borders: &[u32],
    new_borders: &[(u32, PortRef)],
) {
    // [1] Re-point existing borders to their new local-side targets.
    for delta in border_deltas {
        // Locate the port currently connected to FreePort(border_id) and
        // rewire it to delta.new_target. Update free_port_index entry.
        // Implementation detail: scan `partition.subnet.wires` for
        // any `Wire { from: FreePort(border_id), to } | { from: to,
        // to: FreePort(border_id) }` and rewrite `to` to new_target.
        // Exact helper signature may delegate to `partition.subnet.connect`.
        // ... (see TEST-SPEC-0381 for the expected wire-traversal)
        partition.free_port_index.insert(delta.border_id, delta.new_target);
    }

    // [2] Remove resolved borders.
    for bid in resolved_borders {
        partition.free_port_index.remove(bid);
        // Remove any wire of the form FreePort(bid) <-> x from the subnet.
        // (Per R23 and R16, the consumed agents already account for this
        // on the local side; this cleans up the sentinel.)
    }

    // [3] Insert new borders (CON-DUP expansion).
    for (bid, target) in new_borders {
        partition.free_port_index.insert(*bid, *target);
        // Create a wire FreePort(bid) <-> *target in the subnet.
    }
}
```

## Notes

- **Pure-core discipline:** `apply_border_deltas_to_partition` lives in
  `merge/helpers.rs` (pure-core, no tokio). `handle_round_start` lives in
  `worker.rs` (non-pure-core but in this task does not introduce tokio either —
  it is a synchronous handler returning `Vec<WorkerAction>`).
- **R24 ordering:** fixed — apply → reduce → rebuild → diff. Reordering
  breaks R25 (diff would be against pre-reduction state).
- **R26 `has_border_activity`:** uses the same helper as the v1 path
  (`compute_border_activity` from merge/helpers.rs, shipped in TASK-0348
  of the §3.1 bundle). This guarantees the flag has identical semantics
  across v1 and delta paths — important for R7 interop.
- **Wire-level integration (DC-C2):** the async caller that pattern-matches
  `Message::RoundStart` and invokes `handle_round_start` lives in
  `protocol/worker.rs` / `commands.rs`. Updating it is OUT of scope for
  this bundle (2.26-C-wire or 2.26-D).
- **DC-C4 consistency:** this task assumes `state.previous_border_state`
  was seeded from `free_port_index` at Round 0 (TASK-0379 option (b)). If
  spec-critic rules option (a) (empty seed), TASK-0379 amends and this
  task picks up the change automatically via the `compute_outgoing_deltas`
  helper (TASK-0382).
- **No `reduce_border_once` call:** per DC-C3 option (a), `run_grid_delta`
  requires `strict_bsp = true`. In delta mode the worker still calls
  `reduce_all` on its local partition; border redexes are resolved at the
  coordinator via 2.26-B's `BorderResolver`. Lenient delta mode is
  out-of-scope for this bundle.
