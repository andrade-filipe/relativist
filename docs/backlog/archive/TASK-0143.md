## Context

SPEC-11 R5 mandates specific default log levels per module. When the `RUST_LOG` environment variable is not set, Relativist must apply these defaults instead of the `tracing-subscriber` built-in default (which would show everything at `WARN` or higher). The default filter string encodes per-target levels to keep hot paths quiet (reduction and protocol at WARN) while keeping lifecycle events visible (coordinator and worker at INFO).

This task defines the default filter string as a constant and provides a helper function that returns an `EnvFilter`, preferring `RUST_LOG` when set and falling back to the default otherwise.

## Key Types / Signatures

```rust
use tracing_subscriber::EnvFilter;

/// Default per-target log levels (SPEC-11 R5).
///
/// Hot paths (reduction, protocol, net) default to WARN.
/// Lifecycle paths (coordinator, worker, partition) default to INFO.
/// The trailing `warn` sets the global default for any unmatched target.
pub const DEFAULT_LOG_FILTER: &str = "\
    relativist::coordinator=info,\
    relativist::worker=info,\
    relativist::reduction=warn,\
    relativist::protocol=warn,\
    relativist::partition=info,\
    relativist::net=warn,\
    relativist::observability=info,\
    relativist::security=info,\
    warn";

/// Build an `EnvFilter` from `RUST_LOG` if set, or fall back to
/// `DEFAULT_LOG_FILTER` (R4, R5).
///
/// If `RUST_LOG` contains an invalid filter directive, the default
/// filter is used instead and a warning is printed to stderr.
pub fn build_env_filter() -> EnvFilter {
    match EnvFilter::try_from_default_env() {
        Ok(filter) => filter,
        Err(_) => EnvFilter::new(DEFAULT_LOG_FILTER),
    }
}
```

## Notes

- `EnvFilter::try_from_default_env()` reads `RUST_LOG` automatically. If the variable is absent, it returns `Err`, which we catch to apply our default.
- The filter string uses commas as separators. The trailing `warn` is the global base level for any crate/module not explicitly listed.
- This function is deliberately simple and does not support runtime log level changes (R37 explicitly forbids this in v1).
