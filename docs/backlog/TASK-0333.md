# TASK-0333: LambdaCodec encoder (AST + parser + REF-005 mapping)

**Spec:** SPEC-27 R10-R13
**Phase:** v2 — Layer 1 (Encoder/Decoder API), Phase 3 (LambdaCodec)
**Priority:** P0
**Complexity:** M (~180 LoC)
**Depends on:** TASK-0331 (encode-contract validator)

## Description

Implement the encoder side of `LambdaCodec` in
`relativist-core/src/encoding/codec_lambda.rs`. Splits into three sub-deliverables:

1. **AST + parser** (~80 LoC)
   - `enum Term { Var(String), Lam(String, Box<Term>), App(Box<Term>, Box<Term>) }`
   - Hand-written recursive-descent parser for the textual lambda syntax (R12).
     Accepts `λ`, `\` and `lambda` as the abstraction marker. Whitespace
     between tokens is ignored. Application is left-associative; parentheses
     override precedence. Variable names are sequences of ASCII letters,
     digits and `_` (must start with a letter or `_`).
   - Errors: unmatched paren, missing dot after binder, unexpected EOF — each
     mapped to `EncodeError::InvalidInput("parse: <reason>")`.

2. **JSON input dispatch** (~20 LoC)
   - Schema (R12): either `{ "term": "(λx. x)" }` (string form) or
     `{ "ast": { "Lam": ["x", { "Var": "x" }] } }` (serde-derived AST).
     Both MUST be accepted.

3. **REF-005 net builder** (~80 LoC)
   - Pre-pass: count uses of each binder in the body to decide ERA/DUP fan-out
     for the binding port.
   - For each `Lam(x, body)`: create CON `lam`. According to use-count of `x`:
     - 0 uses → connect `ERA.p0` to `lam.p1`.
     - 1 use → directly connect `lam.p1` to the unique use-site of `x`.
     - n>1 uses → build a DUP tree of `n-1` duplicators, root at `lam.p1`,
       leaves at the n use-sites.
   - For each `App(f, arg)`: create CON `app`; recurse to obtain ports for
     `f` and `arg`; connect `f_port ↔ app.p0` (function), `arg_port ↔ app.p2`
     (argument); return `app.p1` (result port).
   - For each `Var(x)`: register a use-site for `x` in the env, return the
     port that the parent will eventually wire (placeholder ID resolved by
     the binder when it encodes its binding chain).
   - Free variables: connect to `FreePort(next_free_id)` and record in a
     side table for later readback (decoder may surface them as `Var(name)`).
   - Top-level wiring: result port returned by recursive encode is wired to
     `FreePort(0)` so reduction does not lose track. `net.root = None`
     (LambdaCodec uses recover-via-FreePort instead of root).

## Acceptance Criteria

1. Parser round-trip: parse → pretty-print → parse yields structurally equal
   AST for at least the 5 edge-case strings in T5-T9.
2. `encode(b"{\"term\":\"(λx. x)\"}")` returns a `Net` with exactly 2 CON
   agents, 1 self-loop wire on the inner CON's auxiliaries, and 0 redexes.
3. `validate_encoded_net(&net)` accepts the encoded net for any beta-redex
   case (T6-T9) and rejects normal-form cases with `E2: no redexes` (this
   is fine — caller handles that case via decode-only).
4. All 716 existing tests still pass; net new tests for parser/encoder
   structure ≥ 8.
5. `Box<dyn Encoder>` works (object safety, R1).
