## Context

R21 phase 3: "When the coordinator determines Global Normal Form (R4), it
sends `FinalStateRequest` to each worker. Each worker responds with
`FinalStateResult` containing its full current partition. The coordinator
performs a final `merge()` (SPEC-05 R1-R11) and produces the output net."

R27: "At convergence (Global Normal Form, R4), the coordinator MUST send
`FinalStateRequest` to every worker."

R29: "The coordinator MUST collect all `FinalStateResult` messages, construct
a `PartitionPlan` from the collected partitions and the `BorderGraph`'s
remaining border map, and invoke `merge()` (SPEC-05 R1-R11) to produce the
final output net. If the `BorderGraph` has zero remaining borders (all
resolved), the merge is trivial (union of agents with no boundary
reconnection)."

This task ships:
1. The `run_grid_delta_final_collect` helper that dispatches
   `FinalStateRequest` to every worker, collects responses, builds a new
   `PartitionPlan`, and calls `merge()`.
2. A helper `reconstruct_partition_plan_from_collected` that composes the
   collected partitions + remaining `BorderGraph` into a `PartitionPlan`
   consumable by `merge()`.

## Key Types / Signatures

```rust
// merge/grid.rs

/// SPEC-19 R21.3, R27, R29 (TASK-0387): Final State Collection +
/// `merge()` at convergence (or at max_rounds cap per R30).
///
/// Dispatches `Message::FinalStateRequest { round }` to every worker
/// via the `WorkerDispatch` trait, collects the returned
/// `Vec<Partition>`, reconstructs a `PartitionPlan` from them + the
/// remaining `BorderGraph`, and invokes v1's `merge()`.
///
/// Under R29, `border_redex_count` SHOULD be 0 when called post-
/// convergence (every border redex was resolved during the loop).
/// Under R30 (max_rounds cap), it MAY be >0; we return the partial
/// merged net regardless — the caller records
/// `metrics.delta_max_rounds_hit = Some(true)`.
fn run_grid_delta_final_collect(
    dispatch: &mut dyn WorkerDispatch,
    coordinator_partition_cache: HashMap<WorkerId, Partition>,
    border_graph: BorderGraph,
    metrics: &mut GridMetrics,
) -> Result<Net, GridError> {
    let round = metrics.rounds as u32;

    // R27 — dispatch FinalStateRequest to every worker.
    let partitions = dispatch.dispatch_final_state_request(round)?;

    // Sanity: size matches cache.
    if partitions.len() != coordinator_partition_cache.len() {
        return Err(GridError::Protocol(format!(
            "FinalStateResult count mismatch: expected {}, got {}",
            coordinator_partition_cache.len(),
            partitions.len()
        )));
    }

    // R29 — reconstruct plan + invoke merge().
    let plan = reconstruct_partition_plan_from_collected(partitions, &border_graph);
    let t_merge = Instant::now();
    let (merged_net, _border_redex_count) = merge(plan);
    metrics.merge_time_per_round.push(t_merge.elapsed());

    Ok(merged_net)
}

fn reconstruct_partition_plan_from_collected(
    mut partitions: Vec<Partition>,
    border_graph: &BorderGraph,
) -> PartitionPlan {
    partitions.sort_by_key(|p| p.worker_id);
    let border_map = border_graph
        .borders
        .iter()
        .map(|(bid, state)| {
            (*bid, /* convert BorderState -> BorderEntry */ todo!("SPEC-05 shape"))
        })
        .collect();
    PartitionPlan {
        partitions,
        border_map,
        // ... other PartitionPlan fields per SPEC-05.
    }
}
```

## Notes

- **R29 triviality claim:** when `border_graph.borders.is_empty()`, merge
  is a pure union of partition agents. The v1 `merge()` handles this
  degenerate case natively (SPEC-05 R1-R11 allow empty border maps).
- **Max-rounds cap (R30) path:** when TASK-0388 sets
  `delta_max_rounds_hit = Some(true)`, this function is STILL called
  (R30: "MUST initiate Final State Collection"). The resulting net
  may still have redexes — workers returned their current partial
  state, coordinator merges them with whatever `BorderGraph` state
  remained. Result is "partial net"; caller checks
  `metrics.delta_max_rounds_hit` to know correctness status.
- **`BorderState` → `BorderEntry` conversion:** requires reading SPEC-05
  to confirm the exact field mapping. If a mismatch arises (e.g.,
  SPEC-05 `BorderEntry` demands data the `BorderGraph` does not carry),
  this is a cross-spec concern flagged to spec-critic. Current bundle
  assumes the mapping is straightforward: `BorderState { side_a,
  side_b, worker_a, worker_b, ..}` → `BorderEntry { worker_a, worker_b
  }` since `side_a`/`side_b` endpoints are embedded in the partition
  subnets.
- **Pure-core discipline preserved:** this function lives in
  `merge/grid.rs` (pure-core) and only calls other pure-core helpers.
  The `WorkerDispatch` trait object is passed by `&mut dyn` — the
  trait itself is pure-core per TASK-0384 design.
- **E2E fixture:** the 4-CON-CON test is the simplest non-trivial
  workload that exercises (i) local reduction in each worker, (ii)
  Round 0 dispatch, (iii) convergence in Round 1, (iv) final merge.
  G1 bit-identity to `run_grid` output is the headline invariant
  (R38 amendment of SPEC-01 G1 under delta mode).
