## Context

pseudocode into actual Rust source code, introduces the empty `BorderGraph`
struct shell with its three private fields, and defines the
`is_principal_pair(a: PortRef, b: PortRef) -> bool` helper used by
TASK-0361 and TASK-0362 to compute the `is_redex` derived bit (R9).

The file is registered in `merge/mod.rs` so subsequent tasks can land
methods. **No methods are implemented in this task** — `BorderGraph` is just
a struct with the three private fields per spec sketch §4.1.

Per R19, the file MUST be pure-core: no `tokio`, no `async`, no I/O imports.
Only `crate::net::{PortRef, DISCONNECTED}`, `crate::partition::WorkerId`, and
`std::collections::{HashMap, HashSet}`.

## Key Types / Signatures

```rust
//! Coordinator-side data structure tracking border-wire connectivity.
//!
//! SPEC-19 §3.2 (R8-R19, item 2.35) — BorderGraph replaces the full merged
//! net for delta-mode rounds. The coordinator updates this as workers report
//! border deltas; principal-vs-principal pairs become detectable border
//! redexes that the coordinator resolves locally (R13-R15) before
//! redistributing the resulting deltas.
//!
//! **Pure-core invariant (R19):** this module MUST NOT import `tokio`,
//! `async_trait`, or anything under `crate::protocol::*`. The grep guard
//! `grep -E '^use\s+(tokio|crate::protocol|async_trait)'` MUST return zero
//! matches against this file. Item 2.26 (delta BSP loop, coordinator
//! integration) is allowed to call into this module from the protocol layer
//! — but the calls go IN here, not OUT.

use crate::net::{PortRef, DISCONNECTED};
use crate::partition::types::WorkerId;
use std::collections::{HashMap, HashSet};

/// State of a single border wire across two worker partitions (SPEC-19 R9).
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct BorderState {
    /// Border identifier from the original `PartitionPlan.borders` map.
    pub border_id: u32,
    /// Current endpoint owned by `worker_a`.
    pub side_a: PortRef,
    /// Current endpoint owned by `worker_b`.
    pub side_b: PortRef,
    /// Worker that owns `side_a`.
    pub worker_a: WorkerId,
    /// Worker that owns `side_b`.
    pub worker_b: WorkerId,
    /// Derived bit: `true` iff both endpoints are principal ports
    /// (`PortRef::AgentPort(_, 0)`). Recomputed by `apply_deltas`.
    pub is_redex: bool,
}

/// Coordinator-side border-connectivity graph (SPEC-19 R8).
///
/// Initialised once from a `PartitionPlan` (TASK-0361, R10), updated on
/// every worker round via `apply_deltas` (TASK-0362, R11+R17), queried for
/// border redexes via `detect_border_redexes` (TASK-0363, R12).
///
/// The internal field set matches SPEC-19 §4.1 verbatim: a `borders` map
/// keyed by `border_id`, a per-worker reverse index `worker_borders` for
/// future coordinator dispatch (item 2.26), and an incremental
/// `active_redexes` set so redex detection is O(|redexes|) rather than
/// O(B) (R18 SHOULD).
#[derive(Debug, Clone)]
pub struct BorderGraph {
    pub(crate) borders: HashMap<u32, BorderState>,
    pub(crate) worker_borders: Vec<Vec<u32>>,
    pub(crate) active_redexes: HashSet<u32>,
}

/// Returns `true` iff both `a` and `b` are principal ports
/// (`PortRef::AgentPort(_, 0)`). The R9 `is_redex` derived bit.
pub(crate) fn is_principal_pair(a: PortRef, b: PortRef) -> bool {
    matches!(
        (a, b),
        (PortRef::AgentPort(_, 0), PortRef::AgentPort(_, 0))
    )
}
```

## Notes

- **Why `pub(crate)` on the `BorderGraph` fields:** keeps the public API
  surface exactly the four documented methods (`from_partition_plan`,
  `apply_deltas`, `detect_border_redexes`, plus `remove_border` and
  `add_border_states`). External callers MUST go through these methods so
  the `active_redexes` invariant cannot drift from `borders`.
- **Why no serde derives on `BorderState`:** the type lives entirely in the
  coordinator's process memory. Workers never see it — they emit
  `BorderDelta` (a separate type defined in SPEC-19 §3.3, shipped under item
  2.26). If a future debugging tool wants to dump a `BorderGraph` snapshot,
  that tool can add a debug-only `Serialize` derive at that point.
- **Why ship `worker_borders` now even though no method in this bundle reads
  it:** spec §4.1 includes it as a structural field, TASK-0361 must
  populate it (R10's "all `BorderState` entries" + the per-worker reverse
  index), and TASK-0364's `add_border_states` updates it for new borders.
  Item 2.26 (delta BSP loop) consumes it for "send `RoundStart` only to
  workers whose border state changed". See bundle DC-2.
- **Why `is_principal_pair` is `pub(crate)` not `pub`:** internal helper;
  external callers should derive the bit from `BorderState.is_redex` field.
