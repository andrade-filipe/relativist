## Context

SPEC-07 Section 4.8 defines the metrics output pipeline. When `--metrics <PATH>` is specified, execution metrics (`GridMetrics`) are written to a file. The format is determined by the file extension: `.json` for JSON (via `serde_json`), `.csv` for CSV (manually formatted), defaulting to JSON for unrecognized extensions. The CSV format has one header line plus one line per round, with columns matching SPEC-07 R29.

## Acceptance Criteria

- [ ] `write_metrics(metrics: &GridMetrics, path: &Path) -> Result<(), RelativistError>` dispatches by extension
- [ ] `.json` extension produces pretty-printed JSON via `serde_json::to_string_pretty`
- [ ] `.csv` extension produces a header line + one data line per round
- [ ] CSV columns: `round,agents,local_interactions,border_interactions,border_redexes,partition_time_ms,compute_time_ms,merge_time_ms,bytes_sent,bytes_received,network_send_time_ms,network_recv_time_ms`
- [ ] Unrecognized or missing extension defaults to JSON
- [ ] Network-related CSV columns are `0` when the vectors are empty (local mode)
- [ ] `GridMetrics` derives `serde::Serialize` (may already be done in TASK-0060)
- [ ] Duration fields in JSON are serialized as floating-point seconds

/// Write metrics in JSON or CSV format, determined by file extension.
pub fn write_metrics(metrics: &GridMetrics, path: &Path) -> Result<(), RelativistError> {
    match path.extension().and_then(|e| e.to_str()) {
        Some("csv") => write_metrics_csv(metrics, path),
        _ => write_metrics_json(metrics, path), // default: JSON
    }
}

/// Write metrics as pretty-printed JSON.
fn write_metrics_json(metrics: &GridMetrics, path: &Path) -> Result<(), RelativistError> {
    let json = serde_json::to_string_pretty(metrics)
        .map_err(|e| RelativistError::Config(format!("JSON serialization failed: {}", e)))?;
    std::fs::write(path, json)?;

}

## Notes

- SPEC-07 R31 requires metrics to be sufficient for reproducing all TCC tables/charts. The CSV format maps directly to what a spreadsheet or Python script would consume.
- The CSV writer is hand-rolled (no csv crate dependency) to avoid adding a dependency for a simple tabular format.
- Durations in CSV are expressed as milliseconds (matching column names `*_ms`). In JSON, durations are serialized as seconds (f64) for consistency with serde defaults.
