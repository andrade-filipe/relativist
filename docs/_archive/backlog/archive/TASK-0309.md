# TASK-0309: Refactor worker.rs to use Transport trait

**Spec:** SPEC-17, Section 3.10 (R36, R37, R38)
**Phase:** v2 — Milestone M1
**Priority:** P0
**Complexity:** S (~50 LoC changes)
**Depends on:** TASK-0302, TASK-0303, TASK-0307
**Blocks:** TASK-0311

## Description

Refactor worker functions to use `Transport` trait instead of hardcoded TCP:

### connect_with_retry (R36)
- Change signature: accept `transport: &mut dyn Transport` instead of `addr: SocketAddr`
- Call `transport.connect()` instead of `TcpStream::connect(addr)`
- Preserve exponential backoff (SPEC-06 R23) and max_attempts logic

### run_worker / run_worker_daemon / run_worker_inner
- Accept `transport: &mut dyn Transport` parameter
- Use `TransportStream` instead of `TcpStream` for the connection
- Pass transport to `connect_with_retry`

### R37: No direct TcpStream imports
- Remove `use tokio::net::TcpStream;` from worker.rs

### R38: Identical observable behavior
- Same retry logic, same message sequences, same timeouts

## Test Migration

Worker tests that create `TcpListener::bind()` for simulating a coordinator need to be updated. Strategy:
- Use `ChannelTransport` for unit tests (no port allocation)
- Keep one TCP integration test via `TcpTransport`

1. `connect_with_retry` accepts `transport: &mut dyn Transport`

3. Retry logic preserved (same backoff sequence)

5. All 690 tests still pass
