**Spec:** SPEC-18 R28, R29, R30, R31, R32 (round-trip identity check),

**Scope:** Atomic commit of the wire compatibility break. Bump
`PROTOCOL_VERSION` from 1 to 2, update the Register handshake to reject
v1 peers with a clear diagnostic, add a `VersionMismatch` variant to
`ProtocolError`, and verify the full v2 round-trip identity holds.

---

## Deliverable

1. **`src/protocol/coordinator.rs`** (or wherever the constant lives —
   likely `src/protocol/types.rs` or `src/protocol/mod.rs`):
   ```rust
   pub const PROTOCOL_VERSION: u8 = 2;  // was 1
   ```

2. **Coordinator Register handshake**: when a worker's `RegisterPayload`
   arrives with `protocol_version != PROTOCOL_VERSION`, respond with:
   ```rust
   Message::RegisterNack(RegisterNackPayload {
       reason: format!(
           "protocol version mismatch: expected {}, got {}",
           PROTOCOL_VERSION, payload.protocol_version
       ),
   })
   ```
   then close the connection (R29).

3. **Worker Register handshake**: when receiving a `RegisterNack` whose
   reason starts with "protocol version mismatch", log an error including
   both expected and received versions (R30 SHOULD), then terminate
   (R30 MUST).

4. **`src/protocol/error.rs`**:
   - Add `ProtocolError::VersionMismatch { expected: u8, received: u8 }`
     (`#[error("protocol version mismatch: expected {expected}, received {received}")]`).
   - Use this internally where the coordinator detects the mismatch
     (separate from the on-the-wire `RegisterNack`).

## Tests

- **T10 protocol version rejection (R29)**: simulate a v1 worker by
  manually constructing `Message::Register(RegisterPayload { protocol_version: 1, .. })`,
  feed it to the coordinator's register handler via an in-memory transport,
  assert the response is `RegisterNack` with reason containing
  "protocol version mismatch", "expected 2", "got 1".
- **T-version-constant**: `assert_eq!(PROTOCOL_VERSION, 2);` — guards
  against accidental rollback.
- **T-version-error-renders**: `ProtocolError::VersionMismatch { expected: 2, received: 1 }
  .to_string()` contains "expected 2" and "received 1".
- **T32 round-trip identity (final v2 contract)**: for each `Message`
  variant, `decode(encode(msg)) == msg` through the **complete** v2 send/recv
  pipeline (bincode v2 + compact PortRef + 9-byte header + LZ4 + frame).
  Smoke covers what TASK-0346's T7/T8 already covered, but this test
  exercises every variant once.
- **T16 v1/v2 mismatch integration**: full coordinator + simulated v1
  worker (TCP loopback or `ChannelTransport`), assert the coordinator
  rejects via RegisterNack and the worker terminates with a non-zero
  exit code.
