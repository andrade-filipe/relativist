## Context

The `PartitionStrategy` trait abstracts the allocation function sigma that assigns each agent to a worker. By defining this as a trait, Relativist decouples the partitioning algorithm from the specific assignment strategy. Any implementation that satisfies C1 (complete coverage) is valid -- the choice affects performance but never correctness (DISC-004 v2, Section 1.6). This trait enables future alternative strategies (topology-aware, redex-aware) without modifying the partitioning pipeline.

## Key Types / Signatures

```rust
use std::collections::HashMap;

/// Trait that abstracts the allocation function sigma: AgentId -> WorkerId.
///
/// Implementations of this trait determine how agents are distributed
/// among workers. The correctness of the system does NOT depend on the
/// strategy chosen (DISC-004 v2, Section 1.6), but performance does.
///
/// Post-conditions:
/// - Every live agent in the net has an entry in the returned map (C1).
/// - Every WorkerId in the map is in range [0, num_workers).
pub trait PartitionStrategy {
    fn allocate(&self, net: &Net, num_workers: u32) -> HashMap<AgentId, WorkerId>;
}
```
