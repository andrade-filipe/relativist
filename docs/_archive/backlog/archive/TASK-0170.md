## Context

After any reduction (local via `reduce` or distributed via `coordinator`), Relativist prints a human-readable Reduction Summary to stdout (R20, R44). This task implements the formatting functions that convert a `ReductionSummary` into the human-readable text output defined in R45-R47.

The summary includes input/output statistics, timing, interaction count, and MIPS. For distributed execution, additional grid metrics are appended (R46). If the net did not reach Normal Form, the reason is indicated (R47).

This task also implements the `--metrics` JSON output (R21): a JSON object with before/after statistics and timing.

## Acceptance Criteria

- [ ] `format_reduction_summary(summary: &ReductionSummary) -> String` produces the text format from R45
- [ ] Output includes `Input:` line with agents, wires, redexes counts
- [ ] Output includes `Output:` line with agents, wires, redexes counts and "(normal form)" or "(NOT normal form: {termination_reason})"
- [ ] Output includes `Interactions:`, `Duration:`, `MIPS:` lines
- [ ] When `summary.rounds` is `Some(...)`, `Rounds:` and `Workers:` lines are appended (R46)
- [ ] When `summary.speedup` is `Some(...)`, `Speedup:`, `Efficiency:`, `Overhead:` lines are appended (R44b: only when --baseline-secs provided)
- [ ] When `summary.normal_form` is `false` and `termination_reason` is `Some(...)`, the Output line includes "(NOT normal form: {reason})" (R47)
- [ ] `format_metrics_json(summary: &ReductionSummary) -> String` produces the JSON format from R21
- [ ] Metrics JSON contains at minimum: `agents_before`, `agents_after`, `wires_before`, `wires_after`, `redexes_before`, `redexes_after`, `normal_form`, `total_interactions`, `duration_secs`, `mips`
- [ ] Duration is formatted as seconds with 3 decimal places (e.g., `1.234s`)
- [ ] MIPS is formatted with 3 decimal places
- [ ] R44a: `duration_secs` for `reduce` subcommand measures wall-clock time of `reduce_all` only (excluding file I/O)

## Notes

- The `format_reduction_summary` function is called by both the `reduce` CLI handler and the `coordinator` handler. It must work for both local and distributed reductions.
- The `===` header and field alignment in R45 should be matched closely but exact column alignment is not critical -- the spec provides the template, not a pixel-perfect requirement.
- MIPS = Millions of Interactions Per Second = `total_interactions as f64 / duration_secs / 1_000_000.0`. If duration is zero (instant reduction), MIPS should display `inf` or a fallback value.
- The metrics JSON (R21) is a separate output from the text summary. It is written to a file specified by `--metrics`, not to stdout. The formatting function produces the string; the CLI handler writes it to the file.
