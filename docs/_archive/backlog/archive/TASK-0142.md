## Context

`ObservabilityConfig` is the single configuration struct passed to `init_tracing()` at startup. It bundles the log format selection, process role, and optional feature-gated fields (metrics registry reference). This struct decouples the CLI/config layer from the observability internals: the CLI parses arguments and environment variables into an `ObservabilityConfig`, which the observability module consumes without knowing about `clap` or argument parsing.

## Key Types / Signatures

```rust
/// Configuration for the observability subsystem (R31).
///
/// Built by the CLI/config layer (SPEC-07) and passed to `init_tracing()`
/// at startup. Feature-gated fields are only present when the corresponding
/// Cargo feature is enabled.
#[derive(Debug, Clone)]
pub struct ObservabilityConfig {
    /// Log output format: Text (human) or Json (machine).
    pub log_format: LogFormat,

    /// Whether this process is a coordinator or worker.
    pub role: ProcessRole,

    /// Port for the metrics/health HTTP server.
    #[cfg(feature = "metrics")]
    pub metrics_port: u16,

    /// Bind address for the metrics/health HTTP server.
    #[cfg(feature = "metrics")]
    pub metrics_bind: String,
}

impl ObservabilityConfig {
    /// Default configuration for a coordinator process.
    pub fn default_coordinator() -> Self {
        Self {
            log_format: LogFormat::default(),
            role: ProcessRole::Coordinator,
            #[cfg(feature = "metrics")]
            metrics_port: 9090,
            #[cfg(feature = "metrics")]
            metrics_bind: "0.0.0.0".to_string(),
        }
    }

    /// Default configuration for a worker process.
    pub fn default_worker() -> Self {
        Self {
            log_format: LogFormat::default(),
            role: ProcessRole::Worker,
            #[cfg(feature = "metrics")]
            metrics_port: 9090,
            #[cfg(feature = "metrics")]
            metrics_bind: "0.0.0.0".to_string(),
        }
    }

    /// Default configuration for local reduction mode (R33a).
    ///
    /// Structured logging applies; HTTP endpoints are NOT started.
    pub fn default_local() -> Self {
        Self {
            log_format: LogFormat::default(),
            role: ProcessRole::Local,
            #[cfg(feature = "metrics")]
            metrics_port: 9090,
            #[cfg(feature = "metrics")]
            metrics_bind: "0.0.0.0".to_string(),
        }
    }
}
```

## Notes

- The `metrics_port` and `metrics_bind` fields are feature-gated to avoid dead code when metrics is disabled. The `#[cfg]` attributes on struct fields are standard Rust practice.
- Workers do not run HTTP servers in v1 (R13), but the `metrics_port` field is still present for forward compatibility. It is simply unused when `role == Worker`.
- The OTel endpoint is configured via the `OTEL_EXPORTER_OTLP_ENDPOINT` environment variable (R27), not via this struct, following OTel conventions.
