## Context

SPEC-07 Section 4.2 (Revised v3) defines the mapping from CLI arguments to internal configuration structures: `GridConfig` (SPEC-05) and `NodeConfig` (SPEC-06). This task implements `build_grid_config`, `build_node_config_coordinator`, `build_node_config_worker`, and `parse_strategy`. These functions translate the raw CLI Args structs into the typed config structs used by the rest of the system. The worker's `--coordinator HOST:PORT` argument is parsed into a `SocketAddr` directly (no separate `parse_host_port` helper needed since `NodeConfig` now uses `bind: SocketAddr`).

## Acceptance Criteria

- [ ] `build_grid_config(args: &CoordinatorArgs) -> GridConfig` correctly maps `workers` and `max_rounds`
- [ ] `build_grid_config_from_local(args: &LocalArgs) -> GridConfig` does the same for local mode
- [ ] `build_node_config_coordinator(args: &CoordinatorArgs) -> NodeConfig` uses `args.bind` directly (SocketAddr), with all defaults from SPEC-06 v3 R36 (no `role` field -- NodeRole removed)
- [ ] `build_node_config_worker(args: &WorkerArgs) -> Result<NodeConfig, RelError>` parses coordinator address via `args.coordinator.parse::<SocketAddr>()` (no `role` field)
- [ ] `parse_strategy(name: &str) -> Result<Box<dyn PartitionStrategy>, RelError>` maps "round-robin" to `ContiguousIdStrategy` (only strategy in v1)
- [ ] Sensible defaults are applied: bind=127.0.0.1:9000 (SPEC-10 R5), max_payload_size=256MiB, worker_connect_timeout=120s, distribute_timeout=60s, collect_timeout=600s
- [ ] Invalid strategy name returns `RelError::Config`
- [ ] Invalid coordinator address format returns `RelError::Config`

use std::net::SocketAddr;
use std::time::Duration;

/// Build GridConfig from coordinator CLI arguments.
pub fn build_grid_config(args: &CoordinatorArgs) -> GridConfig {
    GridConfig {
        num_workers: args.workers,
        max_rounds: args.max_rounds,
    }
}

/// Build GridConfig from local mode CLI arguments.
pub fn build_grid_config_from_local(args: &LocalArgs) -> GridConfig {
    GridConfig {
        num_workers: args.workers,
        max_rounds: args.max_rounds,
    }
}

/// Build NodeConfig for coordinator mode (SPEC-06 v3: no role field, uses bind: SocketAddr).
pub fn build_node_config_coordinator(args: &CoordinatorArgs) -> NodeConfig {
    NodeConfig {
        bind: args.bind,
        num_workers: args.workers,
        max_payload_size: 256 * 1024 * 1024, // 256 MiB
        worker_connect_timeout: Duration::from_secs(120),
        distribute_timeout: Duration::from_secs(60),
        collect_timeout: Duration::from_secs(600),
    }
}

/// Build NodeConfig for worker mode, parsing the coordinator address into SocketAddr.
pub fn build_node_config_worker(args: &WorkerArgs) -> Result<NodeConfig, RelError> {
    let addr: SocketAddr = args.coordinator.parse()
        .map_err(|e| RelError::Config(format!("invalid coordinator address '{}': {}", args.coordinator, e)))?;
    Ok(NodeConfig {
        bind: addr,
        num_workers: 0,
        max_payload_size: 256 * 1024 * 1024,
        worker_connect_timeout: Duration::ZERO,
        distribute_timeout: Duration::ZERO,
        collect_timeout: Duration::ZERO,
    })
}
