## Context

The partitioning subsystem needs two foundational types before any other partition logic can be built. `WorkerId` is a type alias identifying a worker in the grid (0 to n-1). `IdRange` is a struct representing an exclusive, contiguous range of `AgentId` values reserved for a worker to generate new agents during local reduction without coordination. These types are used by `Partition`, `PartitionPlan`, and the ID space partitioning logic.

## Acceptance Criteria

- [ ] `WorkerId` is a public type alias for `u32`
- [ ] `IdRange` struct has `start: AgentId` (inclusive) and `end: AgentId` (exclusive) fields
- [ ] `IdRange` derives `Debug, Clone, Copy, PartialEq, Eq, serde::Serialize, serde::Deserialize`
- [ ] Module is declared in the partition module's `mod.rs`

## Key Types / Signatures

```rust
/// Identifier of a worker in the grid.
/// Values from 0 to n-1, where n is the number of workers.
pub type WorkerId = u32;

/// An exclusive range of AgentIds reserved for a worker.
/// The worker may generate new IDs in the interval [start, end).
#[derive(Debug, Clone, Copy, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
pub struct IdRange {
    /// First AgentId in the range (inclusive).
    pub start: AgentId,
    /// Last AgentId in the range (exclusive).
    pub end: AgentId,
}
```

## Notes

- `IdRange.end` is exclusive. The last worker's range extends to `u32::MAX` (inclusive), so its `end` field should be `u32::MAX`. This edge case is handled by the ID range computation logic (TASK-0047), not by IdRange itself.
