## Context

SPEC-02 Revised v3 (SC-005) added two new MUST-level requirements for agent iteration and counting on `Net`. These formalize the iteration pattern that 6+ successor specs needed (SPEC-03, SPEC-05, SPEC-07, SPEC-09, SPEC-11, SPEC-14). Previously, each consumer implemented its own `net.agents.iter().filter(|s| s.is_some()).count()` or equivalent inline pattern.

`count_live_agents(&self) -> usize` returns the number of live agents in the net (O(A) where A is the arena size). `live_agents(&self) -> impl Iterator<Item = &Agent>` provides an iterator over all live agents, skipping `None` slots.

These are the canonical iteration/counting API on Net. Consumer tasks (TASK-0069, TASK-0070, TASK-0073, TASK-0106, etc.) should call `net.count_live_agents()` and `net.live_agents()` instead of implementing their own helpers.

## Acceptance Criteria

- [ ] `count_live_agents(&self) -> usize` is a `pub` method on `Net`
- [ ] Returns the count of slots where `agents[i].is_some()`
- [ ] Complexity is O(A) where A is the arena length
- [ ] `live_agents(&self) -> impl Iterator<Item = &Agent>` is a `pub` method on `Net`
- [ ] The iterator skips `None` slots using `filter_map(|s| s.as_ref())`
- [ ] Both methods are publicly accessible from the `net` module

## Key Types / Signatures

```rust
impl Net {
    /// Returns the number of live (non-None) agents in the net.
    ///
    /// Complexity: O(A) where A is the arena length (`agents.len()`).
    pub fn count_live_agents(&self) -> usize {
        self.agents.iter().filter(|slot| slot.is_some()).count()
    }

    /// Returns an iterator over all live agents in the net.
    /// Skips `None` slots (removed or never-created agents).
    ///
    /// This encapsulates the internal `Vec<Option<Agent>>` representation.
    pub fn live_agents(&self) -> impl Iterator<Item = &Agent> {
        self.agents.iter().filter_map(|slot| slot.as_ref())
    }
}
```
