# TASK-0001: Convert net module to directory structure

**Spec:** SPEC-02
**Requirements:** (structural prerequisite for all R1-R27)
**Priority:** P0 (critical path)
**Status:** TODO
**Depends on:** none
**Blocked by:** none
**Estimated complexity:** S (< 50 LoC)

## Context

The current `src/net.rs` is a stub file with only a doc comment. SPEC-02 defines many types, operations, and debug assertions that will be organized across multiple sub-modules. This task converts `src/net.rs` into a `src/net/` directory with a `mod.rs` that re-exports all public items. This is a structural prerequisite for all subsequent TASK files in Phase 1.

## Acceptance Criteria

- [ ] `src/net.rs` is replaced by `src/net/mod.rs`
- [ ] `src/net/mod.rs` contains the module-level doc comment from the original `net.rs`
- [ ] `src/net/mod.rs` declares sub-modules: `pub mod types;`, `pub mod net;`, `pub mod debug;`
- [ ] `src/net/mod.rs` re-exports all public items from sub-modules (using `pub use`)
- [ ] `src/lib.rs` compiles with `pub mod net;` unchanged

## Files to Create/Modify

- `src/net.rs` -- **delete** (replaced by directory)
- `src/net/mod.rs` -- **create**: module declarations and re-exports
- `src/net/types.rs` -- **create**: empty file with doc comment placeholder
- `src/net/net.rs` -- **create**: empty file with doc comment placeholder
- `src/net/debug.rs` -- **create**: empty file with doc comment placeholder

## Key Types / Signatures

```rust
// src/net/mod.rs

//! Core types for Interaction Combinator networks (SPEC-02).
//!
//! Defines Symbol, Agent, PortRef, Net, and all operations on the
//! interaction net data structure. This is the foundation of Relativist.

pub mod types;
pub mod net;
pub mod debug;

pub use types::*;
pub use net::*;
```

## Test Expectations

## Dependencies Context

## Notes

- The sub-module names are a suggestion. The key constraint is that types (Symbol, Agent, PortRef, etc.) live in one file, Net struct + operations in another, and debug assertions in a third.
- Re-exports in `mod.rs` ensure that external code can still do `use relativist::net::Symbol;` without knowing the internal module structure.
