**Priority:** P1 (required for end-users to exercise `delta_mode`; not on the

**Blocked by:** None (after TASK-0389).
**Estimated complexity:** S (~60 LoC inc. tests)

## Context

SPEC-19 §3.6 R41 ships the `delta_mode` flag; R42 requires that the absence of
the flag preserves v1 behaviour exactly. This task adds the CLI surface:
`--delta-mode` on the `coordinator` and `local` subcommands, threaded through
`build_grid_config` and `build_grid_config_from_local` into the
`GridConfig.delta_mode` field landed by TASK-0389.

The precedent for this plumbing is `--use-zero-copy` (SPEC-18 §3.5 R36/R37,
TASK-0358): an opt-in `#[arg(long, default_value_t = false)]` bool that is
inert in default builds and only activates its feature when explicitly
requested. We mirror that pattern exactly.

The `worker` subcommand does **not** receive this flag — `delta_mode` is a
coordinator-side scheduling decision; workers learn their operating mode from
the coordinator through the wire protocol (sub-bundle 2.26-A). A
`worker --delta-mode` flag would be a user-facing red herring.

## Notes

- **Worker subcommand excluded by design:** workers don't make scheduling
  decisions — the coordinator is the authority. Adding `--delta-mode` to
  `WorkerArgs` would mislead users and duplicate state. If/when sub-bundle
  2.26-A adds a handshake that signals delta mode to the worker, the signal
  will flow through `Message::InitialPartition` or an equivalent wire variant,
  not via a local worker CLI flag.
- **`compute` subcommand excluded** for the same reason: `compute` uses
  in-process `reduce_all` or an internal `run_grid` invocation without exposing
  the full coordinator surface. If a future spec calls for `compute
  --delta-mode`, it is a separate task.
- **R42 compliance:** `default_value_t = false` + the fact that *nothing* in the
  core read-path uses `GridConfig.delta_mode` as of this sub-bundle means every
  existing integration test, benchmark, and smoke run produces bit-identical
  output. TASK-0391 formalises this as an explicit regression test.
