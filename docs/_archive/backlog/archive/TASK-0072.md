# TASK-0072: Implement n==1 optimization in run_grid

**Spec:** SPEC-05
**Requirements:** R26
**Priority:** P1 (important)
**Status:** TODO
**Depends on:** TASK-0069 (run_grid skeleton)
**Blocked by:** TASK-0071 (full run_grid must work first)
**Estimated complexity:** S (< 50 LoC)

## Context

When `n == 1` (single worker), the split produces one partition with no borders. The entire round degenerates to `reduce_all` on the full net, with no split/merge overhead. SPEC-05 R26 states this SHOULD be optimized to skip the split/merge phases entirely. This task adds an early-return path in `run_grid` for the single-worker case.

## Acceptance Criteria

- [ ] If `config.num_workers == 1`, `run_grid` calls `reduce_all` directly on the net without partitioning
- [ ] Metrics are still populated: `rounds = 1` (or 0 if already in NF), `total_interactions`, `total_interactions_by_rule`, `total_time`, `converged`
- [ ] The optimization is a clean early-return at the top of `run_grid`, before entering the loop
- [ ] Behavior is equivalent to running the full grid cycle with 1 worker (same result, just faster)
- [ ] The module compiles with `cargo check`

## Files to Create/Modify

- `src/merge.rs` -- modify: add n==1 early-return at the start of `run_grid`

## Key Types / Signatures

```rust
// At the start of run_grid, before the loop:

if config.num_workers == 1 {
    let start_time = Instant::now();
    let stats: ReductionStats = reduce_all(&mut current_net);
    let mut metrics = GridMetrics::default();
    metrics.total_interactions = stats.total_interactions;
    metrics.total_interactions_by_rule = reduction_stats_to_by_rule(&stats);
    metrics.total_time = start_time.elapsed();
    metrics.converged = true; // reduce_all always reaches NF for terminating nets
    // No rounds executed (direct reduction)
    return (current_net, metrics);
}
```

## Notes

- This is a SHOULD-level requirement. It is an optimization, not a correctness concern. The full grid cycle with 1 worker would produce the same result, just with unnecessary overhead.
- The `converged` field is set to `true` because `reduce_all` runs to completion for terminating nets. If the net is non-terminating, `reduce_all` itself would not return (there is no timeout in `reduce_all`).
- This task should be implemented AFTER the full grid loop works (TASK-0071), to avoid complicating the initial development.
- **SPEC-03 v3 change (SC-004):** `reduce_all` now returns `ReductionStats` (not `usize`). The `total_interactions` field provides the count. Pseudocode updated accordingly.
- **SPEC-05 v3 change (SC-004):** n==1 path must also populate `total_interactions_by_rule` via `reduction_stats_to_by_rule`. Pseudocode updated accordingly.
