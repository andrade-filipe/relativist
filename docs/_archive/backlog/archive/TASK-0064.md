## Context

During merge, boundary connections are restored via `Net::connect`. To count how many of those reconnections created border redexes (for metrics R13), the merge needs a helper function `is_principal_pair` that checks whether both endpoints are principal ports (`AgentPort(_, 0)`). This is a trivial pattern match but is extracted as a named function for clarity and reuse. The actual redex insertion into the queue is handled automatically by `Net::connect` (SPEC-02, R13); this helper only counts for metrics.

- [ ] Returns `false` for all other combinations (auxiliary ports, FreePort, mixed)

## Key Types / Signatures

```rust
/// Returns true if both ports are principal ports (port index 0).
///
/// Used during merge to count border redexes for metrics (SPEC-05, R12-R13).
/// The actual redex detection is performed by Net::connect (SPEC-02, R13);
/// this function only identifies principal-principal pairs for counting.
fn is_principal_pair(a: PortRef, b: PortRef) -> bool {
    matches!(
        (a, b),
        (PortRef::AgentPort(_, 0), PortRef::AgentPort(_, 0))
    )
}
```
