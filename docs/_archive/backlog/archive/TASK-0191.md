## Context

SPEC-09 Section 3.2.2 (R12) defines the **CON-DUP Expansion** benchmark: N pairs of CON-DUP connected by principal ports, with auxiliary ports connected to FreePorts. This is the ONLY benchmark where the net grows before collapsing (Profile B).

Each CON-DUP interaction creates 4 new agents (+2 balance), which then trigger annihilation cascades. This exercises the commutation rule plus resulting annihilation rules. Profile B behavior (expansion with collapse) is the primary unknown of the TCC -- the Haskell prototype has only 4 TCP datapoints for this benchmark (ARG-004).

Default sizes are smaller than EP because expansion creates O(N) additional agents. Includes Haskell comparison sizes: 50, 100, 1000 (R43).

## Key Types / Signatures

```rust
use crate::bench::{Benchmark, BenchmarkId};
use crate::bench::isomorphism::nets_isomorphic;
use crate::net::{Net, Symbol, PortRef};

/// CON-DUP Expansion: N pairs of CON-DUP connected by principal ports.
/// Profile B (expansion with collapse). Net grows before collapsing.
/// Rules exercised: CON-DUP (commutation), then CON-CON + DUP-DUP (annihilation).
///
/// SPEC-09 R12. Direct mapping from AC-005 `mkExpansionNet`.
/// Primary benchmark for validating Profile B behavior.
pub struct ConDupExpansion;

impl Benchmark for ConDupExpansion {
    fn id(&self) -> BenchmarkId {
        BenchmarkId::ConDupExpansion
    }

    fn describe(&self, size: u32) -> String {
        format!("CON-DUP Expansion: {} pairs, {} initial agents",
                size, size * 2)
    }

    fn make_net(&self, n: u32) -> Net {
        let mut net = Net::with_capacity((n * 2) as usize);
        for i in 0..n {
            let con = net.create_agent(Symbol::Con);
            let dup = net.create_agent(Symbol::Dup);
            // Principal ports: CON.0 <-> DUP.0 (forms CON-DUP redex)
            net.connect(
                PortRef::AgentPort(con, 0),
                PortRef::AgentPort(dup, 0),
            );
            // Auxiliary ports -> FreePorts
            net.connect(PortRef::AgentPort(con, 1), PortRef::FreePort(4 * i));
            net.connect(PortRef::AgentPort(con, 2), PortRef::FreePort(4 * i + 1));
            net.connect(PortRef::AgentPort(dup, 1), PortRef::FreePort(4 * i + 2));
            net.connect(PortRef::AgentPort(dup, 2), PortRef::FreePort(4 * i + 3));
        }
        net
    }

    fn default_sizes(&self) -> Vec<u32> {
        vec![10, 50, 100, 500, 1_000, 5_000]
    }

    fn verify(&self, sequential_result: &Net, distributed_result: &Net) -> bool {
        // CON-DUP expansion may leave a non-trivial normal form
        // (FreePorts reconnected in complex patterns). Use isomorphism.
        nets_isomorphic(sequential_result, distributed_result)
    }
}
```

## Notes

- This is the most computationally interesting benchmark. The CON-DUP rule is the only rule that INCREASES agent count. The expansion creates 4 new agents per CON-DUP interaction, but the subsequent annihilations (CON-CON, DUP-DUP) collapse the net.
- Default sizes are smaller than EP because the expansion factor means the net can grow significantly before collapsing. A size of 5000 may produce nets with > 20K agents at peak.
- The `verify` method uses `nets_isomorphic` rather than agent counting because the normal form may contain remaining FreePorts in a specific topology. This is the first benchmark that exercises the isomorphism verifier with non-trivial results.
