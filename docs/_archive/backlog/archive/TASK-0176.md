## Context

Two tree-structured generators that follow the benchmark definitions in SPEC-09 (R14, R15). These generators produce nets that simulate summation over a tree structure using IC encoding.

`tree_sum(n)` builds an unbalanced (left-leaning) tree of CON agents with `n` work units. Each leaf represents a unit of work, and the tree structure encodes a fold/reduce operation. The Haskell prototype reference is `mkTree` from AC-005.

`tree_sum_balanced(n)` builds a balanced binary tree. The Haskell prototype reference is `mkTreeBalanced` from AC-005. A balanced tree exposes more parallelism for distributed reduction because all branches have similar depth.

Both produce nets where the reduction terminates with a single result value extracted from the root.

## Acceptance Criteria

- [ ] `tree_sum(n: u32) -> Net` generates an unbalanced (left-leaning) tree of CON agents with `n` leaf units
- [ ] `tree_sum_balanced(n: u32) -> Net` generates a balanced binary tree of CON agents with `n` leaf units
- [ ] Each leaf is represented by an ERA agent (unit value) connected to a CON node

## Notes

- The exact structure of these generators depends on the SPEC-09 definitions. The Haskell prototype `mkTree` builds a left-leaning chain: `CON(CON(CON(..., ERA), ERA), ERA)`. The `mkTreeBalanced` variant splits `n` evenly: `CON(tree(n/2), tree(n - n/2))`.
- Edge case: `n = 0` should either produce an empty net or a single ERA agent. Follow the same convention as the Haskell prototype.
- These generators are Profile A/B benchmarks. They are less critical than `con_dup_expansion` (Profile B focus) but important for comparison.
- The `root` field of the net should be set to the root's output port for result extraction.
