## Context

SPEC-09 Section 3.2.6 (R17a, added in Revised v3, SC-004) defines the **ChurchAdd** benchmark: a net encoding Church numeral addition using `build_add(N/2, N - N/2)` from SPEC-14. This is Profile B (expansion with collapse) and the primary demonstration that Relativist performs real computation, not just synthetic graph rewriting.

ChurchAdd is a MUST benchmark. It provides a semantically meaningful Profile B workload that complements the synthetic CON-DUP Expansion benchmark. The result is verified by BOTH `decode_nat` (checking the arithmetic value) AND graph isomorphism with the sequential result.

## Key Types / Signatures

```rust
use crate::bench::{Benchmark, BenchmarkId};
use crate::bench::isomorphism::nets_isomorphic;
use crate::encoding::{build_add, decode_nat};
use crate::net::Net;

/// ChurchAdd: Church numeral addition benchmark.
/// Generates build_add(N/2, N - N/2) using SPEC-14 encoding.
/// Profile B (expansion with collapse).
/// Verified by decode_nat AND graph isomorphism.
///
/// SPEC-09 R17a (MUST). Added in Revised v3.
pub struct ChurchAdd;

impl Benchmark for ChurchAdd {
    fn id(&self) -> BenchmarkId {
        BenchmarkId::ChurchAdd
    }

    fn describe(&self, size: u32) -> String {
        let a = size / 2;
        let b = size - a;
        format!("ChurchAdd: church({}) + church({}) = church({}), Profile B",
                a, b, size)
    }

    fn make_net(&self, n: u32) -> Net {
        let a = (n / 2) as u64;
        let b = (n - n / 2) as u64;
        build_add(a, b)
    }

    fn default_sizes(&self) -> Vec<u32> {
        vec![10, 50, 100, 500]
    }

    fn verify(&self, sequential_result: &Net, distributed_result: &Net) -> bool {
        // Check 1: arithmetic value
        let seq_val = decode_nat(sequential_result);
        let dist_val = decode_nat(distributed_result);
        let value_ok = seq_val.is_some() && dist_val.is_some()
            && seq_val == dist_val;

        // Check 2: structural isomorphism
        let iso_ok = nets_isomorphic(sequential_result, distributed_result);

        value_ok && iso_ok
    }
}
```

## Notes

- Church addition is the primary "real computation" benchmark. It proves that Relativist correctly reduces a Church arithmetic net to the expected normal form, both semantically (decode_nat) and structurally (isomorphism).
- The dual verification (decode_nat + isomorphism) is intentionally redundant: decode_nat catches semantic errors, isomorphism catches structural divergence that might accidentally produce the right value.
- For odd N, the split is floor(N/2) + ceil(N/2). Example: N=7 -> build_add(3, 4).
- This benchmark was added as MUST to resolve OQ-6 (SPEC-09 v3).
