## Context

Workers connecting to a TLS-enabled coordinator need a client TLS configuration. `TlsClientConfig` wraps a `rustls::ClientConfig` pre-built from a PEM-encoded CA certificate file. The worker uses this CA cert to verify the coordinator's server certificate, supporting self-signed certificates (R27). No client certificate is presented (server TLS only, R23). The entire type is feature-gated under `tls`.

## Notes

- The `RootCertStore` is constructed with only the provided CA cert, NOT the system's trust store. This ensures that self-signed certificates work and the worker only trusts the specific CA the operator provides.
- The `TlsClientConfig` is created once at worker startup and cloned (via `Arc`) for each connection attempt (including retries).
- PEM files may contain multiple certificates (a chain). `rustls_pemfile::certs()` returns all of them. For a self-signed CA, there is typically just one.
