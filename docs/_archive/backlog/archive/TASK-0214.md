# TASK-0214: Wire AtomicBool readiness flag in coordinator FSM (R22a)

**Spec:** SPEC-11
**Requirements:** R22a
**Priority:** P1 (important, not blocking)
**Status:** TODO
**Depends on:** TASK-0108, TASK-0154
**Blocked by:** none
**Estimated complexity:** S (< 50 LoC)

## Context

SPEC-11 Revised v2 added R22a (SC-003, SC-012), which specifies that the coordinator FSM must maintain an `AtomicBool` readiness flag (`is_ready`). The FSM MUST set `is_ready = true` when transitioning from `Init` to `WaitingForWorkers`, and MUST set `is_ready = false` when transitioning to `Error` state.

This task wires the `AtomicBool` flag into the coordinator FSM transition function (TASK-0108). The HTTP readiness handler (TASK-0155) reads this flag; this task ensures the FSM writes it correctly.

## Acceptance Criteria

- [ ] Coordinator FSM transition function accepts an `Arc<AtomicBool>` parameter (or stores it in coordinator state)
- [ ] On `Init -> WaitingForWorkers` transition, `is_ready` is set to `true`
- [ ] On any transition to `Error` state, `is_ready` is set to `false`
- [ ] `Ordering::Relaxed` is used for stores (consistency is not critical for readiness probes)
- [ ] The `AtomicBool` is the same instance passed to `spawn_metrics_server()` (TASK-0157)
- [ ] No behavioral changes beyond setting the flag

## Files to Create/Modify

- `src/coordinator/*.rs` -- **modify**: add `is_ready.store()` calls at the appropriate FSM transition points

## Key Types / Signatures

```rust
use std::sync::{Arc, atomic::{AtomicBool, Ordering}};

// In the coordinator FSM transition function:

// On Init -> WaitingForWorkers:
is_ready.store(true, Ordering::Relaxed);
tracing::info!(

tracing::info!(

);
```

## Notes

- The `AtomicBool` must be created in the coordinator startup code (before both the FSM and the HTTP server are started) and shared via `Arc`.
- `Ordering::Relaxed` is sufficient because the readiness probe tolerates brief inconsistency. The flag only needs to eventually reflect the correct state.
- The `Done` state is considered "ready" (R22) so no flag change is needed on `CheckTermination -> Done`.
- R22a was added by SPEC-11 Revised v2 (SC-003, SC-012) to replace the fragile `AtomicU8` ordinal comparison approach.
