# TASK-0344: Compact PortRef serde encoding (R5-R8)

**Phase:** SPEC-18 §3.2 — Custom PortRef serialization (item 2.23)
**Size:** S (~150 LoC inc. tests)
**Depends on:** TASK-0343 (bincode v2 must be in place so varint payload encoding works)
**Spec:** SPEC-18 R5, R6, R7, R8

**Scope:** Implement a custom serde serializer/deserializer for `PortRef` that
shrinks the most common encoding paths to 1-4 bytes (vs 6-9 bytes with default
varint). The DISCONNECTED sentinel (`FreePort(u32::MAX)`) collapses to a
single byte.

---

## Deliverable

1. **`src/net/types.rs` — module `portref_compact`**: serde
   `serialize`/`deserialize` functions implementing the wire format from
   SPEC-18 §4.3:
   ```
   0xFF                        -> DISCONNECTED
   0x00 + varint(id) + u8(pid) -> AgentPort(id, pid)
   0x01 + varint(border)       -> FreePort(border) where border != u32::MAX
   ```
   - Use `serde::Serializer::serialize_bytes` over a small stack buffer
     (max 7 bytes: tag + 5-byte varint + 1-byte port).
   - Varint encoding follows bincode v2 conventions (LEB128-like, single byte
     for values 0-250). Reuse bincode's varint helpers if exposed; otherwise
     implement a private `write_varint`/`read_varint` helper module-local.

2. **`PortRef` field annotation**: apply `#[serde(with = "crate::net::types::portref_compact")]`
   on `PortRef` so every serde-driven serialization (bincode v2, JSON,
   wire payloads carrying `PortRef`) goes through the compact encoding.
   - Verify this composes correctly with the existing `CompactSubnet` serde
     adapter (SPEC-18 §8 Q3).

3. **Tests (inline in `types.rs` or a new `#[cfg(test)] mod compact_portref_tests`)**:
   - **T2 round-trip** (R7): every `PortRef` variant from SPEC-18 §7.1 T2 list
     (`AgentPort(0,0)`, `(16383,2)`, `(16384,1)`, `(u32::MAX-1,2)`, `FreePort(0)`,
     `FreePort(1000)`, `FreePort(u32::MAX)`).
   - **T3 size assertions** (R8):
     - `AgentPort(100, 0)` ≤ 4 bytes.
     - `DISCONNECTED` exactly 1 byte.
     - `AgentPort(0, 0)` exactly 3 bytes (1 tag + 1 varint + 1 port).
   - Adversarial: deserialize a malformed payload (unknown tag byte, truncated
     varint) → returns serde error.

## Acceptance

- `cargo test --workspace` count: 801 → 805+ (≥ +4 new compact PortRef tests).
- All existing tests still pass (the change is observationally equivalent at
  the value level: `decode(encode(p)) == p` is preserved; only on-wire bytes shrink).
- `cargo clippy --workspace --all-targets -- -D warnings` clean.
- `cargo fmt --check` clean.
- Bonus measurement (logged in REVIEW, not enforced): for a representative
  partition from `build_partition_for_tests()`, total wire bytes drop by
  ≥ 30% relative to the post-TASK-0343 baseline. (R8 is SHOULD; this is a
  sanity check that the compact encoding is actually saving bytes.)

## Notes

- The `serde_json` deserializer will see the compact bytes as a byte array,
  which differs from the v1 `PortRef` JSON shape. If any production code path
  relies on serde_json for PortRef (unlikely — JSON is used for codec input,
  not net structure), document the breakage in REVIEW. Otherwise treat the
  custom encoding as wire-only.
