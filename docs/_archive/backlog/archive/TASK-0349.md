# TASK-0349: Populate `has_border_activity` at every WorkerRoundStats build site (R2)

**Bundle:** SPEC-19 §3.1 — Coordinator-Free Round (item 2.34)
**Spec:** SPEC-19 R2 (worker-side reporting)
**Priority:** P0
**Status:** TODO
**Depends on:** TASK-0348 (field + helper must exist)
**Blocked by:** TASK-0348
**Estimated complexity:** S (~50 LoC inc. tests)

## Context

R2 says the worker MUST set `has_border_activity` correctly in **every**
`WorkerRoundStats` it emits. After TASK-0348 the field exists and defaults
to `false` at every construction site. This task replaces those
placeholder `false`s with a real call to
`merge::helpers::compute_border_activity(&partition)` at the two real
production paths:

1. **Distributed worker** (`relativist-core/src/worker.rs` and/or
   `relativist-core/src/protocol/worker.rs`) — runs `reduce_all` on
   the received `Partition`, calls `rebuild_free_port_index`, and
   builds `WorkerRoundStats` before sending `PartitionResult`.
2. **Local in-process simulation** (`relativist-core/src/merge/grid.rs`,
   inside `run_grid`'s per-worker loop, around line 87 where
   `rebuild_free_port_index` is invoked) — the same stats are built for
   the local-simulation path so that R7 compatibility is uniform across
   transport modes.

The order is fixed by R1: `reduce_all → rebuild_free_port_index →
compute_border_activity`. The activity flag MUST reflect the
**post-reduction** partition state, not the pre-reduction one.

## Acceptance Criteria

- [ ] Every place in `relativist-core/src/` that constructs a
      `WorkerRoundStats` for a real round (NOT a test fixture) sets
      `has_border_activity` from
      `compute_border_activity(&partition)`.
- [ ] `compute_border_activity` is called AFTER `rebuild_free_port_index`
      (R1 ordering) — verified by code reading and by an integration
      test (below).
- [ ] No regression on the 850 lib + 4 integration tests baseline.
- [ ] Clippy + fmt clean.
- [ ] Release smoke: `compute add 3 5 → 8` continues to work.

## Files to Modify

- `relativist-core/src/worker.rs` — modify: in the function that builds
  the per-round `WorkerRoundStats` for the distributed path, replace
  the placeholder `has_border_activity: false` with
  `has_border_activity: compute_border_activity(&partition)`.
- `relativist-core/src/protocol/worker.rs` — modify: same change in the
  protocol-loop construction site (if a separate one exists; otherwise
  this file just consumes the worker.rs build).
- `relativist-core/src/merge/grid.rs` — modify: same change inside
  `run_grid`'s per-worker local-simulation loop (around line 90,
  immediately after the `rebuild_free_port_index` call).

## Test Expectations (deferred to test-generator)

- **Worker unit test:** construct a `Partition` whose `free_port_index`
  contains one `AgentPort(_, 0)` entry; run the worker round-build path
  and assert `stats.has_border_activity == true`.
- **Worker unit test (negative):** construct a `Partition` whose
  `free_port_index` is empty (or only `AgentPort(_, 1)` entries) and
  assert `stats.has_border_activity == false`.
- **Local-simulation integration test:** call `run_grid` on a small net
  partitioned into 2 workers where one partition has a border principal
  port and the other does not. Assert that `metrics.worker_stats_per_round[0]`
  records exactly one worker with `has_border_activity == true` and one
  with `has_border_activity == false` for round 0.
- **Ordering test:** assert that `compute_border_activity` is called
  AFTER `rebuild_free_port_index` by constructing a partition where
  `free_port_index` is intentionally stale (empty) but the underlying
  ports show a principal endpoint — after `rebuild_free_port_index` the
  index is repopulated and the activity flag becomes `true`. (This
  catches the bug of computing activity on a stale index.)

  build stats`. A reordering breaks R1.
- Do NOT touch the merge / coordinator skip logic — that is TASK-0351.
- Do NOT add the new config field — that is TASK-0350. This task only
  populates the data; consumption by the coordinator happens later.
