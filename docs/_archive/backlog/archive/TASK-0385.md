## Context

This task fills in `run_grid_delta_inner` — the coordinator's round loop. The
flow is:

1. **Round 0 (R21.1):** dispatch `Message::InitialPartition` to every worker
   via `dispatch.dispatch_initial(&plan)`. No results collected (workers just
   store the partition and transition to `DeltaIdle`, per TASK-0380).
2. **Initialize BorderGraph (R10 from §3.2, already shipped):**
   `let mut border_graph = BorderGraph::from_partition_plan(&plan);`.
3. **Delta round loop (R21.2):**
   - For each round `r = 1, 2, ...`:
     a. Detect current border redexes: `let redexes =
        border_graph.detect_border_redexes();`
     b. Resolve them via 2.26-B's `BorderResolver::resolve(&mut
        border_graph, &redexes, &coordinator_partition_cache)` → returns
        `Vec<BorderResolution>`.
     c. Package resolutions into `RoundStartDispatch` per worker
        (2.26-B's `package_resolutions`).
     d. Call `dispatch.dispatch_round_start(&dispatch_payload)` — the
        dispatch implementation sends `Message::RoundStart` to every
        worker and blocks until every `Message::RoundResult` arrives.
        Returns `Vec<RoundResultPayload>`.
     e. Apply each worker's reported deltas to the `BorderGraph`:
        `for r in &results { border_graph.apply_deltas(r.worker_id,
        &r.border_deltas); }`
     f. Update the coordinator-side partition cache (DC-B1 of 2.26-B
        assumed option (a) — coordinator caches partitions):
        `coordinator_partition_cache.apply_deltas(r.worker_id,
        &r.border_deltas, &resolved_borders, &new_borders);` (re-uses
        TASK-0381's `apply_border_deltas_to_partition` helper).
     g. Accumulate metrics: `metrics.rounds += 1;`
        `metrics.worker_stats_per_round.push(results.iter().map(|r|
        r.stats.clone()).collect());` etc.
     h. **Convergence check (delegated to TASK-0386):** if
        `check_delta_convergence(&results, &border_graph) == true`,
        break out of the loop.
     i. **Max rounds cap (delegated to TASK-0388):** if
        `config.max_rounds.is_some_and(|m| metrics.rounds >= m)`, break
        out of the loop and set `metrics.delta_max_rounds_hit =
        Some(true)`.
4. **Delegate final collection to TASK-0387:**
   `let final_net = run_grid_delta_final_collect(&mut dispatch,
    coordinator_partition_cache, border_graph, metrics)?;`
5. Return `Ok(final_net)`.

## Key Types / Signatures

```rust
// merge/grid.rs

fn run_grid_delta_inner(
    plan: PartitionPlan,
    config: &GridConfig,
    dispatch: &mut dyn WorkerDispatch,
    metrics: &mut GridMetrics,
) -> Result<Net, GridError> {
    use crate::merge::border_graph::BorderGraph;
    use crate::merge::border_resolver::{BorderResolver, package_resolutions};

    // === Round 0: Initial Dispatch (R21.1) ===
    dispatch.dispatch_initial(&plan)?;

    let mut border_graph = BorderGraph::from_partition_plan(&plan);
    let mut coordinator_partition_cache: HashMap<WorkerId, Partition> =
        plan.partitions.iter().map(|p| (p.worker_id, p.clone())).collect();

    // === Rounds 1..N: Delta Rounds (R21.2) ===
    loop {
        // [max_rounds cap check — TASK-0388]
        if check_max_rounds_cap(config, metrics) {
            metrics.delta_max_rounds_hit = Some(true);
            break;
        }

        // Step 3a: detect redexes the coordinator owns.
        let t_partition = Instant::now();
        let redexes = border_graph.detect_border_redexes();

        // Step 3b: resolve them (2.26-B).
        let t_border = Instant::now();
        let resolutions = BorderResolver::resolve(
            &mut border_graph,
            &redexes,
            &coordinator_partition_cache,
        );
        let border_duration = t_border.elapsed();

        // Step 3c: package per-worker.
        let dispatch_payload: RoundStartDispatch =
            package_resolutions(metrics.rounds as u32 + 1, &resolutions);
        metrics.partition_time_per_round.push(t_partition.elapsed());

        // Step 3d: dispatch + block for worker results.
        let t_compute = Instant::now();
        let results = dispatch.dispatch_round_start(&dispatch_payload)?;
        metrics.compute_time_per_round.push(t_compute.elapsed());

        // Step 3e + 3f: apply to border graph + partition cache.
        let t_merge = Instant::now();
        for result in &results {
            border_graph.apply_deltas(result.worker_id, &result.border_deltas);
            if let Some(cached) = coordinator_partition_cache.get_mut(&result.worker_id) {
                apply_border_deltas_to_partition(
                    cached,
                    &result.border_deltas,
                    &[],   // resolved_borders were applied via BorderGraph
                    &[],   // new_borders ditto
                );
            }
        }
        metrics.merge_time_per_round.push(t_merge.elapsed());

        // Step 3g: metrics.
        metrics.rounds += 1;
        metrics.worker_stats_per_round.push(
            results.iter().map(|r| r.stats.clone()).collect()
        );
        metrics.border_redexes_per_round.push(redexes.len() as u64);
        metrics.border_reduce_time_per_round.push(border_duration);
        metrics.border_interactions_per_round.push(resolutions.len() as u64);

        // Step 3h: convergence (TASK-0386).
        if check_delta_convergence(&results, &border_graph) {
            metrics.converged = true;
            break;
        }
    }

    // === Final State Collection (R21.3) — TASK-0387 ===
    let final_net = run_grid_delta_final_collect(
        dispatch,
        coordinator_partition_cache,
        border_graph,
        metrics,
    )?;
    Ok(final_net)
}
```

## Notes

- **DC-B1 consistency:** this task commits to option (a) — coordinator
  caches partitions, refreshed per-round via the same
  `apply_border_deltas_to_partition` the workers use. This is
  symmetric: both sides maintain equivalent views of every partition's
  border state. If spec-critic overrules 2.26-B DC-B1, this task's
  cache management regenerates.
- **Metrics compatibility:** all per-round metric vectors have the same
  shape as v1's `run_grid`, so existing benchmark / reporting code
  works unchanged. The only new fields are `delta_mode` and
  `delta_max_rounds_hit` (TASK-0384).
- **Round 0 semantics:** no worker stats collected. `metrics.rounds`
  starts at 0 and is incremented AFTER each delta round's work, so
  Round 1's completion makes `metrics.rounds == 1`. This matches v1.
- **No border-resolution cascade:** the resolver returns all resolutions
  for the current round's redexes in one pass. If CON-DUP resolution
  creates new borders that are themselves immediately redexes, those
  are NOT resolved within the same round — they surface as new redexes
  in round r+1 (BSP semantics, consistent with strict BSP).
- **Abort-on-dispatch-error:** if `dispatch.dispatch_round_start`
  returns `Err`, the whole loop returns `Err`. The outer `run_grid_delta`
  (TASK-0384) handles the error by returning best-effort (net + metrics
  with `converged = false`). A retry policy is OUT of scope.

## Spec-Critic Amendments (2026-04-17)

| DC | Verdict | Change |
|---|---|---|
| DC-C1 | B (ratified) | Coordinator side: `InitialPartition` is fire-and-forget; no ack wait; immediately enters round-1 dispatch |
| DC-C3 | C (FLIP from task-splitter default A) | Round loop MUST branch on `config.strict_bsp`; BOTH cells of `(delta_mode=true, strict_bsp ∈ {true,false})` implemented |

**Details:**

- **DC-C1 coordinator-side hardening:** per DC-C1 of `docs/spec-reviews/SPEC-19-section-3.3-2.26C-design-choices-2026-04-17.md`, add to "Context": after dispatching `InitialPartition` to every worker via `dispatch.dispatch_initial(&plan)`, the coordinator does NOT block on an ack. It immediately enters the round-1 dispatch loop (`RoundStart` to every worker), then blocks on `RoundResult` collection. A worker whose `InitialPartition` was dropped will be detected via TCP disconnection or Round 1 timeout (SPEC-06 R18-R19). This mirrors the v1 `AssignPartition` → `PartitionResult` contract.

- **DC-C2 cross-pin:** the "Files FORBIDDEN" list explicitly adds `relativist-core/src/protocol/coordinator.rs` async `impl WorkerDispatch` body (deferred to 2.26-D or a follow-up wire bundle). The round loop calls the trait methods SYNCHRONOUSLY.

- **DC-C3 (BLOCKING SEMANTIC FLIP):** the task-splitter's default assumed `config.strict_bsp == true` was hard-asserted at `run_grid_delta` entry (TASK-0384). Per DC-C3, the assert is removed and **the round loop MUST branch on `config.strict_bsp`**:
  - **Delta lenient branch** (`strict_bsp == false`): after collecting `RoundResult`s and applying deltas to `BorderGraph` (steps 3e+3f), IMMEDIATELY invoke `BorderResolver::resolve(&mut border_graph, &border_graph.detect_border_redexes(), &coordinator_partition_cache)` and merge the resolution outputs into THIS round's next-dispatch `RoundStart` payload (via `package_resolutions`). Convergence check (TASK-0386) runs AFTER resolution. In the absence of cross-partition cascades, 1 delta round suffices — matching `R_delta_lenient = 1`.
  - **Delta strict branch** (`strict_bsp == true`): after collecting `RoundResult`s and applying deltas, run `BorderResolver::resolve` and STORE the resolution outputs in a pending buffer; dispatch them as the k+1 round's `RoundStart.border_deltas`. Convergence check runs after apply-only (pre-resolution) via `BorderGraph::detect_border_redexes` for strict semantics. This matches v1 strict mode round discipline and R40 line 244 ("Border redexes are dispatched to workers for resolution in subsequent rounds").
  - **One-round defer (pin):** strict mode defers exactly one round (round k's resolutions dispatched in round k+1's `RoundStart`), matching v1 strict mode. SPEC-19 §3.3 does not pin this; the pin is recorded here to forestall future batching ambiguity. If a future bundle wants multi-round batching, a SPEC-19 amendment is required first.

- **New integration tests** to add to Acceptance Criteria (exercising the 4-cell matrix `delta_mode × strict_bsp`; v1 cells covered by existing `run_grid` tests, so the bundle adds only the two `delta_mode=true` cells plus a G1 parity cross-check):
  - `run_grid_delta_lenient_converges_in_one_round_absent_cascades`: a partition pair whose only inter-partition redex resolves in a single resolution cycle — `strict_bsp = false` + `delta_mode = true` → exactly 1 delta round. Matches `R_delta_lenient = 1`.
  - `run_grid_delta_strict_multi_round_cascade`: same partition pair under `strict_bsp = true` + `delta_mode = true` → ≥ 2 delta rounds (deferred dispatch).
  - `run_grid_delta_result_matches_run_grid_under_both_strict_modes`: final net from `run_grid_delta` is isomorphic to `run_grid`'s output under the SAME `strict_bsp` value (G1 preservation per SPEC-19 R38 amendment). Run both strict=true and strict=false.

- **Cross-bundle note (2.26-B resolver):** `BorderResolver::resolve` must be callable in BOTH lenient (inline, same round) and strict (deferred, stored for round k+1) contexts. The resolver API itself is mode-agnostic per SPEC-19 §4.4 pseudocode; 2.26-C just wires it differently in each branch.

- **Cross-bundle note (2.26-D):** `GridConfig.delta_mode` + `strict_bsp` validation in 2.26-D MUST NOT reject `(delta_mode=true, strict_bsp=false)`.

**Test count delta:** +3 (three new integration tests for the `delta × strict` matrix).
