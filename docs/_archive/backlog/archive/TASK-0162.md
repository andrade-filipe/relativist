## Context

Two summary data structures are needed throughout the I/O layer. `NetSummary` holds statistics about a net (agent count, wire count, redex count, symbol distribution, free ports, normal form status) -- used by the `inspect` subcommand (R29) and as input to the reduction summary. `ReductionSummary` holds the before/after `NetSummary` plus timing and interaction metrics -- used by the `reduce` subcommand (R44-R45) and optionally extended with grid metrics for distributed execution (R46).

This task defines only the data structures. The computation logic (populating these structs) is in TASK-0169 and TASK-0170.

> **Revision note (SPEC-12 v2, SC-011, SC-020):** The `ReductionSummary` struct was redesigned. The `GridReductionInfo` wrapper struct was removed. Grid-specific fields (`rounds`, `workers`, `speedup`, `efficiency`, `overhead_pct`) are now flat `Option` fields on `ReductionSummary` directly. The `max_interactions_reached` field was replaced by `normal_form: bool` and `termination_reason: Option<String>` for more flexible status reporting. The `speedup`, `efficiency`, and `overhead_pct` fields are `None` unless the user provides `--baseline-secs` (R44b). R44a defines `duration_secs` as wall-clock time of `reduce_all` only (excluding file I/O).

## Key Types / Signatures

```rust
/// Summary statistics of a net (SPEC-12, R29).
///
/// Produced by the `inspect` subcommand and used as input/output
/// snapshots in ReductionSummary.
#[derive(Debug, Clone, serde::Serialize)]
pub struct NetSummary {
    pub agents: u32,
    pub wires: u32,
    pub redexes: u32,
    pub con: u32,
    pub dup: u32,
    pub era: u32,
    pub free_ports: u32,
    pub normal_form: bool,
}

/// Summary of a reduction execution (SPEC-12, R44-R47, Section 4.4).
///
/// Printed to stdout after any reduction (local or distributed).
/// Grid-specific fields are None for local `reduce`.
/// Speedup/Efficiency/Overhead are None unless --baseline-secs is provided (R44b).
#[derive(Debug, Clone, serde::Serialize)]
pub struct ReductionSummary {
    pub input: NetSummary,
    pub output: NetSummary,
    pub total_interactions: u64,
    pub duration_secs: f64,
    pub mips: f64,
    pub normal_form: bool,
    /// Reason for stopping if not in Normal Form (e.g., "max-interactions reached").
    pub termination_reason: Option<String>,
    // --- Grid-specific fields (None for local `reduce`) ---
    pub rounds: Option<u32>,
    pub workers: Option<u32>,
    /// Present only if --baseline-secs is provided (R44b).
    pub speedup: Option<f64>,
    /// Present only if --baseline-secs is provided.
    pub efficiency: Option<f64>,
    /// Present only if --baseline-secs is provided.
    pub overhead_pct: Option<f64>,
}
```

## Notes

- `NetSummary` is used in three places: (1) `inspect` output (R29-R31), (2) the `input`/`output` fields of `ReductionSummary`, (3) the `--metrics` JSON output (R21).
- The `mips` field is Millions of Interactions Per Second: `total_interactions as f64 / duration_secs / 1_000_000.0`.
- R44a: `duration_secs` for the `reduce` subcommand measures wall-clock time of `reduce_all` only, excluding file I/O.
- R44b: `speedup`, `efficiency`, and `overhead_pct` are conditional on `--baseline-secs`. If not provided, those fields are `None` and the corresponding output lines are omitted.
- `termination_reason` replaces the old `max_interactions_reached` boolean for more flexible status reporting (R47).
