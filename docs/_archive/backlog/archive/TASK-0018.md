**Blocked by:** none

## Context

`Net` needs structural equality comparison for two purposes: (1) verifying the serialization round-trip identity (R26: `deserialize(serialize(net)) == net`), and (2) testing that operations produce expected results.

**SPEC-02 v3 change:** R26a now mandates that `Net` derives both `PartialEq` and `Eq`. The derive is now included directly in TASK-0008 (Net struct definition). This task becomes a verification task: ensure the derives are present and write the equality-focused tests.

The comparison is structural (field-by-field), NOT isomorphism (which would require agent ID renaming -- that is a separate concept from SPEC-00 Section 6.12, used in G1 testing). Structural equality means: same agents in same slots, same port array, same redex queue, same next_id, same root. Note: for graph isomorphism (structural equivalence modulo ID renaming), use `nets_isomorphic` (SPEC-08).

`PartialEq` and `Eq` are derived rather than manually implemented because all fields already implement both traits (`Vec<Option<Agent>>`, `Vec<PortRef>`, `VecDeque<(u32, u32)>`, `u32`, `Option<PortRef>`).

## Acceptance Criteria

- Unit test: serialization round-trip: `Net::from_bytes(&net.to_bytes()?) == Ok(net)`

## Notes

- Since SPEC-02 Revised v3 (R26a), `PartialEq` and `Eq` are included in the initial Net definition (TASK-0008). This task is now a verification/testing task rather than an implementation task.
