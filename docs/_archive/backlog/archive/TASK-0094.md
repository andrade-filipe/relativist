## Context

SPEC-05 already defines `GridMetrics` with fields for network metrics (`bytes_sent_per_round`, `bytes_received_per_round`, `network_send_time_per_round`, `network_recv_time_per_round`). SPEC-06 concretizes these fields and adds two convenience methods: `total_network_bytes()` (sum of all bytes sent and received) and `network_overhead_fraction()` (ratio of network time to total time, per the formula from DISC-006 v2 Section 1.1). These methods enable the benchmarks (SPEC-09) to calculate communication overhead.

## Acceptance Criteria

- [ ] `total_network_bytes()` method is implemented on `GridMetrics`
- [ ] Returns `sum(bytes_sent_per_round) + sum(bytes_received_per_round)`
- [ ] `network_overhead_fraction()` method is implemented on `GridMetrics`
- [ ] Returns `(sum(send_time) + sum(recv_time)) / total_time`
- [ ] Returns `0.0` if `total_time` is zero (avoid division by zero)
- [ ] Bytes counted include the 8-byte header per frame (R34) -- this is ensured by `send_frame`/`recv_frame` returning total bytes including header

## Key Types / Signatures

```rust
impl GridMetrics {
    /// Returns the total bytes transferred across all rounds (sent + received).
    pub fn total_network_bytes(&self) -> usize {
        self.bytes_sent_per_round.iter().sum::<usize>()
            + self.bytes_received_per_round.iter().sum::<usize>()
    }

    /// Returns the communication overhead as a fraction of total time.
    /// Formula: (sum(send_time) + sum(recv_time)) / total_time
    /// Cf. DISC-006 v2, Section 1.1.
    pub fn network_overhead_fraction(&self) -> f64 {
        let send_total: Duration = self.network_send_time_per_round.iter().sum();
        let recv_total: Duration = self.network_recv_time_per_round.iter().sum();
        let network_total = send_total + recv_total;
        if self.total_time.is_zero() {
            0.0
        } else {
            network_total.as_secs_f64() / self.total_time.as_secs_f64()
        }
    }
}
```

- R35 requires that the metrics be sufficient to calculate communication overhead. The `network_overhead_fraction()` method directly implements the formula from DISC-006 v2.
- This task adds methods to an existing struct, not new fields. The fields are defined in the SPEC-05 phase (Phase 4 tasks).
