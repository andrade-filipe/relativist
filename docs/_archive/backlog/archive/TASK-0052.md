## Context

Each partition maintains a `FreePort index` -- a `HashMap<u32, PortRef>` mapping each `borderId` present in the partition to the local `AgentPort` connected to the corresponding `FreePort(bid)`. This index enables O(1) lookup during merge (SPEC-05), eliminating the O(W) linear scan of the Haskell prototype's `freePortNeighbor` (AC-002, L3). The index is built from the border entries generated during wire classification (TASK-0046).

## Acceptance Criteria

- [ ] Function `build_free_port_index` takes border entries for one partition and returns `HashMap<u32, PortRef>`
- [ ] Each entry maps `borderId -> AgentPort(agent_id, port_id)` for the local side of the border
- [ ] Every borderId in the partition's border entries has exactly one entry in the index
- [ ] No duplicate keys (each borderId appears at most once per partition by C3)

## Notes

- This is a straightforward transformation from `Vec<(AgentId, PortId, u32)>` to `HashMap<u32, PortRef>`. The simplicity is intentional -- the complexity is in the wire classification (TASK-0046) that produces the border entries.
- The FreePort index may become stale during local reduction (reconnections, erasures). See TASK-0055 for lazy reconstruction before merge.
