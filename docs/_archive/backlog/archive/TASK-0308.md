# TASK-0308: Refactor coordinator.rs to use Transport trait

**Spec:** SPEC-17, Section 3.10 (R35, R37, R38)
**Phase:** v2 — Milestone M1
**Priority:** P0
**Complexity:** M (~100 LoC changes)
**Depends on:** TASK-0302, TASK-0303, TASK-0304, TASK-0306, TASK-0307
**Blocks:** TASK-0311

## Description

Refactor all coordinator functions to use `Transport` trait instead of hardcoded TCP:

### accept_workers (R35)
- Change signature: accept `transport: &mut dyn Transport` instead of binding `TcpListener` internally
- Call `transport.listen()` + loop `transport.accept()` instead of `TcpListener::bind()`/`accept()`
- Return `Vec<TransportStream>` instead of `(TcpListener, Vec<TcpStream>)`

### distribute_partitions
- Change `worker_streams: &mut [TcpStream]` → `worker_streams: &mut [TransportStream]`
- `send_frame` already generic — should work without change

### collect_results
- Change `worker_streams: &mut [TcpStream]` → `worker_streams: &mut [TransportStream]`

### shutdown_workers
- Change `worker_streams: &mut [TcpStream]` → `worker_streams: &mut [TransportStream]`

### run_coordinator
- Accept `transport: &mut dyn Transport`
- Pass to `accept_workers`

### R37: No direct TcpListener/TcpStream imports
- Remove `use tokio::net::{TcpListener, TcpStream};` from coordinator.rs

### R38: Identical observable behavior
- Same message sequences, same timeouts, same retry logic

## Critical: Test Migration

All existing coordinator tests use `TcpListener::bind()` and `TcpStream::connect()` directly. These must be migrated to use `TcpTransport` or `ChannelTransport`. This is the highest-risk task in the pipeline.

Strategy: migrate tests to `ChannelTransport` where possible (faster, no port conflicts), keep TCP integration tests via `TcpTransport`.

## Acceptance Criteria

1. `accept_workers` accepts `&mut dyn Transport`
2. All functions use `TransportStream` instead of `TcpStream`
3. No `tokio::net::TcpListener` or `tokio::net::TcpStream` imports in coordinator.rs (R37)
4. Tests: CW1 (all 690 pass), CW2 (accept via ChannelTransport)
5. Test: CW4 (no direct TCP imports — grep verification)
6. G1 distributed tests still pass
