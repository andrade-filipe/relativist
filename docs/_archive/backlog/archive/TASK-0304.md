# TASK-0304: Implement TcpTransport with TCP tuning

**Spec:** SPEC-17, Sections 3.2 + 3.5 (R6-R9, R17-R22, R40, R42-R43)
**Phase:** v2 — Milestone M1
**Priority:** P0
**Complexity:** M (~130 LoC)
**Depends on:** TASK-0300, TASK-0301, TASK-0303
**Blocks:** TASK-0307, TASK-0308

## Description

Create `src/protocol/tcp.rs` implementing `TcpTransport`:

1. **Struct**: holds `bind_addr: SocketAddr`, `config: TransportConfig`, `listener: Option<TcpListener>`
2. **`apply_tuning(&self, stream: &TcpStream)`** (R7, R17-R22):
   - Set `TCP_NODELAY` (R17)
   - Set `SO_SNDBUF` via `socket2::SockRef` (R18)
   - Set `SO_RCVBUF` via `socket2::SockRef` (R19)
   - Set TCP keepalive with `socket2::TcpKeepalive` (R20-R21)
   - Log actual buffer sizes at `debug` level (R43)
3. **`Transport` impl** (R6, R9):
   - `listen()`: bind `TcpListener`, log at `info` (R42)
   - `accept()`: accept connection, apply tuning, return boxed stream
   - `connect()`: connect, apply tuning, return boxed stream
4. **TLS**: when `tls` feature is enabled, wrap stream with `tokio-rustls` (R8). For now, just the non-TLS path — TLS integration exists in TASK-0132/0133 and will be reconnected.

## Acceptance Criteria

1. `TcpTransport::new(addr, config)` constructs correctly
2. `listen()` binds and logs
3. `accept()` returns tuned `TransportStream`
4. `connect()` returns tuned `TransportStream`
5. Tests: TT1 (NODELAY), TT2 (buffer sizes), TT3 (keepalive), TT4 (None buffers), TT5 (keepalive disabled)
6. Test: TR1 (TcpTransport round-trip with `send_frame`/`recv_frame`)
7. Tuning is O(1) per connection (R40)
8. All 690 tests still pass
