# TASK-0310: Add CLI transport flags

**Spec:** SPEC-17, Section 3.8 (R29-R32)
**Phase:** v2 — Milestone M1
**Priority:** P0
**Complexity:** S (~50 LoC)
**Depends on:** TASK-0301, TASK-0302
**Blocks:** TASK-0311

## Description

Add transport-related CLI flags to `CoordinatorArgs` and `WorkerArgs` in `src/config.rs` (R30):

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--transport` | `tcp` \| `unix` | `tcp` | Transport backend selection (R29) |
| `--socket-path` | `PathBuf` | `/tmp/relativist.sock` | UDS path (only with unix) |
| `--tcp-nodelay` | `bool` | `true` | Enable/disable TCP_NODELAY |
| `--send-buffer` | `usize` | `4194304` | SO_SNDBUF size in bytes |
| `--recv-buffer` | `usize` | `4194304` | SO_RCVBUF size in bytes |
| `--keepalive` | `u64` | `30` | TCP keepalive idle in seconds; 0=disabled |

### Validation (R31-R32)
- `--transport=unix` on non-Unix → error with clear message (R31)
- `--socket-path` without `--transport=unix` → warning (R32)

### Config mapping
Update the CLI-to-config mapping in `src/commands.rs` to populate `TransportConfig` from these flags and pass it to `NodeConfig.transport`.

## Acceptance Criteria

1. All flags parse correctly (tests: CL1, CL5)
2. `--transport=unix` on Windows produces error (CL3)
3. `--socket-path` without `--transport=unix` produces warning (CL4)
4. `TransportConfig` correctly populated from CLI flags
5. All 690 tests still pass

## Files to Modify

- `src/config.rs`: add flags to CoordinatorArgs, WorkerArgs
- `src/commands.rs`: update config mapping to populate TransportConfig
